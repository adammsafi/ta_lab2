**RiskLimits dataclass:**
- max_position_pct: Decimal = Decimal("0.15")
- max_portfolio_pct: Decimal = Decimal("0.80")
- daily_loss_pct_threshold: Decimal = Decimal("0.03")
- cb_consecutive_losses_n: int = 3
- cb_loss_threshold_pct: Decimal = Decimal("0.0")
- cb_cooldown_hours: Decimal = Decimal("24.0")
- allow_overrides: bool = True

**RiskCheckResult dataclass:**
- allowed: bool
- adjusted_quantity: Optional[Decimal] = None (None = use original; set = scaled down)
- blocked_reason: Optional[str] = None (None if allowed)

**RiskEngine class:**
Constructor takes `engine: Engine`. Store as `self._engine`. Create logger `logging.getLogger(__name__)`.

`check_order(self, order_qty: Decimal, order_side: str, fill_price: Decimal, asset_id: int, strategy_id: int, current_position_value: Decimal, portfolio_value: Decimal) -> RiskCheckResult`:

Step 1 -- Kill switch check (fast exit):
```python
if self._is_halted():
    self._log_event("position_cap_blocked", "system", "Kill switch active -- all orders blocked",
                    asset_id=asset_id, strategy_id=strategy_id)
    return RiskCheckResult(allowed=False, adjusted_quantity=Decimal("0"), blocked_reason="kill_switch_active")
```

Step 2 -- Circuit breaker check:
```python
if self._is_circuit_breaker_tripped(strategy_id):
    return RiskCheckResult(allowed=False, adjusted_quantity=Decimal("0"), blocked_reason="circuit_breaker_tripped")
```

Step 3 -- Load limits (hot-reload from DB each call):
```python
limits = self._load_limits(asset_id, strategy_id)
```

Step 4 -- Per-asset position cap:
- proposed_value = order_qty * fill_price
- max_position_value = limits.max_position_pct * portfolio_value
- If current_position_value + proposed_value > max_position_value:
  - headroom = max_position_value - current_position_value
  - If headroom <= 0: return blocked (position_cap_exhausted)
  - scaled_qty = headroom / fill_price
  - Log the scaling event to cmc_risk_events
  - return RiskCheckResult(allowed=True, adjusted_quantity=scaled_qty, blocked_reason=None)

Step 5 -- Portfolio utilization cap:
- total_invested = self._get_total_invested_value()
- max_invested = limits.max_portfolio_pct * portfolio_value
- If total_invested + proposed_value > max_invested:
  - headroom = max_invested - total_invested
  - If headroom <= 0: return blocked (portfolio_cap_exhausted)
  - scaled_qty = min(order_qty, headroom / fill_price)
  - Log the scaling event
  - return RiskCheckResult(allowed=True, adjusted_quantity=scaled_qty, blocked_reason=None)

Step 6 -- All checks pass:
return RiskCheckResult(allowed=True)

`check_daily_loss(self) -> bool`:
- Read dim_risk_state: day_open_portfolio_value, last_day_open_date
- Read limits: daily_loss_pct_threshold
- Compute current_value = self._compute_portfolio_value()
- Reset day-open if last_day_open_date < today (UTC): update dim_risk_state with current portfolio value as new day_open
- If day_open_portfolio_value is None or 0: return False (no baseline yet)
- dd_pct = (current_value - day_open_portfolio_value) / day_open_portfolio_value
- If dd_pct < -daily_loss_pct_threshold:
  - Import and call activate_kill_switch(self._engine, reason=f"Daily loss {dd_pct:.2%} exceeded threshold {-daily_loss_pct_threshold:.2%}", trigger_source="daily_loss_stop")
  - return True
- return False

`update_circuit_breaker(self, strategy_id: int, realized_pnl: Decimal) -> bool`:
- Load limits for strategy
- Read current cb_consecutive_losses JSON from dim_risk_state
- Parse JSON: losses_by_strategy = json.loads(cb_consecutive_losses_text)
- current_count = losses_by_strategy.get(str(strategy_id), 0)
- If realized_pnl < -limits.cb_loss_threshold_pct:
  - Increment count
  - If count >= limits.cb_consecutive_losses_n:
    - Write tripped_at timestamp to cb_breaker_tripped_at JSON
    - Log circuit_breaker_tripped event to cmc_risk_events
    - Send Telegram alert via send_critical_alert
    - Return True (tripped)
- Else (profit or break-even):
  - Reset count to 0
- Update dim_risk_state cb_consecutive_losses JSON
- Return False

`reset_circuit_breaker(self, strategy_id: int, reason: str, operator: str) -> None`:
- Reset the count for strategy_id in cb_consecutive_losses
- Clear the tripped_at for strategy_id in cb_breaker_tripped_at
- Log circuit_breaker_reset event to cmc_risk_events

Private helpers:
- `_is_halted(self) -> bool`: SELECT trading_state FROM dim_risk_state WHERE state_id=1
- `_is_circuit_breaker_tripped(self, strategy_id: int) -> bool`: Check cb_breaker_tripped_at JSON for strategy. If tripped, check if cooldown has expired (cb_cooldown_hours). If expired, auto-reset and return False.
- `_load_limits(self, asset_id: int, strategy_id: int) -> RiskLimits`: Query dim_risk_limits with specificity ordering (specific asset+strategy > specific asset > default). Use ORDER BY asset_id NULLS LAST, strategy_id NULLS LAST LIMIT 1.
- `_compute_portfolio_value(self) -> Decimal`: SELECT SUM(ABS(quantity) * COALESCE(last_mark_price, avg_cost_basis)) FROM cmc_positions WHERE exchange != 'aggregate'. Return 0 if no rows.
- `_get_total_invested_value(self) -> Decimal`: Same as _compute_portfolio_value (alias for clarity).
- `_log_event(self, event_type, trigger_source, reason, **kwargs)`: INSERT into cmc_risk_events. Accept optional asset_id, strategy_id, order_id, override_id, metadata kwargs.

CRITICAL: All DB operations use `with self._engine.connect() as conn` for reads and `with self._engine.begin() as conn` for writes. State updates (dim_risk_state) use engine.begin().

CRITICAL: Use `from ta_lab2.notifications.telegram import send_critical_alert` with a try/except ImportError guard -- Telegram may not be configured in all environments. Log warning if import fails.

**KillSwitchStatus dataclass:**
- trading_state: str
- halted_at: Optional[datetime]
- halted_reason: Optional[str]
- halted_by: Optional[str]

**activate_kill_switch(engine: Engine, reason: str, trigger_source: str) -> None:**
All in one engine.begin() transaction:
1. UPDATE dim_risk_state SET trading_state='halted', halted_at=now(), halted_reason=:reason, halted_by=:trigger_source, updated_at=now()
2. UPDATE cmc_orders SET status='cancelled', updated_at=now() WHERE status IN ('created', 'submitted')
3. INSERT into cmc_risk_events (event_type='kill_switch_activated', trigger_source, reason)

After transaction (fire and forget with try/except):
4. Try to import and call send_critical_alert("kill_switch", f"Kill switch activated: {reason}", {"trigger_source": trigger_source})
5. On ImportError or any exception: log warning but do not re-raise

**re_enable_trading(engine: Engine, reason: str, operator: str) -> None:**
Within engine.begin():
1. Verify current state is 'halted'. If already 'active', log warning and return.
2. UPDATE dim_risk_state SET trading_state='active', halted_at=NULL, halted_reason=NULL, halted_by=NULL, updated_at=now()
3. INSERT into cmc_risk_events (event_type='kill_switch_disabled', trigger_source='manual', reason, operator)

**get_kill_switch_status(engine: Engine) -> KillSwitchStatus:**
SELECT trading_state, halted_at, halted_reason, halted_by FROM dim_risk_state WHERE state_id=1
Return KillSwitchStatus dataclass.

**print_kill_switch_status(engine: Engine) -> None:**
Pretty-print status to stdout:
```
Risk State
----------
Trading:  ACTIVE  (or HALTED)
Halted at: (timestamp or N/A)
Reason:   (reason or N/A)
Halted by: (trigger or N/A)
```
