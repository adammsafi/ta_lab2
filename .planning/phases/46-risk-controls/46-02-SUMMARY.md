## Decisions Made

| Decision | Rationale |
|----------|-----------|
| `_is_circuit_breaker_tripped` calls `_load_limits` first | Needs cooldown_hours value before it can check elapsed time; test mocks must follow this order |
| Sell orders skip all cap gates | Reducing exposure is always safe regardless of cap state; only buy orders can increase risk |
| `re_enable_trading` validates non-empty reason+operator | Extra guard so accidental empty-string calls don't silently succeed |
| OverrideManager included in package | Natural companion to kill switch; completes the discretionary override workflow from cmc_risk_overrides |
| Hardcoded RiskLimits defaults as fallback | Ensures risk engine degrades safely if DB has no limit row during initial setup |
