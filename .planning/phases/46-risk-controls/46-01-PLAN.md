Docstring (ASCII only -- no box-drawing or em-dashes):
```
"""risk_controls

Schema changes for Phase 46 (Risk Controls):

1. dim_risk_limits: Runtime-editable config for position caps, daily loss, circuit breaker.
2. dim_risk_state: Single-row live state (kill switch, daily loss tracking, circuit breaker counters).
3. cmc_risk_events: Immutable audit log for all risk events.
4. cmc_risk_overrides: Discretionary override store with sticky/non-sticky support.
"""
```

In `upgrade()`, create tables in this order:

**1. dim_risk_limits** (config table):
- limit_id SERIAL PRIMARY KEY
- created_at TIMESTAMPTZ NOT NULL DEFAULT now()
- updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
- asset_id INTEGER NULL (NULL = applies to all assets)
- strategy_id INTEGER NULL (NULL = applies to all strategies)
- max_position_pct NUMERIC NOT NULL DEFAULT 0.15
- max_portfolio_pct NUMERIC NOT NULL DEFAULT 0.80
- daily_loss_pct_threshold NUMERIC NOT NULL DEFAULT 0.03
- cb_consecutive_losses_n INTEGER NOT NULL DEFAULT 3
- cb_loss_threshold_pct NUMERIC NOT NULL DEFAULT 0.0
- cb_cooldown_hours NUMERIC NOT NULL DEFAULT 24.0
- allow_overrides BOOLEAN NOT NULL DEFAULT TRUE
- CHECK constraints:
  - chk_risk_limits_max_pos: max_position_pct > 0 AND max_position_pct <= 1
  - chk_risk_limits_max_port: max_portfolio_pct > 0 AND max_portfolio_pct <= 1
  - chk_risk_limits_daily: daily_loss_pct_threshold > 0 AND daily_loss_pct_threshold <= 1
  - chk_risk_limits_n: cb_consecutive_losses_n >= 1
  - chk_risk_limits_cooldown: cb_cooldown_hours >= 0
- COMMENT ON TABLE: 'Runtime-editable risk limit configuration. NULL asset_id/strategy_id = portfolio-wide defaults.'

Seed a default row after table creation:
```sql
INSERT INTO public.dim_risk_limits (asset_id, strategy_id)
VALUES (NULL, NULL)
```
This creates one portfolio-wide default row with all the DEFAULT values (15% cap, 3% daily loss, N=3 breaker).

**2. dim_risk_state** (single-row live state):
- state_id INTEGER PRIMARY KEY DEFAULT 1
- trading_state TEXT NOT NULL DEFAULT 'active'
- halted_at TIMESTAMPTZ NULL
- halted_reason TEXT NULL
- halted_by TEXT NULL
- updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
- day_open_portfolio_value NUMERIC NULL
- last_day_open_date DATE NULL
- cb_consecutive_losses TEXT NOT NULL DEFAULT '{}'
- cb_breaker_tripped_at TEXT NOT NULL DEFAULT '{}'
- cb_portfolio_consecutive_losses INTEGER NOT NULL DEFAULT 0
- cb_portfolio_breaker_tripped_at TIMESTAMPTZ NULL
- CHECK constraints:
  - chk_risk_state_trading: trading_state IN ('active', 'halted')
  - chk_risk_state_single: state_id = 1
- COMMENT ON TABLE: 'Single-row live state table for kill switch and circuit breaker. Enforced single-row via CHECK(state_id=1).'

Seed the single row:
```sql
INSERT INTO public.dim_risk_state (state_id) VALUES (1)
ON CONFLICT (state_id) DO NOTHING
```

**3. cmc_risk_events** (immutable audit log):
- event_id UUID PRIMARY KEY DEFAULT gen_random_uuid()
- event_ts TIMESTAMPTZ NOT NULL DEFAULT now()
- event_type TEXT NOT NULL
- trigger_source TEXT NOT NULL
- reason TEXT NOT NULL
- operator TEXT NULL
- asset_id INTEGER NULL
- strategy_id INTEGER NULL
- order_id UUID NULL
- override_id UUID NULL
- metadata TEXT NULL
- CHECK constraints:
  - chk_risk_events_type: event_type IN ('kill_switch_activated', 'kill_switch_disabled', 'position_cap_scaled', 'position_cap_blocked', 'daily_loss_stop_triggered', 'circuit_breaker_tripped', 'circuit_breaker_reset', 'override_created', 'override_applied', 'override_reverted')
  - chk_risk_events_source: trigger_source IN ('manual', 'daily_loss_stop', 'circuit_breaker', 'system')
- Indexes:
  - idx_risk_events_ts: (event_ts DESC)
  - idx_risk_events_type: (event_type, event_ts DESC)
  - idx_risk_events_asset: (asset_id, event_ts DESC) WHERE asset_id IS NOT NULL (partial index)
- COMMENT ON TABLE: 'Immutable audit log for all risk control events. Never DELETE rows.'

**4. cmc_risk_overrides** (discretionary override store):
- override_id UUID PRIMARY KEY DEFAULT gen_random_uuid()
- created_at TIMESTAMPTZ NOT NULL DEFAULT now()
- asset_id INTEGER NOT NULL
- strategy_id INTEGER NOT NULL
- operator TEXT NOT NULL
- reason TEXT NOT NULL
- system_signal TEXT NOT NULL
- override_action TEXT NOT NULL
- sticky BOOLEAN NOT NULL DEFAULT FALSE
- applied_at TIMESTAMPTZ NULL
- reverted_at TIMESTAMPTZ NULL
- revert_reason TEXT NULL
- Indexes:
  - idx_overrides_active: (asset_id, strategy_id) WHERE reverted_at IS NULL (partial index)
- COMMENT ON TABLE: 'Discretionary position overrides with sticky/non-sticky support. Full audit trail.'

NOTE: The override_action column uses free-text validation at the application layer (not a CHECK constraint) per RESEARCH.md recommendation -- this allows adding new action types without migrations.

In `downgrade()`, reverse in order:
1. DROP TABLE IF EXISTS cmc_risk_overrides CASCADE
2. DROP TABLE IF EXISTS cmc_risk_events CASCADE
3. DROP TABLE IF EXISTS dim_risk_state CASCADE
4. DROP TABLE IF EXISTS dim_risk_limits CASCADE
