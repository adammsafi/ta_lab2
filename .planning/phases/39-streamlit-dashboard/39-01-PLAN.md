Create the two query modules. Every function uses `@st.cache_data(ttl=300)` and accepts `_engine` (underscore-prefixed) as first arg.

**pipeline.py** -- functions for Pipeline Monitor page:

1. `load_table_freshness(_engine) -> pd.DataFrame`:
   - Query `asset_data_coverage` grouped by `source_table`: `SELECT source_table, COUNT(DISTINCT id) as n_assets, MAX(last_ts) as latest_data_ts, MAX(updated_at) as last_refresh FROM public.asset_data_coverage GROUP BY source_table ORDER BY source_table`
   - Add `staleness_hours` column: `(pd.Timestamp.utcnow() - pd.to_datetime(df["last_refresh"], utc=True)).dt.total_seconds() / 3600`
   - Return DataFrame with columns: source_table, n_assets, latest_data_ts, last_refresh, staleness_hours

2. `load_stats_status(_engine) -> dict[str, dict[str, int]]`:
   - Stats tables list: `["price_bars_multi_tf_stats", "ema_multi_tf_stats", "ema_multi_tf_cal_stats", "ema_multi_tf_cal_anchor_stats", "returns_ema_stats", "cmc_features_stats"]`
   - For each table, query: `SELECT status, COUNT(*) AS n FROM public.{table} WHERE checked_at >= NOW() - INTERVAL '24 hours' GROUP BY status`
   - IMPORTANT: table names come from a hardcoded allowlist, NOT user input -- safe from injection
   - Return dict mapping table_name -> {status: count}
   - Wrap each query in try/except; on failure set empty dict for that table

3. `load_asset_coverage(_engine) -> pd.DataFrame`:
   - Query: `SELECT a.symbol, c.id, c.source_table, c.granularity, c.n_rows, c.last_ts, c.updated_at FROM public.asset_data_coverage c JOIN public.dim_assets a ON a.id = c.id ORDER BY a.symbol, c.source_table`
   - Apply `pd.to_datetime(df["last_ts"], utc=True)` and same for `updated_at`
   - Add staleness_hours column
   - Return DataFrame

4. `load_alert_history(_engine, days: int = 7) -> pd.DataFrame`:
   - Query all stats tables for recent FAIL/WARN rows: for each stats table, `SELECT '{table}' as stats_table, status, checked_at, check_name FROM public.{table} WHERE status IN ('FAIL', 'WARN') AND checked_at >= NOW() - INTERVAL :d ORDER BY checked_at DESC LIMIT 50`
   - Use same allowlist as load_stats_status
   - Union all results into a single DataFrame sorted by checked_at DESC
   - If check_name column doesn't exist in all tables, use try/except and fall back to NULL
   - Return DataFrame

**research.py** -- functions for Research Explorer page:

1. `load_asset_list(_engine) -> pd.DataFrame`:
   - Query: `SELECT id, symbol FROM public.dim_assets ORDER BY symbol`
   - TTL=3600 (assets change rarely)

2. `load_tf_list(_engine) -> list[str]`:
   - Query: `SELECT tf FROM public.dim_timeframe ORDER BY tf_days_nominal`
   - CRITICAL: use `tf_days_nominal` NOT `tf_days` for ordering
   - TTL=3600

3. `load_ic_results(_engine, asset_id: int, tf: str) -> pd.DataFrame`:
   - Query: `SELECT feature, horizon, return_type, regime_col, regime_label, ic, ic_p_value, ic_t_stat, ic_ir, turnover, n_obs, computed_at FROM public.cmc_ic_results WHERE asset_id = :id AND tf = :tf ORDER BY feature, horizon, return_type`
   - Apply `pd.to_datetime(df["computed_at"], utc=True)` if column present

4. `load_feature_names(_engine, asset_id: int, tf: str) -> list[str]`:
   - Query: `SELECT DISTINCT feature FROM public.cmc_ic_results WHERE asset_id = :id AND tf = :tf ORDER BY feature`
   - TTL=600

5. `load_regimes(_engine, asset_id: int, tf: str) -> pd.DataFrame`:
   - Query: `SELECT ts, l2_label, split_part(l2_label, '-', 1) as trend_state, split_part(l2_label, '-', 2) as vol_state FROM public.cmc_regimes WHERE id = :id AND tf = :tf ORDER BY ts`
   - CRITICAL: derive trend_state and vol_state via split_part -- cmc_regimes has NO such columns
   - Apply `pd.to_datetime(df["ts"], utc=True)`

6. `load_close_prices(_engine, asset_id: int, tf: str) -> pd.Series`:
   - Query: `SELECT ts, close FROM public.cmc_features WHERE id = :id AND tf = :tf ORDER BY ts`
   - Apply `pd.to_datetime(df["ts"], utc=True)`, set_index("ts"), return df["close"]

All query functions must:
- Use `from sqlalchemy import text` for all SQL
- Apply `pd.to_datetime(..., utc=True)` to all timestamp columns after read_sql
- Handle empty result sets gracefully (return empty DataFrame/list)

2. Shared sidebar (runs on every rerun):
   ```python
   with st.sidebar:
       st.title("ta_lab2")
       st.caption("Pipeline Monitor + Research Explorer")
       st.divider()
       ttl_display = st.slider("Cache TTL (s)", min_value=30, max_value=3600, value=300, step=30, key="cache_ttl_display")
       if st.button("Refresh Now", type="primary"):
           st.cache_data.clear()
           st.rerun()
       st.divider()
       st.caption("Cache TTL: 300s (fixed). Use Refresh to clear.")
   ```
