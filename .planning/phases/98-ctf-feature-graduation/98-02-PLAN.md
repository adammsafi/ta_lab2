<objective>
Build a script that evaluates CTF features on a per-asset basis and writes asset-specific tier rows to `dim_feature_selection_asset`. Features passing IC > 0.02 for a specific asset but failing cross-asset consensus get `tier='asset_specific'`.

Purpose: Different assets respond to different CTF indicators. BTC might benefit from `macd_12_26_30d_slope` while ETH does not. This provides per-asset feature customization for downstream consumers (BL optimizer, ML models).

Output: One CLI script that populates dim_feature_selection_asset with per-asset IC winners.
</objective>

  <action>
Create `run_ctf_asset_selection.py` as a CLI script that populates `dim_feature_selection_asset` with per-asset CTF feature selections.

**Algorithm:**

1. **Load global promoted features** from 98-01's output: query `ic_results` for cross-asset median IC > 0.02 CTF features (same query as refresh_ctf_promoted.py). Store as `global_features: set[str]`.

2. **For each tier-1 asset** (all distinct `asset_id` in `ic_results` where feature matches CTF pattern):
   a. Query per-asset IC: for this specific asset_id, find CTF features where `ABS(ic) > 0.02` (not median — single asset, single IC value). Use horizon=1, return_type='arith', regime_col='all'.
   b. Compute `asset_specific_features = per_asset_passing - global_features` (features that pass per-asset but NOT in global set).
   c. Also include `global_features` that pass for this asset (for completeness — superset relationship).
   d. For each asset-specific feature, prepare a row:
      ```python
      {
          "feature_name": feature,
          "asset_id": asset_id,
          "tier": "asset_specific",
          "ic_ir_mean": per_asset_ic_value,
          "pass_rate": None,  # single-asset, not applicable
          "stationarity": None,
          "selected_at": datetime.now(timezone.utc),
          "yaml_version": None,
          "rationale": f"Per-asset IC={ic:.4f} > 0.02 for asset {asset_id}; not in global tier"
      }
      ```

3. **Upsert to dim_feature_selection_asset** using ON CONFLICT (feature_name, asset_id) DO UPDATE:
   ```python
   INSERT INTO public.dim_feature_selection_asset
       (feature_name, asset_id, tier, ic_ir_mean, pass_rate, stationarity, selected_at, rationale)
   VALUES (:feature_name, :asset_id, :tier, :ic_ir_mean, :pass_rate, :stationarity, :selected_at, :rationale)
   ON CONFLICT (feature_name, asset_id) DO UPDATE SET
       tier = EXCLUDED.tier,
       ic_ir_mean = EXCLUDED.ic_ir_mean,
       selected_at = EXCLUDED.selected_at,
       rationale = EXCLUDED.rationale
   ```

4. **Log summary**: "Asset {asset_id}: {N} global features, {M} asset-specific additions, {N+M} total"

**CRITICAL CONSTRAINTS:**
- DO NOT touch `dim_feature_selection` table. Only write to `dim_feature_selection_asset`.
- DO NOT call `save_to_db()` from `feature_selection.py` (it TRUNCATEs dim_feature_selection).
- The "superset" relationship means: for any asset, the effective feature set = global tier (from dim_feature_selection) UNION asset-specific (from dim_feature_selection_asset).
- Empty is fine: if no CTF features pass per-asset IC for a given asset, write zero rows for that asset.

**CLI interface:**
```python
if __name__ == "__main__":
    import argparse
    parser = argparse.ArgumentParser(description="Populate asset-specific CTF feature selection")
    parser.add_argument("--dry-run", action="store_true", help="Show per-asset selections without writing")
    parser.add_argument("--asset-id", type=int, default=None, help="Process single asset")
    parser.add_argument("--ic-threshold", type=float, default=0.02)
    args = parser.parse_args()
```
