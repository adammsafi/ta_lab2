**Cell 19 (markdown): Section -- The Promotion Gate**
- Explain BH correction: When testing multiple hypotheses (multiple horizons, return types), the chance of false positives increases. BH correction controls the false discovery rate.
- "A feature is eligible for promotion if at least one (horizon, return_type) combination has BH-corrected p-value < 0.05."
- "Promotion adds the feature to `dim_feature_registry` with lifecycle='promoted' and generates an Alembic migration stub for adding the column to `cmc_features`."

**Cell 20 (code): Check promotion eligibility**
```python
if target_feature and result_df is not None and len(result_df) > 0:
    # Check if any BH-corrected p-value passes
    bh_col = "ic_p_value_bh" if "ic_p_value_bh" in result_df.columns else "ic_p_value"
    passing = result_df[result_df[bh_col] < 0.05]
    print(f"BH-corrected p < 0.05: {len(passing)} / {len(result_df)} combinations")

    if len(passing) > 0:
        print(f"\nPassing combinations:")
        display(passing[["feature_name", "horizon", "return_type", "ic", bh_col]].round(4))
        print(f"\nThis feature WOULD pass the promotion gate.")
    else:
        print(f"\nNo combinations pass BH correction at alpha=0.05.")
        print(f"This feature would NOT pass the promotion gate -- expected for noise or weak signals.")
else:
    print("No experiment results to evaluate.")
```

```python
all_results = []
for feat_name in experimental:
    print(f"Running: {feat_name}...", end=" ")
    try:
        res = runner.run(
            feat_name,
            asset_ids=[ASSET_ID],
            tf=TF,
            train_start=pd.Timestamp(START_DATE, tz="UTC"),
            train_end=pd.Timestamp(END_DATE, tz="UTC"),
            horizons=HORIZONS,
            dry_run=DRY_RUN,
        )
        all_results.append(res)
        print(f"OK ({len(res)} results)")

        print(f"FAILED: {e}")

if all_results:

```
