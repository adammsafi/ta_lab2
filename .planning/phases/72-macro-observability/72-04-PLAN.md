method that writes all attr_* columns to cmc_drift_metrics, and wire it through
run_drift_report.py (OBSV-04).

Purpose: Operators need to see when drift between paper and backtest is caused by differing
macro regime conditions. If the paper period ran during "adverse" but the backtest training
period was "favorable", this explains performance divergence.

the result without persisting it. This plan fixes both gaps: adds macro_regime_delta AND
creates the missing DB write path for ALL attr_* columns.

```python
def _compute_macro_regime_delta(
    self,
    paper_start: str,
    paper_end: str,
    step2_pnl: float,
) -> tuple[float, dict]:
    """
    Compute macro regime attribution delta.

    Compares the dominant macro_state during the paper period vs the
    full backtest training period. If they differ, the delta represents
    the estimated drift contribution from macro environment changes.

    Returns (delta, details_dict) where details_dict has:
      paper_dominant_state, backtest_dominant_state, paper_state_pct, backtest_state_pct
    """
```

Implementation:
- Query cmc_macro_regimes for the paper period:
  `SELECT macro_state, COUNT(*) AS n FROM cmc_macro_regimes WHERE date BETWEEN :start AND :end AND profile = 'default' GROUP BY macro_state ORDER BY n DESC LIMIT 1`
- Query for the full backtest period (assume backtest period = 1 year before paper_start):
  `SELECT macro_state, COUNT(*) AS n FROM cmc_macro_regimes WHERE date BETWEEN :bt_start AND :bt_end AND profile = 'default' GROUP BY macro_state ORDER BY n DESC LIMIT 1`
  where bt_start = paper_start - 365 days, bt_end = paper_start - 1 day.
- If either query returns no rows, return (0.0, {}) -- no macro data available yet.
- If both return the same dominant state, return (0.0, details) -- no macro divergence.
- If different states: assign a penalty score based on state distance.
  State ordering: favorable(0), constructive(1), neutral(2), cautious(3), adverse(4).
  `state_distance = abs(paper_score - backtest_score)`
  `macro_regime_delta = -state_distance * 0.005 * abs(step2_pnl)` (each state step = ~0.5% of explained PnL)
  This is a heuristic -- captures directional impact without re-running full backtest.
- Return (delta, {paper_dominant_state, backtest_dominant_state, state_distance}).

```python
def persist_attribution(
    self,
    config_id: int,
    asset_id: int,
    metric_date: str,
    result: AttributionResult,
) -> None:
    """
    Write attribution results to the attr_* columns of cmc_drift_metrics.

    Uses UPDATE (not INSERT) because the DriftMetrics row must already exist
    from the daily DriftMonitor run. This is called from
    run_drift_report.py --with-attribution AFTER run_attribution().
    """
    sql = text("""
        UPDATE cmc_drift_metrics
        SET attr_baseline_pnl = :baseline_pnl,
            attr_fee_delta = :fee_delta,
            attr_slippage_delta = :slippage_delta,
            attr_timing_delta = :timing_delta,
            attr_data_revision_delta = :data_revision_delta,
            attr_sizing_delta = :sizing_delta,
            attr_regime_delta = :regime_delta,
            attr_macro_regime_delta = :macro_regime_delta,
            attr_unexplained = :unexplained_residual
        WHERE config_id = :config_id
          AND asset_id = :asset_id
          AND metric_date = :metric_date
    """)
    params = {
        "config_id": config_id,
        "asset_id": asset_id,
        "metric_date": metric_date,
        "baseline_pnl": result.baseline_pnl,
        "fee_delta": result.fee_delta,
        "slippage_delta": result.slippage_delta,
        "timing_delta": result.timing_delta,
        "data_revision_delta": result.data_revision_delta,
        "sizing_delta": result.sizing_delta,
        "regime_delta": result.regime_delta,
        "macro_regime_delta": result.macro_regime_delta,
        "unexplained_residual": result.unexplained_residual,
    }
    with self._engine.begin() as conn:
        row_count = conn.execute(sql, params).rowcount
    if row_count == 0:
        logger.warning(
            "persist_attribution: no matching row for config_id=%d asset_id=%d "
            "metric_date=%s -- run DriftMonitor first to create the base row",
            config_id, asset_id, metric_date,
        )
    else:
        logger.debug(
            "persist_attribution: updated %d row(s) for config_id=%d asset_id=%d date=%s",
            row_count, config_id, asset_id, metric_date,
        )
```
