Pure function that computes updated position fields after a fill event. Uses net-quantity model (NautilusTrader NETTING pattern). All arithmetic in Decimal.

Signature:
```python
def compute_position_update(
    current_qty: Decimal,           # signed: positive=long, negative=short, 0=flat
    current_avg_cost: Decimal,      # per-unit cost basis
    current_realized_pnl: Decimal,  # cumulative realized PnL
    fill_qty: Decimal,              # signed: positive=buy, negative=sell
    fill_price: Decimal,            # execution price for this fill
) -> dict[str, Decimal]:
    """Returns dict with keys: quantity, avg_cost_basis, realized_pnl"""
```

Test cases (input -> expected output):

Case 1 -- New position from flat (buy):
  (0, 0, 0, +1, 50000) -> {qty: 1, cost: 50000, rpnl: 0}

Case 2 -- New position from flat (sell/short):
  (0, 0, 0, -1, 50000) -> {qty: -1, cost: 50000, rpnl: 0}

Case 3 -- Add to long (weighted average):
  (100, 50000, 0, +50, 52000) -> {qty: 150, cost: 50666.666..., rpnl: 0}
  Verify: (100*50000 + 50*52000) / 150 = 7600000/150 = 50666.6666...

Case 4 -- Add to short (weighted average):
  (-100, 50000, 0, -50, 48000) -> {qty: -150, cost: 49333.333..., rpnl: 0}

Case 5 -- Partial close of long (sell some):
  (100, 50000, 0, -30, 52000) -> {qty: 70, cost: 50000, rpnl: 60000}
  Realized: (52000 - 50000) * 30 = 60000

Case 6 -- Full close of long (sell all):
  (100, 50000, 0, -100, 52000) -> {qty: 0, cost: 0, rpnl: 200000}

Case 7 -- Partial close of short (buy some to cover):
  (-100, 50000, 0, +30, 48000) -> {qty: -70, cost: 50000, rpnl: 60000}

Implement using the algorithm from RESEARCH.md Pattern 3 (Position Flip Algorithm):

1. If current_qty == 0: new position -> qty=fill_qty, cost=fill_price, rpnl=current_rpnl
2. If same direction (sign(current_qty) == sign(current_qty + fill_qty)) and new_qty != 0: weighted average cost
3. If reducing or closing: compute realized PnL on closed portion
   - For long: realized = (fill_price - avg_cost) * closed_qty
   - For short: realized = (avg_cost - fill_price) * closed_qty
4. If new_qty == 0: full close -> cost=0
5. If flip (sign changed): open new position at fill_price

Use `from __future__ import annotations` at top for clean type hints.

CRITICAL: All values must be Decimal. The function should NOT import or use float anywhere. Parameters typed as Decimal, return values as Decimal. Use Decimal("0") not 0 for comparisons where needed.
