**Constants:**

- "submitted": ["partial_fill", "filled", "cancelled", "rejected", "expired"]

- "filled": [] (terminal)
- "cancelled": [] (terminal)
- "rejected": [] (terminal)
- "expired": [] (terminal)

**FillData dataclass:**

- fill_qty: Decimal (always positive)

**OrderManager class (all static methods -- stateless):**

`promote_paper_order(engine, paper_order_uuid, environment="sandbox") -> str`:
- Within engine.begin() as conn:
  - SELECT from paper_orders WHERE order_uuid = :uuid. Raise ValueError if not found.
  - Generate order_id = str(uuid.uuid4())
  - INSERT into cmc_orders with status='created', remaining_qty=quantity
  - INSERT 'created' event into cmc_order_events (from_status=NULL, to_status='created', reason='promoted from paper_orders')
- Return order_id

`process_fill(engine, fill_data: FillData) -> str`:
- Wraps _do_process_fill in try/except
- On exception: log error, call _write_dead_letter, re-raise
- Returns fill_id on success

`_do_process_fill(engine, fill_data) -> str`:
Inside single engine.begin() as conn:

Step 1 -- Read order with FOR SHARE lock:
```sql
SELECT order_id, asset_id, exchange, side, quantity,
       filled_qty, remaining_qty, avg_fill_price, status
FROM public.cmc_orders WHERE order_id = :order_id FOR SHARE
```
Raise ValueError if not found. Call _validate_fill_transition.

Step 2 -- Lock position row with FOR UPDATE:
```sql
SELECT quantity, avg_cost_basis, realized_pnl
FROM public.cmc_positions
WHERE asset_id = :asset_id AND exchange = :exchange FOR UPDATE
```
If no row exists, use defaults (qty=0, cost=0, rpnl=0).

Step 3 -- Insert fill:
Generate fill_id = str(uuid.uuid4()). INSERT into cmc_fills with all FillData fields plus side and exchange from the order.

Step 4 -- Update order:
- new_filled_qty = order.filled_qty + fill_data.fill_qty
- new_remaining = order.quantity - new_filled_qty
- Compute weighted avg_fill_price: (old_total + fill_price * fill_qty) / new_filled_qty
  - Handle avg_fill_price being NULL on first fill: treat as 0
- to_status = "filled" if new_remaining <= 0 else "partial_fill"
- UPDATE cmc_orders SET filled_qty, remaining_qty, avg_fill_price, status, updated_at=now()

Step 5 -- Upsert position:
- signed_fill = fill_data.fill_qty if order.side == "buy" else -fill_data.fill_qty
- Call compute_position_update(current_qty, current_cost, current_rpnl, signed_fill, fill_price)
- INSERT INTO cmc_positions ... ON CONFLICT (asset_id, exchange) DO UPDATE SET quantity, avg_cost_basis, realized_pnl, last_fill_id, last_updated

Step 6 -- Insert audit event:
INSERT into cmc_order_events with from_status, to_status, fill_id.

Log success. Return fill_id.

CRITICAL: All Decimal values passed to SQL must be converted with str() -- e.g., `str(fill_data.fill_qty)`. This avoids numpy/Decimal binding issues with psycopg2.

`_validate_fill_transition(from_status, fill_qty, order)`:
- Raise ValueError if from_status is terminal (filled, cancelled, rejected, expired)
- Raise ValueError if fill_qty > remaining_qty

`_write_dead_letter(engine, fill_data, exc)`:
- Uses its OWN engine.begin() (separate connection -- CRITICAL: must not share failed transaction)
- INSERT into cmc_order_dead_letter with operation_type='process_fill', order_id, payload=json.dumps({...}), error_reason=str(exc), error_stacktrace=traceback.format_exc()
- Wrap in try/except -- if DLQ write also fails, log at CRITICAL level. Never swallow the original exception.

`update_order_status(engine, order_id, new_status, reason=None)`:
- Within engine.begin():
  - SELECT status FROM cmc_orders WHERE order_id = :id FOR UPDATE
  - Validate transition: new_status must be in VALID_TRANSITIONS[current_status]. Raise ValueError if not.
  - UPDATE cmc_orders SET status, updated_at
  - INSERT cmc_order_events with from_status, to_status, reason
