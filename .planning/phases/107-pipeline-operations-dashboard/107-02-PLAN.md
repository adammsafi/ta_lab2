**A. Create `src/ta_lab2/dashboard/queries/pipeline_ops.py`:**

Three query functions, all accepting `_engine` (underscore prefix so Streamlit skips hashing):

1. `load_active_run_stages(_engine) -> tuple[dict | None, pd.DataFrame]`
   - Query pipeline_run_log WHERE status='running' ORDER BY started_at DESC LIMIT 1
   - If found, query pipeline_stage_log WHERE run_id = that run_id ORDER BY started_at
   - Return (run_row_dict, stages_df) or (None, empty DataFrame)
   - Use `@st.cache_data(ttl=0)` -- must be real-time for active monitoring

2. `load_run_history(_engine, limit: int = 10) -> pd.DataFrame`
   - JOIN pipeline_run_log with pipeline_stage_log, aggregate per run
   - Columns: run_id, started_at, completed_at, status, total_duration_sec, error_message, stage_count, stages_ok, stages_failed
   - WHERE status IN ('complete', 'failed', 'killed') ORDER BY started_at DESC LIMIT :lim
   - Use `@st.cache_data(ttl=60)` -- short cache for history

3. `load_stage_details(_engine, run_id: str) -> pd.DataFrame`
   - Query pipeline_stage_log WHERE run_id = :run_id ORDER BY started_at
   - Columns: stage_name, started_at, completed_at, status, duration_sec, error_message
   - Use `@st.cache_data(ttl=60)`

4. `is_pipeline_running(_engine) -> bool`
   - Query pipeline_run_log WHERE status='running' AND started_at > NOW() - INTERVAL '4 hours'
   - Returns True if any row exists (prevents concurrent launches)
   - Use `@st.cache_data(ttl=0)` -- must be real-time

Follow the exact pattern from `queries/pipeline.py` for imports and structure.

**B. Create `src/ta_lab2/dashboard/pages/19_pipeline_ops.py`:**

Do NOT call `st.set_page_config()` (only in app.py). Structure the page in three sections:

**Section 1: Active Run Monitor (auto-refresh)**
- Wrap in `@st.fragment(run_every=90)` decorator
- Pass `_engine` (underscore prefix) to the fragment
- Call `load_active_run_stages(_engine)`
- If active run exists:
  - Show run_id, started_at, elapsed time
  - Import `STAGE_ORDER` from `ta_lab2.scripts.run_daily_refresh`
  - Render overall progress bar: `st.progress(completed/total, text=f"Pipeline: {completed}/{total} stages")`
  - Render per-stage status list using STAGE_ORDER as the authoritative ordering:
    - complete -> green circle icon
    - running -> orange circle icon
    - failed -> red circle icon
    - pending (not in stages_df) -> white circle icon
  - Kill button: `st.button("Kill Pipeline", type="secondary")` -> `KILL_SWITCH_FILE.touch()` + `st.warning(...)`
- If no active run: show `st.info("No pipeline run in progress.")`

**Section 2: Operations Panel (trigger buttons)**
- Place OUTSIDE the fragment (buttons cannot be inside auto-refresh fragments -- they re-trigger on each cycle)
- "Run Full Refresh" button (type="primary"):
  - Check `is_pipeline_running(_engine)` first
  - If not running: `subprocess.Popen([sys.executable, str(SCRIPT_PATH), "--all", "--ids", "all"], creationflags=subprocess.DETACHED_PROCESS, stdout=open(".pipeline_stdout.log", "w", encoding="utf-8"), stderr=subprocess.STDOUT)`
  - Show `st.success(f"Pipeline started (PID {proc.pid})")`
  - If already running: `st.warning("Pipeline already running.")`
- "Run From Stage" section:
  - `st.selectbox("Start from stage", STAGE_ORDER)` for stage selection
  - "Run From Stage" button: same Popen pattern but adds `--from-stage {selected_stage}` to args
- Quick actions row (3 columns):
  - "Sync VMs Only": launches with `--sync-vms` flag
  - "Bars + EMAs Only": launches with `--bars --emas --ids all` flags
  - "Signals Only": launches with `--signals` flag

Use `SCRIPT_PATH = Path(__file__).parent.parent / "scripts" / "run_daily_refresh.py"` to locate the script. `KILL_SWITCH_FILE = Path(__file__).parent.parent.parent.parent.parent / ".pipeline_kill"` (5 levels up from pages/ to project root). Double-check the path resolves correctly by counting: pages -> dashboard -> ta_lab2 -> src -> ta_lab2 -> project_root. Actually, use the same KILL_SWITCH_FILE path as defined in run_daily_refresh.py by computing project root. The safest approach: `PROJECT_ROOT = Path(__file__).resolve().parent.parent.parent.parent.parent` then `KILL_SWITCH_FILE = PROJECT_ROOT / ".pipeline_kill"`.

On Windows, subprocess.DETACHED_PROCESS = 0x00000008. Guard with `import platform; creationflags = subprocess.DETACHED_PROCESS if platform.system() == "Windows" else 0`.

**Section 3: Run History**
- Call `load_run_history(_engine)`
- Show a table with columns: started_at, status, total_duration_sec (formatted as mm:ss), stage_count, stages_ok, stages_failed
- For each run row, add an expander that calls `load_stage_details(_engine, run_id)` and shows per-stage timing as a mini table
- Color-code status column (complete=green, failed=red, killed=orange)
