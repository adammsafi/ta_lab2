  Create `src/ta_lab2/analysis/feature_selection.py` with the following public functions. Follow the docstring and typing conventions from `ic.py` (type hints, numpy-style docstrings, logger at module level).

  **1. `test_stationarity(series: pd.Series, alpha: float = 0.05) -> dict`**
  - Run ADF test via `statsmodels.tsa.stattools.adfuller(series_clean, autolag='AIC')`.
  - Run KPSS test via `statsmodels.tsa.stattools.kpss(series_clean, regression='c', nlags='auto')`.
  - CRITICAL: ADF null = unit root (non-stationary), low p-value means stationary. KPSS null = stationary, low p-value means non-stationary. These are OPPOSING nulls.
  - Decision rule: both agree stationary -> "STATIONARY", both agree non-stationary -> "NON_STATIONARY", disagree -> "AMBIGUOUS", fewer than 30 non-NaN observations -> "INSUFFICIENT_DATA".
  - Return dict with keys: `adf_stat`, `adf_pvalue`, `kpss_stat`, `kpss_pvalue`, `result` (one of the four string values above).
  - Wrap KPSS in a `warnings.catch_warnings()` context to suppress the InterpolationWarning that KPSS emits when p-value is outside its table bounds.

  **2. `test_ljungbox_on_ic(rolling_ic_series: pd.Series, lags: int = 10, alpha: float = 0.05) -> dict`**
  - Clean NaN from the series. If fewer than `lags + 1` non-NaN observations, return `{"flag": False, "min_pvalue": None, "n_obs": len(clean)}`.
  - Run `statsmodels.stats.diagnostic.acorr_ljungbox(clean, lags=lags, return_df=True)`.
  - Flag = True if ANY lag's `lb_pvalue < alpha`. This means significant autocorrelation in the IC series, suggesting the IC-IR may be inflated by serial correlation.
  - Return dict: `flag` (bool), `min_pvalue` (float, minimum p-value across lags), `n_obs` (int).

  **3. `compute_monotonicity_score(quintile_cumulative: pd.DataFrame) -> float`**
  - Input: the cumulative returns DataFrame from `compute_quintile_returns()` — columns are quintile labels [1,2,3,4,5], index is timestamps.
  - Take the TERMINAL (last row) cumulative return per quintile: `terminal = quintile_cumulative.iloc[-1]`.
  - Compute Spearman rank correlation between `[1,2,3,4,5]` and the terminal returns: `spearmanr([1,2,3,4,5], terminal.values)`.
  - Return the Spearman rho value (float in [-1, 1]). A value near +1 means perfectly monotonic increasing (Q5 > Q4 > ... > Q1). A value near -1 means perfectly monotonic decreasing (useful for mean-reversion factors).
  - Use `abs()` since both +1 and -1 indicate monotonic predictive power.
  - If the quintile DataFrame has fewer than 2 rows, return 0.0.

  **4. `load_ic_ranking(engine, horizon: int = 1, return_type: str = 'arith') -> pd.DataFrame`**
  - Query ic_results to produce the feature ranking used for selection. SQL pattern from RESEARCH.md:
    ```sql
    SELECT feature,
           AVG(ABS(ic)) AS mean_abs_ic,
           AVG(ABS(ic_ir)) AS mean_abs_ic_ir,
           COUNT(*) AS n_observations,
           COUNT(DISTINCT asset_id || '_' || tf) AS n_asset_tf_pairs,
           SUM(CASE WHEN ABS(ic_ir) > 0.3 THEN 1 ELSE 0 END)::FLOAT / NULLIF(COUNT(*), 0) AS pass_rate
    FROM public.ic_results
    WHERE horizon = :horizon AND return_type = :return_type
      AND regime_col = 'all' AND regime_label = 'all' AND ic IS NOT NULL
    GROUP BY feature
    ORDER BY mean_abs_ic_ir DESC NULLS LAST
    ```
  - Return a DataFrame with columns: feature, mean_abs_ic, mean_abs_ic_ir, n_observations, n_asset_tf_pairs, pass_rate.

  **5. `load_regime_ic(engine, feature: str, horizon: int = 1, return_type: str = 'arith') -> pd.DataFrame`**
  - Query ic_results for regime-specific IC-IR for a single feature.
    ```sql
    SELECT regime_col, regime_label, AVG(ABS(ic_ir)) AS mean_abs_ic_ir, COUNT(*) AS n_obs
    FROM public.ic_results
    WHERE feature = :feature AND horizon = :horizon AND return_type = :return_type
      AND regime_col != 'all' AND ic IS NOT NULL
    GROUP BY regime_col, regime_label
    ORDER BY mean_abs_ic_ir DESC
    ```
  - Return DataFrame with columns: regime_col, regime_label, mean_abs_ic_ir, n_obs.

  **6. `classify_feature_tier(ic_ir_mean: float, pass_rate: float, stationarity: str, regime_ic: pd.DataFrame | None = None, ic_ir_cutoff: float = 0.3) -> str`**
  - Tier classification logic (from CONTEXT.md decisions + RESEARCH.md recommendations):
    - **"active"**: `ic_ir_mean >= ic_ir_cutoff` AND (`stationarity != 'NON_STATIONARY'` OR `ic_ir_mean >= ic_ir_cutoff * 1.5`) AND `pass_rate >= 0.30`
    - **"conditional"**: NOT active, but `regime_ic is not None` and at least one regime has `mean_abs_ic_ir >= ic_ir_cutoff` (regime specialist). Also: `ic_ir_mean` between 0.15 and cutoff with pass_rate >= 0.20.
    - **"watch"**: `ic_ir_mean` between 0.10 and 0.30. Some signal but not strong enough for active.
    - **"archive"**: Everything else (ic_ir_mean < 0.10 across all contexts).
  - Return one of: "active", "conditional", "watch", "archive".

  **7. `build_feature_selection_config(ranking_df: pd.DataFrame, stationarity_results: dict[str, dict], ljungbox_results: dict[str, dict], monotonicity_scores: dict[str, float], regime_ic_map: dict[str, pd.DataFrame], ic_ir_cutoff: float = 0.3) -> dict`**
  - Takes the IC ranking DataFrame (from `load_ic_ranking`), stationarity test results keyed by feature name, Ljung-Box results keyed by feature name, monotonicity scores keyed by feature name, and regime IC DataFrames keyed by feature name.
  - For each feature in ranking_df, calls `classify_feature_tier()` to assign a tier.
  - Builds and returns a nested dict suitable for YAML serialization:
