<task type="auto">
  <name>Task 1: Create run_feature_selection.py CLI orchestrator</name>
  <files>src/ta_lab2/scripts/analysis/run_feature_selection.py</files>
  <action>
  Create `src/ta_lab2/scripts/analysis/run_feature_selection.py` — the main CLI script for feature selection.

  **CLI arguments (argparse):**
  - `--horizon` (int, default=1): IC horizon for ranking.
  - `--return-type` (str, default='arith'): Return type for IC ranking.
  - `--ic-ir-cutoff` (float, default=0.3): IC-IR threshold for active tier.
  - `--tf` (str, default='1D'): Timeframe for stationarity tests and quintile analysis.
  - `--output` (str, default='configs/feature_selection.yaml'): YAML output path.
  - `--skip-quintile` (flag): Skip quintile monotonicity computation (slow step).
  - `--skip-stationarity` (flag): Skip ADF/KPSS stationarity tests.
  - `--skip-ljungbox` (flag): Skip Ljung-Box autocorrelation test.
  - `--top-n` (int, default=50): Only run stationarity + Ljung-Box + quintile on top-N features by IC-IR (performance optimization — no point testing archive-tier features).
  - `--dry-run` (flag): Print ranking and tier assignments without writing YAML or DB.

  **Pipeline steps (in order):**

  Step 1 — Load IC ranking:
  ```python
  ranking_df = load_ic_ranking(engine, horizon=args.horizon, return_type=args.return_type)
  ```
  Log total feature count and top-10 by IC-IR.

  Step 2 — Run stationarity tests (unless --skip-stationarity):
  For each of the top-N features, load the feature series from the `features` table for the given `--tf` timeframe. Use `load_feature_series()` from ic.py to get the series for a representative asset (pick the asset with the most data for that feature). Run `test_stationarity()` on it. Store results in a dict keyed by feature name.

  IMPORTANT: For performance, test stationarity on a SINGLE representative asset per feature (the one with the most rows), NOT all assets. Per-asset stationarity results are stored in ic_results already; this is the aggregate classification for the feature overall.

  Step 3 — Run Ljung-Box on rolling IC series (unless --skip-ljungbox):
  For each of the top-N features, compute rolling IC via `compute_rolling_ic()` from ic.py for the same representative asset. Apply `test_ljungbox_on_ic()` on the resulting series. Store results.

  CRITICAL: Ljung-Box must be on the ROLLING IC SERIES, not the raw feature values. Use `compute_rolling_ic(feature_series, forward_returns_series, window=63)`.

  Step 4 — Load regime-conditional IC:
  For each of the top-N features, call `load_regime_ic()` to get regime-specific IC-IR. Store in a dict keyed by feature name.

  Step 5 — Run quintile monotonicity (unless --skip-quintile):
  For each of the top-N features, load ALL assets for the given TF from the features table, run `compute_quintile_returns()`, then `compute_monotonicity_score()`. This is the slowest step — log progress. Store scores.

  Step 6 — Build config:
  ```python
  config = build_feature_selection_config(
      ranking_df, stationarity_results, ljungbox_results,
      monotonicity_scores, regime_ic_map, ic_ir_cutoff=args.ic_ir_cutoff
  )
  ```

  Step 7 — Write outputs (unless --dry-run):
  ```python
  yaml_path = save_to_yaml(config, args.output)
  n_rows = save_to_db(engine, config, yaml_version="80-initial")
  ```

  Step 8 — Print summary:
  Print a table showing: tier, count, example features. Print totals. Print path to YAML.

  **DB connection pattern:**
  ```python
  from ta_lab2.scripts.refresh_utils import resolve_db_url
  engine = create_engine(resolve_db_url(), poolclass=NullPool)
  ```

  **Logging:** Use standard logging pattern (`logging.basicConfig(level=logging.INFO, format=...)`). Log each step start/end with timing.

  **Error handling:** If a stationarity test fails for a specific feature (e.g., singular matrix), catch the exception, log a warning, and mark it as INSUFFICIENT_DATA. Do NOT let one feature crash the whole pipeline.
  </action>
  <verify>

  Run: `python -m ta_lab2.scripts.analysis.run_feature_selection --help` — should print all CLI arguments.
  </verify>
  <done>run_feature_selection.py executes the full pipeline (load IC ranking, stationarity tests, Ljung-Box, regime IC, quintile monotonicity, tier classification) and prints a dry-run summary showing features classified into tiers.</done>
