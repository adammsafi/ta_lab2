<objective>
Create the concordance script that compares IC-IR rankings with MDA feature importance rankings to cross-validate the feature selection.

Purpose: IC-IR and MDA measure predictive power through different lenses (rank correlation vs classification accuracy). Agreement between them is strong evidence. This script identifies high-confidence features that both methods rank highly, and flags disagreements for investigation.

Output: `run_concordance.py` CLI that produces a concordance report (printed to console + optional CSV).
</objective>

  <name>Task 1: Create run_concordance.py CLI</name>

  <action>
  Create `src/ta_lab2/scripts/analysis/run_concordance.py` — CLI script that cross-validates IC-IR ranking against MDA importance.

  **CLI arguments (argparse):**
  - `--asset-ids` (comma-separated ints, required): Asset IDs for MDA computation. E.g., "1,1027" for BTC/ETH.
  - `--tf` (str, default='1D'): Timeframe.
  - `--start` (str, required): Training start date (ISO format).
  - `--end` (str, required): Training end date (ISO format).
  - `--horizon` (int, default=1): IC horizon for ranking comparison.
  - `--top-n` (int, default=30): Top N features from IC-IR ranking to include in concordance.
  - `--n-splits` (int, default=5): Number of purged K-fold splits for MDA.
  - `--cluster-threshold` (float, default=0.5): Spearman correlation threshold for feature clustering.
  - `--output-csv` (str, optional): Path to write concordance CSV.

  **Pipeline steps:**

  Step 1 — Load IC-IR ranking:
  ```python
  from ta_lab2.analysis.feature_selection import load_ic_ranking
  ic_ranking = load_ic_ranking(engine, horizon=args.horizon)
  top_features = ic_ranking.head(args.top_n)['feature'].tolist()
  ```

  Step 2 — Load feature data for MDA:
  Load from the `features` table for the given asset_ids, tf, and date range. Only load the top-N feature columns (plus close for labels). Follow the pattern from `run_feature_importance.py`:
  - Build binary labels: `y = (ret_arith > 0).astype(int)` where ret_arith comes from the features table.
  - Build t1_series with 1-bar lag for PurgedKFoldSplitter.
  - Drop rows with NaN in any feature column.

  Step 3 — Run MDA:
  ```python
  from ta_lab2.ml.feature_importance import compute_mda, cluster_features
  from sklearn.ensemble import RandomForestClassifier
  from ta_lab2.backtests.cv import PurgedKFoldSplitter

  model = RandomForestClassifier(n_estimators=100, max_depth=5, random_state=42, n_jobs=-1)
  mda_importance = compute_mda(model, X, y, t1_series, n_splits=args.n_splits, n_repeats=3)
  ```
  mda_importance is a pd.Series sorted descending by importance.

  Step 4 — Compute concordance:
  - Create a merged DataFrame with columns: feature, ic_ir_rank (1=best), mda_rank (1=best).
  - Compute Spearman rank correlation between ic_ir_rank and mda_rank: `spearmanr(merged['ic_ir_rank'], merged['mda_rank'])`.
  - Identify features in BOTH top-20 of IC-IR and top-20 of MDA: these are "high-confidence" features.
  - Identify features in IC-IR top-20 but NOT MDA top-20 (and vice versa): these are "divergent" features.

  Step 5 — Feature clustering:
  ```python
  clusters = cluster_features(X[top_features], threshold=args.cluster_threshold)
  ```
  For each cluster with >1 feature, select the feature with the highest IC-IR. Report cluster membership.

  Step 6 — Print concordance report:
  ```
  === Concordance Report ===
  IC-IR vs MDA Spearman rho: 0.72 (p=0.001)
  High-confidence (both top-20): [feature_list]
  IC-IR only (top-20): [feature_list]
  MDA only (top-20): [feature_list]

  === Feature Clusters ===
  Cluster 1: [rsi_14, rsi_7, rsi_21] -> Best: rsi_14 (IC-IR=0.45)
  Cluster 2: [vol_30d, vol_63d] -> Best: vol_30d (IC-IR=0.38)

  === Merged Ranking ===
  Feature          | IC-IR Rank | MDA Rank | Agreement | Cluster
  rsi_14           | 1          | 3        | AGREE     | 1
  ...
  ```

  Step 7 — Optional CSV output:
  If `--output-csv` is given, write the merged ranking DataFrame.

  **DB connection:**
  ```python
  engine = create_engine(resolve_db_url(), poolclass=NullPool)
  ```

  **Error handling:**
  - If no ic_results data: print warning and exit gracefully.
  - If MDA fails (e.g., not enough observations for PurgedKFold): log warning, report IC-IR only ranking.
  - If feature not in features table (e.g., AMA features from ic_results but not in features table): skip those features in the MDA step and note in the report.
