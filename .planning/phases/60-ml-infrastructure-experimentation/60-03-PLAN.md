  In this branch:
  1. Get the expression string: `expr = compute.get('expression', '')`
  2. If empty, raise ValueError: `"Feature '{name}' has mode='expression' but no 'expression' key"`
  3. Extract $col references via regex: `dollar_refs = re.findall(r'\$(\w+)', expr)`
  4. Validate each ref is a valid identifier
  5. If params exist, substitute first combo into expression before validation (same pattern as inline mode)
  6. Replace $col refs with placeholder `_` and validate with `ast.parse(substituted, mode='eval')`
  7. Import `re` at top of file if not already imported

  Also update the error message in the final `elif mode is not None:` branch to say:
  `"Must be 'inline', 'dotpath', or 'expression'."`

  Update `_expand_params` to handle expression mode parameter substitution:
  - In the `if compute.get("mode") == "inline":` branch, also handle `mode == "expression"` with the same substitution logic (expression uses {param} format strings just like inline mode).
  - Change the condition to: `if compute.get("mode") in ("inline", "expression"):`

  CRITICAL: Do NOT break existing inline or dotpath validation. This is an additive change only.
  </action>
