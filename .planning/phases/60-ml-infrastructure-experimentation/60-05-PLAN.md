  <name>Task 1: Create RegimeRouter module</name>
  <files>src/ta_lab2/ml/regime_router.py</files>
  <action>
  Create `src/ta_lab2/ml/regime_router.py` with:

  1. `load_regimes(conn, asset_id: int, tf: str, start: pd.Timestamp, end: pd.Timestamp) -> pd.Series`:
     - SQL: `SELECT ts, l2_label FROM public.cmc_regimes WHERE id = :id AND tf = :tf AND ts BETWEEN :start AND :end ORDER BY ts`
     - Use `pd.to_datetime(df['ts'], utc=True)` (MEMORY.md pitfall)
     - Set index to ts, return `l2_label` series with NaN filled as `'Unknown'`

  2. Class `RegimeRouter`:
     - `__init__(self, base_model, min_samples: int = 30, regime_col: str = 'l2_label')`:
       - Store base_model (sklearn-compatible estimator)
       - `self.models: dict[str, Any] = {}` -- per-regime sub-models
       - `self.min_samples = min_samples`

     - `fit(self, X: pd.DataFrame, y: np.ndarray, regimes: pd.Series) -> 'RegimeRouter'`:
       - For each unique regime label in regimes:
         - Build boolean mask: `mask = (regimes == regime_label).values`
         - If `mask.sum() < self.min_samples`: log warning, skip (will use global fallback)
         - Otherwise: clone base_model, fit on X[mask], y[mask], store in self.models[regime_label]
       - Always train a global fallback: clone base_model, fit on ALL X, y, store in self.models['__global__']
       - Log regime sample counts at INFO level
       - Return self

     - `predict(self, X: pd.DataFrame, regimes: pd.Series) -> np.ndarray`:
       - For each row, look up model by regime label, fall back to '__global__'
       - Use vectorized approach: group by regime, predict per group, then reassemble
       - Return np.ndarray of predictions

     - `predict_proba(self, X: pd.DataFrame, regimes: pd.Series) -> np.ndarray`:
       - Same grouping approach but call predict_proba per group
       - Return (n_samples, n_classes) array

     - `get_regime_stats(self) -> dict`:
       - Return dict with per-regime info: model type, n_training_samples, feature names

  CRITICAL: Always pass DataFrame slices (not numpy arrays) to model.fit/predict to avoid LightGBM warnings (Pitfall 5).
  CRITICAL: Handle case where regimes series has values not seen during training -- fall back to global.
  </action>
  <verify>
  ```python
  from ta_lab2.ml.regime_router import RegimeRouter, load_regimes
  import pandas as pd, numpy as np
  from sklearn.ensemble import RandomForestClassifier

  # Smoke test
  np.random.seed(42)
  n = 150
  X = pd.DataFrame({'f1': np.random.randn(n), 'f2': np.random.randn(n)})
  y = (X['f1'] > 0).astype(int).values
  regimes = pd.Series(np.random.choice(['Up', 'Down', 'Sideways'], n))

  router = RegimeRouter(RandomForestClassifier(n_estimators=10, random_state=42), min_samples=10)
  router.fit(X, y, regimes)
  preds = router.predict(X, regimes)
  assert len(preds) == n
  assert '__global__' in router.models
  print(f'Regime models: {list(router.models.keys())}')
  print('RegimeRouter OK')
  ```
  </verify>

  Create `src/ta_lab2/scripts/ml/__init__.py` (empty, docstring only).

  Create `src/ta_lab2/scripts/ml/run_feature_importance.py` as a `__main__`-runnable CLI:

  Arguments (argparse):
  - `--asset-ids`: comma-separated list of asset IDs (default: "1,2" for BTC, ETH)
  - `--tf`: timeframe (default: "1D")
  - `--start`: train start date (default: "2023-01-01")
  - `--end`: train end date (default: "2025-12-31")
  - `--n-splits`: CV folds (default: 5)
  - `--scoring`: sklearn scoring metric (default: "accuracy")
  - `--mode`: "mda", "sfi", or "both" (default: "both")
  - `--model`: "rf" or "lgbm" (default: "rf")
  - `--output-csv`: optional path to write CSV results
  - `--log-experiment`: flag to log to cmc_ml_experiments (default: False)

  Logic:
  1. Connect to DB via `get_db_url()` from ta_lab2 config
  2. Load cmc_features for given asset_ids, tf, date range
     - Exclude PK columns: id, ts, tf, alignment_source, tf_days, asset_class
     - Exclude raw OHLCV: open, high, low, close, volume
     - Drop columns with all NaN
     - Build labels: simple binary (positive returns = 1, else 0) from existing `ret_arith` column in cmc_features
     - Build t1_series: ts + 1 day (fixed horizon label)
  3. Run compute_mda and/or compute_sfi
  4. Print results: top 20 and bottom 10 features by importance
  5. If --output-csv, write full results to CSV
  6. If --log-experiment, use ExperimentTracker.log_run() to persist

  Handle import errors for lightgbm gracefully -- fall back to RandomForest if lgbm not available.
  Use NullPool for engine (multiprocessing safety).
  </action>
