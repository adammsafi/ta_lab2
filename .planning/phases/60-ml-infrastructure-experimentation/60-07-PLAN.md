  <files>src/ta_lab2/scripts/ml/run_regime_routing.py, src/ta_lab2/scripts/ml/run_double_ensemble.py</files>
  <action>
  **run_regime_routing.py** -- CLI for MLINFRA-03 regime-routed backtest:

  Arguments (argparse):
  - `--asset-ids`: comma-separated (default: "1")
  - `--tf`: timeframe (default: "1D")
  - `--start`, `--end`: date range
  - `--n-splits`: CV folds (default: 5)
  - `--min-regime-samples`: minimum samples per regime (default: 30)
  - `--log-experiment`: flag to log to tracker

  Logic:
  1. Connect to DB, load cmc_features (same filtering as run_feature_importance)
  2. Build labels: binary (ret_arith > 0), build t1_series
  3. Load regimes: `load_regimes(conn, asset_id, tf, start, end)` -- join by ts index
  4. Train RegimeRouter with LGBMClassifier (or RandomForest fallback)
  5. Evaluate with PurgedKFoldSplitter:
     - For each fold: fit router on train, predict on test, compute accuracy
     - CRITICAL: Empty fold guard
  6. Train single global model (same base estimator) for baseline comparison
  7. Print comparison: regime-routed accuracy vs global accuracy, per-regime breakdown
  8. If --log-experiment, log both runs to ExperimentTracker (one for global, one for regime-routed)

  **run_double_ensemble.py** -- CLI for MLINFRA-04 concept drift evaluation:

  Arguments (argparse):
  - `--asset-ids`: comma-separated (default: "1")
  - `--tf`: timeframe (default: "1D")
  - `--start`, `--end`: date range
  - `--window-size`: DoubleEnsemble window (default: 60)
  - `--stride`: DoubleEnsemble stride (default: 15)
  - `--n-splits`: CV folds (default: 5)
  - `--log-experiment`: flag

  Logic:
  1. Connect to DB, load cmc_features (same filtering)
  2. Build labels and t1_series
  3. Evaluate DoubleEnsemble with PurgedKFoldSplitter:
     - For each fold: fit DoubleEnsemble on X_train, predict on X_test, compute accuracy
     - CRITICAL: Empty fold guard
  4. Evaluate static LGBMClassifier baseline with same CV
  5. Print comparison: DoubleEnsemble accuracy vs static model accuracy
  6. If --log-experiment, log both runs to ExperimentTracker

  Both scripts:
  - Use `get_db_url()` for DB connection
  - Use `NullPool` for engine
  - Handle LightGBM import error gracefully (fall back to RandomForest)
  - Always pass DataFrames to models (not numpy arrays)
  - Print clear comparison tables with accuracy, precision, recall

<task type="auto">
  <name>Task 2: Create Optuna sweep CLI script</name>
  <files>src/ta_lab2/scripts/ml/run_optuna_sweep.py</files>
  <action>
  Create `src/ta_lab2/scripts/ml/run_optuna_sweep.py` -- CLI for MLINFRA-06:

  Arguments (argparse):
  - `--asset-ids`: comma-separated (default: "1")
  - `--tf`: timeframe (default: "1D")
  - `--start`, `--end`: date range
  - `--n-trials`: Optuna trial count (default: 50)
  - `--n-splits`: CV folds (default: 5)
  - `--study-name`: Optuna study name (default: "lgbm_sweep")
  - `--storage`: Optuna storage URL (default: "sqlite:///optuna.db")
  - `--direction`: "maximize" or "minimize" (default: "maximize")
  - `--grid-comparison`: flag to also run grid search equivalent for efficiency comparison
  - `--log-experiment`: flag to log to tracker

  Logic:
  1. Connect to DB, load cmc_features (same filtering)
  2. Build labels and t1_series
  3. Define objective function:
     - Trial suggests: n_estimators (50-300, step 50), num_leaves (10-63), learning_rate (0.01-0.3 log), min_child_samples (5-50)
     - Create LGBMClassifier with suggested params
     - Evaluate with PurgedKFoldSplitter (n_splits folds)
     - CRITICAL: Empty fold guard in each fold
     - Return mean OOS accuracy
  4. Create Optuna study with TPE sampler (seed=42), storage URL, load_if_exists=True
  5. Run study.optimize(objective, n_trials=n_trials)
  6. Print: best params, best value, n_trials
  7. If --grid-comparison:
     - Define grid: n_estimators=[50,100,150,200,250,300], num_leaves=[10,20,31,50,63], learning_rate=[0.01,0.05,0.1,0.3], min_child_samples=[5,10,20,50]
     - Calculate total grid size: product of all grid dimensions
     - Find how many Optuna trials were needed to reach within 1% of best value
     - Print efficiency comparison: grid size vs Optuna trials to near-optimal
  8. If --log-experiment, log to ExperimentTracker with optuna_study_name, optuna_n_trials, optuna_best_params

  Import optuna at top. If not installed, raise ImportError with "pip install optuna==4.7.0".
  Use `optuna.logging.set_verbosity(optuna.logging.WARNING)` to reduce noise.
  </action>

  </verify>
