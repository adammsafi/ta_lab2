  <action>
Replace the per-(id, tf, venue_id) architecture with per-ID batch processing. This follows the exact same pattern as the EMA returns rewrite (Plan 01):

1. **Change key resolution**: Replace `_load_keys()` returning `List[Tuple[int, str, int]]` (id, tf, venue_id) with `_load_ids()` returning `List[int]`.

2. **Bulk watermark preload**: Add `_load_watermarks(engine, state_table, id_) -> dict`:
   ```sql
   SELECT tf, venue_id, last_timestamp
   FROM {state_table}
   WHERE id = :id
   ```
   Returns dict keyed by (tf, venue_id) -> last_timestamp. Compute `min_last_ts` as seed point.

3. **Source-advance skip**: Check if source table has rows newer than min_last_ts:
   ```sql
   SELECT 1 FROM {bars_table}
   WHERE id = :id AND "timestamp" > :min_last_ts
   LIMIT 1
   ```
   Skip ID if no new data. Optionally also filter by alignment_source if src_alignment_source is set.

4. **Rewrite `_run_one_key()` -> `_run_one_id()`**: The new function executes ONE SQL CTE per ID that:
   - Seeds from min_last_ts - 2 canonical rows (same seed logic, minimum across all keys)
   - Pulls ALL (tf, venue_id) combos from source bars for this ID
   - Adds PARTITION BY (tf, venue_id) to ALL LAG window functions:
     ```sql
     -- Unified LAG (all rolls)
     LAG(s.close) OVER (PARTITION BY s.tf, s.venue_id ORDER BY s."timestamp") AS prev_close_u,
     -- Canonical LAG (same roll partition)
     LAG(s.close) OVER (PARTITION BY s.tf, s.venue_id, s.roll ORDER BY s."timestamp") AS prev_close_c,
     ```
   - The calc CTE, calc2 CTE, and to_insert CTE all gain PARTITION BY (tf, venue_id) in their window functions
   - Uses a watermark VALUES CTE to filter: only insert rows where "timestamp" > per-key last_timestamp
   - INSERT uses same ON CONFLICT DO UPDATE SET pattern

5. **Bulk state update**: After INSERT, update all state rows for this ID:
   ```sql
   INSERT INTO {state_table} (id, venue_id, tf, last_timestamp, updated_at)
   SELECT id, venue_id, tf, MAX("timestamp"), now()
   FROM {out_table}
   WHERE id = :id
   GROUP BY id, venue_id, tf
   ON CONFLICT (id, venue_id, tf) DO UPDATE SET
     last_timestamp = GREATEST(EXCLUDED.last_timestamp, {state_table}.last_timestamp),
     updated_at = now()
   ```

6. **Update main()**: Load IDs, loop over IDs, keep --full-refresh and --workers and --venue-ids and --src-alignment-source flags.

7. **_ensure_state_rows**: Update to batch-insert state rows per ID (one INSERT with all tf/venue combos), not per-key loop.

CRITICAL: Bar returns has additional columns not in EMA returns:
- bar_seq, pos_in_bar, count_days, count_days_remaining (metadata columns)
- range, range_pct, true_range, true_range_pct (price range columns)
- gap_bars (bar_seq delta, canonical partition only)
All of these must be correctly computed in the batch CTE. The gap_bars uses `bar_seq - prev_bar_seq` which needs PARTITION BY (tf, venue_id, roll) for canonical, PARTITION BY (tf, venue_id) for unified.

Keep the `_as_filter` logic for filtering source bars by alignment_source when reading from _u tables.
