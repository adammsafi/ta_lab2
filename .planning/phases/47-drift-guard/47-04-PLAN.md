**attribution.py:**

1. `AttributionResult` frozen dataclass:
   - baseline_pnl: float (zero fees + zero slippage backtest P&L)
   - fee_delta: float (step 1: fee model contribution)
   - slippage_delta: float (step 2: slippage contribution)
   - timing_delta: float (step 3: execution timing contribution)
   - data_revision_delta: float (step 4: data revision contribution)
   - sizing_delta: float (step 5: position sizing rounding)
   - regime_delta: float (step 6: regime label differences)
   - unexplained_residual: float (paper_pnl - total_explained)
   - total_explained_pnl: float (baseline + sum of all deltas)
   - paper_pnl: float (actual paper executor P&L)

2. `DriftAttributor` class:
   - `__init__(self, engine: Engine)`: Store engine.
   - `run_attribution(self, config_id: int, signal_id: int, signal_type: str, asset_id: int, paper_start: str, paper_end: str, paper_pnl: float) -> AttributionResult`:
     Sequential OAT in fixed order. Each step runs a backtest with one more cost source added:

     **Step 0 (Baseline):** Run backtest with CostModel(fee_bps=0, slippage_bps=0, funding_bps_day=0). Result = baseline_pnl.

     **Step 1 (+Fees):** Run backtest with CostModel(fee_bps=config_fee_bps, slippage_bps=0). fee_delta = step1_pnl - baseline_pnl.

     **Step 2 (+Slippage):** Run backtest with CostModel(fee_bps=config_fee_bps, slippage_bps=config_slippage_bps). slippage_delta = step2_pnl - step1_pnl.

     **Step 3 (+Timing):** For V1, timing delta is 0 because both use next-bar-open execution. Log "timing_delta=0 (same execution model)". Future enhancement: compare exact fill timestamps.

     **Step 4 (+Data revision):** Run PIT replay vs current-data replay comparison. data_revision_delta = abs(pit_pnl - current_pnl). For V1 this is likely 0. Use the DriftMonitor's data_revision_pnl_diff if available, otherwise run both replays.

     **Step 5 (+Sizing):** Compare theoretical vs actual position sizes. For V1, sizing_delta = 0 (both use same sizing model). Future: capture actual quantity vs theoretical quantity from fills.

     **Step 6 (+Regime):** Run replay with --no-regime vs with regime. regime_delta = regime_pnl - no_regime_pnl. If regime data is not available or signal_type doesn't support regime, set to 0.

     **Residual:** unexplained = paper_pnl - (baseline + fee_delta + slippage_delta + timing_delta + data_revision_delta + sizing_delta + regime_delta).

     IMPORTANT: Steps 3, 5 are placeholders for V1 (always 0). Steps 0, 1, 2 are the primary attribution sources. Step 4 uses existing data_revision_pnl_diff. Step 6 depends on regime support.

     Use SignalBacktester.run_backtest() for each replay. Load cost config from dim_executor_config. Handle backtest failures gracefully (log exception, set delta to 0 for that step).

   - `_load_config_cost_params(self, config_id: int) -> dict`: Query dim_executor_config for fee_bps, slippage_mode, slippage_base_bps.

   - `_run_replay_with_cost(self, signal_type, signal_id, asset_id, start, end, cost_model: CostModel) -> float | None`: Run SignalBacktester, extract cumulative P&L. Return None on failure.

   GUARD: Add minimum trade count guard. If paper_trade_count < 10 (from research pitfall 7), skip attribution and return an AttributionResult with all zeros and paper_pnl, logging "Insufficient trade history (N=X, minimum=10)".
