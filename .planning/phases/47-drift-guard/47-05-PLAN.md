**run_drift_report.py** -- weekly drift report CLI:

```python
"""Weekly drift report generator -- produces Markdown + Plotly charts.

Usage:
    python -m ta_lab2.scripts.drift.run_drift_report
    python -m ta_lab2.scripts.drift.run_drift_report --week-start 2025-01-01 --week-end 2025-01-07
    python -m ta_lab2.scripts.drift.run_drift_report --output-dir reports/drift --verbose
"""
```

argparse arguments:
- `--week-start`: ISO date for report start (default: 7 days ago)
- `--week-end`: ISO date for report end (default: today)
- `--output-dir`: Report output directory (default: reports/drift)
- `--db-url`: Override database URL
- `--verbose` / `-v`: Set logging to DEBUG
- `--with-attribution`: Also run full attribution decomposition for the report period (default: skip -- attribution is compute-heavy)

main() function:
1. Parse args, configure logging.
2. Create engine (NullPool).
3. If --with-attribution: run DriftAttributor for each config and update attr_* columns in cmc_drift_metrics for the report period.
4. Instantiate ReportGenerator(engine).
5. Call generator.generate_weekly_report(week_start, week_end, output_dir).
6. Print path to generated report.
7. Return exit code 0 on success.

NOTE: The weekly report is NOT wired into --all pipeline. It's invoked manually or from cron (per research: "Weekly digest NOT in --all").
