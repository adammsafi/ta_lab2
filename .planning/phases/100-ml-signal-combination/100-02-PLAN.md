  <files>

  <action>
Create `src/ta_lab2/ml/shap_analysis.py` with class `RankerShapAnalyzer`:

**Constructor `__init__(self, model, feature_names)`:**
- Stores the fitted LGBMRanker model and feature name list.

**Method `compute_shap_values(X_sample, max_samples=500)`:**
- Subsample X to `max_samples` rows if larger (memory safety -- interaction tensor is O(n * f * f)).
- Create `shap.TreeExplainer(self.model)`.
- Compute `explainer.shap_values(X_sample)` -- returns (n, f) array of SHAP values.
- Store as `self.shap_values_`.
- Return the SHAP values array.

**Method `compute_interaction_values(X_sample, max_samples=500)`:**
- Subsample to `max_samples` rows.
- Create `shap.TreeExplainer(self.model)`.
- Compute `explainer.shap_interaction_values(X_sample)` -- returns (n, f, f) tensor.
- Store as `self.interaction_values_`.
- Compute `mean_abs_interactions = np.abs(interaction_values).mean(axis=0)` -- (f, f) matrix.
- Zero the diagonal (self-interactions are just main effects).
- Store as `self.mean_abs_interactions_`.
- Return the interaction tensor.

**Method `top_interaction_pairs(k=5)`:**
- From `self.mean_abs_interactions_`, extract upper triangle.
- Sort by mean absolute interaction strength descending.
- Return list of dicts: `[{"feature_a": name_a, "feature_b": name_b, "mean_abs_interaction": val}, ...]` for top k pairs.

**Method `top_shap_features(k=20)`:**
- From `self.shap_values_`, compute `np.abs(shap_values).mean(axis=0)` per feature.
- Sort descending, return top k as list of dicts: `[{"feature": name, "mean_abs_shap": val}, ...]`.

**Method `generate_report(top_k_interactions=5, top_k_features=20)`:**
- Returns a markdown string with:
  - "## SHAP Feature Importance (Top {k})" table
  - "## SHAP Interaction Pairs (Top {k})" table
  - "## Recommendations" section with actionable notes

**Method `update_feature_selection(yaml_path, engine=None, top_k_interactions=3)`:**
- Load existing `feature_selection.yaml`.
- Add an `interactions` key with the top k pairs (feature_a, feature_b, strength).
- Write back to yaml_path with `yaml.dump()`.
- If engine provided, also update `dim_feature_selection` notes column for the interacting features with "SHAP interaction partner: {other_feature}".
- This satisfies the ML-02 requirement: "at least one finding feeds back into feature_selection.yaml or dim_feature_selection".

**Lazy-import shap** inside methods (same pattern as lightgbm in double_ensemble.py). Informative ImportError if shap not installed.
