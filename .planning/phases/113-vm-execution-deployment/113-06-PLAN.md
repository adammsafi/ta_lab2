**Architecture (from CONTEXT.md):** Single event-driven service. WebSocket loop is the main event loop (always watching positions for stops/TPs). Signal processing triggers when new signals detected via watermark (executor_processed_at IS NULL).

**Startup sequence:**
```python
def main():
    # 1. Parse args (--dry-run, --log-level)
    # 2. Configure logging to stdout (journald captures it)
    # 3. Create SQLAlchemy engine with NullPool (existing pattern)
    #    DB URL from env: EXECUTOR_DB_URL or default postgresql+psycopg2://hluser:hlpass@127.0.0.1:5432/hyperliquid
    # 4. Initialize PriceCache
    # 5. Start WebSocket feeds (start_all_feeds)
    #    - HL: always on (primary exchange)
    #    - Kraken/Coinbase: configure via env vars KRAKEN_SYMBOLS, COINBASE_PRODUCTS
    # 6. Wait for initial prices (up to 30s for PriceCache to populate)
    # 7. Initialize PaperExecutor with vm_mode=True, price_cache=price_cache
    # 8. Initialize StopMonitor
    # 9. Start StopMonitor thread
    # 10. Enter main signal processing loop
    # 11. Handle SIGTERM/SIGINT for graceful shutdown
```

**Main signal processing loop:**
```python
def _signal_loop(executor, engine, price_cache, config):
    """Main loop: check for new signals, execute, sleep, repeat."""
    POLL_INTERVAL = 30  # seconds between signal checks
    STALENESS_THRESHOLD_HOURS = 26  # from dim_executor_config.cadence_hours
    
    consecutive_errors = 0
    MAX_CONSECUTIVE_ERRORS = 10
    
    while True:
        try:
            # 1. Check for unprocessed signals (watermark query)
            # 2. If new signals found:
            #    a. Run executor.run(dry_run=False)
            #    b. Reset consecutive_errors = 0
            # 3. If no new signals:
            #    a. Check signal staleness (last signal ts vs now)
            #    b. If stale: log warning, send Telegram alert (once per cadence)
            #    c. Do NOT open new trades on stale signals
            #    d. Stop/TP monitoring continues (StopMonitor thread is independent)
            # 4. Sleep POLL_INTERVAL
            
        except Exception as exc:
            consecutive_errors += 1
            logger.error("Signal loop error (%d/%d): %s", consecutive_errors, MAX_CONSECUTIVE_ERRORS, exc)
            if consecutive_errors >= MAX_CONSECUTIVE_ERRORS:
                _send_telegram_alert(f"EXECUTOR CRASH LOOP: {consecutive_errors} consecutive errors. Last: {exc}")
                sys.exit(1)  # systemd will restart, StartLimitBurst protects against infinite loop
            time.sleep(min(30 * consecutive_errors, 300))  # backoff
```

**Stale signal handling (from CONTEXT.md):**
- No new trades opened on stale signals
- Existing positions protected via WebSocket monitoring (stops/TPs continue)
- Telegram alert fired using existing cadence_hours threshold (default 26h) per strategy
- No defensive mode / position reduction -- just halt new trades and alert

**PriceCache staleness monitoring:**
- Every 5 minutes, check `price_cache.stale_symbols(max_age_seconds=120)`
- If HL prices stale > 2 min, send Telegram alert (HL SDK has no auto-reconnect)
- Log stale symbols at WARNING level

**Graceful shutdown:**
```python
import signal
_shutdown = threading.Event()
signal.signal(signal.SIGTERM, lambda s, f: _shutdown.set())
signal.signal(signal.SIGINT, lambda s, f: _shutdown.set())
```
- On shutdown signal: stop StopMonitor, log final state, exit cleanly

**Crash-loop detection:**
Track start times in a file (`/tmp/executor_starts.json`). If 5 starts within 5 minutes, send Telegram alert and exit with code 2 (distinguishes from normal restart). Systemd's StartLimitBurst=5 / StartLimitIntervalSec=300 provides the OS-level protection; this is application-level awareness.

**Logging:** All output to stdout/stderr. systemd journald captures it via `StandardOutput=journal`. Use structured log format: `%(asctime)s [%(levelname)s] %(name)s: %(message)s`.
