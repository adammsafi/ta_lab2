        self._running.clear()

Extend `get_current_price()` in PositionSizer with a 3-tier VM-aware fallback chain as specified in RESEARCH.md Pattern 9.

**Current state:** `get_current_price()` queries `exchange_price_feed` and falls back to `price_bars_multi_tf_u`. The `price_bars_multi_tf_u` table does NOT exist on the VM -- this is a CRITICAL BLOCKER identified in RESEARCH.md.

**New fallback chain (VM-aware):**
1. **PriceCache (WebSocket live tick)** -- sub-second freshness. Check `price_cache.get(symbol)` first. Only available when PriceCache is provided (VM mode).
2. **exchange_price_feed** -- existing fallback, works on both local and VM.
3. **hl_assets.mark_px** -- HL collector updates this every 6h on the VM. Query: `SELECT mark_px FROM hyperliquid.hl_assets WHERE symbol = :symbol LIMIT 1`.
4. **hl_candles latest close** -- last resort on VM. Query: `SELECT close FROM hyperliquid.hl_candles WHERE asset_id = :asset_id ORDER BY ts DESC LIMIT 1`.
5. **price_bars_multi_tf_u** -- existing fallback, only on local (skip on VM).

**Implementation approach:**
- Add an optional `price_cache: PriceCache | None = None` parameter to PositionSizer.__init__() (default None for backward compatibility).
- Add an optional `vm_mode: bool = False` parameter to PositionSizer.__init__().
- In `get_current_price()`, check tiers in order:
  1. If price_cache is set: try price_cache.get(symbol). Return if not None and not stale (< 120s).
  2. Try exchange_price_feed (existing code, unchanged).
  3. If vm_mode: try hl_assets.mark_px via `hyperliquid.hl_assets` table.
  4. If vm_mode: try hl_candles latest close.
  5. If NOT vm_mode: try price_bars_multi_tf_u (existing code, unchanged).
- Log which tier provided the price at DEBUG level.

**Zero behavior change for existing callers:** Default price_cache=None and vm_mode=False means existing local executor code is completely unchanged. Only the VM executor service will pass these parameters.

**Symbol mapping:** PriceCache uses exchange symbols (e.g., "BTC"), but position_sizer may receive asset_id. Add a `_resolve_symbol(asset_id, conn)` helper that queries dim_listings. Cache the mapping.
