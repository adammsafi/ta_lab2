<task type="auto">
  <name>Task 2: Create create_vm_tables.sh DDL extraction and deployment script</name>
  <files>deploy/executor/create_vm_tables.sh</files>
  <action>
Create a bash script that:

1. **Reads vm_table_list.txt** -- strips comments and blank lines to get table names.

2. **Extracts DDL from local DB** using `pg_dump --schema-only -t tablename` for each table. Uses `MARKETDATA_DB_URL` from db_config.env (parse host/port/db/user/pass). Captures all DDL into a single SQL file.

3. **Post-processes the DDL:**
   - Strips FK constraints referencing tables NOT in the VM list (e.g., FK to features, price_bars). Use `sed` or `grep -v` to remove `ALTER TABLE ... ADD CONSTRAINT ... FOREIGN KEY ... REFERENCES <non-vm-table>` lines.
   - Updates exchange_price_feed CHECK constraint to add 'hyperliquid' to the allowed values. Research says CHECK constraint must be updated.
   - Wraps each CREATE TABLE in `CREATE TABLE IF NOT EXISTS` for idempotency.

4. **Pipes DDL to VM** via SSH:
   ```bash
   SSH_KEY=~/Downloads/oracle_sg_keys/ssh-key-2026-03-10.key
   VM_HOST=161.118.209.59
   VM_USER=ubuntu
   VM_DB=hyperliquid
   VM_DB_USER=hluser
   VM_DB_PASS=hlpass
   
   cat combined_ddl.sql | ssh -i $SSH_KEY $VM_USER@$VM_HOST \
     "PGPASSWORD=$VM_DB_PASS psql -h 127.0.0.1 -U $VM_DB_USER -d $VM_DB"
   ```

5. **Seeds dimension tables** by running COPY TO STDOUT on local for dim_timeframe, dim_venues, dim_signals, dim_sessions, then piping each to VM via SSH + COPY FROM STDIN (same pattern as sync_hl_from_vm.py but reversed direction).

6. **Seeds dim_executor_config and dim_risk_limits** -- same COPY pattern. These contain the runtime config the executor needs.

**Important details:**
- Use `ServerAliveInterval=30` in SSH options (from research: prevents timeout on large DDL).
- Script should be idempotent (safe to re-run).
- Follow the pattern of deploy/tvc/setup_vm.sh for structure and conventions.
- Include `set -euo pipefail` at top.
- Source db_config.env for local DB credentials.
- Do NOT create a Python venv or install packages -- that's a separate plan.
  </action>
