**Signal tables** -- import SIGNAL_TABLE_MAP from signal_reader:
```python
from ta_lab2.executor.signal_reader import SIGNAL_TABLE_MAP
SIGNAL_TABLES = list(SIGNAL_TABLE_MAP.values())
# ["signals_ema_crossover", "signals_rsi_mean_revert", ...]
```

**Core sync flow for each table:**
1. Query local DB for watermark: `SELECT MAX(ts) FROM {vm_table}_sync_wm` on VM (or NULL for first run).
2. Local COPY TO STDOUT with WHERE ts > watermark (incremental) or no WHERE (--full):
   ```python
   copy_sql = f"COPY (SELECT * FROM public.{table} WHERE ts > '{watermark}') TO STDOUT WITH CSV HEADER"
   ```
   Run via local `psql -c` piped to temp file.
3. SSH pipe temp file to VM: `cat tempfile | ssh ... "PGPASSWORD=... psql ... -c \"COPY _staging_{table} FROM STDIN WITH CSV HEADER\""`.
4. VM-side upsert: `INSERT INTO {table} SELECT * FROM _staging_{table} ON CONFLICT DO UPDATE SET ...` (update all non-PK columns).
5. Update watermark on VM.

**Staging table approach:** For each signal table, create a temp staging table with `CREATE TEMP TABLE _staging_{table} (LIKE {table} INCLUDING ALL) ON COMMIT DELETE ROWS;` via SSH before the COPY FROM STDIN.

**CLI args:**
- `--full` : full resync (no watermark filter)
- `--dry-run` : report row counts only
- `--table <name>` : sync only one signal table
- No args: incremental sync of all 7 tables

**Watermark tracking:** Use a simple `sync_signal_watermarks` table on VM:
```sql
CREATE TABLE IF NOT EXISTS sync_signal_watermarks (
    table_name TEXT PRIMARY KEY,
    last_synced_ts TIMESTAMPTZ,
    synced_at TIMESTAMPTZ DEFAULT NOW()
);
```
Create it if not exists at script start.

**Logging:** Print table name, row count synced, duration for each table. Use the same _ssh_cmd() and _vm_psql() helper pattern as sync_hl_from_vm.py.

**Error handling:** If one table fails, log error and continue to next table. Return exit code 1 if any table failed.
