Implement PurgedKFoldSplitter and CPCVSplitter as sklearn-compatible cross-validation splitters using TDD.

Purpose: CV-01 through CV-03 require leakage-free CV splitters that handle overlapping labels (time series data). Implementing from scratch avoids mlfinlab (discontinued, known bug #295).

  <behavior>
    PurgedKFoldSplitter:
    - PurgedKFoldSplitter(n_splits=5, t1_series=None) -> raises ValueError("t1_series is required")
    - PurgedKFoldSplitter(n_splits=5, t1_series=valid_series).get_n_splits() -> 5
    - split(X) yields exactly n_splits (train_idx, test_idx) tuples
    - For each fold: set(train_idx) & set(test_idx) == empty set (no overlap)
    - Purge: no train observation has t1[i] > test_period_start (label end leaks into test)
    - Embargo: no train obs in [test_end, test_end + embargo_size) range
    - embargo_frac=0.01 by default: embargo_size = max(1, int(0.01 * n))
    - Works with sklearn cross_val_score(DummyClassifier(), X, y, cv=splitter)

    CPCVSplitter:
    - CPCVSplitter(n_splits=6, n_test_splits=2, t1_series=valid_series).get_n_splits() -> 15 (C(6,2))
    - CPCVSplitter(n_splits=10, n_test_splits=2, ...).get_n_splits() -> 45 (C(10,2))
    - CPCVSplitter(n_splits=6, n_test_splits=2, t1_series=None) -> raises ValueError
    - split(X) yields exactly C(n_splits, n_test_splits) (train_idx, test_idx) tuples
    - Each combination: test is union of n_test_splits fold groups, train is complement minus purge/embargo
    - No train-test overlap in any combination
    - Embargo applies after last test group only
  </behavior>
  <implementation>
    Create `src/ta_lab2/backtests/cv.py` with:

    1. `PurgedKFoldSplitter(BaseCrossValidator)`:
       - __init__(n_splits=5, t1_series=None, embargo_frac=0.01): raise ValueError if t1_series is None
       - get_n_splits(X=None, y=None, groups=None) -> int: return self.n_splits
       - split(X, y=None, groups=None): yields (train_idx, test_idx) with purge + embargo
         - Divide indices into n_splits equal folds
         - For each fold as test: build train from all non-test indices
         - Purge: remove train obs where self.t1.iloc[idx] > test_start_timestamp
         - Embargo: remove train obs in [test_end, test_end + embargo_size) range
         - embargo_size = max(1, int(self.embargo_frac * n))
       - Post-construction validation: assert t1_series index is monotonically increasing

    2. `CPCVSplitter(BaseCrossValidator)`:
       - __init__(n_splits=6, n_test_splits=2, t1_series=None, embargo_frac=0.01)
       - Uses itertools.combinations(range(n_splits), n_test_splits) for all combos
       - get_n_splits() -> len(self._combos) = C(n_splits, n_test_splits)
       - split(): for each combo, test = union of combo fold groups, train = complement minus purge/embargo
       - Purge uses min(test_start_timestamps) across all test groups in combo
       - Embargo after last test group only (max test_end across groups)

    NOTE: Do NOT inherit from or modify the existing `splitters.py` (which has expanding_walk_forward and fixed_date_splits). The cv.py module is a new, independent file.

    NOTE: This is library-only code. Do NOT create any CLI scripts, pipeline integrations, or wiring into backtest_from_signals.py. The splitters are consumed by callers in future phases.

    Follow TDD: write failing tests first (RED), implement to pass (GREEN), refactor if needed.
  </implementation>
</feature>
