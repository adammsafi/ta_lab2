  <name>Task 1: Create Alembic revision for psr column rename</name>
  <files>alembic/versions/XXXX_psr_column_rename.py</files>
  <action>
Generate a new Alembic revision using `alembic revision -m "psr_column_rename"` from the project root. The filename will contain an auto-generated revision hash (XXXX is a placeholder).

In the generated file, implement upgrade():
1. Query information_schema.columns to check if `psr` column exists on `cmc_backtest_metrics`
2. If it exists: rename `psr` to `psr_legacy` using `op.alter_column('cmc_backtest_metrics', 'psr', new_column_name='psr_legacy', schema='public')`
3. If it does NOT exist: add `psr_legacy` as new nullable NUMERIC column
4. In both cases: add new `psr` column as nullable NUMERIC

Implement downgrade():
CRITICAL: The `psr` column does NOT currently exist in the database. The upgrade has two branches (rename existing vs create fresh), so the downgrade must be safe regardless of which branch ran. Use unconditional drops:
1. `op.execute("ALTER TABLE cmc_backtest_metrics DROP COLUMN IF EXISTS psr")` -- drop the new psr column
2. `op.execute("ALTER TABLE cmc_backtest_metrics DROP COLUMN IF EXISTS psr_legacy")` -- drop psr_legacy

Do NOT rename psr_legacy back to psr -- that would create a phantom `psr` column that never existed in the pre-migration state. The goal of downgrade is to return the table to its exact pre-migration state: no `psr` column, no `psr_legacy` column.

CRITICAL: The `down_revision` must be `'25f2b3c90f65'` (the baseline). Use `sa.Column('psr_legacy', sa.Numeric(), nullable=True)` and `sa.Column('psr', sa.Numeric(), nullable=True)`.

Use `from alembic import op` and `import sqlalchemy as sa` and `from sqlalchemy import text`. The bind for information_schema query is `op.get_bind()`.
  </action>
