Create `src/ta_lab2/scripts/backtests/compute_psr.py` as a standalone CLI script for computing/recomputing PSR on historical backtest runs.

The script must:

1. Accept CLI args via argparse:
   - `--run-id UUID` : compute PSR for a single run
   - `--all` : compute PSR for all runs that do NOT have a psr_results row yet
   - `--recompute` : force recompute even if psr_results row exists (use with --all)
   - `--sr-star FLOAT` : benchmark Sharpe (default 0.0, annualized -- converted internally to per-bar)
   - `--dry-run` : print what would be computed, do not write to DB

2. **Return reconstruction from cmc_backtest_trades (NOT price pct_change):**

   For each run_id, reconstruct a bar-level return series from trade data. This approach accounts for cash periods (when not in a trade, return = 0), unlike simple price pct_change which assumes always-invested.

   Implementation:
   a. Load run metadata: `SELECT asset_id, start_ts, end_ts FROM cmc_backtest_runs WHERE run_id = :run_id`
   b. Load price bars for the date range: `SELECT ts FROM cmc_features WHERE id = :asset_id AND tf = '1D' AND ts BETWEEN :start_ts AND :end_ts ORDER BY ts` (to get the bar timestamps)
   c. Load trades: `SELECT entry_ts, exit_ts, pnl_pct FROM cmc_backtest_trades WHERE run_id = :run_id ORDER BY entry_ts`
   d. Build bar-level return series:
      ```python
      import numpy as np
      import pandas as pd

      def reconstruct_bar_returns(bar_timestamps, trades_df):
          """
          Build per-bar return series from trade records.

          For bars within a trade's [entry_ts, exit_ts] range:
            spread the trade's pnl_pct evenly across holding bars.
          For bars outside any trade: return = 0 (cash).

          Args:
              bar_timestamps: pd.DatetimeIndex of bar timestamps
              trades_df: DataFrame with entry_ts, exit_ts, pnl_pct columns

          Returns:
              pd.Series of per-bar returns indexed by bar_timestamps
          """
          returns = pd.Series(0.0, index=bar_timestamps)
          for _, trade in trades_df.iterrows():
              # Find bars within this trade's holding period
              mask = (bar_timestamps >= trade["entry_ts"]) & (bar_timestamps <= trade["exit_ts"])
              n_holding_bars = mask.sum()
              if n_holding_bars > 0 and trade["pnl_pct"] is not None:
                  # Spread trade pnl_pct evenly across holding bars
                  per_bar_ret = trade["pnl_pct"] / n_holding_bars
                  returns.loc[mask] = per_bar_ret
          return returns
      ```
   e. If no trades exist for the run, log a warning and skip.
   f. Compute PSR: `compute_psr(returns.values, sr_star=sr_star_per_bar)`
   g. Compute MinTRL: `min_trl(returns.values, sr_star=sr_star_per_bar, target_psr=0.95, freq_per_year=365)`

3. Write to psr_results with `return_source='trade_reconstruction'`:
   - ON CONFLICT DO UPDATE on (run_id, formula_version)
   - Set return_source = 'trade_reconstruction' to distinguish from auto-compute's 'portfolio'
   - Also update cmc_backtest_metrics.psr column

4. Use NullPool for DB connection (matching project pattern).
5. Use resolve_db_url() from refresh_utils.
6. Log summary at end: "Computed PSR for N runs, skipped M (already computed), failed K"

Invocation pattern: `python -m ta_lab2.scripts.backtests.compute_psr --run-id <UUID>` or `python -m ta_lab2.scripts.backtests.compute_psr --all`

Handle the sr_star conversion: if user passes --sr-star 1.0 (annual), convert to per-bar: `sr_star_per_bar = sr_star / np.sqrt(365)` (for daily backtests with freq_per_year=365).

IMPORTANT: The `reconstruct_bar_returns` function produces an approximation of the portfolio returns. It differs from `pf.returns()` (which accounts for exact position sizing, fees deducted per-bar, and partial fills). This difference is acceptable and is made explicit via the `return_source` column.
