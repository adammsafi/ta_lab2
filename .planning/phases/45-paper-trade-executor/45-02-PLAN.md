Implement FillSimulator following RESEARCH.md "Recommended Slippage Model":

compute_fill_price algorithm:
1. If slippage_mode == "zero": return base_price unchanged
2. If slippage_mode == "fixed": base_bps only (no noise)
   - offset = float(base_price) * (slippage_base_bps / 10_000)
   - buy: base_price + offset; sell: base_price - offset
3. If slippage_mode == "lognormal":
   - base_bps = slippage_base_bps * (1 + volume_impact_factor * order_fraction)
   - noise = rng.lognormal(mean=0.0, sigma=slippage_noise_sigma)
   - total_bps = base_bps * noise
   - offset = float(base_price) * (total_bps / 10_000)
   - buy: + offset (adverse); sell: - offset (adverse)
4. Return Decimal(str(round(fill, 8)))

simulate_fill algorithm:
1. Check rejection: if rng.random() < rejection_rate, return None
2. Compute fill_price via compute_fill_price
3. Check partial fill: if rng.random() < partial_fill_rate:
   - fill_pct = partial_fill_min_pct + rng.random() * (1 - partial_fill_min_pct)
   - fill_qty = Decimal(str(round(float(order_qty) * fill_pct, 8)))
   - is_partial = True
4. Else: fill_qty = order_qty, is_partial = False
5. Return FillResult(fill_qty, fill_price, is_partial)

CRITICAL: Use numpy.random.default_rng(seed=config.seed) for all random calls.
CRITICAL: Prices in Decimal. Intermediate float math is OK for slippage computation but final price must be Decimal(str(round(...))).
