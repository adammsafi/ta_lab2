Create the `scripts/integration/` package with `__init__.py` (empty) and `smoke_test.py`.

Follow the `run_preflight_check.py` pattern exactly:
- namedtuple-based check results with PASS/WARN/FAIL status
- NullPool engine (project convention for CLI scripts)
- resolve_db_url() from refresh_utils for DB URL resolution
- Exit 0 = all pass, Exit 1 = any fail
- ASCII-only output (NO UTF-8 box-drawing chars -- Windows cp1252 safety)
- All file I/O with encoding='utf-8'

Test assets: BTC (id=1), ETH (id=52), plus 2 dynamically discovered from dim_executor_config (query at runtime, fallback to hardcoded [825, 5426] if dim_executor_config has fewer than 2 additional active assets).

Define SmokeCheck namedtuple with (name, query, validator) fields.
Define SmokeResult namedtuple with (name, status, detail) fields.

Validator helpers:
- `_val_count_gte(expected, label)`: row[0] >= expected
- `_val_count_zero(label)`: row[0] == 0
- `_val_stale_check(label, hours=48)`: MAX(ts) >= NOW() - hours (use pd.Timestamp for tz handling, same as preflight)
- `_val_range(col_label, lo, hi)`: lo <= row[0] <= hi

Pipeline stage checks (9 stages, ~25-30 checks total):

1. **bars** (price_bars_multi_tf_u):
   - Non-zero row count for test assets WHERE tf='1D' AND ts >= NOW()-48h
   - No NULL close values
   - close > 0

2. **emas** (ema_multi_tf_u):
   - Non-zero row count for test assets WHERE tf='1D' AND ts >= NOW()-48h
   - d1 > 0, d2 IS NOT NULL

3. **features** (features):
   - Non-zero row count for test assets WHERE tf='1D' AND ts >= NOW()-48h
   - No all-NULL feature rows (check a known column like rsi_14 IS NOT NULL for at least 1 row)

4. **garch** (garch_forecasts, garch_forecasts_latest):
   - garch_forecasts has rows for BTC (id=1) with created_at >= NOW()-48h
   - garch_forecasts_latest: BTC forecast_vol_1d > 0 AND < 5.0

5. **signals** (signals_ema_crossover):
   - Non-zero row count for test assets with ts >= NOW()-48h
   - signal_value IN (-1, 0, 1) -- no out-of-range values

6. **stop_calibrations** (stop_calibrations):
   - Row count >= 1
   - sl_p50 > 0 AND sl_p50 < 1.0 for any row

7. **portfolio_allocations** (portfolio_allocations):
   - Row count >= 1
   - weight_mv BETWEEN -1 AND 1 for all rows

8. **executor** (orders, fills):
   - orders table accessible (COUNT(*) >= 0 -- existence check, not recency)
   - fills table: if rows exist, fill_price > 0

9. **drift** (drift_metrics):
   - Table accessible (COUNT(*) >= 0 -- existence check)
   - If rows exist: tracking_error_pct >= 0

Add a Step 0 check group that verifies prerequisites:
- DB connectivity (simple SELECT 1)
- Alembic at head (query alembic_version table, compare to expected latest revision -- or just verify the table exists with a row; exact head comparison is fragile)

CLI flags:
- `--db-url`: Override database URL
- `--verbose`: Print all check details (not just failures)
- `--ids`: Comma-separated asset IDs to test (overrides default BTC+ETH+discovered)

Main function: iterate all checks, collect results, print summary table, exit with appropriate code.

Print format:
```
=== v1.2.0 Pipeline Smoke Test ===

[PASS] BTC price bars: non-zero row count (42 recent 1D bars)
[PASS] BTC price bars: no NULL close (0 NULL close prices)
[FAIL] GARCH forecasts: BTC has recent forecast (no rows for garch_forecasts BTC horizon-1)
...

============================================================
SMOKE TEST RESULT: 24/25 PASS, 1 FAIL
============================================================
```
