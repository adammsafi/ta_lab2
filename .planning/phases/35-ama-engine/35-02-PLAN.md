Create pure computation functions for all 4 AMA indicators. These take numpy arrays or pandas Series as input and return computed values. No DB code, no side effects.

**compute_kama(close: np.ndarray, er_period: int, fast_period: int, slow_period: int) -> tuple[np.ndarray, np.ndarray]**
- Returns (kama, er) arrays, both length len(close)
- Initialize with np.full(n, np.nan) for warmup guard
- ER[i] = abs(close[i] - close[i - er_period + 1]) / sum(abs(diff(close[i-er_period+1:i+1])))
  - When volatility == 0: ER = 0.0
- SC[i] = (ER[i] * (fast_sc - slow_sc) + slow_sc) ** 2
  - fast_sc = 2.0 / (fast_period + 1), slow_sc = 2.0 / (slow_period + 1)
- kama[er_period - 1] = close[er_period - 1] (seed)
- kama[i] = kama[i-1] + SC[i] * (close[i] - kama[i-1])
- NOTE: ER indexing starts at er_period - 1 (0-based). KAMA starts at same index.

**compute_dema(close: pd.Series, period: int) -> pd.Series**
- alpha = 2.0 / (period + 1) -- matches existing EMA convention in ema_operations.py
- ema1 = close.ewm(alpha=alpha, adjust=False).mean()
- ema2 = ema1.ewm(alpha=alpha, adjust=False).mean()
- dema = 2 * ema1 - ema2
- Warmup guard: dema.iloc[:max(0, 2 * period - 1)] = np.nan
- Return dema

**compute_tema(close: pd.Series, period: int) -> pd.Series**
- Same alpha convention
- ema1, ema2, ema3 = successive ewm applications
- tema = 3 * ema1 - 3 * ema2 + ema3
- Warmup guard: tema.iloc[:max(0, 3 * period - 1)] = np.nan
- Return tema

**compute_hma(close: pd.Series, period: int) -> pd.Series**
- Internal helper `_wma(series: pd.Series, period: int) -> pd.Series`:
  - weights = np.arange(1, period + 1, dtype=float)
  - Use series.rolling(window=period, min_periods=period).apply(lambda x: np.dot(x, weights) / weights.sum(), raw=True)
- half_period = max(1, int(period / 2))
- sqrt_period = max(2, int(math.sqrt(period)))
- hma = _wma(2 * _wma(close, half_period) - _wma(close, period), sqrt_period)
- NaN naturally from rolling min_periods, no additional guard needed
- Return hma
- CRITICAL: Use rolling().apply() with raw=True, NOT ewm(). HMA is WMA-based.

Also include a dispatcher function:
**compute_ama(close: pd.Series, indicator: str, params: dict) -> tuple[pd.Series, pd.Series | None]**
- Dispatches to the correct function based on indicator name
- Returns (ama_values, er_or_none) -- er is only non-None for KAMA
- For KAMA: convert close to np.ndarray, call compute_kama, wrap results back in pd.Series
- For DEMA/TEMA/HMA: call respective function, return (result, None)

Import math for sqrt in HMA. Import numpy and pandas.
