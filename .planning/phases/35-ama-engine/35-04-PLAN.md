**base_ama_refresher.py** - BaseAMARefresher class:

Mirrors BaseEMARefresher but adapted for AMA PK structure. Key differences:
- Instead of `periods: list[int]`, works with `param_sets: list[AMAParamSet]`
- State management uses AMAStateManager (not EMAStateManager)
- Worker tasks carry indicator + params info instead of period lists

Class `BaseAMARefresher(ABC)`:

1. `__init__(self, feature_class, state_table, output_table)`: Store references
2. `create_argument_parser()` classmethod: Standard CLI args:
   - --ids (comma-separated or "all")
   - --tf (optional, specific TF)
   - --all-tfs (all TFs from dim_timeframe)
   - --full-rebuild (clear state first)
   - --indicators (optional filter: "KAMA,DEMA" to run subset)
   - --verbose, --dry-run
   - --num-processes (default: min(6, cpu_count()))
   - --db-url

3. `run(self, args)`: Main execution flow:
   a. Resolve DB URL, create engine
   b. Ensure state table exists (AMAStateManager.ensure_state_table)
   c. Populate dim_ama_params (idempotent)
   d. Resolve IDs (from --ids arg using load_all_ids)
   e. Resolve TFs (from --tf or --all-tfs using dim_timeframe)
   f. Filter param_sets by --indicators if provided
   g. If --full-rebuild: clear state for affected IDs
   h. For each (asset_id, tf) combination:
      - For each param_set:
        - Load state watermark
        - If no new bars since watermark: skip
      - Compute AMAs via feature_class.compute_for_asset_tf()
      - Add derivatives via feature_class.add_derivatives()
      - Write to DB via feature_class.write_to_db()
      - Update state watermarks
   i. Use multiprocessing with NullPool for parallel asset processing

4. Worker function pattern: Process one asset across all TFs and param_sets. Each worker creates its own DB engine with NullPool (required for multiprocessing).

**refresh_cmc_ama_multi_tf.py** - Concrete refresher script:

Minimal script that:
1. Creates a MultiTFAMARefresher (subclass of BaseAMARefresher)
2. Configures it with:
   - feature_class = MultiTFAMAFeature
   - state_table = "public.cmc_ama_multi_tf_state"
   - output_table = "public.cmc_ama_multi_tf"
3. Parses CLI args and calls run()

Should be runnable as:
- `python -m ta_lab2.scripts.amas.refresh_cmc_ama_multi_tf --ids 1 --tf 1D`
- `python -m ta_lab2.scripts.amas.refresh_cmc_ama_multi_tf --ids all --all-tfs`
- `python -m ta_lab2.scripts.amas.refresh_cmc_ama_multi_tf --ids 1 --all-tfs --indicators KAMA`

Import from existing utilities:
- `from ta_lab2.scripts.bars.common_snapshot_contract import load_all_ids`
- `from ta_lab2.scripts.refresh_utils import resolve_db_url`
- `from ta_lab2.scripts.emas.logging_config import setup_logging, add_logging_args`

CRITICAL: Workers MUST use NullPool: `create_engine(db_url, poolclass=NullPool)`. This prevents "too many clients" in multiprocessing.
