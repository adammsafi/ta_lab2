Create BaseAMAFeature as a sibling of BaseEMAFeature (NOT a subclass — different PK columns require different _get_pk_columns() and _pg_upsert() logic). Read `src/ta_lab2/features/m_tf/base_ema_feature.py` for the structural template.

Class design:

```python
class BaseAMAFeature(ABC):
    """Abstract base for AMA feature computation modules."""
```

**Config dataclass** `AMAFeatureConfig`:
- `param_sets: list[AMAParamSet]` (from ama_params.py)
- `output_schema: str`
- `output_table: str`

**Abstract methods** (subclasses implement):
- `_load_bars(engine, asset_id, tf, tf_days, start_ts)` -> DataFrame with close prices
- `_get_timeframes(engine)` -> list of TFSpec
- `_get_source_table_info()` -> dict

**Concrete methods** (provided by base):

1. `compute_for_asset_tf(engine, asset_id, tf, tf_days, param_sets, start_ts=None) -> pd.DataFrame`:
   - Load bars via `_load_bars()`
   - For each param_set in param_sets:
     - Call `compute_ama(close_series, param_set.indicator, param_set.params)` from ama_computations.py
     - Get (ama_values, er_or_none)
     - Build rows with: id, ts, tf, indicator, params_hash, tf_days, roll, ama, er
   - Return combined DataFrame for all param_sets

2. `add_derivatives(df: pd.DataFrame) -> pd.DataFrame`:
   - Group by (id, tf, indicator, params_hash, roll)
   - Within each group, compute:
     - d1 = ama.diff(1), d2 = d1.diff(1)
     - For non-roll (roll=FALSE): d1, d2 on canonical rows only
     - For all rows: d1_roll, d2_roll on the full timeline (sorted by ts)
   - Mirror the derivative logic from `ema_operations.py:compute_derivatives()` and `compute_rolling_derivatives_canonical()`

3. `write_to_db(engine, df: pd.DataFrame, schema: str, table: str)`:
   - Use scoped DELETE + INSERT per (ids, tf) batch (matching the Feature write pattern from MEMORY.md)
   - Filter DataFrame columns to match actual DB columns via `_get_table_columns(engine, schema, table)` query against information_schema
   - DELETE FROM table WHERE id IN (...) AND tf = ... then INSERT
   - Use `_pg_upsert()` with ON CONFLICT (id, ts, tf, indicator, params_hash) DO UPDATE for safety

4. `_get_table_columns(engine, schema, table) -> list[str]`: query information_schema.columns for column names (same pattern as BaseEMAFeature)

5. `_get_pk_columns() -> list[str]`: returns `["id", "ts", "tf", "indicator", "params_hash"]`

CRITICAL pitfalls to handle:
- tz-aware timestamps: use `.tolist()` for tz-aware objects, NOT `.values` (returns tz-naive on Windows)
- Workers use NullPool
- The `er` column is NULL for non-KAMA indicators — make sure to set er=np.nan when indicator != "KAMA"

**__init__.py**: Empty file.

**ama_state_manager.py**: Mirror EMAStateManager but with (id, tf, indicator, params_hash) PK instead of (id, tf, period).

Class `AMAStateManager`:
- `__init__(self, engine: Engine, state_table: str)`: Store engine and state table name. The state table is specified by the caller (each refresher has its own state table: cmc_ama_multi_tf_state, cmc_ama_multi_tf_cal_us_state, etc.)

- `ensure_state_table(self)`: CREATE TABLE IF NOT EXISTS with DDL:
  ```sql
  CREATE TABLE IF NOT EXISTS {state_table} (
      id INTEGER NOT NULL, tf TEXT NOT NULL,
      indicator TEXT NOT NULL, params_hash TEXT NOT NULL,
      last_canonical_ts TIMESTAMPTZ,
      updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
      PRIMARY KEY (id, tf, indicator, params_hash)
  );
  ```

- `load_state(self, asset_id: int, tf: str, indicator: str, params_hash: str) -> Optional[datetime]`:
  - SELECT last_canonical_ts FROM state_table WHERE id=:id AND tf=:tf AND indicator=:indicator AND params_hash=:params_hash
  - Return the timestamp or None
  - CRITICAL: Use `.tolist()` or explicit UTC handling on the returned timestamp (Windows tz pitfall)

- `save_state(self, asset_id: int, tf: str, indicator: str, params_hash: str, last_ts: datetime)`:
  - INSERT INTO state_table ... ON CONFLICT (id, tf, indicator, params_hash) DO UPDATE SET last_canonical_ts = EXCLUDED.last_canonical_ts, updated_at = NOW()

- `load_all_states(self, asset_id: int) -> pd.DataFrame`:
  - SELECT * FROM state_table WHERE id = :id
  - Returns DataFrame with tf, indicator, params_hash, last_canonical_ts columns

- `clear_state(self, asset_id: int, tf: str | None = None, indicator: str | None = None)`:
  - DELETE FROM state_table WHERE id = :id (and optionally tf/indicator filters)
  - Used for --full-rebuild scenarios

CRITICAL: Do NOT import or reuse EMAStateManager. Create a standalone class with AMA-specific PK columns. The EMAStateManager hardcodes `period INTEGER` in its DDL.
