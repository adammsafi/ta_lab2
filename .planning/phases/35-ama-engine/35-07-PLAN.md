  <files>src/ta_lab2/scripts/returns/refresh_returns_zscore.py</files>
  <action>
Extend the existing z-score script to support AMA returns tables. Read the existing file to understand the _BAR_TABLES and _EMA_TABLES patterns, then add _AMA_TABLES following the same structure.

Add these constants after the existing _EMA_TABLES list:

```python
# -- AMA returns tables (6) -------------------------------------------------

_AMA_CANONICAL_BASE = [
    ("ret_arith_ama", "ret_arith_ama_zscore"),
    ("ret_log_ama", "ret_log_ama_zscore"),
]

_AMA_ROLL_BASE = [
    ("ret_arith_ama_roll", "ret_arith_ama_roll_zscore"),
    ("ret_log_ama_roll", "ret_log_ama_roll_zscore"),
]
```

Then define _AMA_TABLES as a list of 6 TableConfig objects:
1. cmc_returns_ama_multi_tf -- pk_cols: ["id", "ts", "tf", "indicator", "params_hash"], key_cols: ["id", "tf", "indicator", "params_hash"], ts_col: "ts"
2. cmc_returns_ama_multi_tf_u -- pk_cols adds "alignment_source", key_cols adds "alignment_source"
3. cmc_returns_ama_multi_tf_cal_us -- same pk/key pattern as #1
4. cmc_returns_ama_multi_tf_cal_iso
5. cmc_returns_ama_multi_tf_cal_anchor_us
6. cmc_returns_ama_multi_tf_cal_anchor_iso

CRITICAL: key_cols MUST include "indicator" and "params_hash" -- this ensures z-scores are computed per (id, tf, indicator, params_hash) group. Without these, z-scores would aggregate across different AMA types and parameter sets, producing garbage.

Extend the --tables CLI choices:
- Add "amas" to the choices list (currently: "bars", "emas", "all")
- Update "all" to include _AMA_TABLES
- Add elif block for "amas" that selects _AMA_TABLES

AMA returns have only 4 base z-score pairs (2 canonical + 2 roll) per window, vs EMA's 8 (4+4). This is correct -- AMA has no _ema_bar columns.

Update the docstring to mention AMA z-scores.
