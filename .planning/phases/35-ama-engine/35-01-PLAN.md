**create_cmc_ama_multi_tf.sql** - Main value table + state table:
- Table `cmc_ama_multi_tf` with columns: id INTEGER NOT NULL, tf TEXT NOT NULL, ts TIMESTAMPTZ NOT NULL, indicator TEXT NOT NULL, params_hash TEXT NOT NULL, tf_days INTEGER, roll BOOLEAN NOT NULL DEFAULT FALSE, ama DOUBLE PRECISION, d1 DOUBLE PRECISION, d2 DOUBLE PRECISION, d1_roll DOUBLE PRECISION, d2_roll DOUBLE PRECISION, er DOUBLE PRECISION, is_partial_end BOOLEAN, ingested_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
- PK: (id, ts, tf, indicator, params_hash)
- Index: ix_cmc_ama_multi_tf_indicator ON (indicator, params_hash, tf, ts)
- Index: ix_cmc_ama_multi_tf_roll_true ON (id, tf, indicator, params_hash, ts) WHERE roll = TRUE
- State table `cmc_ama_multi_tf_state` with columns: id INTEGER NOT NULL, tf TEXT NOT NULL, indicator TEXT NOT NULL, params_hash TEXT NOT NULL, last_canonical_ts TIMESTAMPTZ, updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(). PK: (id, tf, indicator, params_hash)

**create_cmc_ama_multi_tf_cal.sql** - Calendar variant tables (cal_us + cal_iso):
- Two tables: `cmc_ama_multi_tf_cal_us` and `cmc_ama_multi_tf_cal_iso`
- Same column schema as main table
- Each has its own state table: `cmc_ama_multi_tf_cal_us_state`, `cmc_ama_multi_tf_cal_iso_state`
- Same indexes with adjusted names

**create_cmc_ama_multi_tf_cal_anchor.sql** - Anchor variant tables:
- Two tables: `cmc_ama_multi_tf_cal_anchor_us` and `cmc_ama_multi_tf_cal_anchor_iso`
- Same column schema + state tables + indexes

**create_cmc_ama_multi_tf_u.sql** - Unified value table:
- Table `cmc_ama_multi_tf_u` with same columns as main table PLUS `alignment_source TEXT NOT NULL DEFAULT 'multi_tf'`
- PK: (id, ts, tf, indicator, params_hash, alignment_source) -- alignment_source IN PK on _u tables
- Index on (alignment_source, id, tf, ts)

CRITICAL: No `period` column anywhere. No `ema_bar` column. AMA tables have `ama` column, not `ema`. The `er` column is for KAMA's Efficiency Ratio (NULL for non-KAMA indicators). Wrap each file in BEGIN/COMMIT for transactional DDL.

No `_ema_bar` columns (AMA has no bar-space variant per research open question resolution).

**create_cmc_returns_ama_multi_tf.sql** - Main returns table + state table:
- PK: (id, ts, tf, indicator, params_hash)
- Columns pattern (mirror EMA returns exactly but with _ama suffix instead of _ema):
  - gap_days, gap_days_roll
  - Canonical (roll=FALSE): delta1_ama, delta2_ama, ret_arith_ama, delta_ret_arith_ama, ret_log_ama, delta_ret_log_ama
  - Roll (ALL rows): delta1_ama_roll, delta2_ama_roll, ret_arith_ama_roll, delta_ret_arith_ama_roll, ret_log_ama_roll, delta_ret_log_ama_roll
  - Z-scores 30/90/365 for: ret_arith_ama, ret_log_ama (canonical) + ret_arith_ama_roll, ret_log_ama_roll (roll) = 4 base x 3 windows = 12 z-score columns
  - is_outlier BOOLEAN
  - ingested_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
- State table: cmc_returns_ama_multi_tf_state with PK (id, tf, indicator, params_hash)

**create_cmc_returns_ama_multi_tf_cal.sql** - cal_us + cal_iso returns tables + state tables

**create_cmc_returns_ama_multi_tf_cal_anchor.sql** - cal_anchor_us + cal_anchor_iso returns tables + state tables

**create_cmc_returns_ama_multi_tf_u.sql** - Unified returns table:
- Same as main returns + alignment_source TEXT NOT NULL DEFAULT 'multi_tf'
- PK: (id, ts, tf, indicator, params_hash, alignment_source)

**create_dim_ama_params.sql** - Parameter lookup:
- Table `dim_ama_params` with columns: indicator TEXT NOT NULL, params_hash TEXT NOT NULL, params_json JSONB NOT NULL, label TEXT, created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
- PK: (indicator, params_hash)
- This allows human-readable queries: SELECT label FROM dim_ama_params WHERE indicator='KAMA' AND params_hash='...'

IMPORTANT: AMA returns have NO `_ema_bar` columns. Only `_ama` columns. This simplifies the DDL vs EMA returns which have both `_ema` and `_ema_bar` column families.
