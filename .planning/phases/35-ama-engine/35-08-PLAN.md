<task type="auto">
  <name>Task 1: Create run_all_ama_refreshes.py orchestrator</name>
  <files>src/ta_lab2/scripts/amas/run_all_ama_refreshes.py</files>
  <action>
Create the all-in-one AMA orchestrator. Mirror `src/ta_lab2/scripts/emas/run_all_ema_refreshes.py` structure. This script runs the complete AMA pipeline in logical order:

1. AMA values (multi_tf)
2. AMA values (cal us/iso)
3. AMA values (cal_anchor us/iso)
4. AMA returns (all sources)
5. AMA _u sync (values)
6. AMA returns _u sync
7. Z-scores (AMA returns)

**RefresherConfig** for AMA value refreshers:
```python
ALL_AMA_VALUE_REFRESHERS = [
    RefresherConfig(name="multi_tf", script_path="refresh_cmc_ama_multi_tf.py", description="Multi-TF AMAs"),
    RefresherConfig(name="cal", script_path="refresh_cmc_ama_multi_tf_cal_from_bars.py", description="Calendar AMAs", supports_scheme=True),
    RefresherConfig(name="cal_anchor", script_path="refresh_cmc_ama_multi_tf_cal_anchor_from_bars.py", description="Calendar anchor AMAs", supports_scheme=True),
]
```

**Post-value steps** (run after ALL value refreshers complete):
```python
POST_STEPS = [
    PostStep(name="returns", script="refresh_cmc_returns_ama.py", description="AMA returns"),
    PostStep(name="sync_values", script="sync_cmc_ama_multi_tf_u.py", description="Sync AMA values to _u"),
    PostStep(name="sync_returns", script="sync_cmc_returns_ama_multi_tf_u.py", description="Sync AMA returns to _u"),
    PostStep(name="zscores", module="ta_lab2.scripts.returns.refresh_returns_zscore", args=["--tables", "amas"], description="AMA z-scores"),
]
```

**CLI arguments** (mirror EMA orchestrator):
- --ids: comma-separated or "all"
- --tf: specific TF
- --all-tfs: all TFs
- --only: run specific refreshers (multi_tf, cal, cal_anchor)
- --num-processes / -n: parallel workers
- --full-rebuild: clear state
- --continue-on-error: keep going on failure
- --dry-run, --verbose
- --db-url
- --indicators: filter to specific AMA types

**Execution flow:**
1. Parse args
2. For each value refresher in order (or filtered by --only):
   - Build subprocess command with forwarded args (--ids, --tf, --all-tfs, etc.)
   - Run via subprocess.run() with timeout (TIMEOUT_AMAS = 3600)
   - Track RefresherResult (name, success, duration, returncode)
   - If failure and not --continue-on-error: stop
3. If any value refreshers succeeded, run post-steps:
   - Returns: pass same --ids, --tf args
   - Sync: run each sync script
   - Z-scores: run refresh_returns_zscore with --tables amas
4. Print summary of all steps

Runnable as:
- `python -m ta_lab2.scripts.amas.run_all_ama_refreshes --ids all --all-tfs`
- `python -m ta_lab2.scripts.amas.run_all_ama_refreshes --ids 1 --tf 1D --only multi_tf`

Subprocess timeout: Use TIMEOUT_AMAS = 3600 (1 hour, matching EMA tier).

       script_dir = Path(__file__).parent / "amas"
       cmd = [sys.executable, str(script_dir / "run_all_ama_refreshes.py")]
       # Forward IDs
       if ids_for_amas is None:
           ids_str = "all"
       elif len(ids_for_amas) == 0:
           print("[INFO] No IDs - skipping AMA refresh")
           return ComponentResult(component="amas", success=True, duration_sec=0.0, returncode=0)
       else:
           ids_str = ",".join(str(i) for i in ids_for_amas)
       cmd.extend(["--ids", ids_str])
       # Forward --all-tfs (AMAs always run all TFs in daily refresh)
       cmd.append("--all-tfs")
       if args.verbose: cmd.append("--verbose")
       if args.num_processes: cmd.extend(["--num-processes", str(args.num_processes)])
       if getattr(args, "dry_run", False): cmd.append("--dry-run")
       # ... subprocess.run() pattern identical to run_ema_refreshers
