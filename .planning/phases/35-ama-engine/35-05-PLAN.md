  <action>
Create AMAReturnsFeature that reads AMA values from a source table and computes returns. Read the EMA returns patterns from the codebase for structural reference.

Class `AMAReturnsFeature`:

**Constructor:**
- `__init__(self, source_table: str, returns_table: str, state_table: str)`:
  - source_table: e.g., "public.cmc_ama_multi_tf"
  - returns_table: e.g., "public.cmc_returns_ama_multi_tf"
  - state_table: e.g., "public.cmc_returns_ama_multi_tf_state"

**Core method: `compute_returns(self, engine, asset_id: int, tf: str) -> pd.DataFrame`:**

1. Load AMA values: SELECT id, ts, tf, tf_days, indicator, params_hash, roll, ama FROM {source_table} WHERE id = :id AND tf = :tf ORDER BY indicator, params_hash, ts

2. Group by (id, tf, indicator, params_hash) to process each param set separately.

3. For each group, compute returns using the SAME column pattern as EMA returns (but with _ama suffix instead of _ema, and NO _ema_bar columns):

   **Roll columns (ALL rows, unified timeline sorted by ts):**
   - delta1_ama_roll = ama.diff(1) -- first difference
   - delta2_ama_roll = delta1_ama_roll.diff(1) -- second difference
   - ret_arith_ama_roll = ama.pct_change(1) -- arithmetic return
   - delta_ret_arith_ama_roll = ret_arith_ama_roll.diff(1)
   - ret_log_ama_roll = np.log(ama / ama.shift(1)) -- log return
   - delta_ret_log_ama_roll = ret_log_ama_roll.diff(1)

   **Canonical columns (roll=FALSE rows only):**
   - Filter to roll=FALSE subset, compute same metrics on that subset:
   - delta1_ama, delta2_ama, ret_arith_ama, delta_ret_arith_ama, ret_log_ama, delta_ret_log_ama
   - These columns are NULL for roll=TRUE rows

   **Gap columns:**
   - gap_days_roll: (ts - ts.shift(1)).dt.total_seconds() / 86400 on all rows
   - gap_days: same but only on roll=FALSE subset

4. Combine all groups into single DataFrame with columns matching DDL exactly.

**Refresh method: `refresh(self, engine, asset_ids: list[int], tfs: list[str])`:**
- For each (asset_id, tf):
  - Compute returns
  - Write to returns_table using scoped DELETE + INSERT per (id, tf)
  - Update state table

CRITICAL: The AMA returns have NO `_ema_bar` column family. Only `_ama` columns. This means 12 return columns total (6 canonical + 6 roll), not the 24 that EMA returns have. Z-score columns (12 total) are handled by refresh_returns_zscore.py in a later plan.
