Create `validate_signal_anomalies.py` as a standalone CLI script (invokable via `python -m ta_lab2.scripts.signals.validate_signal_anomalies`).

**SignalAnomalyGate class:**

Constructor: `__init__(self, engine: Engine, lookback_days: int = 90, zscore_threshold: float = 2.0, crowded_pct: float = 0.40, cooldown_hours: int = 4)`

**Method 1: check_signal_count_anomalies()**
For each of the 3 signal tables (signals_ema_crossover, signals_rsi, signals_atr_breakout):

1. Query historical baseline: daily count of NEW signals (position_state='open') for each day in the last `lookback_days`, excluding today:
   ```sql
   SELECT DATE(ts) as day, COUNT(*) as cnt
   FROM {table}
   WHERE ts >= CURRENT_DATE - INTERVAL ':days days'
     AND ts < CURRENT_DATE
     AND position_state = 'open'
   GROUP BY DATE(ts)
   ORDER BY day
   ```

2. Compute mean and std of daily counts. If fewer than 30 days of data, fall back to full history (no lookback filter).

3. Query today's count:
   ```sql
   SELECT COUNT(*) as cnt
   FROM {table}
   WHERE ts >= CURRENT_DATE
     AND position_state = 'open'
   ```

4. Compute z-score: `(today_count - mean) / max(std, 1e-6)`. Flag if `abs(zscore) > zscore_threshold`.

5. Log result to signal_anomaly_log (anomaly_type='count_anomaly').

Return list of anomalies found.

**Method 2: check_crowded_signals()**

1. Query all open signals from TODAY across all 3 tables (UNION ALL):
   ```sql
   SELECT id as asset_id, direction
   FROM signals_ema_crossover
   WHERE ts >= CURRENT_DATE AND position_state = 'open'
   UNION ALL
   SELECT id as asset_id, direction
   FROM signals_rsi
   WHERE ts >= CURRENT_DATE AND position_state = 'open'
   UNION ALL
   SELECT id as asset_id, direction
   FROM signals_atr_breakout
   WHERE ts >= CURRENT_DATE AND position_state = 'open'
   ```

2. Group by (asset_id, direction), count signals per group.

3. Total open signals today = sum of all groups. For each group, compute pct = group_count / total.

4. Flag groups where pct > crowded_pct (default 0.40).

5. Log each flagged group to signal_anomaly_log (anomaly_type='crowded_signal').

Return list of crowded groups.

**Method 3: run_gate() -> tuple[bool, list[dict]]**

Orchestrates both checks. Returns (has_anomalies, anomaly_details).

- If ANY count anomaly or crowded signal is detected: has_anomalies=True.
- Send throttled Telegram CRITICAL alert for each anomaly:
  - Check pipeline_alert_log for recent alert with type='signal_gate_blocked', alert_key=signal_type within cooldown_hours
  - If not throttled: send via telegram.send_alert() with severity="critical"
  - Title: "Signal Gate Blocked" with emoji header
  - Message includes: which table, today's count vs baseline mean+std, z-score, or crowded pct
  - Log to pipeline_alert_log

**CLI interface** using argparse:
- `--db-url` (optional, resolve_db_url() fallback)
- `--lookback-days` (default 90)
- `--zscore-threshold` (default 2.0)
- `--crowded-pct` (default 0.40)
- `--dry-run` (compute but don't log to DB or send alerts)
- `--verbose`

**Return codes:**
- 0 = all signals clean, proceed to executor
- 1 = script error (DB failure, import error)
- 2 = anomalies detected, signals BLOCKED

**Critical anti-patterns from research:**
- Do NOT use `ts >= NOW() - INTERVAL '90 days'` for baseline (includes today's partial data). Use `ts < CURRENT_DATE`.
- Do NOT count closed signals in crowded check. Filter `position_state = 'open'`.
- Do NOT raise exceptions for anomalies. Use return codes.
- Do NOT trip the kill switch (`dim_risk_state`). This is a pre-execution gate only.

**DB safety:**
- Wrap signal_anomaly_log and pipeline_alert_log INSERTs in try/except (OperationalError, ProgrammingError) with logger.warning -- tables may not exist if migration hasn't run.
- Use `encoding='utf-8'` for any string operations.
