**Step 1: Create the baseline revision**

Run from project root:
```bash
alembic revision -m "baseline"
```

This generates `alembic/versions/<rev>_baseline.py`. The file will have auto-generated revision ID and empty upgrade()/downgrade() bodies (both `pass`).

Verify the generated file has:
- `down_revision = None` (this is the first revision)
- `upgrade()` contains only `pass` (or add a comment `# baseline no-op — existing schema unchanged`)
- `downgrade()` contains only `pass` (or add a comment `# no rollback defined for baseline`)

Do NOT add any `op.create_table()` or other DDL operations. The baseline represents the current schema as-is.

**Step 2: Verify revision chain**

Run:
```bash
alembic history
```
Expected output: `<base> -> <rev-id> (head), baseline`

This must show exactly ONE revision. If it shows more, something is wrong.

**Step 3: Stamp the production database**

Run from project root (DB must be accessible via resolve_db_url()):
```bash
alembic stamp head
```

This creates the `alembic_version` table in the production database and writes the baseline revision ID into it. No DDL is executed on any other table.

Expected output: `INFO [alembic.runtime.migration] Running stamp_revision  -> <rev-id>`

**Step 4: Verify stamp**

Run:
```bash
alembic current
```
Expected output: `<rev-id> (head)`

This confirms the database knows it is at the baseline revision.

Record the revision ID and both command outputs — these go in the SUMMARY.md.

**Part B: Add Schema Migrations section to CONTRIBUTING.md**

Append a new section `## Schema Migrations (Alembic)` to the end of CONTRIBUTING.md. Include:

1. **Quick reference** — the 5-step workflow:
   ```
   1. alembic revision -m "add_foo_to_bar"
   2. Edit upgrade() and downgrade() in the generated file
   3. git add alembic/versions/<rev>_add_foo_to_bar.py && git commit
   4. alembic upgrade head  (on the live DB)
   5. alembic current  (verify)
   ```

2. **Conventions:**
   - Write all revisions by hand (no `--autogenerate`)
   - Always implement `downgrade()` (even if just `pass` with a comment explaining why)
   - Descriptive slugs in revision messages (e.g., `add_is_outlier_to_cmc_vol`)
   - Run from project root (alembic.ini must be in CWD)

3. **Gotchas:**
   - **No autogenerate:** Without ORM models, `--autogenerate` compares against an empty schema and generates `op.create_table()` for all 50+ tables. Never use it.
   - **Windows encoding:** env.py already handles this (`encoding='utf-8'` in fileConfig). If you open SQL files in Python migration code, always use `encoding='utf-8'`.
   - **CWD matters:** Run all alembic commands from the project root where `alembic.ini` lives. Use `alembic -c /path/to/alembic.ini` if running from elsewhere.
   - **stamp vs upgrade:** `stamp` = "record where we are now (no SQL)". `upgrade` = "run pending migrations". Use `stamp` only during initial setup or disaster recovery. Use `upgrade` for applying new migrations.

4. **Historical migrations:** Reference `sql/migration/CATALOG.md` for the 17 legacy SQL files applied before Alembic was adopted.

Add a new `alembic-history` job to `.github/workflows/ci.yml`. Place it after the existing `version-check` job.

The job structure:
```yaml
  alembic-history:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: "3.12"
      - name: Install dependencies
        run: pip install -e ".[dev]"
      - name: Verify alembic revision chain
        run: alembic history
```

**Why this works without a DB:** `alembic history` reads only from the filesystem (revision files in `alembic/versions/`). It parses revision IDs and `down_revision` pointers to validate the chain is intact. No database connection is needed.

**What it catches:** Corrupted revision chain (e.g., missing `down_revision` reference), syntax errors in revision files, missing files referenced by the chain.

Do NOT add `alembic current` to CI — that requires a live database connection.
