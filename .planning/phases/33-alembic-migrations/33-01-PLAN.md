**Step 1: Run alembic init**

Run `alembic init alembic` from the project root (`C:/Users/asafi/Downloads/ta_lab2`). This generates the default alembic/ directory, alembic.ini, and template files.

NOTE: Do NOT use `--template pyproject` — the pyproject template appends a `[tool.alembic]` section to pyproject.toml which is redundant since alembic.ini already provides all config. The standard template is cleaner.

**Step 2: Customize alembic.ini**

Replace the generated alembic.ini with the following (preserving the logging sections from the generated file). Key changes from default:
- `sqlalchemy.url` set to placeholder `driver://user:pass@localhost/dbname` (real URL loaded in env.py)
- Add `prepend_sys_path = .` so project root is on sys.path
- Add `output_encoding = utf-8` for Windows compatibility
- Add `[post_write_hooks]` section for ruff auto-formatting of new revision files
- Keep all default logging configuration

The full alembic.ini content is specified in the RESEARCH.md — use that as the reference. Key sections:

```ini
[alembic]
script_location = %(here)s/alembic
prepend_sys_path = .
output_encoding = utf-8
sqlalchemy.url = driver://user:pass@localhost/dbname

[post_write_hooks]
hooks = ruff
ruff.type = exec
ruff.executable = ruff
ruff.options = check --fix REVISION_SCRIPT_FILENAME
```

**Step 3: Customize alembic/env.py**

Replace the generated env.py entirely with the project-specific version. Critical requirements:
1. `sys.path.insert(0, _PROJECT_ROOT)` to ensure refresh_utils is importable
2. `from src.ta_lab2.scripts.refresh_utils import resolve_db_url` — uses the project's existing URL resolution
3. `fileConfig(config.config_file_name, encoding="utf-8")` — MANDATORY for Windows (prevents UnicodeDecodeError with cp1252)
4. `target_metadata = None` — explicitly disables autogenerate (no ORM models)
5. `pool.NullPool` in run_migrations_online — matches project-wide pattern
6. Both `run_migrations_offline()` and `run_migrations_online()` implemented (offline costs nothing)

Use the verified env.py from RESEARCH.md (the "Verified env.py for This Project" section) as the exact template.

**Step 4: Create alembic/versions/ directory**

Ensure `alembic/versions/` directory exists (should be created by `alembic init`). If not, create it. This is where revision files will live.

**Step 5: Verify structure**

The final file tree should be:
```
alembic/
  env.py          (customized)
  script.py.mako  (default from init)
  README          (default from init)
  versions/       (empty directory)

```

Run `alembic history` from the project root. Expected: empty output (no revisions yet). Must exit 0.
