    Read all 3 generate_signals_*.py files to understand their current structure, then modify each to add optional CUSUM pre-filtering.

    **Pattern for each generator (same integration approach):**

    1. Add import at top of file:
       ```python
       from ta_lab2.labeling.cusum_filter import cusum_filter, get_cusum_threshold
       ```

    2. Add CUSUM parameters to the generator class or function signature. Each generator is a class with a `generate_for_ids()` method. Add two params to the params dict or to the method signature:
       - `cusum_enabled: bool = False`
       - `cusum_threshold_multiplier: float = 2.0`

    3. Inside the generation loop (after loading features DataFrame), add CUSUM filtering step:
       ```python
       if cusum_enabled:
           close_series = features_df['close']  # or whatever the close price column is
           threshold = get_cusum_threshold(close_series, multiplier=cusum_threshold_multiplier)
           cusum_events = cusum_filter(close_series, threshold)
           n_before = len(features_df)
           features_df = features_df.loc[features_df.index.isin(cusum_events)]
           logger.info(f"CUSUM filter: {n_before} -> {len(features_df)} bars ({len(features_df)/n_before:.1%} retained)")
       ```

    4. The rest of signal generation proceeds as before — it just operates on fewer bars.

    **Key design decisions:**
    - CUSUM filtering happens BEFORE signal generation logic, not after
    - When cusum_enabled=False (default), zero code path changes — existing behavior 100% preserved
    - The close price column name varies by generator — check each file to find the correct column
    - Log the filtering stats so users can see the reduction

    **IMPORTANT:** Read each file carefully before modifying. The 3 generators may have different class structures. The CUSUM integration point is wherever the feature DataFrame is loaded and before signals are computed. Add the cusum params alongside existing params like `regime_enabled`.
