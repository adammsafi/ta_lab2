    **Pattern for each generator (same integration approach):**

    1. Add import at top of file:
       ```python
       from ta_lab2.labeling.cusum_filter import cusum_filter, get_cusum_threshold
       ```

    2. Add CUSUM parameters to the generator class or function signature. Each generator is a class with a `generate_for_ids()` method. Add two params to the params dict or to the method signature:
       - `cusum_enabled: bool = False`
       - `cusum_threshold_multiplier: float = 2.0`

    3. Inside the generation loop (after loading features DataFrame), add CUSUM filtering step:
       ```python
       if cusum_enabled:
           close_series = features_df['close']  # or whatever the close price column is
           threshold = get_cusum_threshold(close_series, multiplier=cusum_threshold_multiplier)
           cusum_events = cusum_filter(close_series, threshold)
           n_before = len(features_df)
           features_df = features_df.loc[features_df.index.isin(cusum_events)]
           logger.info(f"CUSUM filter: {n_before} -> {len(features_df)} bars ({len(features_df)/n_before:.1%} retained)")
       ```

    4. The rest of signal generation proceeds as before -- it just operates on fewer bars.

    **For each refresh script, add to argparse:**
    ```python
    parser.add_argument(
        "--cusum",
        action="store_true",
        default=False,
        help="Enable CUSUM event filter as pre-filter (reduces noise trades)",
    )
    parser.add_argument(
        "--cusum-multiplier",
        type=float,
        default=2.0,
        help="CUSUM threshold multiplier on daily vol (default: 2.0)",
    )
    ```

    **Steps:**

    1. **Baseline (no CUSUM):**
       Run EMA crossover signal generation for BTC without CUSUM:
       ```bash
       python -m ta_lab2.scripts.signals.refresh_cmc_signals_ema_crossover --ids 1 --tf 1D --full-refresh
       ```
       Query baseline trade count and compute trade-level Sharpe:
       ```python
       import pandas as pd
       from sqlalchemy import create_engine, text
       import numpy as np
       import os

       engine = create_engine(os.environ["TARGET_DB_URL"])
       with engine.connect() as conn:
           df = pd.read_sql(text(
               "SELECT pnl_pct FROM cmc_signals_ema_crossover "
               "WHERE id = 1 AND position_state = 'closed'"
           ), conn)
       baseline_n = len(df)
       baseline_sharpe = df['pnl_pct'].mean() / df['pnl_pct'].std() * np.sqrt(baseline_n) if len(df) > 1 else 0.0
       print(f"Baseline: {baseline_n} trades, Sharpe={baseline_sharpe:.3f}")
       ```
       Record: baseline_trade_count, baseline_sharpe

    2. **CUSUM-filtered:**
       Run with CUSUM enabled:
       ```bash
       python -m ta_lab2.scripts.signals.refresh_cmc_signals_ema_crossover --ids 1 --tf 1D --full-refresh --cusum --cusum-multiplier 2.0
       ```
       Query CUSUM trade count and Sharpe using the same Python approach as step 1.
       Record: cusum_trade_count, cusum_sharpe

    3. **Compute and log comparison:**
       ```
       trade_reduction = (baseline_trade_count - cusum_trade_count) / baseline_trade_count
       sharpe_change = cusum_sharpe - baseline_sharpe
       ```
       Log: "CUSUM A/B: trades {baseline} -> {cusum} ({reduction:.1%} reduction), Sharpe {baseline:.3f} -> {cusum:.3f}"

    4. **Acceptance:** trade_reduction should be 20-40%. If outside range, try adjusting cusum_multiplier (1.5 or 3.0) and document the optimal multiplier. The Sharpe should be maintained or improved (cusum_sharpe >= baseline_sharpe * 0.9 is acceptable).

    5. **Restore baseline signals** after comparison by re-running without CUSUM:
       ```bash
       python -m ta_lab2.scripts.signals.refresh_cmc_signals_ema_crossover --ids 1 --tf 1D --full-refresh
       ```
