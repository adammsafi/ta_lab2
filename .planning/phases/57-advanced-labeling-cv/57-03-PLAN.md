    Create `src/ta_lab2/scripts/labeling/` package.

    **__init__.py:** Empty or minimal (no special exports needed for scripts).

    **refresh_triple_barrier_labels.py** — follows pattern from `refresh_cmc_signals_ema_crossover.py`:

    **CLI arguments (argparse):**
    - `--ids`: Comma-separated asset IDs (e.g., '1,52,1027')
    - `--all`: Process all assets in dim_listings
    - `--tf`: Timeframe (default '1D')
    - `--pt`: Profit-taking multiplier (default 1.0)
    - `--sl`: Stop-loss multiplier (default 1.0)
    - `--vertical-bars`: Vertical barrier in bars (default 10)
    - `--vol-span`: EWM vol span (default 100)
    - `--full-refresh`: Delete existing labels for these params before inserting
    - `--dry-run`: Compute but don't write to DB
    - `--cusum-filter`: Optional flag to use CUSUM filter for event selection instead of every bar
    - `--cusum-multiplier`: CUSUM threshold multiplier (default 2.0, only used with --cusum-filter)

    **Main flow:**
    1. Parse args, create engine (NullPool for one-shot connections)
    2. Resolve asset IDs (from --ids or --all via dim_listings query)
    3. For each asset_id:
       a. Load close prices: `SELECT ts, close FROM cmc_price_bars_multi_tf_u WHERE id = :id AND tf = :tf ORDER BY ts`
       b. Build close Series with tz-aware UTC DatetimeIndex
       c. Compute `vol = get_daily_vol(close, span=args.vol_span)`
       d. Determine event timestamps:
          - If `--cusum-filter`: use `cusum_filter(close, get_cusum_threshold(close, args.cusum_multiplier))` from cusum_filter module
          - Else: use every bar where vol is not NaN (i.e., `vol.dropna().index`)
       e. Compute `labels = apply_triple_barriers(close, events, pt_sl=[args.pt, args.sl], target=vol, num_bars=args.vertical_bars)`
       f. Build DataFrame for DB insert with columns matching cmc_triple_barrier_labels schema
       g. Log: asset_id, n_events, label distribution {+1: N, -1: N, 0: N}

    4. Write to DB using temp table + upsert pattern:
       - Create temp table
       - COPY DataFrame to temp
       - `INSERT INTO cmc_triple_barrier_labels SELECT ... FROM temp ON CONFLICT ON CONSTRAINT uq_triple_barrier_key DO UPDATE SET ret=EXCLUDED.ret, bin=EXCLUDED.bin, t1=EXCLUDED.t1, barrier_type=EXCLUDED.barrier_type, daily_vol=EXCLUDED.daily_vol, target=EXCLUDED.target, computed_at=EXCLUDED.computed_at`
       - This allows re-labeling with same params to overwrite

    5. If `--full-refresh`: DELETE FROM cmc_triple_barrier_labels WHERE asset_id IN (...) AND tf = :tf AND pt_multiplier = :pt AND sl_multiplier = :sl AND vertical_bars = :vb BEFORE inserting

    6. Summary log: total assets, total labels, label distribution, elapsed time

    **Usage:**
    ```
    python -m ta_lab2.scripts.labeling.refresh_triple_barrier_labels --ids 1,52 --tf 1D --pt 1.0 --sl 1.0 --vertical-bars 10
    python -m ta_lab2.scripts.labeling.refresh_triple_barrier_labels --all --tf 1D --cusum-filter
    ```

    **DB write pattern:** Follow the existing write patterns in the codebase. Use SQLAlchemy `text()` for raw SQL. Use `pandas.DataFrame.to_sql()` with temp table for bulk insert, or use `executemany` with parameterized INSERT. The upsert via ON CONFLICT is critical for idempotency.

    **Logging:** Use `logging.getLogger(__name__)` with INFO level for progress, DEBUG for per-asset details.
