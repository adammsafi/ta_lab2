    Create `src/ta_lab2/labeling/meta_labeler.py` implementing the secondary RF classifier for meta-labeling.

    **MetaLabeler class:**

    ```python
    class MetaLabeler:
        """
        Secondary RF classifier for meta-labeling (AFML Ch.3).

        Primary model (EMA/RSI/ATR signal) provides direction (+1/-1).
        MetaLabeler predicts whether to take the trade {0, 1} and
        outputs trade_probability for position sizing.
        """
    ```

    **Constructor params:**
    - `n_estimators: int = 100`
    - `max_features: str = "sqrt"`
    - `class_weight: str = "balanced_subsample"` -- handles imbalanced outcomes
    - `n_jobs: int = -1` (use all cores)
    - `random_state: int = 42` (reproducibility)

    **Internally creates:**
    - `self.model = RandomForestClassifier(...)` with the above params
    - `self.scaler = StandardScaler()`
    - `self._is_fitted = False`

    **Methods:**

    1. `fit(X: pd.DataFrame, y: pd.Series) -> "MetaLabeler"`:
       - X: features at signal entry timestamps (from cmc_features)
       - y: {0, 1} where 1 = primary signal direction matched triple barrier label
       - `X_scaled = self.scaler.fit_transform(X)`
       - `self.model.fit(X_scaled, y)`
       - `self._is_fitted = True`
       - Return self

    2. `predict_proba(X: pd.DataFrame) -> pd.Series`:
       - Raises if not fitted
       - `X_scaled = self.scaler.transform(X)`
       - `proba = self.model.predict_proba(X_scaled)[:, 1]`
       - Return `pd.Series(proba, index=X.index, name='trade_probability')`

    3. `predict(X: pd.DataFrame, threshold: float = 0.5) -> pd.Series`:
       - Returns binary {0, 1} based on threshold
       - `proba = self.predict_proba(X)`
       - Return `(proba >= threshold).astype(int)`

    4. `feature_importance(X: pd.DataFrame) -> pd.Series`:
       - Returns `pd.Series(self.model.feature_importances_, index=X.columns, name='importance').sort_values(ascending=False)`

    5. `evaluate(X: pd.DataFrame, y: pd.Series) -> dict`:
       - Computes and returns: accuracy, precision, recall, f1, auc_roc (if binary)
       - Uses `sklearn.metrics` for computation
       - Returns dict with these metrics

    **Static method:**

    6. `construct_meta_labels(primary_side: pd.Series, triple_barrier_bin: pd.Series) -> pd.Series`:
       - `primary_side`: +1 (long) or -1 (short) from signal generator
       - `triple_barrier_bin`: +1, -1, 0 from triple barrier labels
       - For meta-labeling: y = 1 if primary_side * triple_barrier_bin > 0 (signal direction was correct)
       - y = 0 if primary_side * triple_barrier_bin <= 0 (signal was wrong or timed out)
       - Special case: if triple_barrier_bin == 0 (timeout), y = 0 (no clear outcome)
       - Return pd.Series of {0, 1}

    **Module imports:** `from sklearn.ensemble import RandomForestClassifier`, `from sklearn.preprocessing import StandardScaler`, `from sklearn.metrics import accuracy_score, precision_score, recall_score, f1_score, roc_auc_score`

  <action>
    Create `src/ta_lab2/scripts/labeling/run_meta_labeling.py` -- Part 1: CLI interface, data loading, and alignment logic.

    **CLI arguments:**
    - `--ids`: Comma-separated asset IDs
    - `--all`: All assets with triple barrier labels
    - `--tf`: Timeframe (default '1D')
    - `--signal-type`: Primary signal type ('ema_crossover', 'rsi_mean_revert', 'atr_breakout')
    - `--pt`: pt_multiplier for triple barrier labels to load (default 1.0)
    - `--sl`: sl_multiplier for triple barrier labels to load (default 1.0)
    - `--vertical-bars`: vertical_bars for labels to load (default 10)
    - `--n-estimators`: RF trees (default 100)
    - `--n-folds`: PurgedKFold folds for evaluation (default 5)
    - `--dry-run`: Train and evaluate but don't persist to DB

    **Data loading functions (defined as helper functions in the module):**

    1. `_load_triple_barrier_labels(engine, asset_id, tf, pt, sl, vb) -> pd.DataFrame`:
       ```sql
       SELECT t0, t1, bin FROM cmc_triple_barrier_labels
       WHERE asset_id = :id AND tf = :tf AND pt_multiplier = :pt AND sl_multiplier = :sl AND vertical_bars = :vb
       ORDER BY t0
       ```
       Build t1_series: `pd.Series(labels['t1'].values, index=labels['t0'])`

    2. `_load_signal_directions(engine, asset_id, signal_type, tf) -> pd.DataFrame`:
       Query the appropriate signal table based on signal_type:
       - ema_crossover: `cmc_signals_ema_crossover`
       - rsi_mean_revert: `cmc_signals_rsi_mean_revert`
       - atr_breakout: `cmc_signals_atr_breakout`
       Get signal timestamps (entry_ts), direction (+1 for 'long', -1 for 'short'), filtering to position_state='open' records (entry events only).

    3. `_align_signals_to_labels(labels_df, signals_df) -> pd.DataFrame`:
       Inner-join on t0/entry_ts timestamp. Only events where both a signal AND a triple barrier label exist.

    4. `_load_features(engine, asset_id, tf, timestamps) -> pd.DataFrame`:
       ```sql
       SELECT * FROM cmc_features WHERE id = :id AND tf = :tf AND ts IN (aligned timestamps)
       ```
       Select numeric feature columns only (drop id, ts, tf, text columns). Drop NaN rows and log how many dropped.

    **Main function `main()`:**
    - Parse args, create engine (NullPool)
    - Resolve asset IDs
    - For each asset_id: call _load_triple_barrier_labels, _load_signal_directions, _align_signals_to_labels, _load_features
    - Log alignment stats: N labels, N signals, N aligned, N features available
    - If alignment produces < 30 samples: log warning, skip asset

    **Error handling:**
    - If no triple barrier labels: log warning "No labels found -- run refresh_triple_barrier_labels first", skip
    - If no signals: log warning "No signals found -- run signal refresh first", skip
    - If < 30 aligned: log warning "Too few aligned samples (N) -- need at least 30", skip
