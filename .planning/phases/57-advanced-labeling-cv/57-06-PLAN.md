  <files>src/ta_lab2/scripts/labeling/run_cpcv_backtest.py</files>
  <action>
    Create `src/ta_lab2/scripts/labeling/run_cpcv_backtest.py`.

    **CLI arguments:**
    - `--ids`: Comma-separated asset IDs (required — typically 1 or 2 assets per run)
    - `--tf`: Timeframe (default '1D')
    - `--signal-type`: Signal strategy ('ema_crossover', 'rsi_mean_revert', 'atr_breakout')
    - `--pt`: pt_multiplier for triple barrier labels (default 1.0)
    - `--sl`: sl_multiplier for triple barrier labels (default 1.0)
    - `--vertical-bars`: vertical_bars for labels (default 10)
    - `--n-splits`: Number of fold groups (default 6)
    - `--n-test-splits`: Number of test fold groups (default 2, gives C(6,2)=15)
    - `--embargo-frac`: Embargo fraction (default 0.01)
    - `--fee-bps`: Fee in basis points for backtest (default 10)
    - `--slippage-bps`: Slippage in basis points (default 5)

    **Pipeline:**

    1. **Load data:**
       a. Load close prices from `cmc_price_bars_multi_tf_u` for the asset/tf
       b. Load features from `cmc_features` for the asset/tf
       c. Load EMA data from `cmc_ema_multi_tf_u` for the asset/tf (needed for EMA crossover signals)
       d. Load triple barrier labels from `cmc_triple_barrier_labels` for the specified params
       e. Build t1_series: `pd.Series(labels['t1'].values, index=labels['t0'])`

    2. **Align all data to common timestamps:**
       - Inner join features and close prices on timestamp
       - Ensure t1_series covers the feature timestamps (drop any feature rows without labels)

    3. **Create CPCVSplitter:**
       ```python
       from ta_lab2.backtests.cv import CPCVSplitter
       splitter = CPCVSplitter(
           n_splits=args.n_splits,
           n_test_splits=args.n_test_splits,
           t1_series=t1_series,
           embargo_frac=args.embargo_frac,
       )
       ```

    4. **Run backtest per split:**
       For each (train_idx, test_idx) from `splitter.split(X)`:
       a. Map test_idx back to timestamps: `test_timestamps = features_df.index[test_idx]`
       b. Extract test fold close prices and features
       c. Generate signals on the test fold using the appropriate signal generator:
          - Read the signal generator files to understand how to call them. The signal generators take a features DataFrame and produce entry/exit signals.
          - For EMA: `from ta_lab2.signals.ema_trend import make_signals` then `signals = make_signals(test_features)`
          - For RSI: `from ta_lab2.signals.rsi_mean_revert import make_signals`
          - For ATR: `from ta_lab2.signals.breakout_atr import make_signals`
       d. Compute returns from signals:
          - Signal = +1 when entry, -1 when exit (or use position series)
          - Daily returns = position * close.pct_change()
          - Sharpe = mean(returns) / std(returns) * sqrt(252) (annualized)
       e. Record OOS Sharpe for this split

    5. **Alternative simpler approach if signal generators are complex:**
       Instead of calling signal generators directly, load pre-computed signals from the signal tables:
       ```sql
       SELECT ts, signal, position FROM cmc_signals_{signal_type} WHERE id = :id AND tf = :tf ORDER BY ts
       ```
       Filter to test fold timestamps, compute returns from position * price changes.
       This avoids needing to re-generate signals and is more consistent with how signals are actually used.

       **Prefer this approach** — it tests the actual persisted signals, not regenerated ones.

    6. **Aggregate OOS Sharpe distribution:**
       ```python
       result = {
           'asset_id': asset_id,
           'signal_type': args.signal_type,
           'n_splits': splitter.get_n_splits(),  # e.g., 15 for CPCV(6,2)
           'sharpe_distribution': oos_sharpes,
           'mean_sharpe': np.mean(oos_sharpes),
           'median_sharpe': np.median(oos_sharpes),
           'sharpe_p10': np.percentile(oos_sharpes, 10),
           'sharpe_p25': np.percentile(oos_sharpes, 25),
           'sharpe_p75': np.percentile(oos_sharpes, 75),
           'sharpe_p90': np.percentile(oos_sharpes, 90),
           'sharpe_std': np.std(oos_sharpes),
           'pct_positive_sharpe': sum(1 for s in oos_sharpes if s > 0) / len(oos_sharpes),
       }
       ```

    7. **Output:**
       - Print formatted table to stdout
       - Log per-split Sharpe values
       - Write results to JSON file: `.planning/phases/57-advanced-labeling-cv/cpcv_results_{asset_id}_{signal_type}.json`
       - This is a research/evaluation script — does NOT persist to a DB table (no new migration needed)

    **Usage:**
    ```
    python -m ta_lab2.scripts.labeling.run_cpcv_backtest --ids 1 --tf 1D --signal-type ema_crossover
    python -m ta_lab2.scripts.labeling.run_cpcv_backtest --ids 1 --tf 1D --signal-type rsi_mean_revert --n-splits 6 --n-test-splits 2
    ```

    **Error handling:**
    - If no triple barrier labels: error with message to run refresh_triple_barrier_labels first
    - If no signals for signal_type: error with message to run signal refresh first
    - If < n_splits * 10 aligned timestamps: error, not enough data for CPCV
    - If any fold produces NaN Sharpe (zero std): log warning, record as NaN, exclude from distribution stats

    **Key bridge insight (from research open question #3):**
    The CPCVSplitter yields integer indices. Map them to timestamps via `features_df.index[test_idx]`. Use those timestamps to slice the pre-computed signal position series, then compute returns.
