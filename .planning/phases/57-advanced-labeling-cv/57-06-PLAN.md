    **CRITICAL: Pre-condition checks (run BEFORE pipeline):**

    Before starting the CPCV pipeline, verify that required data exists:

    ```python
    def _check_preconditions(engine, asset_id, tf, pt, sl, vb):
        """Verify triple barrier labels exist. Exit with clear message if not."""
        with engine.connect() as conn:
            label_count = conn.execute(text(
                "SELECT count(*) FROM cmc_triple_barrier_labels "
                "WHERE asset_id = :id AND tf = :tf AND pt_multiplier = :pt "
                "AND sl_multiplier = :sl AND vertical_bars = :vb"
            ), {"id": asset_id, "tf": tf, "pt": pt, "sl": sl, "vb": vb}).scalar()

            if label_count == 0:
                logger.error(
                    f"No triple barrier labels for asset {asset_id} tf={tf} "
                    f"pt={pt} sl={sl} vb={vb}. "
                    f"Run first: python -m ta_lab2.scripts.labeling.refresh_triple_barrier_labels "
                    f"--ids {asset_id} --tf {tf} --pt {pt} --sl {sl} --vertical-bars {vb}"
                )
                return False

            logger.info(f"Pre-check passed: {label_count} triple barrier labels found")
            return True
    ```

    Call `_check_preconditions()` at the start of processing each asset. If it returns False, skip the asset with a clear error message.

    **CRITICAL: Build features_df with pre-joined EMA columns BEFORE the CPCV loop:**

    The `make_signals()` function for `ema_crossover` (in `src/ta_lab2/signals/ema_trend.py`) expects columns named `ema_21` and `ema_50` in the input DataFrame. However, `cmc_features` does NOT contain EMA columns -- EMAs live in `cmc_ema_multi_tf_u` with a `period` column (values like 9, 21, 50) and an `ema` column holding the value.

    **You MUST pivot and join EMA data into features_df before the CPCV splitter loop:**

    ```python
    def _build_features_with_ema(engine, asset_id, tf):
        """
        Load cmc_features and pre-join pivoted EMA columns from cmc_ema_multi_tf_u.

        cmc_ema_multi_tf_u has columns: id, ts, tf, period, ema, d1, d2, ...
        The 'period' column contains values like 9, 21, 50, 200.
        The 'ema' column contains the EMA value for that period.

        We pivot on 'period' to produce wide-format columns: ema_9, ema_21, ema_50, etc.
        Then merge into features_df on (ts).

        This MUST happen BEFORE the CPCV loop so that each test fold slice
        (features_df.iloc[test_idx]) already contains the EMA columns.
        """
        # 1. Load features
        features_df = pd.read_sql(text(
            "SELECT * FROM cmc_features WHERE id = :id AND tf = :tf ORDER BY ts"
        ), engine, params={"id": asset_id, "tf": tf})
        features_df["ts"] = pd.to_datetime(features_df["ts"], utc=True)
        features_df = features_df.set_index("ts").sort_index()

        # 2. Load EMA data for required periods
        ema_periods = [9, 21, 50]  # periods needed by make_signals()
        ema_df = pd.read_sql(text(
            "SELECT ts, period, ema FROM cmc_ema_multi_tf_u "
            "WHERE id = :id AND tf = :tf AND period IN :periods ORDER BY ts"
        ), engine, params={"id": asset_id, "tf": tf, "periods": tuple(ema_periods)})
        ema_df["ts"] = pd.to_datetime(ema_df["ts"], utc=True)

        # 3. Pivot: rows=(ts), columns=(period), values=(ema)
        #    Result: columns named 9, 21, 50 with EMA values
        ema_wide = ema_df.pivot(index="ts", columns="period", values="ema")
        # Rename columns: 9 -> "ema_9", 21 -> "ema_21", 50 -> "ema_50"
        ema_wide.columns = [f"ema_{int(p)}" for p in ema_wide.columns]

        # 4. Merge into features_df
        features_df = features_df.join(ema_wide, how="left")

        logger.info(
            f"Built features_df: {len(features_df)} rows, "
            f"EMA columns: {[c for c in features_df.columns if c.startswith('ema_')]}"
        )
        return features_df
    ```

    **The resulting `features_df` is the FULL pre-joined frame.** Inside the CPCV loop, slicing is simply:
    ```python
    test_features_df = features_df.iloc[test_idx]
    test_close = test_features_df["close"]
    ```
    This ensures `make_signals(test_features_df)` has access to `ema_21`, `ema_50` columns.

    For `rsi_mean_revert` and `atr_breakout`, the required columns (`rsi_14`, `atr_14`) are already in `cmc_features`, so no extra join is needed. But the `_build_features_with_ema()` function should be called unconditionally (the extra EMA columns are harmless for non-EMA signal types).

    **Pipeline:**

    1. **Load data:**
       a. Call `_build_features_with_ema(engine, asset_id, tf)` to get the full pre-joined features_df
       b. Load close prices from features_df["close"] (already available)
       c. Load triple barrier labels from `cmc_triple_barrier_labels` for the specified params

    2. **Align all data to common timestamps:**
       - Ensure t1_series covers the feature timestamps (drop any feature rows without labels)

    3. **Create CPCVSplitter:**
       ```python
       from ta_lab2.backtests.cv import CPCVSplitter
       splitter = CPCVSplitter(
           n_splits=args.n_splits,
           n_test_splits=args.n_test_splits,
           t1_series=t1_series,
           embargo_frac=args.embargo_frac,

       ```

    4. **Run backtest per split using make_signals() (NOT pre-computed signal tables):**

       **IMPORTANT DATA MODEL NOTE:** The signal tables (cmc_signals_ema_crossover, etc.) store entry/exit EVENT records with `position_state` ('open'/'closed'), NOT a continuous position float series. They cannot be directly sliced to a test fold to compute returns. Instead, call make_signals() on each test fold's feature DataFrame to generate fresh entries/exits.

       For each (train_idx, test_idx) from `splitter.split(X)`:
       a. Slice test fold from the FULL pre-joined features_df: `test_features_df = features_df.iloc[test_idx]`
       b. Extract test fold close prices: `test_close = test_features_df["close"]`
       c. Generate signals on the test fold by calling the appropriate make_signals():
          ```python
          if signal_type == 'ema_crossover':
              from ta_lab2.signals.ema_trend import make_signals
              entries, exits, size = make_signals(test_features_df)
          elif signal_type == 'rsi_mean_revert':
              from ta_lab2.signals.rsi_mean_revert import make_signals
              entries, exits, size = make_signals(test_features_df)
          elif signal_type == 'atr_breakout':
              from ta_lab2.signals.breakout_atr import make_signals
              entries, exits, size = make_signals(test_features_df)
          ```
          Each make_signals() returns `(entries: bool Series, exits: bool Series, size: Optional[Series])`.
          For ema_crossover, make_signals() expects `ema_21` and `ema_50` columns in the DataFrame -- these are present because we pre-joined them in step 1.
       d. Derive continuous position series from entries/exits:
          ```python
          position = pd.Series(0.0, index=test_close.index)
          in_position = False
          for ts in test_close.index:
              if entries.get(ts, False) and not in_position:
                  in_position = True
              elif exits.get(ts, False) and in_position:
                  in_position = False
              position.loc[ts] = 1.0 if in_position else 0.0
          ```
          (Or vectorize: `position = entries.astype(int).cumsum() - exits.astype(int).cumsum(); position = position.clip(0, 1).astype(float)` -- but test carefully for correctness.)
       e. Compute returns:
          ```python
          price_returns = test_close.pct_change().fillna(0)
          # Apply transaction costs at entry/exit points
          cost_per_trade = (args.fee_bps + args.slippage_bps) / 1e4
          trade_costs = (entries.astype(int) + exits.astype(int)) * cost_per_trade
          strategy_returns = position.shift(1).fillna(0) * price_returns - trade_costs
          ```
       f. Compute OOS Sharpe:
          ```python
          if strategy_returns.std() > 0:
              sharpe = strategy_returns.mean() / strategy_returns.std() * np.sqrt(252)
          else:
              sharpe = 0.0
          ```
       g. Record OOS Sharpe for this split. Log: split index, n_test_bars, n_trades, Sharpe.

    5. **Aggregate OOS Sharpe distribution:**
       ```python
       result = {
           'asset_id': asset_id,
           'signal_type': args.signal_type,
           'n_splits': splitter.get_n_splits(),  # e.g., 15 for CPCV(6,2)
           'sharpe_distribution': oos_sharpes,
           'mean_sharpe': np.mean(oos_sharpes),
           'median_sharpe': np.median(oos_sharpes),
           'sharpe_p10': np.percentile(oos_sharpes, 10),
           'sharpe_p25': np.percentile(oos_sharpes, 25),
           'sharpe_p75': np.percentile(oos_sharpes, 75),
           'sharpe_p90': np.percentile(oos_sharpes, 90),
           'sharpe_std': np.std(oos_sharpes),
           'pct_positive_sharpe': sum(1 for s in oos_sharpes if s > 0) / len(oos_sharpes),
       }
       ```
