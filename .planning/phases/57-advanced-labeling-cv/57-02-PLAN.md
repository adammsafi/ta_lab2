  <files>src/ta_lab2/labeling/cusum_filter.py</files>
  <action>
    Create `src/ta_lab2/labeling/cusum_filter.py` implementing the symmetric CUSUM filter from AFML Ch.17.

    **Main function -- `cusum_filter(raw_series: pd.Series, threshold: float) -> pd.DatetimeIndex`:**
    - Takes a close price series (NOT log returns -- the function computes diff internally)
    - Computes `diff = raw_series.diff().dropna()`
    - Tracks s_pos (positive cumsum) and s_neg (negative cumsum)
    - When s_pos >= threshold: append timestamp to events, reset s_pos = 0
    - When s_neg <= -threshold: append timestamp to events, reset s_neg = 0
    - Return `pd.DatetimeIndex(t_events)`

    **Helper function -- `get_cusum_threshold(close: pd.Series, multiplier: float = 2.0, vol_span: int = 100) -> float`:**
    - Computes `np.log(close / close.shift(1)).dropna().ewm(span=vol_span).std().mean() * multiplier` directly (keeps cusum_filter fully independent -- no circular import)
    - Returns scalar float threshold
    - This is the recommended per-asset threshold calibration from research

    **Validation helper -- `validate_cusum_density(n_events: int, n_bars: int, target_min: float = 0.05, target_max: float = 0.60) -> dict`:**
    - Returns dict with `density` (n_events/n_bars), `within_target` (bool), `recommendation` (str)
    - If density > 0.60: "threshold too low -- increase multiplier"
    - If density < 0.05: "threshold too high -- decrease multiplier"
    - This helps calibrate threshold per asset (research pitfall #3)

    Module docstring: Reference AFML Ch.17, explain symmetric CUSUM vs one-sided, explain reset-to-zero mechanism.
  </action>

  <name>Task 2: Implement trend scanning labels (standalone library)</name>
  <files>src/ta_lab2/labeling/trend_scanning.py</files>
  <action>
    Create `src/ta_lab2/labeling/trend_scanning.py` implementing OLS t-value trend scanning from AFML "Machine Learning for Asset Managers" Ch.2.

    **NOTE:** Trend scanning is a standalone library module. It is NOT wired to any downstream consumer in Phase 57. It will be consumed by future phases (e.g., trend-following strategy development). The cv.py key_link (t1_series) is a capability -- the module CAN produce t1_series compatible with PurgedKFoldSplitter, but no Phase 57 plan actually calls it.

    **Main function -- `trend_scanning_labels(price_series: pd.Series, look_forward_window: int = 20, min_sample_length: int = 5, min_tvalue_threshold: float = 0.0) -> pd.DataFrame`:**
    - Works on log-prices: `close = np.log(price_series)`
    - For each index position i from 0 to len(close) - min_sample_length:
      a. Set t0 = close.index[i]
      b. For each window length L from min_sample_length to min(look_forward_window, remaining bars):
         - Extract y = close.iloc[i:i+L].values
         - x = np.arange(len(y))
         - Use `scipy.stats.linregress(x, y)` -- returns (slope, intercept, rvalue, pvalue, stderr)
         - Compute t_val = slope / stderr if stderr > 1e-15 else 0.0
         - Track the L with maximum |t_val|
      c. Record: t1 = close.index[i + best_L - 1], tvalue = best_tval
      d. bin = int(np.sign(best_tval)) if abs(best_tval) > min_tvalue_threshold else 0
    - Return DataFrame with columns: ['t1', 'tvalue', 'bin']
      - Index = price_series.index (label start timestamps)
      - t1 = optimal regression end timestamp
      - tvalue = t-statistic at optimal L
      - bin = label {-1, 0, +1}

    **Performance note in docstring:** The naive O(n * L) loop is slow for large series (>10K bars). For production, apply on CUSUM-filtered events (not every bar) to reduce n. Add a `t_events: pd.DatetimeIndex | None = None` parameter -- if provided, only compute for those timestamps instead of every bar.

    **Sample weight function -- `get_trend_weights(trend_df: pd.DataFrame) -> pd.Series`:**
    - Returns `np.abs(trend_df['tvalue'])` normalized to [0, 1] range
    - Higher |t-value| = higher confidence = higher sample weight
    - Can be used as sample_weight in sklearn RF training

    **t1_series helper -- `get_t1_series(trend_df: pd.DataFrame) -> pd.Series`:**
    - Returns `pd.Series(trend_df['t1'].values, index=trend_df.index)`
    - This is the t1_series format expected by PurgedKFoldSplitter
    - Ensure both index and values are tz-aware UTC

    Import: `from scipy.stats import linregress`

    Module docstring: Reference AFML, explain that trend scanning is an alternative to triple barrier for trend-following, and that t-values double as sample weights. Note this is a standalone library for future phase consumption.
  </action>
