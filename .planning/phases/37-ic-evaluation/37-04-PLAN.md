  <action>
Add three DB helper functions to `src/ta_lab2/analysis/ic.py`. These are placed in the library module (not the CLI) so that notebooks and the Streamlit dashboard can also use them.

1. `load_feature_series(conn, asset_id, tf, feature_col, train_start, train_end) -> tuple[pd.Series, pd.Series]`:
   - Loads a single feature column + close from `cmc_features` for the given asset/tf/time range.
   - Uses `sqlalchemy.text()` for the query. The feature column name must be dynamically inserted into the SQL string because parameterized queries cannot be used for column names. Validate the column name first using `get_columns()` from `ta_lab2.scripts.sync_utils`.
   - SQL: `SELECT ts, {feature_col}, close FROM public.cmc_features WHERE id = :id AND tf = :tf AND ts >= :start AND ts <= :end ORDER BY ts`
   - CRITICAL: Apply `df['ts'] = pd.to_datetime(df['ts'], utc=True)` immediately after pd.read_sql. This fixes the mixed-tz-offset object dtype issue.
   - Set index to 'ts'. Return (feature_series, close_series).
   - If feature_col not in available columns, raise ValueError with available columns listed.

2. `load_regimes_for_asset(conn, asset_id, tf, train_start, train_end) -> pd.DataFrame`:
   - Loads regime labels from `cmc_regimes`, parsing trend_state and vol_state from l2_label.
   - SQL uses `split_part(l2_label, '-', 1) AS trend_state` and `split_part(l2_label, '-', 2) AS vol_state`.
   - Filter: `WHERE id = :id AND tf = :tf AND ts >= :start AND ts <= :end AND l2_label IS NOT NULL`.
   - CRITICAL: `cmc_regimes` has NO `trend_state` or `vol_state` columns. They MUST be derived from `l2_label` via `split_part()` in SQL. Do NOT reference trend_state or vol_state as actual column names in the WHERE clause.
   - Apply `df['ts'] = pd.to_datetime(df['ts'], utc=True)` immediately after loading.
   - Set index to 'ts'. Return DataFrame with columns: regime_key, trend_state, vol_state.
   - If no rows returned (asset has no regime data), return an empty DataFrame with the correct columns.

3. `save_ic_results(conn, rows, *, overwrite=False) -> int`:
   - Persists IC result rows to `cmc_ic_results`.
   - `rows` is a list of dicts with keys matching cmc_ic_results columns: asset_id, tf, feature, horizon, horizon_days, return_type, regime_col, regime_label, train_start, train_end, ic, ic_t_stat, ic_p_value, ic_ir, ic_ir_t_stat, turnover, n_obs.
   - When overwrite=False (default): `ON CONFLICT (asset_id, tf, feature, horizon, return_type, regime_col, regime_label, train_start, train_end) DO NOTHING` -- keeps historical evaluations.
   - When overwrite=True: `ON CONFLICT ... DO UPDATE SET ic = EXCLUDED.ic, ic_t_stat = EXCLUDED.ic_t_stat, ...` -- upsert with updated values and computed_at = now().
   - Uses `sqlalchemy.text()` with named parameters. Loops over rows, executing one INSERT per row.
   - Returns count of rows written (rowcount).
   - CRITICAL: Before inserting, convert any numpy scalars to Python native types using `float(v) if hasattr(v, 'item') else v` pattern (per MEMORY.md numpy scalar normalization). Also convert pd.Timestamp to Python datetime for train_start/train_end. Convert NaN ic values to None for SQL NULL.

Implementation:

Module docstring: Brief description matching compute_psr.py style.

Imports:
- `argparse, logging, sys` (stdlib)

- `from ta_lab2.time.dim_timeframe import DimTimeframe`

`main() -> int`:
1. Create argparse parser with:
   - `--asset-id` (required, int): Asset ID
   - `--tf` (default='1D', str): Timeframe
   - `--feature` (nargs='+', str): One or more feature column names
   - `--all-features` (action='store_true'): Score all numeric features in cmc_features
   - `--train-start` (required, str): Training window start (ISO date)
   - `--train-end` (required, str): Training window end (ISO date)
   - `--horizons` (nargs='+', type=int, default=None): Custom horizons (default [1,2,3,5,10,20,60])
   - `--return-types` (nargs='+', default=None): Custom return types (default ['arith', 'log'])
   - `--rolling-window` (type=int, default=63): Rolling IC window
   - `--regime` (action='store_true'): Enable regime breakdown (both trend_state and vol_state)
   - `--overwrite` (action='store_true'): Upsert instead of append
   - `--dry-run` (action='store_true'): Compute but do not write
   - `--verbose` / `-v` (action='store_true'): DEBUG logging

2. Parse args. Convert train_start/train_end to pd.Timestamp with tz='UTC'.

3. Create engine: `create_engine(resolve_db_url(), poolclass=pool.NullPool)`

4. Look up tf_days_nominal via DimTimeframe: `DimTimeframe(engine).tf_days(args.tf)`. Wrap in try/except; if it fails (e.g., custom TF), default to 1 with a warning.

5. `with engine.begin() as conn:`:
   a. Determine feature list:
      - If --all-features: load column names from cmc_features via get_columns(), exclude non-feature columns (id, ts, tf, close, open, high, low, volume, ingested_at).
      - Else: use args.feature list.

   b. For each feature:
      - Call `load_feature_series(conn, asset_id, tf, feature, train_start, train_end)` to get (feature_series, close_series).
      - If no data returned, log warning and skip.
      - Call `compute_ic(feature_series, close_series, train_start, train_end, horizons=args.horizons, return_types=return_types, rolling_window=args.rolling_window, tf_days_nominal=tf_days_nominal)`.
      - Add `feature` column to result DataFrame.
      - If `--regime`:
        - Load regimes: `load_regimes_for_asset(conn, asset_id, tf, train_start, train_end)`.
        - For each regime_col in ['trend_state', 'vol_state']:
          - Call `compute_ic_by_regime(feature_series, close_series, regimes_df, train_start, train_end, horizons=args.horizons, return_types=return_types, regime_col=regime_col, tf_days_nominal=tf_days_nominal)`.
          - Add `feature` column.
        - Concatenate with the full-sample results.
      - Else: Add `regime_col='all'` and `regime_label='all'` columns to results.

   c. Prepare rows for persistence:
      - Add `asset_id`, `tf`, `train_start`, `train_end` columns.
      - Compute `horizon_days` = horizon * tf_days_nominal for each row.
      - Convert DataFrame rows to list of dicts.

   d. Log summary: number of rows computed, top-5 features by mean |IC| for horizon=1 arith.

   e. If not dry_run:
      - Call `save_ic_results(conn, rows, overwrite=args.overwrite)`.
      - Log number of rows written.

6. Return 0 if no errors, 1 if any failures.

Add `if __name__ == "__main__": sys.exit(main())` at bottom.
