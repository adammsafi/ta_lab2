  <name>IC core computation library (Spearman IC, rolling IC, IC-IR, turnover, significance)</name>
  <files>src/ta_lab2/analysis/ic.py, tests/analysis/test_ic.py</files>
  <behavior>
    compute_forward_returns:
    - compute_forward_returns(close_100_bars, horizon=5, log=False) -> Series of length 100, last 5 are NaN
    - compute_forward_returns(close_100_bars, horizon=5, log=True) -> log returns, last 5 are NaN
    - Forward returns are computed on the FULL series (before any train window slicing)

    compute_ic (single feature, all horizons):
    - compute_ic(feature, close, train_start, train_end) -> DataFrame with 14 rows (7 horizons x 2 return types)
    - compute_ic(feature, close) -> TypeError (train_start/train_end are REQUIRED positional-or-keyword args with no default)
    - Each row has columns: horizon, return_type, ic, ic_t_stat, ic_p_value, ic_ir, ic_ir_t_stat, turnover, n_obs
    - For a known predictive feature (e.g. linearly increasing signal): IC > 0 for horizon=1 arith
    - For a random feature: |IC| close to 0 (within statistical noise)
    - Bars within `horizon * tf_days_nominal` days of train_end have their forward returns nulled (look-ahead prevention)
    - horizon=60 on a 50-bar window: NaN IC (fewer bars than horizon)
    - Both 'arith' and 'log' return types computed by default
    - horizons parameter defaults to [1, 2, 3, 5, 10, 20, 60]
    - Custom horizons: compute_ic(..., horizons=[1, 5]) -> 4 rows (2 horizons x 2 return types)
    - Custom return types: compute_ic(..., return_types=['arith']) -> 7 rows

    compute_rolling_ic:
    - compute_rolling_ic(feature, fwd_ret, window=63) -> (rolling_ic_series, ic_ir_float, ic_ir_tstat_float)
    - Uses vectorized rank-then-correlate pattern: feature.rolling(window).rank(), fwd_ret.rolling(window).rank(), then rolling().corr()
    - ic_ir = mean(rolling_ic) / std(rolling_ic)
    - ic_ir_tstat = mean * sqrt(n) / std  (equivalent to ttest_1samp t-stat on rolling IC)
    - With fewer than 5 valid rolling IC values: ic_ir=NaN, ic_ir_tstat=NaN
    - rolling_ic_series has NaN for the first window-1 bars

    IC significance:
    - t_stat = ic * sqrt(n - 2) / sqrt(max(1 - ic^2, 1e-15))
    - p_value = 2 * (1 - norm.cdf(abs(t_stat)))  (two-sided)
    - When |ic| = 1.0: denominator guarded by 1e-15 floor, t_stat is large but finite

    compute_feature_turnover:
    - compute_feature_turnover(stable_feature_series) -> float close to 0 (high rank autocorrelation)
    - compute_feature_turnover(random_feature_series) -> float close to 1 (low rank autocorrelation)
    - turnover = 1 - spearmanr(ranks[:-1], ranks[1:]).statistic
    - With fewer than 20 obs: returns NaN
  </behavior>

    Then implement `src/ta_lab2/analysis/ic.py` (GREEN phase):

    1. `compute_forward_returns(close, horizon, log=False) -> pd.Series`:
       Compute on FULL close series. close.shift(-horizon) / close - 1 for arith, np.log variant for log.

    2. `_compute_single_ic(feature, fwd_ret, train_start, train_end, horizon, tf_days_nominal, min_obs=20) -> dict`:
       Internal. Filter to train window. Apply boundary mask: bars where `bar_ts + timedelta(days=horizon * tf_days_nominal) > train_end` get NaN forward returns. Align, dropna. If n < min_obs: return NaN dict. Compute spearmanr. t_stat with denom guard. p_value via norm.cdf.

    3. `compute_rolling_ic(feature, fwd_ret, window=63) -> tuple[pd.Series, float, float]`:
       Vectorized: feat_rank = feature.rolling(window).rank(); fwd_rank = fwd_ret.rolling(window).rank(); rolling_ic = feat_rank.rolling(window).corr(fwd_rank). Compute ic_ir and ic_ir_tstat from valid rolling IC values.

    4. `compute_feature_turnover(feature, min_obs=20) -> float`:
       ranks = feature.dropna().rank(); spearmanr(ranks[:-1], ranks[1:]); return 1 - statistic.

    5. `compute_ic(feature, close, train_start, train_end, *, horizons=None, return_types=None, rolling_window=63, tf_days_nominal=1, min_obs=20) -> pd.DataFrame`:
       PUBLIC API. train_start and train_end have NO default (TypeError when omitted). Defaults: horizons=[1,2,3,5,10,20,60], return_types=['arith','log']. For each (horizon, return_type): compute forward returns globally, call _compute_single_ic for point IC, call compute_rolling_ic for IC-IR, call compute_feature_turnover once per feature. Assemble rows into DataFrame.

    CRITICAL: train_start and train_end must be regular parameters with NO default value, so omitting them raises TypeError naturally. Do NOT use sentinel values or manual raises -- Python's own TypeError on missing required args is the correct behavior.

    CRITICAL: The boundary masking must explicitly null forward returns for bars where `bar_ts + pd.Timedelta(days=horizon * tf_days_nominal) > train_end`. This prevents look-ahead bias even when the forward return was computed on the full series.

    CRITICAL: Rolling IC must use the vectorized rank-then-correlate pattern, NOT a per-window loop with spearmanr(). The research verified this is 30x faster.
