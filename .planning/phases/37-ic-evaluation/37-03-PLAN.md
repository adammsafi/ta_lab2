1. `compute_ic_by_regime(feature, close, regimes_df, train_start, train_end, *, horizons=None, return_types=None, rolling_window=63, tf_days_nominal=1, regime_col='trend_state', min_obs_per_regime=30, min_obs=20) -> pd.DataFrame`:
   - `regimes_df` is a DataFrame indexed by ts (UTC) with columns including `regime_col` (e.g. 'trend_state', 'vol_state'). This is NOT loaded from DB here -- the caller provides it. The DB helper in Plan 04 handles loading and parsing l2_label.
   - If `regimes_df` is empty or None: fall back to full-sample IC by calling `compute_ic(feature, close, train_start, train_end, ...)` and adding `regime_col=regime_col, regime_label='all'` columns to the result.
   - Otherwise: for each unique label in `regimes_df[regime_col]`, filter feature and close to only those timestamps where the regime is active. If the filtered subset has fewer than `min_obs_per_regime` observations, skip that regime (do not add a row). Compute IC on the filtered subset using `compute_ic()` and add `regime_col` and `regime_label` columns.
   - Return concatenated DataFrame of all regime results.
   - CRITICAL: The library function does NOT load from the database. It accepts a pre-built `regimes_df`. The l2_label parsing (`split('-')` to get trend_state/vol_state) happens in the CLI/DB helper layer (Plan 04), NOT here.

   - `features_df` is a DataFrame indexed by ts (UTC) with multiple feature columns.
   - `feature_cols` defaults to all numeric columns in features_df except 'close' (if present).
   - Loops over each feature column, calls `compute_ic(features_df[col], close, train_start, train_end, ...)`, adds a `feature` column with the column name, and concatenates all results.
   - Returns a single DataFrame with an additional `feature` column.
