This task builds the GenericOneDayBarBuilder class, source spec loading, and state table migration. Before any building occurs, a pre-flight check fixes CTE templates that are missing venue_id in their INSERT column lists.

**Pre-flight CTE template venue_id fix (CRITICAL):**

The CMC and TVC CTE templates stored in dim_data_sources have `ON CONFLICT (id, venue_id, tf, bar_seq, "timestamp")` but do NOT include `venue_id` in their INSERT column lists. Since price_bars_1d.venue_id is NOT NULL (after migration), these inserts will fail.

Add a `_preflight_fix_cte_templates(conn)` function that runs ONCE at startup:
1. Query `SELECT source_key, src_cte_template FROM dim_data_sources`
2. For each template, check if the INSERT column list in the `ins AS (INSERT INTO {dst} (...)` section includes `venue_id`
3. For CMC: if `venue_id` is missing from the INSERT column list, UPDATE the template to add `venue_id` to the column list AND add `1::smallint AS venue_id` to the SELECT list (right after `source_file` in final CTE). The CMC template already produces `name` as src_name -- add venue_id before the ON CONFLICT clause.
4. For TVC: same fix -- add `venue_id` to INSERT columns AND add `11::smallint AS venue_id` to the SELECT. The TVC template already has `venue` and `venue_rank` -- add `venue_id` next to them.
5. Log which templates were fixed. If already correct, log "templates OK" and skip.

**Detection logic (IMPORTANT -- do NOT use SQL LIKE for detection):**

The naive SQL pattern `NOT LIKE '%venue_id%INSERT%'` is WRONG because `venue_id` appears in the ON CONFLICT clause AFTER the INSERT keyword, so the pattern `venue_id...INSERT` never matches and `NOT LIKE` is always TRUE. This would cause the UPDATE to fire on every single startup.

Instead, do the detection in Python:
```python
rows = fetchall(conn, "SELECT source_key, src_cte_template FROM dim_data_sources")
for source_key, template_text in rows:
    if source_key == 'cmc' and '1::smallint AS venue_id' not in template_text:
        # Fix CMC template: add venue_id to INSERT cols and SELECT
        fixed = _add_venue_id_to_template(template_text, '1::smallint AS venue_id')
        execute(conn, "UPDATE dim_data_sources SET src_cte_template = %s WHERE source_key = %s",
                [fixed, source_key])
        log.info("Fixed CMC CTE template: added venue_id column")
    elif source_key == 'tvc' and '11::smallint AS venue_id' not in template_text:
        # Fix TVC template: add venue_id to INSERT cols and SELECT
        fixed = _add_venue_id_to_template(template_text, '11::smallint AS venue_id')
        execute(conn, "UPDATE dim_data_sources SET src_cte_template = %s WHERE source_key = %s",
                [fixed, source_key])
        log.info("Fixed TVC CTE template: added venue_id column")
    else:
        log.info(f"Template for {source_key} already has venue_id -- skipping")
conn.commit()
```

The `_add_venue_id_to_template(template_text, venue_id_expr)` helper does the string surgery:
- Find the INSERT column list and add `venue_id` after the last column before the closing paren
- Find the corresponding SELECT and add `venue_id_expr` in the matching position

This is idempotent: if `1::smallint AS venue_id` is already present in the CMC template, the check short-circuits. No spurious UPDATEs on every startup.

IMPORTANT: The HL template already includes venue_id (it has `2::smallint AS venue_id` in src_filtered and carries it through). Only CMC and TVC need fixing.

To implement the string replacement:
- For CMC: after `repaired_timehigh, repaired_timelow, repaired_high, repaired_low` in the INSERT list, add `, venue_id`. In the SELECT, after `repaired_high, repaired_low` add `, 1::smallint AS venue_id`.
- For TVC: after `venue, venue_rank` in the INSERT list, add `, venue_id`. In the SELECT, after `venue, venue_rank` add `, 11::smallint AS venue_id`.

Call `_preflight_fix_cte_templates(conn)` once in `main()` before the source loop.

**State table migration `_migrate_state_table_pk(conn, state_table, source_spec_list)`:**

CRITICAL: CMC and TVC share the state table with overlapping IDs (same dim_assets.id can appear in both cmc_price_histories7 and tvc_price_histories). A naive "backfill all NULL venue_id rows to X" will corrupt the other source's state.

The migration function must accept a LIST of all source specs and discriminate by source:
1. Check if `venue_id` column exists in state table (query information_schema.columns)
2. If not: `ALTER TABLE ADD COLUMN venue_id SMALLINT NOT NULL DEFAULT 1`
3. For each source spec, backfill venue_id using source membership:
   - CMC (venue_id=1): `UPDATE state SET venue_id = 1 WHERE venue_id = 1 AND id IN (SELECT DISTINCT id FROM cmc_price_histories7)` -- DEFAULT 1 means CMC rows are already correct if column was added with DEFAULT 1
   - TVC (venue_id=11): `UPDATE state SET venue_id = 11 WHERE id IN (SELECT DISTINCT id FROM tvc_price_histories) AND venue_id = 1` -- but only for IDs that are NOT also in CMC. For IDs in BOTH sources: duplicate the state row (INSERT a new row with venue_id=11 copying from the existing venue_id=1 row)
   - HL (venue_id=2): `UPDATE state SET venue_id = 2 WHERE id IN (SELECT DISTINCT dai.id FROM dim_asset_identifiers dai JOIN hyperliquid.hl_candles c ON dai.id_type = 'HL' AND dai.id_value::int = c.asset_id WHERE c.interval = '1d') AND venue_id = 1` -- same overlap handling as TVC
4. For IDs that appear in BOTH CMC and TVC/HL: the CMC row keeps venue_id=1 (from DEFAULT). Create NEW rows for TVC/HL:
   ```sql
   INSERT INTO {state} (id, venue_id, tf, last_src_ts, daily_min_seen, last_run_ts, last_upserted, ...)
   SELECT id, {other_venue_id}, tf, NULL, NULL, now(), 0, ...
   FROM {state}
   WHERE id IN (SELECT DISTINCT id FROM {other_source_table})
     AND venue_id = 1
   ON CONFLICT (id, venue_id, tf) DO NOTHING;
   ```
   Setting last_src_ts=NULL for the new source row triggers a full build for that source (correct behavior -- we've never built those IDs from that source before).
5. Then check if PK includes venue_id (query pg_index + pg_attribute for the PK constraint name)
6. If PK is still (id, tf): DROP old PK, ADD new PK (id, venue_id, tf)
7. Must be idempotent: if already migrated, no-op

Call `_migrate_state_table_pk()` ONCE in `main()` before the source loop, passing all source specs.

  def _check_for_backfill_generic(self, conn, id_: int, spec: dict) -> datetime | None:
      if spec.get("join_clause"):
          # HL path: JOIN through dim_asset_identifiers
          sql = """
              SELECT MIN(c.{ts_col})
              FROM {src} c
              JOIN dim_asset_identifiers dai
                ON dai.id_type = 'HL' AND dai.id_value::int = c.asset_id
              WHERE dai.id = %s AND c.interval = '1d'
          """.format(ts_col=spec["ts_column"], src=spec["source_table"])
      else:
          # CMC/TVC path: direct id column
          sql = """
              SELECT MIN({ts_col})
              FROM {src}
              WHERE id = %s
          """.format(ts_col=spec["ts_column"], src=spec["source_table"])
      row = fetchone(conn, sql, [id_])
      return row[0] if row else None
