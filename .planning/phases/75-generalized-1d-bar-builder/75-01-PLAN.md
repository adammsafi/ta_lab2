  <name>Task 1: Rewrite refresh_price_bars_1d.py as generalized source-aware builder</name>
  <files>src/ta_lab2/scripts/bars/refresh_price_bars_1d.py</files>
  <action>
Rewrite `refresh_price_bars_1d.py` to replace the CMC-only `OneDayBarBuilder` with a generalized `GenericOneDayBarBuilder` that handles all sources. The new builder reads config from `dim_data_sources` at runtime.

**CLI changes:**
- Add `--source` argument accepting `cmc`, `tvc`, `hl`, or `all` (default: `cmc` for backwards compat)
- When `--source all`, iterate over all source_keys from dim_data_sources and build each independently
- Keep existing `--ids`, `--db-url`, `--full-rebuild`, `--keep-rejects` arguments
- For `--ids all`, use `id_loader_sql` from the source spec (NOT the old `load_all_ids()` function)
- For `--ids 1,2,3` (explicit), use those IDs directly regardless of source

**Source spec loading:**
- Add `_load_source_spec(conn, source_key: str) -> dict` function that queries dim_data_sources
- Load columns: source_key, source_name, source_table, venue_id, default_venue, ohlc_repair, has_market_cap, has_timehigh, id_loader_sql, src_cte_template, join_clause, id_filter_sql, ts_column, conflict_columns, src_name_label, description
- Raise ValueError with helpful message if source_key not found

**State table migration to (id, venue_id, tf) PK:**
- Change `ensure_state_table_exists()` DDL to use `PRIMARY KEY (id, venue_id, tf)` with `venue_id SMALLINT NOT NULL DEFAULT 1`
- Add `_migrate_state_table_pk(conn, state_table, venue_id)` function that:
  1. Checks if `venue_id` column exists in state table (query information_schema.columns)
  2. If not: ALTER TABLE ADD COLUMN venue_id SMALLINT NOT NULL DEFAULT 1
  3. Then check if PK includes venue_id (query pg_index + pg_attribute for the PK constraint name)
  4. If PK is still (id, tf): DROP old PK, ADD new PK (id, venue_id, tf)
  5. Backfill existing CMC rows with venue_id=1, TVC rows with venue_id=11 (from spec)
  CRITICAL: This migration must be idempotent. If already migrated, it should be a no-op.
- Call `_migrate_state_table_pk()` once at startup before processing IDs, not inside per-ID loop

**Generalized build_bars_for_id():**
- Accept `spec` dict as instance attribute (set during __init__)
- Use `spec["venue_id"]` instead of hardcoded values
- Build SQL from template: `sql = spec["src_cte_template"].format(dst=OUTPUT_TABLE, src=spec["source_table"])` -- only include `src=` if `{src}` appears in template (HL template has no `{src}`)
- Build params based on `spec["ohlc_repair"]`:
  - If True (CMC): 11 params = [id_, time_max, time_max, id_, time_min, time_min, time_max, time_max, last_src_ts, last_src_ts, lookback_days]
  - If False (TVC, HL): 6 params = [id_, id_, last_src_ts, last_src_ts, time_max, time_max]
- State queries use `WHERE id = %s AND venue_id = %s AND tf = '1D'` with spec["venue_id"]
- State upsert uses `ON CONFLICT (id, venue_id, tf) DO UPDATE SET ...`
- Coverage tracking: use `spec["source_table"]` and `spec["ts_column"]` for MIN/MAX/COUNT queries

**Generalized backfill detection:**
- Rename `_check_for_backfill()` to `_check_for_backfill_generic()` accepting spec dict
- Use `spec["ts_column"]` (either "timestamp" or "ts") in the MIN query
- Use `spec["source_table"]` as the source table
- `_handle_backfill()`: delete from both dst and state, filtering by venue_id
- `_get_state()`: add venue_id parameter to WHERE clause

**Post-build sync for TVC/HL:**
- Preserve `_sync_1d_to_multi_tf()` function for TVC and HL sources
- After builder.run() completes, call sync if `spec["source_key"] != "cmc"` (CMC writes directly; TVC/HL need the sync from price_bars_1d to price_bars_multi_tf)
- The existing ON CONFLICT in the sync functions differs between TVC (`venue`) and HL (`venue_id`). Use the HL pattern (`venue_id`) since that's the current PK after the venue_id migration. Update the sync SQL to:
  ```sql
  INSERT INTO public.price_bars_multi_tf
  SELECT * FROM public.price_bars_1d
  WHERE src_name = %s
  ON CONFLICT (id, tf, bar_seq, venue_id, "timestamp") DO UPDATE SET
      open = EXCLUDED.open, high = EXCLUDED.high, low = EXCLUDED.low,
      close = EXCLUDED.close, volume = EXCLUDED.volume, ...
  ```
  Pass `spec["src_name_label"]` as the src_name filter param.

**ID loading:**
- Add `_load_ids_for_source(conn, spec: dict) -> list[int]` that executes `spec["id_loader_sql"]`
- For `--ids all`, call this function. For explicit IDs, use them directly.
- NOTE: The HL id_loader_sql in dim_data_sources loads ALL HL assets (not just Y-marked from HL_YN.csv). This is intentional -- the generalized builder drops the CSV dependency. If the user wants to restrict IDs, they pass `--ids 1,2,3` explicitly.

**What NOT to change:**
- Keep `BaseBarBuilder` inheritance and template method pattern
- Keep `BarBuilderConfig` dataclass usage
- Keep the `common_snapshot_contract` imports for resolve_db_url, get_engine, parse_ids, ensure_coverage_table, upsert_coverage
- Do NOT modify the CTE templates in dim_data_sources -- they are the source of truth
- Do NOT modify base_bar_builder.py or bar_builder_config.py

**main() function:**
```python
def main() -> None:
    parser = GenericOneDayBarBuilder.create_argument_parser()
    args = parser.parse_args()

    db_url = resolve_db_url(args.db_url)
    engine = get_engine(db_url)
    conn = connect(db_url)

    sources = _resolve_sources(conn, args.source)  # ['cmc'] or ['cmc','tvc','hl']
    for source_key in sources:
        spec = _load_source_spec(conn, source_key)
        # Migrate state table PK if needed (idempotent)
        _migrate_state_table_pk(conn, "public.price_bars_1d_state", spec["venue_id"])

        ids = _resolve_ids(conn, args, spec)
        config = BarBuilderConfig(
            db_url=db_url, ids=ids,
            daily_table=spec["source_table"],
            bars_table="public.price_bars_1d",
            state_table="public.price_bars_1d_state",
            full_rebuild=args.full_rebuild,
            keep_rejects=getattr(args, "keep_rejects", False),
            rejects_table="public.price_bars_1d_rejects",
            num_processes=getattr(args, "num_processes", 6),
            log_level="INFO", log_file=None, tz=None,
        )
        builder = GenericOneDayBarBuilder(config=config, engine=engine, source_spec=spec)
        builder.run()

        # Sync 1D to multi_tf for non-CMC sources
        if source_key != "cmc":
            _sync_1d_to_multi_tf(db_url, spec["src_name_label"])

    conn.close()
```
