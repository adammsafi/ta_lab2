**Page layout (top to bottom):**

1. **Alert banners** (outside fragment -- read risk_state for kill switch / drift pause):
   - Load risk_state via load_risk_state(engine)
   - If trading_state == 'halted': st.error() with halted_by, halted_at, halted_reason
   - If drift_paused: st.warning() with drift_paused_at, drift_paused_reason
   - These MUST be at the top, impossible to miss

2. **Sidebar controls** (outside fragment -- st.sidebar not allowed in fragment):
   - st.sidebar toggle "Per-Strategy Breakdown" (default False)
   - This toggle affects whether positions table groups by strategy

3. **Auto-refreshing content section** wrapped in `@st.fragment(run_every=AUTO_REFRESH_SECONDS)`:
   The fragment function receives `_engine` and `show_per_strategy` as parameters.

   a. **Portfolio Summary KPIs** (st.columns(4)):
      - Total Portfolio Value: from risk_state day_open_portfolio_value + sum of unrealized_pnl
      - Daily P&L: latest row from daily PnL series
      - Cumulative P&L: last cumulative_pnl value
      - Open Positions: count of positions where quantity != 0

   b. **PnL + Drawdown Chart** (stacked two-panel):
      - Load daily_pnl_series
      - Call build_pnl_drawdown_chart()
      - st.plotly_chart(fig, use_container_width=True, theme=None)
      - chart_download_button for export

   c. **Drawdown KPIs** (st.columns(3)):
      - Peak Equity: max of cumulative_pnl series
      - Current Drawdown: latest drawdown_pct value, formatted as %
      - Max Historical Drawdown: min of drawdown_pct series, formatted as %

   d. **Position Table** (DASH-L02):
      - Load open_positions
      - If show_per_strategy: show all columns including strategy grouping
      - Else: show aggregate view
      - Columns as specified in CONTEXT.md: Asset (symbol), Side (derived: 'Long' if qty > 0 else 'Short'), Qty (quantity), Avg Cost (avg_cost_basis), Current Price (last_mark_price), Unrealized PnL (unrealized_pnl), % of Portfolio (computed: abs(qty * last_mark_price) / total_value * 100), Strategy (config_name), Entry Date (entry_date), Realized PnL (realized_pnl), Signal Type (signal_type), Regime Label (regime_label from JOIN)
      - Format numeric columns: prices to 2 decimals, PnL to 2 decimals, pct to 1 decimal
      - st.dataframe with use_container_width=True

   e. **Trade Log** (last 20 fills):
      - Load recent_fills
      - st.subheader("Recent Trades")
      - st.dataframe with columns: Time (filled_at), Asset (symbol), Side, Qty (fill_qty), Price (fill_price), Fee (fee_amount), Signal ID
      - If empty: st.info("No fills recorded yet.")

   f. **Refresh caption**: st.caption with last update time and auto-refresh interval

**Error handling**: Wrap each section in try/except, display st.warning on failure. Use `except Exception as exc: # noqa: BLE001` pattern matching existing pages.

  <action>
Create `src/ta_lab2/dashboard/pages/7_risk_controls.py` following the existing page pattern.

Module-level constant:
```python
AUTO_REFRESH_SECONDS = 900
```

**Page layout (top to bottom):**

1. **Alert banners** (outside fragment):
   - Same kill switch / drift pause banners as trading page
   - Load risk_state and check trading_state and drift_paused

2. **Sidebar controls** (outside fragment):
   - st.sidebar selectbox for event_type filter (None = all, plus each event_type value from the CHECK constraint)
   - st.sidebar slider for event history days (7, 14, 30, 60, 90 -- default 30)

3. **Auto-refreshing content section** wrapped in `@st.fragment(run_every=AUTO_REFRESH_SECONDS)`:
   Fragment receives `_engine`, `event_type_filter`, `event_days`.

   a. **Risk Status Cards** (st.columns(4)):
      - Kill Switch: st.metric with value "ACTIVE" (green) or "HALTED" (red), use delta for time since halted
      - Drift Pause: st.metric with "OK" or "PAUSED" + time since paused
      - Circuit Breaker: Parse cb_breaker_tripped_at (TEXT JSON via json.loads()), st.metric showing number of tripped keys, "0 tripped" when none
      - Last Update: risk_state updated_at timestamp

   b. **Proximity Gauges** (st.columns(2)):
      - Daily Loss Gauge:
        - Load risk_limits to get daily_loss_pct_threshold
        - Compute consumed from risk_state day_open_portfolio_value vs current portfolio value (approximation from positions sum)
        - st.progress(pct, text=f"Daily Loss: {consumed:.1%} / {cap:.1%} cap")
        - Color context via st.caption: green "Within limits" if < 70%, yellow "Approaching limit" if 70-90%, red "Near limit" if > 90%
      - Position Utilization Gauge:
        - Load positions, compute max position as % of portfolio
        - st.progress(pct, text=f"Largest Position: {max_pct:.1%} / {max_position_pct:.0%} cap")

   c. **Circuit Breaker Details** (st.expander):
      - Parse cb_consecutive_losses JSON: show each asset:strategy key with its consecutive loss count
      - Parse cb_breaker_tripped_at JSON: show each tripped key with timestamp
      - If no keys tripped: "No circuit breakers active"
      - Show cb_consecutive_losses_n from limits (the threshold)

   d. **Risk Event History** (filterable table):
      - st.subheader("Risk Event History")
      - Load risk_events with days and event_type filters from sidebar
      - Display as st.dataframe with columns: Time (event_ts), Type (event_type), Source (trigger_source), Reason, Operator, Asset ID, Strategy ID
      - If empty: st.info("No risk events in the selected period.")

   e. **Refresh caption** with auto-refresh interval

**Error handling**: Same try/except pattern as trading page.

**Import json** at top for cb_consecutive_losses and cb_breaker_tripped_at parsing.
