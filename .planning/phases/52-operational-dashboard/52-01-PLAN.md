- Module docstring, `from __future__ import annotations`, pandas + streamlit + sqlalchemy text imports

**queries/trading.py** (TTL: 120s for positions/fills, 300s for daily PnL):

1. `load_open_positions(_engine) -> pd.DataFrame` (TTL 120s):
   - SELECT from cmc_positions p JOIN dim_assets a ON a.id = p.asset_id LEFT JOIN dim_executor_config ec ON ec.config_id = p.strategy_id
   - WHERE p.quantity != 0 AND p.exchange = 'paper'
   - Columns: symbol, asset_id, exchange, strategy_id, config_name, signal_type, quantity, avg_cost_basis, last_mark_price, unrealized_pnl, unrealized_pnl_pct, realized_pnl, last_updated, created_at (as entry_date)
   - LEFT JOIN cmc_regimes r ON r.id = p.asset_id AND r.tf = '1D' to get l2_label as regime_label (use subquery to get latest regime per asset: WHERE r.ts = (SELECT MAX(ts) FROM cmc_regimes WHERE id = p.asset_id AND tf = '1D'))
   - ORDER BY a.symbol, p.strategy_id
   - Coerce last_updated and entry_date with pd.to_datetime(utc=True)

2. `load_recent_fills(_engine, limit: int = 20) -> pd.DataFrame` (TTL 120s):
   - SELECT from cmc_fills f JOIN cmc_orders o ON o.order_id = f.order_id JOIN dim_assets a ON a.id = o.asset_id
   - WHERE o.exchange = 'paper'
   - Columns: filled_at, symbol, side, fill_qty, fill_price, fee_amount, order_avg_price (o.avg_fill_price), signal_id
   - ORDER BY f.filled_at DESC LIMIT :limit
   - Coerce filled_at with pd.to_datetime(utc=True)

3. `load_daily_pnl_series(_engine) -> pd.DataFrame` (TTL 300s):
   - Aggregate fills by day: DATE(f.filled_at AT TIME ZONE 'UTC') as trade_date
   - SUM(CASE f.side WHEN 'sell' THEN fill_qty * fill_price - fee_amount WHEN 'buy' THEN -(fill_qty * fill_price + fee_amount) END) as daily_realized_pnl
   - JOIN cmc_orders for exchange='paper' filter
   - GROUP BY trade_date, ORDER BY trade_date ASC
   - After loading: compute cumulative_pnl = daily_realized_pnl.cumsum()
   - Compute peak_equity = cumulative_pnl.cummax()
   - Compute drawdown_pct = (cumulative_pnl - peak_equity) / peak_equity.where(peak_equity != 0, 1)

**queries/risk.py** (TTL: 60s for state, 300s for limits, 120s for events):

1. `load_risk_state(_engine) -> dict` (TTL 60s):
   - SELECT trading_state, halted_at, halted_reason, halted_by, drift_paused, drift_paused_at, drift_paused_reason, drift_auto_escalate_after_days, day_open_portfolio_value, last_day_open_date, cb_consecutive_losses, cb_breaker_tripped_at, updated_at
   - FROM dim_risk_state WHERE state_id = 1
   - Return dict(row._mapping) or empty dict

2. `load_risk_limits(_engine) -> dict` (TTL 300s):
   - SELECT max_position_pct, max_portfolio_pct, daily_loss_pct_threshold, cb_consecutive_losses_n, cb_cooldown_hours, drift_tracking_error_threshold_5d, drift_tracking_error_threshold_30d
   - FROM dim_risk_limits WHERE asset_id IS NULL AND strategy_id IS NULL LIMIT 1
   - Return dict or empty dict

3. `load_risk_events(_engine, days: int = 30, event_type: str | None = None) -> pd.DataFrame` (TTL 120s):
   - SELECT event_id, event_ts, event_type, trigger_source, reason, operator, asset_id, strategy_id, metadata
   - FROM cmc_risk_events WHERE event_ts >= NOW() - INTERVAL ':days days'
   - Optional filter AND event_type = :event_type when provided
   - ORDER BY event_ts DESC LIMIT 200

**queries/drift.py** (TTL: 300s):

1. `load_drift_timeseries(_engine, config_id: int, days: int = 30) -> pd.DataFrame` (TTL 300s):
   - SELECT metric_date, tracking_error_5d, tracking_error_30d, paper_cumulative_pnl, replay_pit_cumulative_pnl, threshold_breach, drift_pct_of_threshold
   - FROM cmc_drift_metrics WHERE config_id = :config_id AND metric_date >= CURRENT_DATE - :days
   - ORDER BY metric_date ASC

2. `load_drift_summary(_engine) -> pd.DataFrame` (TTL 300s):
   - SELECT config_id, asset_id, signal_type, days_monitored, breach_count, avg_tracking_error_5d, max_tracking_error_5d, avg_tracking_error_30d, max_tracking_error_30d, avg_absolute_pnl_diff, avg_sharpe_divergence, last_metric_date, current_tracking_error_5d
   - FROM v_drift_summary ORDER BY config_id, asset_id

3. `load_executor_configs(_engine) -> pd.DataFrame` (TTL 300s):
   - SELECT config_id, config_name, signal_type, is_active FROM dim_executor_config WHERE is_active = TRUE ORDER BY config_id
   - Used by drift page to populate config selector

**queries/executor.py** (TTL: 120s for run log, 300s for config):

1. `load_executor_run_log(_engine, limit: int = 50) -> pd.DataFrame` (TTL 120s):
   - SELECT run_id, started_at, finished_at, config_ids, dry_run, replay_historical, status, signals_read, orders_generated, fills_processed, skipped_no_delta, error_message
   - FROM cmc_executor_run_log ORDER BY started_at DESC LIMIT :limit
   - Coerce started_at, finished_at with pd.to_datetime(utc=True)

2. `load_executor_config(_engine) -> pd.DataFrame` (TTL 300s):
   - SELECT config_id, config_name, signal_type, signal_id, is_active, exchange, environment, sizing_mode, position_fraction, max_position_fraction, fill_price_mode, slippage_mode, slippage_base_bps, cadence_hours
   - FROM dim_executor_config ORDER BY config_id

All modules must wrap SQL in sqlalchemy text() and follow the exact pattern from queries/pipeline.py. Use try/except around DB calls at query level is NOT needed (pages handle exceptions). Exception handling at page level, not query level -- consistent with existing pattern.
