Create `src/ta_lab2/analysis/vol_sizer.py` with pure library functions for vol-based position sizing and vectorbt backtest integration.

**Functions:**

1. `compute_vol_sized_position(close, atr_14, init_cash, risk_budget, max_position_pct=0.30) -> float`
   - ATR is in DOLLAR terms (from cmc_features). atr_pct = atr_14 / close.
   - position_pct = min(risk_budget / atr_pct, max_position_pct)
   - position_units = position_pct * init_cash / close
   - Guard: if atr_14 is None or atr_14 <= 0 or close <= 0, return 0.0

2. `compute_realized_vol_position(rolling_std, close, init_cash, risk_budget, max_position_pct=0.30) -> float`
   - rolling_std is the 20d rolling std of daily returns (already a percentage)
   - position_pct = min(risk_budget / rolling_std, max_position_pct)
   - Guard: if rolling_std <= 0, return 0.0

3. `run_vol_sized_backtest(price, entries, exits, vol_series, vol_type, risk_budget, max_position_pct=0.30, init_cash=1000.0, fee_bps=16, sl_stop=None) -> vbt.Portfolio`
   - vol_type: "atr" or "realized" -- determines how vol_series is interpreted
   - If vol_type == "atr": vol_series is dollar ATR, compute atr_pct = vol_series / price
   - If vol_type == "realized": vol_series is already a percentage (rolling std)
   - Compute position_pct = np.minimum(risk_budget / vol_pct.values, max_position_pct)
   - position_units = position_pct * init_cash / price.values
   - size_array = np.where(entries.values, position_units, np.nan) -- only entry bars get size
   - CRITICAL: strip tz from price index before vectorbt (vbt 0.28.1 tz boundary issue)
   - If sl_stop is not None: pass sl_stop=sl_stop to vbt.Portfolio.from_signals()
   - Return vbt.Portfolio.from_signals(price_no_tz, entries=entries.to_numpy(), exits=exits.to_numpy(), size=size_array, direction="longonly", freq="D", init_cash=init_cash, fees=fee_bps/1e4, sl_stop=sl_stop)

4. `worst_n_day_returns(returns, n_values=[1, 3, 5, 10]) -> dict`
   - Compute mean of worst N daily returns for tail risk characterization
   - sorted_rets = np.sort(returns)
   - Return dict: {f"worst_{n}_day_mean": float(sorted_rets[:n].mean()) for n in n_values}
   - Result is a FLAT dict (keys like "worst_1_day_mean", "worst_5_day_mean", etc.)

5. `compute_comparison_metrics(portfolio, returns_series=None) -> dict`
   - Extract standard comparison metrics from a vbt.Portfolio
   - sharpe: from performance.py sharpe() on portfolio returns (annualized with freq="D")
   - sortino: from performance.py sortino()
   - calmar: from performance.py calmar()
   - max_dd: from performance.py max_drawdown()
   - total_return: pf.total_return()
   - n_trades: len(pf.trades.records)
   - win_rate: from performance.py hit_rate() on trade returns
   - recovery_bars: max consecutive drawdown bars (compute from drawdown series: find longest streak where drawdown < 0, i.e., equity below prior peak)
   - worst_n_day: **merged flat** into the top-level return dict via `result = {..., **worst_n_day_returns(pf_returns)}`
   - The return dict has keys like "sharpe", "sortino", "calmar", "max_dd", "total_return", "n_trades", "win_rate", "recovery_bars", "worst_1_day_mean", "worst_3_day_mean", "worst_5_day_mean", "worst_10_day_mean" -- all at the top level, NO nested dicts.

   Recovery bars computation:
   ```python
   # Compute max consecutive drawdown bars
   equity = (1 + pf_returns).cumprod()
   running_max = equity.cummax()
   in_drawdown = equity < running_max
   # Find longest consecutive True streak
   groups = (~in_drawdown).cumsum()
   if in_drawdown.any():
       recovery_bars = int(in_drawdown.groupby(groups).sum().max())
   else:
       recovery_bars = 0
   ```

**Imports:** numpy, pandas, vectorbt, ta_lab2.analysis.performance (sharpe, sortino, calmar, max_drawdown, equity_from_returns, hit_rate)
