Create `src/ta_lab2/scripts/analysis/run_ctf_feature_selection.py` following the structure of `run_feature_selection.py` with CTF-specific adaptations.

**Module docstring:**
```
CTF feature selection: tier classification, AMA comparison, config pruning.

Pipeline steps:
  1. Load CTF IC ranking from ic_results (filter to CTF feature names)
  2. Run stationarity tests on top-N CTF features
  3. Run Ljung-Box on rolling IC series for top-N features
  4. Classify all CTF features into tiers (active/conditional/watch/archive)
  5. Compare CTF IC-IR vs AMA IC-IR (redundancy + head-to-head)
  6. Persist tier assignments to dim_ctf_feature_selection
  7. Write ctf_config_pruned.yaml (remove all-archive combos)
  8. Write comparison report (md + json)

Usage:
    python -m ta_lab2.scripts.analysis.run_ctf_feature_selection --all
    python -m ta_lab2.scripts.analysis.run_ctf_feature_selection --dry-run --top-n 50
    python -m ta_lab2.scripts.analysis.run_ctf_feature_selection --skip-stationarity
```

**Imports:**
```python
from ta_lab2.analysis.feature_selection import (
    classify_feature_tier,
    test_ljungbox_on_ic,
    test_stationarity,
)
from ta_lab2.analysis.ic import compute_rolling_ic, load_feature_series
from ta_lab2.features.cross_timeframe import load_ctf_features
from ta_lab2.scripts.refresh_utils import resolve_db_url
```

**Key helper functions:**

1. **_get_ctf_feature_names(engine) -> set[str]:**
   - Query dim_ctf_indicators for all indicator_name values
   - Build the set of all possible CTF feature name patterns: {indicator_name}_{ref_tf_lower}_{composite} for all indicators, ref_tfs from ctf_config.yaml, and composites (ref_value, base_value, slope, divergence, agreement, crossover)
   - Return the set (used to filter ic_results)

2. **_load_ctf_ic_ranking(engine, horizon=1, return_type="arith") -> pd.DataFrame:**
   - Query ic_results: SELECT feature, AVG(ABS(ic_ir)) AS mean_abs_ic_ir, AVG(ABS(ic)) AS mean_abs_ic, COUNT(*) AS n_obs, SUM(CASE WHEN ABS(ic_ir) >= 0.5 THEN 1 ELSE 0 END)::float / COUNT(*) AS pass_rate FROM ic_results WHERE horizon = :horizon AND return_type = :return_type AND feature = ANY(:ctf_features) GROUP BY feature ORDER BY mean_abs_ic_ir DESC
   - The :ctf_features param comes from _get_ctf_feature_names()
   - Return ranking DataFrame

3. **_load_ama_ic_ranking(engine, horizon=1, return_type="arith") -> pd.DataFrame:**
   - Same query as above but WHERE feature NOT IN (ctf_features) -- i.e., the AMA/features IC results from Phase 80
   - Used for comparison

4. **save_ctf_to_db(engine, results: list[dict]) -> int:**
   - Upsert to dim_ctf_feature_selection using ON CONFLICT (feature_name, base_tf) DO UPDATE
   - Each dict has: feature_name, base_tf, tier, ic_ir_mean, pass_rate, stationarity, ljung_box_flag, selected_at, yaml_version, rationale
   - Use text() + engine.begin() pattern
   - Return count of rows written
   - IMPORTANT: Do NOT call save_to_db() from feature_selection.py (that truncates dim_feature_selection)

5. **_build_comparison_report(ctf_ranking, ama_ranking, ctf_tiers, ...) -> tuple[str, dict]:**
   - Generate markdown report string and JSON dict
   - Sections: Summary Statistics (tier counts for CTF vs AMA), Top CTF Features by IC-IR, Redundancy Analysis (Spearman correlation between CTF and AMA IC series where both exist for same asset/tf), Head-to-Head (best CTF IC-IR vs best AMA IC-IR), Pruning Recommendations
   - For redundancy: compute correlation between CTF feature IC values and nearest AMA feature IC values (by same asset_id/tf) -- query ic_results for both feature sets, compute Spearman rank correlation on mean_abs_ic_ir vectors. A CTF feature is redundant if corr > 0.7 with any AMA feature.

6. **_prune_ctf_config(original_yaml_path, archive_indicators: dict[str, set[str]], output_path) -> None:**
   - Load original ctf_config.yaml
   - For each indicator in the indicators section, check if ALL composites at ALL base_tf x ref_tf combinations are in archive_indicators
   - If so, remove the indicator's ref_tf entries from timeframe_pairs (but keep the indicator in the indicators section for reference)
   - Actually: prune at the ref_tf level within timeframe_pairs. For each base_tf entry, only include ref_tfs that have at least one non-archive feature for ANY indicator. If a base_tf entry has no ref_tfs left, still keep it (per context decision: "Keep all 6 base TFs regardless of results")
   - Write to output_path (configs/ctf_config_pruned.yaml)
   - IMPORTANT: use encoding='utf-8' for yaml.dump on Windows

**main() pipeline steps:**

Step 1: Load CTF IC ranking
- Call _load_ctf_ic_ranking(engine, horizon=args.horizon, return_type=args.return_type)
- Print summary: N features found, top 10 by IC-IR

Step 2: Stationarity tests (if not --skip-stationarity)
- For top-N features by IC-IR (default 50, per research recommendation):
  - Parse indicator name and base_tf from feature name (e.g., rsi_14_7d_slope -> indicator rsi_14, ref_tf 7d)
  - Load feature values via load_ctf_features() for representative asset (BTC, id=1) at 1D base_tf
  - Run test_stationarity() on the column
- Store results in stationarity_results dict

Step 3: Ljung-Box tests (if not --skip-ljungbox)
- For top-N features:
  - Load rolling IC from ic_results for this feature
  - Run test_ljungbox_on_ic() on the rolling IC series
- Store results in ljungbox_results dict

Step 4: Classify all features into tiers
- For each CTF feature in ranking:
  - Call classify_feature_tier(ic_ir_mean, pass_rate, stationarity, regime_ic=None, ic_ir_cutoff=0.5)
  - Store tier assignment
- Build summary: count per tier

Step 5: Compare CTF vs AMA
- Call _load_ama_ic_ranking() for Phase 80 features
- Run _build_comparison_report()

Step 6: Persist to dim_ctf_feature_selection
- Build list of dicts with all tier assignments
- For base_tf: extract from the feature's IC results tf column (the ic_results.tf stores base_tf)
- Actually: each feature appears in ic_results at MULTIPLE base_tfs. Classify per (feature, base_tf) pair. The dim_ctf_feature_selection PK is (feature_name, base_tf).
- So: group ctf IC results by (feature, tf), compute per-group IC-IR and pass_rate, classify each group separately
- Call save_ctf_to_db()

Step 7: Prune config
- Identify archive indicators: for each indicator, collect the set of ref_tfs where ALL composites are archive across ALL base_tfs
- Call _prune_ctf_config()

Step 8: Write reports
- Write markdown to reports/ctf/ctf_ic_comparison_report.md
- Write JSON to reports/ctf/ctf_ic_comparison_report.json
- mkdir -p reports/ctf/ before writing

Step 9: Print terminal summary
- Print tier counts, top 10 active features, pruning summary, comparison headline

**CLI flags:**
- --all: run full pipeline (required unless --dry-run)
- --horizon: IC horizon for ranking (default 1)
- --return-type: IC return type (default "arith")
- --ic-ir-cutoff: tier classification cutoff (default 0.5, per CONTEXT.md)
- --top-n: number of features for stationarity/ljung-box (default 50)
- --dry-run: print ranking without writing to DB or files
- --skip-stationarity: skip ADF/KPSS tests
- --skip-ljungbox: skip Ljung-Box tests
- --yaml-version: version tag for dim_ctf_feature_selection (default "phase92_v1")
