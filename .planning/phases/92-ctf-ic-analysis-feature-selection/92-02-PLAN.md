**Module docstring:**
```
Batch IC sweep for Cross-Timeframe (CTF) features across all assets x base TFs.

Loads CTF pivot features via load_ctf_features(), computes IC for all feature
columns using batch_compute_ic(), and persists results to ic_results.

Usage:
    python -m ta_lab2.scripts.analysis.run_ctf_ic_sweep --all
    python -m ta_lab2.scripts.analysis.run_ctf_ic_sweep --assets 1 1027 --base-tf 1D
    python -m ta_lab2.scripts.analysis.run_ctf_ic_sweep --dry-run --min-bars 500
```

**Imports:**
```python
from ta_lab2.analysis.ic import batch_compute_ic, save_ic_results
from ta_lab2.features.cross_timeframe import load_ctf_features
from ta_lab2.scripts.refresh_utils import resolve_db_url
from ta_lab2.time.dim_timeframe import DimTimeframe
```

**Frozen dataclass (MANDATORY for Windows multiprocessing):**
```python
@dataclass(frozen=True)
class CTFICWorkerTask:
    asset_id: int
    base_tf: str
    n_rows: int
    db_url: str
    horizons: tuple        # tuple[int, ...] for pickling
    return_types: tuple    # tuple[str, ...]
    rolling_window: int
    tf_days_nominal: int
    overwrite: bool
```

**Discovery function _discover_ctf_pairs():**
- Query ctf table: SELECT id AS asset_id, base_tf, COUNT(DISTINCT ts) AS n_ts FROM public.ctf WHERE alignment_source = 'multi_tf' AND venue_id = 1 GROUP BY id, base_tf HAVING COUNT(DISTINCT ts) >= :min_bars ORDER BY id, base_tf
- Return list of (asset_id, base_tf, n_ts) tuples
- If --assets provided, filter to those asset IDs
- If --base-tf provided, filter to those base TFs

**Close loader _load_close_for_asset():**
- Query: SELECT ts, close FROM public.features WHERE id = :asset_id AND tf = :base_tf ORDER BY ts
- Fix tz: pd.to_datetime(df["ts"], utc=True)
- Return pd.Series indexed by UTC ts

**Train window _get_train_window():**
- Query: SELECT MIN(ts) AS train_start, MAX(ts) AS train_end FROM public.ctf WHERE id = :asset_id AND base_tf = :base_tf AND alignment_source = 'multi_tf' AND venue_id = 1
- Return (train_start, train_end) as UTC timestamps

**Module-level worker function (MANDATORY for Pool pickling):**
```python
def _ctf_ic_worker(task: CTFICWorkerTask) -> dict:
```
- Creates NullPool engine from task.db_url
- Calls load_ctf_features(conn, task.asset_id, task.base_tf, train_start, train_end)
- Calls _load_close_for_asset(conn, task.asset_id, task.base_tf)
- Pre-filters: drop columns where notna count < 50 (avoid IC on near-empty columns)
- Calls batch_compute_ic(ctf_df, close, train_start, train_end, horizons=list(task.horizons), return_types=list(task.return_types), rolling_window=task.rolling_window, tf_days_nominal=task.tf_days_nominal)
- Converts IC output to rows via _rows_from_ic_df() (same pattern as run_ic_sweep.py: list of dicts with asset_id, tf=base_tf, feature, horizon, return_type, ic, ic_t_stat, ic_p_value, ic_ir, ic_ir_t_stat, turnover, n_obs, train_start, train_end)
- Persists via save_ic_results(conn, rows, overwrite=task.overwrite)
- Returns summary dict: {asset_id, base_tf, n_features, n_ic_rows, elapsed_s}
- Wraps entire body in try/except: on error, log warning and return error dict

**_rows_from_ic_df() helper:**
- Follows the same pattern as run_ic_sweep.py's _rows_from_ic_df
- Each row dict contains: asset_id, tf (=base_tf), feature, horizon, return_type, ic, ic_t_stat, ic_p_value, ic_ir, ic_ir_t_stat, turnover, n_obs, train_start, train_end
- Use _to_python() for numpy scalar conversion (import from feature_selection or copy the hasattr(v, 'item') pattern)

**main() function:**
- argparse with flags:
  - --all: sweep all qualifying pairs
  - --assets: list of asset IDs (int nargs='+')
  - --base-tf: list of base TFs (str nargs='+')
  - --min-bars: minimum rows (default 500)
  - --horizons: IC horizons (default [1, 5, 10, 21] -- matching CONTEXT.md)
  - --rolling-window: rolling IC window (default 63)
  - --dry-run: list pairs without computing
  - --workers: number of parallel workers (default 4)
  - --overwrite: overwrite existing IC results (default True, store_true)
- Load DimTimeframe from DB: dim_tf = DimTimeframe.from_db(db_url)
- Discover pairs
- If --dry-run: print pairs and exit
- Build CTFICWorkerTask list (one per asset_id x base_tf pair)
  - tf_days_nominal = dim_tf.tf_days(base_tf) for each task
- Run with Pool(processes=args.workers, maxtasksperchild=1)
- Print summary: total pairs, successful, failed, total IC rows, elapsed time

**CRITICAL constraints:**
- horizons default [1, 5, 10, 21] (CONTEXT.md says match Phase 80 forward return horizons: 1d, 5d, 10d, 21d)
- return_types default ["arith", "log"]
- NullPool for multiprocessing (MANDATORY on Windows)
- maxtasksperchild=1 (MANDATORY on Windows to avoid memory leaks)
- All timestamps must be UTC-aware
- The tf column in ic_results stores the base_tf (e.g., "1D"), not the ref_tf
