<task type="auto">
  <name>Task 1: GARCH comparison report generator</name>
  <files>src/ta_lab2/scripts/garch/run_garch_comparison.py</files>
  <action>
Create CLI script that generates a comprehensive comparison report evaluating
GARCH variants against existing range-based volatility estimators.

**CLI interface (argparse):**
- `--ids`: Comma-separated asset IDs or "all" (default: "all")
- `--tf`: Timeframe (default: "1D")
- `--venue-id`: Venue ID (default: 1)
- `--db-url`: Database URL override
- `--output-dir`: Directory for report output (default: current directory)
- `--eval-window`: OOS evaluation window in days (default: 63)
- `--train-window`: Training window in days (default: 252)
- `--verbose`: Verbose output

**Report generation flow:**

1. Load returns data from `returns_bars_multi_tf` for each asset
2. Load existing vol estimators from `features` table (columns: `vol_parkinson_20`, `vol_gk_20`, `atr_14`) -- these are the range-based baselines
3. For each asset with sufficient history (>= train_window + eval_window):
   a. Compute realized vol proxy (5-day rolling std) via garch_evaluator
   b. Run rolling OOS evaluation for all 4 GARCH variants via `evaluate_all_models`
   c. Compute RMSE, QLIKE, MZ R2 for each GARCH variant against realized vol proxy
   d. Compute same metrics for Parkinson, GK, ATR against the same realized vol proxy
   e. Record per-asset results
4. Aggregate across assets:
   a. Mean and median RMSE/QLIKE/MZ R2 per estimator
   b. Count of assets where each estimator ranks #1 (by combined score)
   c. Distribution of per-asset improvement: GARCH best vs range best (median improvement %)
5. Generate outputs:
   a. **Markdown report** (`garch_comparison_report.md`):
      - Header with date, asset count, evaluation parameters
      - Aggregate results table (all estimators, sorted by combined score)
      - Per-asset top-3 table (which assets benefit most from GARCH)
      - Per-asset bottom-3 table (where range estimators are better)
      - Conclusion: whether GARCH adds value overall and for which assets
   b. **CSV files**:
      - `garch_comparison_aggregate.csv`: one row per estimator with all metrics
      - `garch_comparison_per_asset.csv`: one row per (asset, estimator) with all metrics
   c. Print summary to stdout

**Range-based estimator RMSE computation:**
- Load vol estimates from features table aligned to the same dates as GARCH OOS forecasts
- Parkinson, GK are daily vol estimates; ATR_14 needs conversion (atr_14 / close to get fractional vol)
- All comparisons against the same realized vol proxy (5-day rolling std)

**Per-asset granularity (Phase 80 lesson):**
The report MUST show per-asset results, not just aggregates. Phase 80 showed that
aggregation masks significant variation. Include:
- Per-asset best estimator identification
- Per-asset GARCH improvement over range-best
- Histogram summary of improvement distribution (as text, e.g. "12 assets show >10% improvement, 5 show degradation")
