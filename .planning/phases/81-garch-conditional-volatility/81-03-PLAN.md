  <name>Task 1: GARCH evaluation framework</name>
  <files>src/ta_lab2/analysis/garch_evaluator.py</files>
  <action>
Create `garch_evaluator.py` as a pure computation module (no DB, no CLI). It provides
accuracy metrics and rolling out-of-sample evaluation for volatility forecasts.

**Functions to implement:**

1. `compute_realized_vol_proxy(returns: pd.Series, window: int = 5) -> pd.Series`:
   - 5-day rolling standard deviation of returns (daily scale)
   - Do NOT use single-day squared returns (too noisy per research)
   - Do NOT use Parkinson/GK as proxy (that is what we compare against)
   - Returns pd.Series aligned to input index

2. `rmse_loss(sigma_forecast: np.ndarray, realized_vol: np.ndarray) -> float`:
   - `sqrt(mean((sigma_forecast - realized_vol)^2))`
   - Both inputs must be same scale (decimal daily vol)
   - Clip both to min 1e-8 before computation

3. `qlike_loss(sigma_forecast: np.ndarray, realized_vol: np.ndarray) -> float`:
   - `QLIKE = mean(log(sigma^2) + realized^2 / sigma^2)` per Patton (2011)
   - Clip both sigma^2 and realized^2 to min 1e-16 to avoid log(0) and div-by-zero (research pitfall #6)
   - Return float; lower is better

4. `mincer_zarnowitz_r2(sigma_forecast: np.ndarray, realized_vol: np.ndarray) -> float`:
   - OLS regression: `realized_vol = alpha + beta * sigma_forecast + epsilon`
   - Perfect forecast: alpha=0, beta=1, R2=1.0
   - Use `statsmodels.api.OLS` with `add_constant` (already installed)
   - Return R-squared as float

5. `combined_score(rmse: float, qlike: float, rmse_weight: float = 0.5) -> float`:
   - Weighted combination: `rmse_weight * rmse + (1 - rmse_weight) * qlike`
   - Simple combined metric for ranking

6. `rolling_oos_evaluate(returns: pd.Series, model_type: str, train_window: int = 252, eval_window: int = 63, step: int = 21) -> pd.DataFrame`:
   - Expanding window OOS evaluation (train on [0:t], forecast t+1, evaluate)
   - For each step:
     a. Fit GARCH on returns[:train_end] using `fit_single_variant` from garch_engine
     b. Generate h=1 forecast
     c. Record (date, model_type, forecast_vol, realized_vol)
   - After all steps, compute RMSE, QLIKE, MZ R2 on the OOS forecast series
   - Return DataFrame with columns: [date, model_type, forecast_vol, realized_vol, rmse_cumulative, qlike_cumulative]
   - Handle fit failures gracefully (skip that step, log warning)

7. `evaluate_all_models(returns: pd.Series, train_window: int = 252, eval_window: int = 63) -> dict[str, dict]`:
   - Run rolling_oos_evaluate for all 4 model types
   - Return dict[model_type -> {rmse, qlike, mz_r2, combined, n_forecasts}]

**Critical notes:**
- Forecast alignment: forecast for bar t is evaluated against realized vol at bar t+1 (avoid look-ahead bias per research pitfall #5). The h.1 forecast from fitting through bar t predicts t+1.
- Import garch_engine functions lazily (try/except) to handle missing arch dependency
- Use `from __future__ import annotations`
- Add proper docstrings with parameter descriptions

<task type="auto">
  <name>Task 2: Inverse-RMSE blend weight system</name>
  <files>src/ta_lab2/analysis/garch_blend.py</files>
  <action>
Create `garch_blend.py` providing the forecast combination system. This module
computes inverse-RMSE blend weights and produces a blended volatility estimate
combining GARCH variants with range-based estimators (Parkinson, GK, ATR).

**Components to implement:**

1. `@dataclass BlendConfig`:
   - `eval_window: int = 63` (trailing days for RMSE computation)
   - `min_weight: float = 0.05` (minimum weight floor per estimator)
   - `mode: str = "sizing_only"` (one of: 'sizing_only', 'sizing_and_limits', 'advisory')
   - `garch_model_types: list[str]` (default: all 4 model types)
   - `include_range_estimators: bool = True` (include Parkinson/GK in blend)

2. `compute_blend_weights(rmse_dict: dict[str, float], min_weight: float = 0.05) -> dict[str, float]`:
   - Input: dict mapping estimator name -> RMSE over trailing window
   - Compute inverse-RMSE: `1/rmse` for each estimator with rmse > 0
   - Normalize to sum to 1.0
   - Apply minimum weight floor and renormalize (Bates-Granger/Timmermann)
   - Return normalized weights dict

3. `compute_trailing_rmse(forecasts_df: pd.DataFrame, realized_vol: pd.Series, window: int = 63) -> dict[str, float]`:
   - Input: DataFrame with columns [ts, estimator_name, forecast_vol], plus realized_vol Series
   - Compute RMSE over last `window` days for each estimator
   - Return dict[estimator_name -> rmse]
   - Estimator names include GARCH model types + 'parkinson' + 'garman_klass' + 'atr_14'

4. `get_blended_vol(asset_id: int, venue_id: int, tf: str, engine, config: BlendConfig | None = None) -> dict`:
   - Reads latest forecasts from `garch_forecasts_latest` for the asset
   - Reads trailing RMSE data (from garch_diagnostics or computed on the fly from stored forecasts)
   - Computes blend weights via `compute_blend_weights`
   - Returns dict with keys:
     - `blended_vol`: weighted average of all estimator forecasts (float)
     - `weights`: dict[estimator_name -> weight]
     - `components`: dict[estimator_name -> forecast_vol]
     - `mode`: the BlendConfig mode
   - If no GARCH forecasts available, returns None (caller falls back to range-based vol)

5. `blend_vol_simple(garch_vol: float, range_vol: float, garch_weight: float) -> float`:
   - Simple two-input blend: `garch_weight * garch_vol + (1 - garch_weight) * range_vol`
   - Used by vol_sizer.py for inline blending without full DB lookup
   - Validates inputs (both > 0, weight in [0, 1])

**Design notes:**
- This module has BOTH pure computation functions (compute_blend_weights, blend_vol_simple)
  and DB-aware functions (get_blended_vol, compute_trailing_rmse)
- DB-aware functions use SQLAlchemy Engine parameter, same pattern as other scripts
- Use `from ta_lab2.analysis.garch_evaluator import rmse_loss` for consistency
- The get_blended_vol function is the main entry point for vol_sizer.py (Plan 04)
