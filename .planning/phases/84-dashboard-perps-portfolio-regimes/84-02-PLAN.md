Create `src/ta_lab2/dashboard/queries/regimes.py` following the exact pattern from `queries/research.py`:
- `from __future__ import annotations`, `import pandas as pd`, `import streamlit as st`, `from sqlalchemy import text`
- All functions: `@st.cache_data(ttl=900)`, `_engine` prefix

Implement these 4 functions:

1. `load_regime_all_assets(_engine, tf: str = "1D", days_back: int = 365) -> pd.DataFrame` -- regime states for ALL assets for heatmap.
   CRITICAL: trend_state derived via `split_part(l2_label, '-', 1)` -- regimes table has NO trend_state column.
   ```sql
   SELECT r.id, da.symbol, r.ts,
          split_part(r.l2_label, '-', 1) AS trend_state,
          split_part(r.l2_label, '-', 2) AS vol_state,
          r.regime_key
   FROM public.regimes r
   JOIN public.dim_assets da ON da.id = r.id
   WHERE r.tf = :tf
     AND r.ts >= NOW() - (:days_back || ' days')::interval
   ORDER BY r.id, r.ts
   ```
   Convert ts: `pd.to_datetime(df["ts"], utc=True)`

2. `load_regime_stats_summary(_engine, tf: str = "1D") -> pd.DataFrame` -- per-asset regime stats.
   NOTE: avg_ret_1d is NULL for all rows in current data -- still include it but don't rely on it.
   ```sql
   SELECT rs.id, da.symbol, rs.regime_key, rs.n_bars, rs.pct_of_history, rs.avg_ret_1d
   FROM public.regime_stats rs
   JOIN public.dim_assets da ON da.id = rs.id
   WHERE rs.tf = :tf
   ORDER BY rs.id, rs.n_bars DESC
   ```

3. `load_regime_flips_recent(_engine, tf: str = "1D", limit: int = 50) -> pd.DataFrame` -- most recent regime flips across all assets.
   ```sql
   SELECT rf.id, da.symbol, rf.ts, rf.old_regime, rf.new_regime, rf.duration_bars
   FROM public.regime_flips rf
   JOIN public.dim_assets da ON da.id = rf.id
   WHERE rf.tf = :tf
   ORDER BY rf.ts DESC
   LIMIT :limit
   ```
   Convert ts: `pd.to_datetime(df["ts"], utc=True)`

4. `load_regime_comovement(_engine, tf: str = "1D") -> pd.DataFrame` -- per-asset EMA comovement data.
   WARNING: Only 21 rows (7 assets x 3 EMA pairs). This is NOT cross-asset correlation. This tracks how EMAs within a single asset co-move.
   ```sql
   SELECT rc.id, da.symbol, rc.ema_a, rc.ema_b,
          rc.correlation, rc.sign_agree_rate, rc.best_lead_lag, rc.n_obs
   FROM public.regime_comovement rc
   JOIN public.dim_assets da ON da.id = rc.id
   WHERE rc.tf = :tf
   ORDER BY rc.id, rc.ema_a, rc.ema_b
   ```
