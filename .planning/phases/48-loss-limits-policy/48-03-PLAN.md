Create `src/ta_lab2/scripts/analysis/run_var_simulation.py`. Follow the existing pattern from `run_bakeoff_scoring.py` for CLI structure, logging, and report output.

**Usage:**
```
python -m ta_lab2.scripts.analysis.run_var_simulation
python -m ta_lab2.scripts.analysis.run_var_simulation --strategies ema_trend_17_77 ema_trend_21_50
python -m ta_lab2.scripts.analysis.run_var_simulation --dry-run
```

**Arguments:**
- `--strategies`: Space-separated strategy names (default: all 4 from bakeoff: ema_trend_17_77, ema_trend_21_50, rsi_mean_revert, breakout_atr)
- `--asset-ids`: Space-separated asset IDs (default: 1 2 for BTC/ETH)
- `--confidence-levels`: Confidence levels (default: 0.95 0.99)
- `--output-dir`: Report output directory (default: `reports/loss_limits/`)
- `--dry-run`: Print configuration and exit without running

**Data loading:**
1. First check if `cmc_backtest_metrics` has pre-computed VaR values:
   ```sql
   SELECT var_95, expected_shortfall FROM public.cmc_backtest_metrics
   WHERE signal_type = :signal_type AND asset_id = :asset_id
   LIMIT 5
   ```
2. If var_95 columns are populated (non-NULL), log "Using pre-computed VaR from cmc_backtest_metrics" and use those directly.
3. If NULL, load trade-level returns from `cmc_backtest_trades`:
   ```sql
   SELECT bt.return_pct
   FROM public.cmc_backtest_trades bt
   JOIN public.cmc_backtest_runs r ON bt.run_id = r.run_id
   WHERE r.signal_type = :signal_type AND r.asset_id = :asset_id
   ORDER BY bt.exit_ts
   ```
4. If both sources are empty, fall back to daily bar returns from `cmc_returns_bars_multi_tf_u`:
   ```sql
   SELECT ret_arith FROM public.cmc_returns_bars_multi_tf_u
   WHERE id = :asset_id AND tf = '1D'
   ORDER BY ts
   ```
   Log WARNING: "No backtest data found; using raw bar returns as proxy."

**Computation:**
- For each (strategy, asset_id, confidence_level): call `compute_var_suite()` from var_simulator
- Collect all VaRResult objects

**Report generation (VAR_REPORT.md):**
Write to `{output_dir}/VAR_REPORT.md`:
```markdown
# VaR Analysis Report

Generated: {timestamp}
Data source: {cmc_backtest_metrics | cmc_backtest_trades | cmc_returns_bars_multi_tf_u}

## Summary

| Strategy | Asset | Confidence | Hist VaR | CF VaR | Normal VaR | CVaR | CF Reliable | Skew | Excess Kurt |
|----------|-------|------------|----------|--------|------------|------|-------------|------|-------------|
...

## Key Finding: Historical vs Cornish-Fisher Divergence

{For each strategy where CF diverges from historical by > 20%:}
- {strategy}: At 99%, historical VaR = X%, CF VaR = Y%, divergence = Z%
  {If CF unreliable: "WARNING: Excess kurtosis = K > 8; CF VaR is unreliable for this strategy."}

## Recommended Daily Loss Cap

Based on {method} VaR:
- Recommended daily_loss_pct_threshold: {value}%
- Derivation: median of {method} VaR across strategies, capped at 15%
```

**Chart generation (var_comparison.html):**
Use Plotly grouped bar chart:
- X axis: strategy x confidence level
- Y axis: VaR value (negative, so flip for readability)
- Grouped bars: Historical, Parametric Normal, Cornish-Fisher
- Color: distinct colors per VaR method
- Save as `{output_dir}/charts/var_comparison.html` via `fig.write_html()`

Create `src/ta_lab2/scripts/analysis/run_stop_simulation.py`. Follow the pattern from `run_bakeoff_scoring.py`.

**Usage:**
```
python -m ta_lab2.scripts.analysis.run_stop_simulation
python -m ta_lab2.scripts.analysis.run_stop_simulation --strategies ema_trend_17_77 --asset-ids 1
python -m ta_lab2.scripts.analysis.run_stop_simulation --write-to-db
python -m ta_lab2.scripts.analysis.run_stop_simulation --dry-run
```

**Arguments:**
- `--strategies`: Space-separated (default: all 4)
- `--asset-ids`: Space-separated (default: 1 2)
- `--thresholds`: Stop thresholds as decimals (default: 0.01 0.03 0.05 0.07 0.10 0.15)
- `--time-bars`: Time-stop bar counts (default: 5 10 20 30)
- `--output-dir`: Report output directory (default: `reports/loss_limits/`)
- `--write-to-db`: If set, write optimal stop parameters to dim_risk_limits
- `--dry-run`: Print configuration and exit

**Data loading:**
Load signal entries/exits and price data for each (strategy, asset_id):

1. Load price from `cmc_price_bars_multi_tf_u`:
   ```sql
   SELECT ts, close FROM public.cmc_price_bars_multi_tf_u
   WHERE id = :asset_id AND tf = '1D'
   ORDER BY ts
   ```
   Create pd.Series with ts as index, close as values.

2. Load entry/exit signals from signal tables. Use the signal_type -> table mapping:
   - ema_trend_* -> cmc_signals_ema_crossover
   - rsi_* -> cmc_signals_rsi
   - breakout_atr* -> cmc_signals_atr_breakout

   ```sql
   SELECT entry_ts, exit_ts FROM public.{signal_table}
   WHERE asset_id = :asset_id AND direction = 'long'
   ORDER BY entry_ts
   ```
   Convert to boolean entry/exit Series aligned to price index.

3. If signal table is empty or does not exist, fall back to synthetic entries every 30 bars (for illustrative purposes). Log WARNING.

**Computation:**
- For each (strategy, asset_id): call `sweep_stops()` from stop_simulator
- Concatenate all results with strategy + asset_id columns

**Optimal stop selection:**
- Group by (stop_type, threshold)
- For each stop_type, find the threshold that maximizes Sharpe while keeping MaxDD < 2x the no-stop MaxDD
- If no threshold satisfies the constraint, select the one with best Sharpe-to-MaxDD ratio
- Log the optimal: "Optimal {stop_type} stop for {strategy}: {threshold}% (Sharpe={X}, MaxDD={Y}%)"

**Report generation (STOP_SIMULATION_REPORT.md):**
Write to `{output_dir}/STOP_SIMULATION_REPORT.md`:
```markdown
# Stop-Loss Simulation Report

Generated: {timestamp}

## Sweep Results

### {Strategy} on {Asset}

| Stop Type | Threshold | Sharpe | MaxDD | Return | Trades | Win Rate | Recovery (bars) | Opp. Cost |
|-----------|-----------|--------|-------|--------|--------|----------|-----------------|-----------|
| baseline  | none      | X      | X     | X      | X      | X        | X               | 0.0       |
| hard      | 1%        | X      | X     | X      | X      | X        | X               | X         |
...

### Whipsaw Analysis

Stops below {X}% significantly increase trade count (>2x baseline) with worse Sharpe.
This is consistent with BTC 1D average daily range of 3-5%.

## Optimal Stop Parameters

| Strategy | Asset | Best Hard | Best Trailing | Best Time |
|----------|-------|-----------|---------------|-----------|
...

## Recommendation

{Summary of findings: which stop type and threshold offers best risk-reward tradeoff}
```

**Chart generation (stop_heatmap.html):**
Use Plotly heatmap (or grouped bar chart):
- X axis: threshold %
- Y axis: stop type (hard, trailing, time)
- Color/Z: Sharpe ratio (or MaxDD)
- Create two subplots: one for Sharpe, one for MaxDD
- Save as `{output_dir}/charts/stop_heatmap.html`

**--write-to-db behavior:**
If --write-to-db is set:
1. For the optimal trailing stop threshold (best overall): UPDATE dim_risk_limits aggregate row's daily_loss_pct_threshold
2. Log: "Updated dim_risk_limits aggregate row: daily_loss_pct_threshold = {value}"
3. Use `WHERE pool_name = 'aggregate' OR (pool_name IS NULL AND asset_id IS NULL AND strategy_id IS NULL)` to target the correct row
4. If no aggregate row exists, INSERT one
