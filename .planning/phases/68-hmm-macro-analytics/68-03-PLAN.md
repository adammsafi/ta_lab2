**Key methods:**

1. `_load_rule_based_regimes(self) -> pd.DataFrame`:
   - SELECT date, macro_state FROM cmc_macro_regimes WHERE profile = 'default' ORDER BY date ASC.
   - Return DataFrame with date index and macro_state column.
   - If table is empty or does not exist, return empty DataFrame (log warning, do not crash).

2. `_load_hmm_regimes(self, n_states: int | None = None) -> pd.DataFrame`:
   - If n_states is None, use the BIC winner: SELECT date, state_label FROM cmc_hmm_regimes WHERE is_bic_winner = true ORDER BY date ASC. If no BIC winner flagged, default to n_states=2.
   - Otherwise: SELECT date, state_label FROM cmc_hmm_regimes WHERE n_states = :n_states ORDER BY date ASC. Use the most recent model_run_date.
   - Convert state_label (int) to string for consistency (e.g., "state_0", "state_1", "state_2").
   - Return DataFrame with date index and state_label column.

3. `_compute_transition_matrix(self, labels: pd.Series) -> pd.DataFrame`:
   - Count transitions: for each consecutive pair (labels[i], labels[i+1]), increment count[from_state][to_state].
   - Build a square DataFrame with states as both index and columns.
   - Row-normalize: each row sums to 1.0. Handle zero-count rows (set to 0.0, not NaN).
   - Return the transition probability matrix.

4. `compute_static(self, regime_source: str) -> pd.DataFrame`:
   - Load regime labels based on regime_source ("rule_based" or "hmm").
   - Compute transition matrix on full history.
   - Flatten to rows for DB storage: each cell (from_state, to_state, probability) becomes a row.
   - Add columns: regime_source, window_type="static", window_end_date=max(dates), transition_count, total_from_count, window_days=len(labels).
   - Return DataFrame shaped for cmc_macro_transition_probs upsert.

5. `compute_rolling(self, regime_source: str) -> pd.DataFrame`:
   - Load regime labels.
   - For each window_end_date starting from min(date) + rolling_window_days to max(date):
     - Slice labels for [window_end_date - rolling_window_days, window_end_date].
     - Compute transition matrix for this window.
     - Flatten to rows with window_type="rolling", window_end_date, window_days=rolling_window_days.
   - Concatenate all windows.
   - To manage output size: step by 1 day (every day gets a rolling matrix). If performance is an issue, make step configurable (default=1).
   - Return DataFrame shaped for cmc_macro_transition_probs upsert.

6. `compute_all(self) -> pd.DataFrame`:
   - Run compute_static for both "rule_based" and "hmm".
   - Run compute_rolling for both "rule_based" and "hmm".
   - Concatenate all results.
   - Log: total transition rows computed, sources processed.
   - Return combined DataFrame.

7. `upsert_results(self, df: pd.DataFrame) -> int`:
   - Temp table + ON CONFLICT (regime_source, window_type, window_end_date, from_state, to_state) DO UPDATE.
   - Follow refresh_macro_features.py upsert pattern.
   - Return row count.

**Part A: Create `src/ta_lab2/scripts/macro/refresh_macro_analytics.py`**

This is the unified CLI entry point for all three Phase 68 analytical tools. Follow the exact pattern of `refresh_macro_features.py`.

**CLI arguments:**
- `--full`: Force HMM refit + full lead-lag rescan (instead of incremental/weekly)
- `--dry-run`: Compute but do not write to DB
- `--verbose`: Set logging to DEBUG
- `--hmm-only`: Run only HMM classifier
- `--lead-lag-only`: Run only lead-lag analysis
- `--transition-only`: Run only transition probabilities
- `--covariance-type {full,diag}`: Override HMM covariance type (default: full)
- `--force-refit`: Force HMM model refit regardless of weekly cadence

If none of --hmm-only, --lead-lag-only, --transition-only are specified, run all three.

**Main flow:**

```python
def main() -> int:
    args = parse_args()
    setup_logging(args.verbose)
    engine = get_engine()

    run_all = not (args.hmm_only or args.lead_lag_only or args.transition_only)
    results = {}

    # 1. HMM classifier
    if run_all or args.hmm_only:
        logger.info("=== HMM Classifier ===")
        try:
            hmm = HMMClassifier(engine, covariance_type=args.covariance_type)
            df_hmm = hmm.fit_and_predict(force_refit=args.force_refit or args.full)
            logger.info(f"HMM: {len(df_hmm)} rows computed")
            if not args.dry_run and not df_hmm.empty:
                count = hmm.upsert_results(df_hmm)
                logger.info(f"HMM: upserted {count} rows")
            # Optional: run comparison
            try:
                comparison = hmm.compare_with_rule_based()
                logger.info(f"HMM vs rule-based: kappa={comparison['kappa']:.3f}, "
                           f"n_dates={comparison['n_aligned_dates']}")
            except Exception as e:
                logger.warning(f"HMM comparison skipped (rule-based data may not exist): {e}")
            results["hmm"] = "success"
        except Exception as e:
            logger.error(f"HMM classifier failed: {e}")
            results["hmm"] = f"failed: {e}"

    # 2. Lead-lag analysis
    if run_all or args.lead_lag_only:
        logger.info("=== Lead-Lag Analysis ===")
        try:
            lla = LeadLagAnalyzer(engine)
            df_ll = lla.scan_all()
            logger.info(f"Lead-lag: {len(df_ll)} feature-asset pairs scanned")
            sig_count = df_ll["is_significant"].sum() if not df_ll.empty else 0
            logger.info(f"Lead-lag: {sig_count} significant pairs")
            if not args.dry_run and not df_ll.empty:
                count = lla.upsert_results(df_ll)
                logger.info(f"Lead-lag: upserted {count} rows")
            results["lead_lag"] = "success"
        except Exception as e:
            logger.error(f"Lead-lag analysis failed: {e}")
            results["lead_lag"] = f"failed: {e}"

    # 3. Transition probabilities
    if run_all or args.transition_only:
        logger.info("=== Transition Probabilities ===")
        try:
            tpm = TransitionProbMatrix(engine)
            df_tp = tpm.compute_all()
            logger.info(f"Transition probs: {len(df_tp)} rows computed")
            if not args.dry_run and not df_tp.empty:
                count = tpm.upsert_results(df_tp)
                logger.info(f"Transition probs: upserted {count} rows")
            results["transition_probs"] = "success"
        except Exception as e:
            logger.error(f"Transition probabilities failed: {e}")
            results["transition_probs"] = f"failed: {e}"

    # Summary
    logger.info(f"Macro analytics complete: {results}")
    failures = [k for k, v in results.items() if "failed" in str(v)]
    return 1 if failures else 0
```
