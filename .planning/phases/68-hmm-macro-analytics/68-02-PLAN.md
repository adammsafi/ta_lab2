**NOTE on covariance_type:** The default is `"diag"` (diagonal covariance) because with 38 features, `"full"` covariance requires estimating O(n^2) parameters per state which may be unstable with limited training data. Users can opt into `"full"` covariance explicitly if they have sufficient data and want to capture feature cross-correlations.

**Class structure:**

```python
class HMMClassifier:
    def __init__(self, engine: Engine, covariance_type: str = "diag"):
        self.engine = engine
        self.covariance_type = covariance_type  # "diag" (default, safe for 38 features) or "full" (opt-in)
```

**Key methods:**

1. `_load_features(self, end_date: str | None = None) -> pd.DataFrame`:
   - SELECT date + _HMM_CANDIDATE_COLUMNS from `fred.fred_macro_features` ORDER BY date ASC.
   - Filter to only columns that actually exist in the table (gracefully handle Phase 66 not being fully applied -- use `_get_table_columns()` pattern or catch KeyError).
   - Set `date` as index.
   - Return the DataFrame.

2. `_fit_best_hmm(self, X: np.ndarray, n_states: int, random_seed: int = 42) -> GaussianHMM | None`:
   - Multiple random restarts (_N_RESTARTS). For each restart: create GaussianHMM with `n_components=n_states, covariance_type=self.covariance_type, n_iter=_N_ITER, tol=1e-4`, random_state from RNG. Fit on X. Score. Keep best by log-likelihood.
   - Return best model, or None if all restarts failed.
   - Wrap each restart in try/except (EM can diverge). Skip non-finite scores.

3. `fit_and_predict(self, force_refit: bool = False, end_date: str | None = None) -> pd.DataFrame`:
   - Load features.
   - Drop rows with any NaN (`dropna(how="any")`). Log how many rows remain vs total.
   - If fewer than _MIN_TRAIN_ROWS remain, raise ValueError with clear message.
   - StandardScaler fit_transform on the cleaned data.
   - Check if refit is needed: query MAX(model_run_date) from cmc_hmm_regimes. If model_run_date is less than _REFIT_INTERVAL_DAYS ago and not force_refit, skip fitting -- just predict from loaded model. For first run or force_refit, always fit.
   - For the expanding window approach: use ALL data from start up to end_date for fitting. This means each run uses all available history (no look-ahead because we only fit on data <= end_date).
   - For each n_states in _N_STATES_OPTIONS:
     - Call _fit_best_hmm.
     - Get BIC via model.bic(X_scaled) and AIC via model.aic(X_scaled).
     - Predict state labels: model.predict(X_scaled).
     - Get posterior probs: model.predict_proba(X_scaled) -- take max probability per row.
     - Extract state means: model.means_ (shape: n_states x n_features). Store as JSON string.
   - Determine BIC winner: compare BIC values across n_states options. LOWER BIC = better fit. Flag the winner.
   - Build output DataFrame with columns: date, n_states, model_run_date (today), state_label, state_probability, bic, aic, is_bic_winner, state_means_json, covariance_type, n_features.
   - Each date produces one row per n_states (so 2 rows per date: one for 2-state, one for 3-state).
   - Return the DataFrame.

4. `upsert_results(self, df: pd.DataFrame) -> int`:
   - Follow the temp table + ON CONFLICT pattern from refresh_macro_features.py.
   - Create temp table, COPY data in, INSERT INTO cmc_hmm_regimes ... ON CONFLICT (date, n_states, model_run_date) DO UPDATE SET all columns.
   - Use _to_python() and _sanitize_dataframe() helpers (copy from refresh_macro_features.py or import if accessible).
   - Return row count.

5. `compare_with_rule_based(self, n_states: int | None = None) -> dict`:
   - Load latest HMM labels (filter to BIC winner if n_states not specified).
   - Load rule-based labels from cmc_macro_regimes (macro_state column).
   - Align by date.
   - Compute confusion matrix (sklearn.metrics.confusion_matrix) and Cohen's kappa (sklearn.metrics.cohen_kappa_score).
   - Return dict with keys: confusion_matrix, kappa, n_aligned_dates, hmm_n_states.
   - NOTE: HMM states are indices (0,1,2) while rule-based are strings (favorable/constructive/neutral/cautious/adverse). For kappa, both just need to be consistent labels -- sklearn handles mixed types. But document that HMM state 0 != "favorable" automatically -- the comparison is about structural agreement, not label identity.

**Lead-lag convention (MUST document in docstring):**
- `col_a` = macro feature (reference), `col_b` = asset returns (shifted)
- `best_lag < 0`: macro feature leads asset returns (macro is predictive)
- `best_lag > 0`: asset returns lead macro feature (macro is lagging)

**Class structure:**

```python
class LeadLagAnalyzer:
    def __init__(self, engine: Engine, lag_range: range | None = None):
        self.engine = engine
        self.lag_range = lag_range or _DEFAULT_LAG_RANGE
```

**Key methods:**

1. `_load_macro_features(self) -> pd.DataFrame`:
   - SELECT date + all float columns from `fred.fred_macro_features` ORDER BY date ASC.
   - Use the same _HMM_CANDIDATE_COLUMNS list (import from hmm_classifier or define a shared constant in __init__.py or a _constants.py file -- prefer importing from hmm_classifier to avoid duplication).
   - Set `date` as index (DatetimeIndex, tz-aware UTC).

2. `_load_asset_returns(self, asset_ids: list[int] | None = None) -> pd.DataFrame`:
   - For each asset_id in (asset_ids or _DEFAULT_ASSET_IDS):
     - SELECT ts, ret_cc (or the close-to-close return column) FROM cmc_returns_bars_multi_tf WHERE id = :id AND tf = '1D' ORDER BY ts ASC.
     - Rename column to e.g. `btc_1d_return` or `eth_1d_return` (use a mapping: {1: "btc", 2: "eth"}).
   - Merge all asset returns on ts (outer join).
   - Set ts as index (DatetimeIndex, tz-aware UTC).
   - CRITICAL: Align with macro features by date. Use `.loc` on the intersection of both indices. Handle tz-awareness: macro features are DATE (tz-naive), returns are TIMESTAMP WITH TZ. Normalize both to date-level index before joining: `df.index = pd.to_datetime(df.index).normalize().tz_localize(None)`.

3. `scan_all(self, asset_ids: list[int] | None = None) -> pd.DataFrame`:
   - Load macro features and asset returns.
   - Build a combined DataFrame aligned by date index.
   - For each macro_feature column x each asset_return column:
     - Call `lead_lag_max_corr(combined_df, col_a=macro_col, col_b=asset_col, lags=self.lag_range)`.
     - Extract best_lag, best_corr, corr_by_lag.
     - Compute N (number of non-NaN overlapping observations for that pair).
     - Compute Bartlett significance threshold: `2.0 / np.sqrt(N)`.
     - Flag is_significant: `abs(best_corr) > bartlett_threshold`.
     - Store corr_by_lag as JSON string (dict of lag -> corr).
   - Build result DataFrame with columns matching cmc_macro_lead_lag_results schema:
     - macro_feature, asset_col, computed_at (today's date), best_lag, best_corr, is_significant, n_obs, lag_range_min, lag_range_max, corr_by_lag_json.
   - Log: total pairs scanned, significant pairs count, top 5 by absolute correlation.
   - Return the DataFrame.

4. `upsert_results(self, df: pd.DataFrame) -> int`:
   - Temp table + ON CONFLICT (macro_feature, asset_col, computed_at) DO UPDATE pattern.
   - Follow same upsert style as refresh_macro_features.py.
   - Use _to_python() / _sanitize_dataframe() helpers.
   - Return row count.

**Bartlett threshold explanation (document in docstring):**
For a cross-correlation of N observations, values beyond +/- 2/sqrt(N) are statistically significant at approximately the 95% confidence level under the null hypothesis of no correlation. This is a standard threshold from time series analysis (Bartlett, 1946).
