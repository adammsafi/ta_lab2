2. Generate the migration: `alembic revision -m "hmm_macro_analytics_tables"`. Then hand-write the table definitions.

3. The migration creates THREE tables in the PUBLIC schema (no schema= argument):

**Table 1: `cmc_hmm_regimes`**
- `date` DATE NOT NULL
- `n_states` INTEGER NOT NULL (2 or 3)
- `model_run_date` DATE NOT NULL (date the model was last fit)
- `state_label` INTEGER nullable (HMM state index: 0, 1, 2)
- `state_probability` FLOAT nullable (posterior probability of assigned state)
- `bic` FLOAT nullable (Bayesian Information Criterion for this model)
- `aic` FLOAT nullable (Akaike Information Criterion for this model)
- `is_bic_winner` BOOLEAN NOT NULL server_default='false' (true if this n_states had best BIC)
- `state_means_json` TEXT nullable (JSON string of state mean vectors for interpretation)
- `covariance_type` TEXT NOT NULL server_default='full'
- `n_features` INTEGER nullable (number of input features used)
- `ingested_at` TIMESTAMP WITH TIME ZONE NOT NULL, server_default=func.now()
- PK: (date, n_states, model_run_date)
- Index: idx_cmc_hmm_regimes_date on (date DESC)
- Index: idx_cmc_hmm_regimes_winner on (is_bic_winner) WHERE is_bic_winner = true (partial index)

**Table 2: `cmc_macro_lead_lag_results`**
- `macro_feature` TEXT NOT NULL (column name from fred_macro_features, e.g. 'vixcls')
- `asset_col` TEXT NOT NULL (e.g. 'btc_1d_return' or 'eth_1d_return')
- `computed_at` DATE NOT NULL (date this analysis was run)
- `best_lag` INTEGER nullable (lag with highest absolute correlation; negative = macro leads)
- `best_corr` FLOAT nullable (correlation at best_lag)
- `is_significant` BOOLEAN nullable (true if |best_corr| > 2/sqrt(N))
- `n_obs` INTEGER nullable (number of overlapping observations)
- `lag_range_min` INTEGER nullable (min lag tested, e.g. -60)
- `lag_range_max` INTEGER nullable (max lag tested, e.g. +60)
- `corr_by_lag_json` TEXT nullable (JSON string of {lag: corr} for full correlation profile)
- `ingested_at` TIMESTAMP WITH TIME ZONE NOT NULL, server_default=func.now()
- PK: (macro_feature, asset_col, computed_at)
- Index: idx_lead_lag_significant on (is_significant) WHERE is_significant = true (partial index)

**Table 3: `cmc_macro_transition_probs`**
- `regime_source` TEXT NOT NULL ('rule_based' or 'hmm')
- `window_type` TEXT NOT NULL ('static' or 'rolling')
- `window_end_date` DATE NOT NULL (for static: max date in history; for rolling: end of rolling window)
- `from_state` TEXT NOT NULL (regime label being transitioned FROM)
- `to_state` TEXT NOT NULL (regime label being transitioned TO)
- `probability` FLOAT NOT NULL (row-normalized transition probability, 0.0 to 1.0)
- `transition_count` INTEGER NOT NULL (raw count of from->to transitions in window)
- `total_from_count` INTEGER NOT NULL (total transitions from from_state in window)
- `window_days` INTEGER nullable (for rolling: window size in days; for static: total history days)
- `ingested_at` TIMESTAMP WITH TIME ZONE NOT NULL, server_default=func.now()
- PK: (regime_source, window_type, window_end_date, from_state, to_state)
- Index: idx_transition_source_date on (regime_source, window_end_date DESC)

4. The downgrade function must drop all three tables in reverse creation order.

5. Follow the exact import and style pattern from the Phase 66 migration file: `from alembic import op` and `import sqlalchemy as sa`, docstring with phase reference, revision identifiers at top.
  </action>
