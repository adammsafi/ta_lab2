When a policy table key contains glob characters (`*`, `?`, or `[`), use `fnmatch.fnmatch(regime_key, k)` instead of token-based matching. If the key does NOT contain glob characters, fall through to the existing token-based `all(t in regime_key for t in tokens)` check. This ensures zero regression for existing L0-L2 entries.

Updated `_match_policy()`:
```python
def _match_policy(
    regime_key: str, table: Mapping[str, Mapping[str, object]]
) -> Dict[str, object]:
    for k, v in table.items():
        # Glob pattern matching (e.g. '*-RiskOff-*') -- check first
        if "*" in k or "?" in k or "[" in k:
            if fnmatch.fnmatch(regime_key, k):
                return dict(v)
            continue
        # Existing token-based substring matching (no glob chars)
        tokens = [t for t in k.split("-") if t]  # ignore empty fragments
        if all(t in regime_key for t in tokens):
            return dict(v)
    # fallback
    return {
        "size_mult": 0.8,
        "stop_mult": 1.5,
        "setups": ["pullback"],
        "orders": "mixed",
    }
```
