**Change 1: Add `_load_regime_for_asset()` helper method to PaperExecutor**

Add this method in the internal helpers section (after `_load_active_configs`, around line 236):

```python
    def _load_regime_for_asset(
        self, conn, asset_id: int
    ) -> dict:
        """
        Read the latest regime row from cmc_regimes for the given asset,
        including L0-L2 labels, L4 macro label, gross_cap, and size_mult.

        Returns a dict with keys: l0_label, l1_label, l2_label, l4_label,
        gross_cap, size_mult. All label fields may be None.
        If no regime row exists, returns defaults (all labels None,
        gross_cap=1.0, size_mult=1.0).
        """
        defaults = {
            "l0_label": None,
            "l1_label": None,
            "l2_label": None,
            "l4_label": None,
            "gross_cap": 1.0,
            "size_mult": 1.0,
        }
        try:
            row = conn.execute(
                text(
                    """
                    SELECT l0_label, l1_label, l2_label, l4_label,
                           gross_cap, size_mult
                    FROM public.cmc_regimes
                    WHERE id = :asset_id AND tf = '1D'
                    ORDER BY ts DESC
                    LIMIT 1
                    """
                ),
                {"asset_id": asset_id},
            ).fetchone()
        except Exception as exc:  # noqa: BLE001
            self.logger.debug(
                "_load_regime_for_asset: query failed for asset_id=%d: %s",
                asset_id,
                exc,
            )
            return defaults

        if row is None:
            return defaults

        return {
            "l0_label": row.l0_label,
            "l1_label": row.l1_label,
            "l2_label": row.l2_label,
            "l4_label": row.l4_label,
            "gross_cap": float(row.gross_cap) if row.gross_cap is not None else 1.0,
            "size_mult": float(row.size_mult) if row.size_mult is not None else 1.0,
        }
```

Note: The method returns a dict (not a tuple) for clarity since we now return 6 fields. All callers destructure by key name.

**Change 2: Apply gross_cap scaling in `_process_asset_signal()` BEFORE RiskEngine**

In `_process_asset_signal()`, AFTER `target_qty` is computed by `PositionSizer.compute_target_position()` (line 434) and BEFORE the current position query `pos_row = conn.execute(...)` (line 437), insert the regime loading and gross_cap scaling:

```python
        # --- L4 macro regime: read full regime context and apply gross_cap ---
        regime_info = self._load_regime_for_asset(conn, asset_id)
        l4_gross_cap = regime_info["gross_cap"]
        if l4_gross_cap < 1.0:
            original_target = target_qty
            target_qty = target_qty * Decimal(str(l4_gross_cap))
            self.logger.info(
                "_process_asset_signal: asset_id=%d L4 gross_cap=%.2f "
                "scaling target_qty %.6f -> %.6f (l4=%s)",
                asset_id,
                l4_gross_cap,
                float(original_target),
                float(target_qty),
                regime_info["l4_label"],
            )
```

This applies the adaptive gross_cap BEFORE the delta computation and RiskEngine gate, so the risk engine sees the already-scaled-down target. The gross_cap scaling reduces the target BEFORE computing delta, which is the correct behavior: it reduces the desired position size, not the order delta.

        self._current_l4_size_mult: Optional[float] = None
