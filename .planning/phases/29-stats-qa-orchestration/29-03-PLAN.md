  <action>
    Create `src/ta_lab2/scripts/stats/weekly_digest.py` as a standalone script with `if __name__ == "__main__": main()`.

    1. Define DIGEST_TABLES list of (label, table_name) tuples:
       - ("bars", "public.price_bars_multi_tf_stats")
       - ("ema_multi_tf", "public.ema_multi_tf_stats")
       - ("ema_cal", "public.ema_multi_tf_cal_stats")
       - ("ema_cal_anchor", "public.ema_multi_tf_cal_anchor_stats")
       - ("returns_ema", "public.returns_ema_stats")
       - ("features", "public.cmc_features_stats")
       - ("audit", "public.audit_results")

    2. Create `query_period_status(engine, table, interval_start, interval_end)` function:
       - Query: SELECT status, COUNT(*) as n FROM {table} WHERE checked_at >= {start} AND checked_at < {end} GROUP BY status
       - Return dict[str, int] mapping status -> count
       - Use sqlalchemy.text() with bound parameters for interval values

    3. Create `build_weekly_summary(engine)` function:
       - For each table in DIGEST_TABLES, query this week (last 7 days) and last week (7-14 days ago)
       - Build per-table line: "{STATUS_ICON} {label}: {n_pass} pass / {n_warn} warn / {n_fail} fail"
       - STATUS_ICON: "PASS" if no FAIL and no WARN, "WARN" if no FAIL but has WARN, "FAIL" if has FAIL
       - Build top-level summary: "Overall: {total_pass} pass / {total_warn} warn / {total_fail} fail"

    4. Create `build_weekly_delta(this_week, last_week)` function:
       - Compare aggregate FAIL counts this week vs last week
       - If total FAIL increased: "Delta: +{N} new FAILs since last week"
       - If total FAIL decreased: "Delta: -{N} fewer FAILs since last week"
       - If unchanged: "Delta: No change in FAIL count"
       - Also note WARN count changes
       - Return a string with the delta summary

    5. Create `format_digest(summary_lines, delta_lines)` function:
       - Combine into a formatted message suitable for Telegram
       - Keep total message under 4000 characters (leave room for HTML formatting)
       - If message too long, truncate per-table detail to top 5 tables by FAIL count
       - Include header: "Weekly QC Digest" and timestamp

    6. Create `send_digest(engine)` function:
       - Build summary and delta
       - Print to stdout (always)
       - If telegram.is_configured(): send via telegram.send_alert(title="Weekly QC Digest", message=..., severity=overall_severity)
       - overall_severity: "critical" if any FAIL, "warning" if any WARN, "info" if all PASS

    7. Create `main()` with argparse:
       - --db-url: Database URL (default: from config/env)
       - --no-telegram: Skip Telegram delivery, print to stdout only
       - --verbose: Show per-test detail (not just per-table)
       - Resolve db_url via resolve_db_url from refresh_utils
       - Call send_digest()

    IMPORTANT: For week-over-week delta, compare AGGREGATE counts (total FAIL this week vs last week), NOT individual row comparisons. The delete-before-insert pattern means old rows for un-impacted keys may not be present.

    IMPORTANT: Telegram has a 4096 character limit per message. If the full digest exceeds this, split into: (1) top-level summary message, (2) per-table breakdown as a second message. Use telegram.send_message() for the second part if needed.

    IMPORTANT: Handle the case where a stats table doesn't exist yet (e.g., if a runner has never been executed). Wrap each table query in try/except and note "no data" for that table.
