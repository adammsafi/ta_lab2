    ```python
        cols = [col.strip() for col in pk_str.split(",") if col.strip()]
    ```

    ```python
        if df.empty:
            return 0
        self._ensure_output_table()
        df_write = df.copy()
        # Stamp alignment_source BEFORE to_sql -- _pg_upsert uses df columns for INSERT
        if self.config.alignment_source:
            df_write["alignment_source"] = self.config.alignment_source
        # ... existing dedup/filter logic ...
        rows = df_write.to_sql(...)  # alignment_source is now in the DataFrame
        ...
    ```

    **Part A: Row count parity verification**

    NOTE: This row count check validates the EXISTING data in ema_multi_tf_u that was populated by the old sync script. The redirected builders have not yet been executed to write new data. The purpose is to confirm that the _u table already contains parity data from the sync process BEFORE the old sync path is disabled. Once confirmed, the builders will maintain this data going forward via direct writes.

    Run a SQL query against the database:

    ```sql
    WITH siloed AS (
        SELECT 'multi_tf' AS alignment_source, COUNT(*) AS n_rows FROM ema_multi_tf
        UNION ALL
        SELECT 'multi_tf_cal_us', COUNT(*) FROM ema_multi_tf_cal_us
        UNION ALL
        SELECT 'multi_tf_cal_iso', COUNT(*) FROM ema_multi_tf_cal_iso
        UNION ALL
        SELECT 'multi_tf_cal_anchor_us', COUNT(*) FROM ema_multi_tf_cal_anchor_us
        UNION ALL
        SELECT 'multi_tf_cal_anchor_iso', COUNT(*) FROM ema_multi_tf_cal_anchor_iso
    ),
    unified AS (
        SELECT alignment_source, COUNT(*) AS n_rows
        FROM ema_multi_tf_u
        GROUP BY alignment_source
    )
    SELECT
        s.alignment_source,
        s.n_rows AS siloed_count,
        COALESCE(u.n_rows, 0) AS unified_count,
        COALESCE(u.n_rows, 0) - s.n_rows AS deficit,
        CASE WHEN COALESCE(u.n_rows, 0) >= s.n_rows THEN 'MATCH' ELSE 'DEFICIT' END AS status
    FROM siloed s
    LEFT JOIN unified u ON s.alignment_source = u.alignment_source
    ORDER BY s.alignment_source;
    ```

    Save output to `.planning/phases/77-direct-to-u-remaining-families/77-02-row-count-verification.txt`.

    All 5 alignment_sources must show MATCH or surplus. This confirms the existing sync'd data has parity before the old sync path is retired.
