       - CalAnchorISOAMAFeature (line ~277): Change `bars_table: str = "price_bars_multi_tf_cal_anchor_iso"` to `bars_table: str = "price_bars_multi_tf_u"`

    **B. Add alignment_source filter to bar-read SQL queries:**

    All 3 files have `preload_all_bars()` and some have `_load_bars()` methods with SQL like:
    ```sql
    SELECT ... FROM {self.bars_schema}.{self.bars_table}
    WHERE id = :id AND venue_id = :venue_id
    ```

    For each such SQL query, add an alignment_source filter. The alignment_source value comes from `self.config.alignment_source` (AMAFeatureConfig).

    The pattern to apply in EVERY preload_all_bars() and _load_bars() method across all 3 files:
    - Build the alignment_source filter conditionally: if `self.config.alignment_source` is not None, append `AND alignment_source = :alignment_source` to the WHERE clause and add `alignment_source` to the params dict.
    - This is the same pattern used by the WRITE path in base_ama_feature.py (line ~433-446).

    Example for preload_all_bars in ama_multi_timeframe.py:
    ```python
    alignment_filter = ""
    params = {"id": asset_id, "venue_id": venue_id}
    if self.config.alignment_source:
        alignment_filter = "AND alignment_source = :alignment_source"
        params["alignment_source"] = self.config.alignment_source

    sql = text(
        f"""
        SELECT id, venue_id, "timestamp" AS ts, tf, tf_days, is_partial_end AS roll, close, is_partial_end
        FROM {self.bars_schema}.{self.bars_table}
        WHERE id = :id AND venue_id = :venue_id
        {alignment_filter}
        ORDER BY tf, "timestamp"
        """
    )
    ```

    Apply the same pattern to ALL SQL queries that read from self.bars_table in all 3 files:
    - ama_multi_timeframe.py: preload_all_bars (1 query)
    - ama_multi_tf_cal.py: CalUSAMAFeature.preload_all_bars, CalUSAMAFeature._load_bars, CalISOAMAFeature.preload_all_bars, CalISOAMAFeature._load_bars (4 queries)
    - ama_multi_tf_cal_anchor.py: CalAnchorUSAMAFeature.preload_all_bars, CalAnchorUSAMAFeature._load_bars, CalAnchorISOAMAFeature.preload_all_bars, CalAnchorISOAMAFeature._load_bars (4 queries)

    Total: 9 SQL queries to update.

    IMPORTANT: The _load_bars() methods in cal/cal_anchor files build a WHERE clause dynamically. Append the alignment_source filter in the same way, after the existing WHERE conditions.
  </action>
