    1. For `build_views()`: compute a per-asset composite score. For each asset, extract its IC-IR row from the DataFrame, weight the signal_scores columns by those IC-IR values, and produce a composite score. Then cross-sectional z-score the composites across ALL assets (single call, not per-asset). This avoids the degenerate z-score anti-pattern.
    2. Concretely: compute `weighted_composite = (signal_scores * ic_ir_matrix).sum(axis=1)` where ic_ir_matrix is aligned to signal_scores columns and reindexed to signal_scores index (missing assets get universal fallback). Then z-score this composite cross-sectionally.
    3. View confidences: use the mean IC-IR per asset (mean of its row) as the confidence basis, normalized to [min_view_confidence, max_view_confidence].
    4. The existing `signals_to_mu()` method: add a parallel path when `ic_ir` is a DataFrame. Same weighted composite approach: for each asset, weight its features by its own IC-IR row, then z-score cross-sectionally.

    ```python
    def _per_asset_composite(
        self,

        """
        Compute per-asset IC-IR weighted composite and per-asset mean confidence.
    ```

    - Clip IC-IR to >= 0.
    - Per-asset weighted composite: for each row, multiply signal_scores by aligned IC-IR, sum across features.
    - Mean IR per asset = aligned IC-IR row means (used for confidence).
    - Return composite Series and mean_ir Series.

    **PositionSizer.compute_target_position() extension**:
    - Add `**kwargs` to the method signature (after `signal_confidence`)
    - Add new sizing_mode branch `elif sizing_mode == "target_vol":` BEFORE the fixed_fraction fallback
    - Implementation:
      ```
      target_ann_vol = getattr(config, "target_annual_vol", None)
      garch_vol = kwargs.get("garch_vol")  # daily decimal vol from get_blended_vol()

      if target_ann_vol and target_ann_vol > 0 and garch_vol:
          # Annualize daily GARCH vol: daily_vol * sqrt(252)
          current_ann_vol = float(garch_vol) * (252 ** 0.5)
          if current_ann_vol > 1e-6:
              vol_scalar = float(target_ann_vol) / current_ann_vol
              fraction = Decimal(str(config.position_fraction)) * Decimal(str(vol_scalar))
          else:
              # Near-zero vol: fallback to fixed_fraction
              fraction = Decimal(str(config.position_fraction))
              logger.warning("target_vol: current_ann_vol near zero (%.8f), using fixed_fraction", current_ann_vol)
      else:
          # Fallback: no GARCH vol available or target not configured
          fraction = Decimal(str(config.position_fraction))
          if target_ann_vol and not garch_vol:
              logger.info("target_vol: no garch_vol provided, falling back to fixed_fraction")
      ```
    - Apply the same max_fraction hard cap that exists for other modes
    - Update the module-level `compute_target_position()` wrapper to also accept **kwargs and pass through

    CRITICAL: Do NOT import get_blended_vol inside position_sizer.py. The GARCH vol is passed as a kwarg from the caller (run_paper_executor's _process_asset_signal loop where the engine is available). This follows the research recommendation and avoids violating the project pattern of not constructing engines inside static methods.

    CRITICAL: The annualization factor is sqrt(252). garch_blend.get_blended_vol() returns daily decimal vol. Forgetting to annualize causes positions ~15x too large (Pitfall 2 from research).

    - Update class docstring to mention the 4th sizing mode: `target_vol`
