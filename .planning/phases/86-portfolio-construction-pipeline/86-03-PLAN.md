    Extend run_parity_check.py with a `--bakeoff-winners` flag that auto-discovers signal IDs from the bake-off results and runs parity checks for each.

    **New CLI flag**:
    - `--bakeoff-winners`: action="store_true", help="Auto-discover winning signal IDs from strategy_bakeoff_results and run parity for each"
    - When `--bakeoff-winners` is used, `--signal-id` becomes optional (not required)
    - `--slippage-mode` default changes to "fixed" when --bakeoff-winners is used (because historical replay has fill price differences)

    **Auto-discovery logic** (new function `_discover_bakeoff_winners(engine) -> list[dict]`):
    1. Query `strategy_bakeoff_results` for winners:
       ```sql
       SELECT DISTINCT sr.strategy_name, sr.asset_id, sr.signal_id
       FROM strategy_bakeoff_results sr
       WHERE sr.is_winner = true
         AND sr.signal_id IS NOT NULL
       ORDER BY sr.strategy_name, sr.asset_id
       ```
    2. If signal_id column is NULL or doesn't exist in strategy_bakeoff_results (MEDIUM confidence from research), try fallback:
       ```sql
       SELECT DISTINCT sr.strategy_name, sr.asset_id
       FROM strategy_bakeoff_results sr
       WHERE sr.is_winner = true
       ```
       Then look up signal_id from dim_signals: `SELECT signal_id FROM dim_signals WHERE signal_type = :strategy_name`
    3. Return list of dicts with keys: strategy_name, asset_id, signal_id
    4. If no winners found, print error and return exit code 1

    **Multi-signal parity loop** (in main()):
    When `--bakeoff-winners` is set:
    1. Call `_discover_bakeoff_winners(engine)`
    2. For each winner, call `checker.check(config_id=args.config_id, signal_id=winner["signal_id"], ...)`
    3. Print each report
    4. At end, print summary: N/M signals passed parity
    5. Exit 0 if ALL pass, exit 1 if ANY fail

Phase 82 bake-off results are in strategy_bakeoff_results, not backtest_trades. A backtest run linking step may be needed."
