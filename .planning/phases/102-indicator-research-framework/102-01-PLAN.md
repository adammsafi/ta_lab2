  <name>Task 2: Core statistical functions — permutation_ic_test, fdr_control, log_trials_to_registry</name>
  <files>src/ta_lab2/analysis/multiple_testing.py</files>
  <action>
Create new module `src/ta_lab2/analysis/multiple_testing.py` with three functions:

1. **permutation_ic_test(feature, fwd_returns, n_perms=10_000, seed=42) -> dict**
   - Drop NaN pairs from both arrays
   - Compute observed IC via spearmanr
   - Shuffle fwd_returns n_perms times using np.random.default_rng(seed).permutation()
   - Compute |IC| on each shuffle
   - p_value = fraction of null |IC| >= observed |IC|
   - Return dict: {ic_obs, p_value, percentile_95, passes (bool: |ic_obs| >= pct_95), n_perms, n_obs}
   - Edge case: if fewer than 20 valid observations after NaN drop, return passes=False with p_value=1.0

2. **fdr_control(p_values, alpha=0.05) -> dict**
   - Use statsmodels.stats.multitest.fdrcorrection(p_arr, alpha=alpha, method='indep')
   - Return dict: {rejected (bool array), p_adjusted (array), n_rejected (int), alpha}
   - Edge case: empty p_values array returns empty arrays and n_rejected=0

3. **log_trials_to_registry(conn, ic_rows, source_table='ic_results') -> int**
   - Takes a SQLAlchemy connection and list of ic_row dicts (same format as save_ic_results)
   - Maps ic_row fields to trial_registry columns:
     - feature -> indicator_name
     - '' -> param_set (default for non-parametric features)
     - tf, asset_id, horizon, return_type mapped directly
     - alignment_source -> venue_id via lookup: 'multi_tf' maps to 1 (default)
     - ic -> ic_observed, ic_p_value -> ic_p_value, n_obs -> n_obs
   - Only insert rows where horizon=1 AND return_type='arith' (permutation scope reduction per research)
   - Uses INSERT...ON CONFLICT DO UPDATE SET ic_observed=EXCLUDED.ic_observed, ic_p_value=EXCLUDED.ic_p_value, sweep_ts=EXCLUDED.sweep_ts, n_obs=EXCLUDED.n_obs
   - MUST NOT overwrite perm_p_value, fdr_p_adjusted, passes_fdr, bb_ci_lo, bb_ci_hi, bb_block_len (preserve expensive computations)
   - Returns count of rows upserted
   - Use temp table + executemany pattern consistent with existing upsert patterns in the project

Add module docstring explaining the multiple-comparison testing harness purpose. Add proper type hints and logging via logging.getLogger(__name__).
  </action>
