1. **dim_macro_events** -- Scheduled event calendar:
   - `event_id SERIAL PRIMARY KEY`
   - `event_type TEXT NOT NULL` -- 'fomc', 'cpi', 'nfp'
   - `event_ts TIMESTAMPTZ NOT NULL` -- exact announcement timestamp in UTC
   - `data_period TEXT NOT NULL` -- e.g. '2026-01' for January 2026 data

2. **dim_macro_gate_state** -- Live gate state (one row per gate):
   - `gate_id TEXT PRIMARY KEY` -- 'fomc', 'cpi', 'nfp', 'vix', 'carry', 'credit', 'freshness', 'composite'
   - `gate_state TEXT NOT NULL DEFAULT 'normal'` -- 'normal', 'reduce', 'flatten'
   - `size_mult NUMERIC NOT NULL DEFAULT 1.0`
   - `triggered_at TIMESTAMPTZ`
   - `trigger_reason TEXT`
   - `cleared_at TIMESTAMPTZ`
   - `updated_at TIMESTAMPTZ NOT NULL DEFAULT now()`
   - `cooldown_ends_at TIMESTAMPTZ`
   - `CONSTRAINT chk_macro_gate_state CHECK (gate_state IN ('normal', 'reduce', 'flatten'))`
   - `CONSTRAINT chk_macro_gate_size CHECK (size_mult >= 0.0 AND size_mult <= 1.0)`

3. **cmc_macro_stress_history** -- Composite score time series:
   - `ts TIMESTAMPTZ NOT NULL PRIMARY KEY`
   - `composite_score NUMERIC NOT NULL` -- 0-100
   - `stress_tier TEXT NOT NULL` -- 'calm', 'elevated', 'stressed', 'crisis'
   - `vix_percentile NUMERIC`

   - `dexjpus_zscore_raw NUMERIC`
   - `ingested_at TIMESTAMPTZ NOT NULL DEFAULT now()`

  <name>Task 2: Event calendar seed script with FRED API auto-fetch</name>
  <files>src/ta_lab2/scripts/risk/seed_macro_events.py</files>
  <action>
Create `src/ta_lab2/scripts/risk/seed_macro_events.py` -- a CLI script that seeds dim_macro_events with FOMC, CPI, and NFP dates for 2026-2027 and optionally auto-fetches upcoming dates from the FRED API.

The script must:

1. **Hardcoded FOMC 2026-2027 dates** (from federalreserve.gov -- no machine-readable API):
   Use announcement time = 2pm ET on day 2 of meeting. Convert to UTC:
   - EST (Nov-Mar): 2pm ET = 19:00 UTC
   - EDT (Mar-Nov): 2pm ET = 18:00 UTC

   FOMC 2026: Jan 28 (19:00), Mar 18 (18:00), Apr 29 (18:00), Jun 17 (18:00), Jul 29 (18:00), Sep 16 (18:00), Oct 28 (18:00), Dec 9 (19:00)
   FOMC 2027: Jan 27 (19:00), Mar 17 (18:00), Apr 28 (18:00), Jun 9 (18:00), Jul 28 (18:00), Sep 15 (18:00), Oct 27 (18:00), Dec 8 (19:00)

   Store each with `event_type='fomc'`, `source='fed_gov'`, `data_period` = month of meeting.

2. **CPI release dates via FRED API** (`release_id=10`):
   Use `requests.get("https://api.stlouisfed.org/fred/release/dates", params={release_id: 10, ...})`.
   CPI releases at 8:30am ET (13:30 UTC winter / 12:30 UTC summer).
   Fetch for 2026 and 2027. Store with `event_type='cpi'`, `source='fred_api'`.
   `data_period` = the month BEFORE the release month (e.g., Feb 11 release = Jan data = '2026-01').

3. **NFP (Employment Situation) dates via FRED API** (`release_id=50`):
   Same pattern. NFP releases at 8:30am ET on first Friday of each month.
   Store with `event_type='nfp'`, `source='fred_api'`.
   `data_period` = the month BEFORE the release month.

4. **Upsert pattern**: Use temp table + `ON CONFLICT (event_type, event_ts) DO NOTHING` to be idempotent.

5. **CLI interface**: `argparse` with:
   - `--fomc-only` -- only seed FOMC dates (no API calls)
   - `--fetch-api` -- fetch CPI + NFP from FRED API (requires FRED_API_KEY env var)
   - `--dry-run` -- print events that would be inserted, do not write
   - `--year` -- specific year to seed (default: both 2026 and 2027)

6. **FRED API key**: Read from `os.environ.get("FRED_API_KEY")`. If missing and `--fetch-api` requested, exit with clear error message.

7. **Logging**: Use `logging.getLogger(__name__)` with INFO-level event counts.

8. **Engine**: Use `from ta_lab2.db import get_engine` pattern (same as other scripts in this directory).

Script should be runnable as `python -m ta_lab2.scripts.risk.seed_macro_events --fomc-only` (no external deps for FOMC) and `python -m ta_lab2.scripts.risk.seed_macro_events --fetch-api` (needs FRED_API_KEY).
  </action>
