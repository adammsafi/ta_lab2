1. Add optional `macro_gate_evaluator` parameter to `RiskEngine.__init__()`:

   ```python
   def __init__(
       self,
       engine: Engine,
       macro_gate_evaluator: Optional["MacroGateEvaluator"] = None,
   ) -> None:
       self._engine = engine
       self._macro_gate_evaluator = macro_gate_evaluator
   ```
   Add TYPE_CHECKING import: `from ta_lab2.risk.macro_gate_evaluator import MacroGateEvaluator`

2. Add `check_macro_gates()` method to RiskEngine:
   ```python
   def check_macro_gates(self) -> tuple[str, float]:
       """Read macro gate state. Returns (state, size_mult).

       If no MacroGateEvaluator is configured, returns ('normal', 1.0) (no-op).
       Delegates to MacroGateEvaluator.check_order_gates() for lightweight DB read.
       """
       if self._macro_gate_evaluator is None:
           return ("normal", 1.0)
       return self._macro_gate_evaluator.check_order_gates()
   ```

3. Insert Gate 1.7 into `check_order()` AFTER Gate 1.5 (tail risk) and BEFORE Gate 2 (circuit breaker).
   The insertion point is after the `if tail_state == "reduce" and order_side.lower() == "buy":` block
   and before the `# Gate 2: Circuit breaker` comment.

   ```python
   # Gate 1.7: Macro event / stress gates
   macro_state, macro_size_mult = self.check_macro_gates()
   if macro_state == "flatten":
       self._log_event(
           event_type="macro_stress_gate_triggered",
           trigger_source="macro_gate",
           reason="Order blocked by macro gate FLATTEN state",
           asset_id=asset_id,
           strategy_id=strategy_id,
           metadata={"macro_state": "flatten", "order_side": order_side},
       )
       return RiskCheckResult(
           allowed=False,
           blocked_reason="Macro gate: FLATTEN state active -- all new orders blocked",
       )
   if macro_state == "reduce" and order_side.lower() == "buy":
       # Apply macro gate size_mult to buy orders
       # Use worst-of with any prior tail risk adjustment
       order_qty = (order_qty * Decimal(str(macro_size_mult))).quantize(
           Decimal("0.00000001")
       )
       order_notional = order_qty * fill_price
       logger.info(
           "Macro gate REDUCE: buy order quantity scaled by %.2f to %s",
           macro_size_mult, order_qty,
       )
   ```

   IMPORTANT: The tail risk (Gate 1.5) already reduces order_qty before Gate 1.7 runs.
   So if tail_risk gives size_mult=0.5 and macro_gate gives size_mult=0.6, the order gets
   multiplied by 0.5 first (tail risk), then by 0.6 (macro gate), resulting in 0.3 effective.
   This is correct worst-of (tighten-only) semantics because both reductions stack.

4. Update the docstring for `check_order()` to mention Gate 1.7 in the gate chain:
   ```
   Gates are checked in priority order:
     1.  Kill switch
     1.5 Tail risk
     1.7 Macro event/stress gates (FOMC, VIX, carry, credit, freshness, composite)
     2.  Circuit breaker
     3.  Per-asset position cap
     4.  Portfolio utilization cap
     1.6 Margin/liquidation check
     5.  All pass
   ```

5. Update the class-level docstring to mention Gate 1.7 integration.

**Create `src/ta_lab2/scripts/risk/evaluate_macro_gates.py`:**

CLI script to run MacroGateEvaluator.evaluate() and print results. Follow the pattern of `kill_switch_cli.py`.

Features:
- `argparse` with:
  - `--dry-run`: evaluate but do not update dim_macro_gate_state
  - `--json`: output results as JSON (for pipeline integration)
  - No other flags needed -- evaluate() handles everything.

- Main logic:
  1. `from ta_lab2.db import get_engine` to get DB engine.
  2. Instantiate `MacroGateEvaluator(engine)`.
  3. Call `evaluator.evaluate()`.
  4. Print summary table:
     ```
     Gate             State     Size Mult  Reason
     ----             -----     ---------  ------
     fomc             normal    1.00
     cpi              normal    1.00
     nfp              normal    1.00
     vix              reduce    0.50       VIX=32.5 > 30.0
     carry            normal    1.00
     credit           normal    1.00
     freshness        normal    1.00
     composite        elevated  0.80       Score=34.2

     Overall: REDUCE (size_mult=0.50)
     Active gates: vix
     ```
  5. If `--json`: output `{"state": "reduce", "size_mult": 0.5, "active_gates": ["vix"], "details": "..."}`.

- Runnable as: `python -m ta_lab2.scripts.risk.evaluate_macro_gates`

**Create `src/ta_lab2/scripts/risk/macro_gate_cli.py`:**

CLI for override management. Follow the pattern of `override_cli.py`.

Subcommands via `argparse` with `add_subparsers()`:

1. `create` -- Create a gate override:
   - `--gate-id` (required): one of fomc, cpi, nfp, vix, carry, credit, freshness, composite
   - `--type` (required): 'disable_gate', 'force_normal', 'force_reduce'
   - `--reason` (required): free-text reason
   - `--operator` (default: 'cli'): who created the override
   - `--expires-hours` (default: 24): hours until auto-expiry
   - Calls `GateOverrideManager.create_override(...)`.
   - Prints: `Override created: {override_id}, expires at {expires_at}`

2. `list` -- List active overrides:
   - `--gate-id` (optional): filter by gate_id
   - `--all`: include expired/reverted overrides
   - Calls `GateOverrideManager.get_active_overrides(...)`.
   - Prints table of overrides.

3. `revert` -- Revert an override:
   - `--override-id` (required): UUID of the override
   - `--reason` (required): why reverting
   - `--operator` (default: 'cli')
   - Calls `GateOverrideManager.revert_override(...)`.
   - Prints success/failure.

4. `status` -- Show current gate states:
   - Reads dim_macro_gate_state directly (no evaluation).
   - Prints table of all 8 gates with state, size_mult, triggered_at, cooldown_ends_at.

- Runnable as: `python -m ta_lab2.scripts.risk.macro_gate_cli status`
