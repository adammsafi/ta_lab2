**Data classes:**

```python
@dataclass
class MacroGateResult:
    state: str           # 'normal', 'reduce', 'flatten'
    size_mult: float     # 0.0 - 1.0
    active_gates: list   # list of gate_id strings that are firing
    details: str         # human-readable summary
```

**MacroGateEvaluator class:**

```python
class MacroGateEvaluator:
    def __init__(self, engine: Engine) -> None:
        self._engine = engine
        self._override_manager = GateOverrideManager(engine)
```

Import GateOverrideManager at module level: `from ta_lab2.risk.macro_gate_overrides import GateOverrideManager`. The override manager is used by evaluate() to check for active overrides per gate before applying gate logic.

**Public methods:**

1. `evaluate() -> MacroGateResult` -- Main evaluation entry point. Called once per executor cycle.
   - Reads latest row from `fred.fred_macro_features` (ORDER BY date DESC LIMIT 1) to get: `vixcls`, `dexjpus_daily_zscore`, `hy_oas_30d_zscore`, `nfci_level`, `us_jp_rate_spread`.
   - Calls each individual gate method.
   - Applies override logic via `self._override_manager.check_override(gate_id)` for each gate.
   - Calls `self._override_manager.expire_stale_overrides()` at the start to clean up expired overrides.
   - Updates dim_macro_gate_state for each gate.
   - Computes and persists composite stress score.
   - Returns MacroGateResult with worst-of (min) semantics across all gates.

2. `check_order_gates() -> tuple[str, float]` -- Lightweight read from dim_macro_gate_state.
   - Called per-order by RiskEngine (no expensive re-evaluation).
   - Reads all rows from dim_macro_gate_state.
   - Returns (worst_state, min_size_mult) across all gates.
   - If any gate is 'flatten', return ('flatten', 0.0).
   - If any gate is 'reduce', return ('reduce', min(size_mult of reduce gates)).
   - Otherwise return ('normal', 1.0).

**Private gate methods (each returns tuple[str, float]):**

3. `_check_event_gate(event_type: str, size_mult: float, window_hours: float) -> tuple[str, float]`
   - Queries dim_macro_events for events within +/- window_hours of now().
   - If found, return ('reduce', size_mult).
   - FOMC: window_hours=24.0, size_mult=0.5
   - CPI: window_hours=24.0, size_mult=0.7
   - NFP: window_hours=12.0, size_mult=0.75
   - If event_type is 'fomc', also check for events where event_ts is in the window.
   - Return ('normal', 1.0) if no event in window.

4. `_check_vix_gate(vixcls: Optional[float]) -> tuple[str, float]`
   - If vixcls is None, return ('normal', 1.0) -- no data = no gate.
   - Read thresholds from dim_macro_gate_state metadata or use defaults:
     - `vix_reduce_threshold = 30.0` (GATE-03)
     - `vix_flatten_threshold = None` (disabled by default per CONTEXT.md open question)
     - `vix_reduce_mult = 0.5`
   - If vix_flatten_threshold is not None and vixcls >= vix_flatten_threshold: return ('flatten', 0.0)
   - If vixcls >= vix_reduce_threshold: return ('reduce', vix_reduce_mult)
   - CRITICAL: Do NOT use `vix_regime` from fred_macro_features (uses 15/25 thresholds, not 30/40).

5. `_check_carry_gate(dexjpus_zscore: Optional[float], rate_spread: Optional[float]) -> tuple[str, float]`
   - If dexjpus_zscore is None, return ('normal', 1.0).
   - SIGN CONVENTION: DEXJPUS = JPY per USD. JPY strengthening (carry unwind) = negative z-score.
     The carry gate should trigger when JPY is strengthening sharply (dexjpus_zscore < -2.0).
     Use `carry_signal = -dexjpus_zscore` so that carry unwind = positive carry_signal.
   - Thresholds (from GATE-04):
     - carry_signal > 2.0 -> ('reduce', 0.6)
     - carry_flatten_threshold: defaults to NULL (disabled), DB-configurable -- matching VIX FLATTEN treatment per CONTEXT.md open question. Only trigger ('flatten', 0.0) if carry_flatten_threshold is not None and carry_signal > carry_flatten_threshold.
   - Also verify rate_spread is positive (if provided, carry trade is active).
   - Add detailed code comments explaining the sign convention.
   - Store carry_flatten_threshold as module-level constant `CARRY_FLATTEN_THRESHOLD: Optional[float] = None` for future DB-config migration.

6. `_check_credit_gate(hy_oas_zscore: Optional[float]) -> tuple[str, float]`
   - If hy_oas_zscore is None, return ('normal', 1.0).
   - From GATE-06:
     - hy_oas_zscore > 1.5 -> ('reduce', 0.7)
     - hy_oas_zscore > 2.5 -> ('reduce', 0.4)
   - Credit stress does NOT trigger FLATTEN (only reduces).

7. `_check_freshness_gate() -> tuple[str, float]`
   - Per-series freshness check (NOT aggregate -- per CONTEXT.md).
   - Query `fred.fred_macro_features` for `MAX(date)` and compute business days stale using
     `pandas.tseries.holiday.USFederalHolidayCalendar` + `pd.bdate_range`.
   - Check canary columns: vixcls, dexjpus (mapped to their source series).
   - From GATE-05:
     - business_days_stale > 3 (approx 48 calendar hours): return ('reduce', 0.8) + log WARNING
     - business_days_stale > 6 (approx 96 calendar hours): return ('reduce', 0.5) + disable macro regime
   - Friday data is treated as fresh through the weekend (business-day calendar handles this automatically since Saturday/Sunday are not business days).
   - Log which specific series are stale.
   - Auto re-enable when fresh data arrives (per CONTEXT.md).

8. `_compute_composite_score(vixcls, hy_oas_zscore, carry_zscore, nfci_level) -> tuple[float, str]`
   - Returns (score_0_100, tier_string).
   - Component normalization (clamped 0-100):
     - VIX: (vixcls - 10.0) / 70.0 * 100.0 (range [10, 80])
     - HY OAS: (hy_oas_zscore + 2.0) / 6.0 * 100.0 (range [-2, +4])
     - Carry: (abs(carry_zscore) + 0.0) / 3.0 * 100.0 (range [0, 3] of absolute value)
     - NFCI: (nfci_level + 1.5) / 4.0 * 100.0 (range [-1.5, +2.5])
   - Weights: VIX=0.40, HY_OAS=0.25, carry=0.20, NFCI=0.15
   - Tier boundaries: 0-25 calm, 25-50 elevated, 50-75 stressed, 75+ crisis
   - Tier size_mults: calm=1.0, elevated=0.8, stressed=0.6, crisis=0.4
   - Persist to cmc_macro_stress_history via INSERT with all component values.

**State management methods:**

9. `_update_gate_state(gate_id, new_state, new_size_mult, reason, cooldown_hours=4.0) -> None`
   - UPSERT to dim_macro_gate_state using ON CONFLICT (gate_id) DO UPDATE.
   - Set cooldown_ends_at = now + cooldown_hours when entering reduce/flatten.
   - When transitioning to normal, only clear if now >= cooldown_ends_at (prevents oscillation).
   - On state TRANSITIONS (old != new): send Telegram alert via `_send_gate_transition_alert()`.
   - On state transitions: log to cmc_risk_events with appropriate event_type and trigger_source='macro_gate'.

10. `_send_gate_transition_alert(old_state, new_state, gate_id, reason) -> None`
    - Follow the `_try_telegram_alert()` pattern from paper_executor.py.
    - Lazy import `from ta_lab2.notifications.telegram import send_alert`.
    - severity='critical' for FLATTEN transitions, 'warning' for REDUCE, 'info' for return to normal.
    - Wrap in try/except -- alert failure must never crash the evaluator.

11. `_log_risk_event(event_type, gate_id, reason, metadata=None) -> None`
    - INSERT into cmc_risk_events matching existing column schema.
    - event_type: one of the Phase 71 types added in the migration.
    - trigger_source: 'macro_gate'.

**Anti-patterns to avoid:**
- Do NOT read `vix_regime` from fred_macro_features (uses wrong thresholds 15/25).
- Do NOT hardcode VIX > 40 as FLATTEN (must be configurable, default NULL/disabled).
- Do NOT hardcode carry FLATTEN threshold (must be configurable, default NULL/disabled).
- Do NOT use aggregate freshness (must be per-series).
- Do NOT embed event window logic in risk_engine.py (keep it here).
- Store all thresholds as module-level constants with clear docstrings for future DB-config migration.

**GateOverrideManager class:**

```python
class GateOverrideManager:
    def __init__(self, engine: Engine) -> None:
        self._engine = engine
```

**Methods:**

1. `create_override(gate_id, operator, reason, override_type, expires_hours=24.0) -> str`
   - INSERT into dim_macro_gate_overrides.
   - `override_type` must be one of: 'disable_gate', 'force_normal', 'force_reduce'.
   - `expires_at = now() + timedelta(hours=expires_hours)`.
   - Log to cmc_risk_events with event_type='macro_gate_override_created', trigger_source='macro_gate'.
   - Return the override_id as string.

2. `get_active_overrides(gate_id: Optional[str] = None) -> list[dict]`
   - SELECT from dim_macro_gate_overrides WHERE reverted_at IS NULL AND expires_at > now().
   - If gate_id provided, filter by gate_id.
   - Return list of dicts with all columns.

3. `revert_override(override_id: str, reason: str, operator: str) -> bool`
   - UPDATE dim_macro_gate_overrides SET reverted_at = now(), revert_reason = reason.
   - Log to cmc_risk_events with event_type='macro_gate_override_expired'.
   - Return True if row was updated, False if not found.

4. `check_override(gate_id: str) -> Optional[str]`
   - Called by MacroGateEvaluator during evaluate().
   - Returns the override_type if an active (non-reverted, non-expired) override exists for this gate_id.
   - Returns None if no active override.
   - If override_type is 'disable_gate', the gate is skipped (returns 'normal', 1.0).
   - If override_type is 'force_normal', the gate is forced to normal.
   - If override_type is 'force_reduce', the gate is forced to reduce (with size_mult from the gate's default).

5. `expire_stale_overrides() -> int`
   - Called periodically (e.g., by evaluate()).
   - SELECT overrides WHERE expires_at < now() AND reverted_at IS NULL.
   - For each: revert with reason='Auto-expired' and operator='system'.
   - Return count of expired overrides.

Follow existing patterns:
- Use `sqlalchemy.text()` for all queries (not ORM).
- Use `self._engine.begin()` for writes, `self._engine.connect()` for reads.
- Logger: `logging.getLogger(__name__)`.
