  <name>Task 3: Restructure ci.yml -- lint/format/mypy/version-check as separate jobs</name>
  <files>.github/workflows/ci.yml</files>
  <action>
    Replace the entire `lint:` job block (lines 43-62) with four separate jobs. Keep the existing `test:` job (lines 13-41) completely unchanged.

    The new jobs to add after the `test:` job:

    **Job 1: `lint` -- Ruff lint (BLOCKING)**
    ```yaml
      lint:
        runs-on: ubuntu-latest
        steps:
          - name: Checkout
            uses: actions/checkout@v4
          - name: Set up Python
            uses: actions/setup-python@v5
            with:
              python-version: "3.12"
          - name: Install ruff
            run: pip install "ruff>=0.9.0"
          - name: Ruff lint
            run: ruff check src/ --output-format=github
    ```

    CRITICAL: No `|| true` -- this is now a hard gate. The `--output-format=github` flag produces inline PR annotations on violating lines.

    **Job 2: `format` -- Ruff format check (BLOCKING)**
    ```yaml
      format:
        runs-on: ubuntu-latest
        steps:
          - name: Checkout
            uses: actions/checkout@v4
          - name: Set up Python
            uses: actions/setup-python@v5
            with:
              python-version: "3.12"
          - name: Install ruff
            run: pip install "ruff>=0.9.0"
          - name: Ruff format check
            run: ruff format --check src/
    ```

    **Job 3: `mypy` -- Type checking (NON-BLOCKING)**
    ```yaml
      mypy:
        runs-on: ubuntu-latest
        continue-on-error: true
        steps:
          - name: Checkout
            uses: actions/checkout@v4
          - name: Set up Python
            uses: actions/setup-python@v5
            with:
              python-version: "3.12"
          - name: Install dependencies
            run: |
              pip install "mypy>=1.14" "pandas-stubs>=2.2"
              pip install -e ".[dev]"
          - name: mypy (features + regimes)
            run: mypy src/ta_lab2/features/ src/ta_lab2/regimes/
    ```

    Note: `continue-on-error: true` makes this non-blocking. The 15 baseline errors will show as warnings but not fail the PR. The `[tool.mypy]` config in pyproject.toml provides `ignore_missing_imports` and `check_untyped_defs` automatically (mypy reads pyproject.toml by default).

    **Job 4: `version-check` -- Version consistency (BLOCKING)**
    ```yaml
      version-check:
        runs-on: ubuntu-latest
        steps:
          - name: Checkout
            uses: actions/checkout@v4
          - name: Check version consistency
            run: |
              PYPROJECT_VER=$(grep -oP '^version = "\K[^"]+' pyproject.toml)
              README_VER=$(head -1 README.md | grep -oP 'v\K[0-9]+\.[0-9]+\.[0-9]+')
              if [ "$PYPROJECT_VER" != "$README_VER" ]; then
                echo "::error::Version mismatch: pyproject.toml=$PYPROJECT_VER, README=$README_VER"
                exit 1
              fi
              echo "Version consistent: $PYPROJECT_VER"
    ```

    The full ci.yml should have these jobs: `test`, `lint`, `format`, `mypy`, `version-check`. All are independent (no `needs:` dependencies between them) -- they all run in parallel on every PR.
