2. **Signals Generated** -- Query signals for today. For each active config in dim_executor_config, query the signal table. Use SIGNAL_TABLE_MAP pattern from signal_reader.py: `{'ema_crossover': 'cmc_signals_ema_crossover', 'rsi_mean_reversion': 'cmc_signals_rsi', 'atr_breakout': 'cmc_signals_atr_breakout'}`. For each table, query `WHERE generated_at::date = :log_date`. Wrap each in try/except (table may not exist). Format as table with strategy, asset, direction, signal_ts.

3. **Orders & Fills** -- Query `cmc_fills f JOIN cmc_orders o ON f.order_id = o.order_id WHERE f.filled_at::date = :log_date`. Show strategy_id, asset_id, side, fill_qty, fill_price. Compute slippage_bps inline: `ABS(f.fill_price::float - pb.open::float) / NULLIF(pb.open::float, 0) * 10000` by joining to `cmc_price_bars_multi_tf pb ON pb.id = o.asset_id AND pb.tf = '1D' AND pb.ts::date = f.filled_at::date`.

4. **Current Positions** -- Query `cmc_positions WHERE quantity != 0`. Show asset_id, strategy_id, quantity, avg_entry_price.

5. **P&L Summary** -- Daily: sum of signed fill values for today. Cumulative: sum since validation_start. Use `cmc_fills` with signed cash flow: `CASE WHEN o.side = 'sell' THEN f.fill_price * f.fill_qty ELSE -f.fill_price * f.fill_qty END`.

6. **Drift Metrics** -- Query `cmc_drift_metrics WHERE metric_date = :log_date`. Show config_id, tracking_error_5d, tracking_error_30d, threshold_breach, paper_cumulative_pnl.

7. **Risk State** -- Query `dim_risk_state WHERE state_id = 1`. Show trading_state, halted_at, halted_reason, drift_paused, drift_paused_at.

8. **Anomalies** section -- placeholder text: "See audit report for full gap detection."

9. **Notes** section -- human-fillable with `[ ] Reviewed by: ___________ Date: ___________`

**`audit_checker.py`** -- `AuditChecker` class.

Constructor: `__init__(self, engine: Engine)`

**`AuditFinding` dataclass:**
- `check_name: str`
- `status: str` ("PASS" or "FAIL")
- `count: int` (number of anomalies)
- `details: list[dict]` (rows of anomaly data)

**Public method:** `run_audit(self, start_date: date, end_date: date) -> tuple[list[AuditFinding], AuditSummary]`

Returns list of findings + AuditSummary (imported from gate_framework.py).

**6 checks (all via `text()`):**

1. **Missing executor run days** -- `generate_series(:start_date::date, :end_date::date, '1 day'::interval)` LEFT JOIN `cmc_executor_run_log` (status IN success, no_signals). Missing = no join match. FAIL if count > 0.

2. **Error runs** -- `cmc_executor_run_log WHERE status IN ('failed', 'stale_signal') AND started_at::date BETWEEN :start_date AND :end_date`. FAIL if count > 0.

3. **Orphaned orders** -- `cmc_orders WHERE status IN ('created', 'submitted') AND created_at < now() - interval '2 days' AND NOT EXISTS (SELECT 1 FROM cmc_fills WHERE order_id = ...)`. FAIL if count > 0.

4. **Position/fill consistency** -- `cmc_positions WHERE quantity != 0 AND NOT EXISTS (SELECT 1 FROM cmc_fills f JOIN cmc_orders o ON f.order_id = o.order_id WHERE o.asset_id = p.asset_id)`. FAIL if count > 0.

5. **Stale price data** -- `cmc_price_bars_multi_tf WHERE tf = '1D' GROUP BY id HAVING MAX(ts) < now() - interval '28 hours'`. FAIL if count > 0.

6. **Drift metric gaps** -- executor ran successfully but no drift metrics for that day. Use CTE with exec_days from cmc_executor_run_log, LEFT JOIN cmc_drift_metrics. FAIL if count > 0.

AuditSummary computation:
```python
n_anomalies = sum(f.count for f in findings if f.status == "FAIL")
AuditSummary(n_anomalies=n_anomalies, n_signed_off=0, all_signed_off=(n_anomalies == 0))
```

**`generate_report(self, findings: list[AuditFinding], summary: AuditSummary, start_date: date, end_date: date, output_dir: str) -> str`**

Writes Markdown audit report with:
- Summary table (check name, status, count)
- Details per failed check (rows formatted as tables)
- Sign-off section: `[ ] All anomalies reviewed and explained\nOperator: ___________ Date: ___________`
