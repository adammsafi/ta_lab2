Create the `src/ta_lab2/validation/` package.

**`gate_framework.py`** must contain:

1. **`GateStatus(str, Enum)`** -- PASS, CONDITIONAL, FAIL values.

2. **`GateResult` dataclass** (NOT frozen -- may be updated post-construction):
   - `gate_id: str` (e.g., "VAL-02", "BT-01")
   - `gate_name: str` (human-readable)
   - `threshold: str` (what criterion requires)
   - `measured_value: str` (what was actually measured)
   - `status: GateStatus`
   - `evidence_sources: list[str]` (DB tables / file paths)
   - `mitigation: str | None = None` (for CONDITIONAL: what mitigation exists)
   - `notes: str = ""`

3. **`score_gate(measured: float | None, threshold: float, direction: str) -> GateStatus`**
   - direction "above" (higher is better): PASS if measured >= threshold
   - direction "below" (lower is better): PASS if measured <= threshold
   - None measured -> FAIL

4. **Query helpers** (all take `engine` + `start_date` + `end_date` params):
   - `query_distinct_executor_run_days(engine, start, end) -> list[date]` -- distinct dates from cmc_executor_run_log WHERE status IN ('success', 'no_signals')
   - `query_max_tracking_error_5d(engine, start, end) -> float | None` -- MAX(tracking_error_5d) from cmc_drift_metrics
   - `query_mean_slippage_bps(engine, start, end) -> tuple[float, int]` -- mean abs slippage in bps from cmc_fills JOIN cmc_price_bars_multi_tf (fill_price vs bar open). Returns (mean_bps, n_fills). For paper trading fills use bar open price as reference. Use `ABS(f.fill_price::float - pb.open::float) / NULLIF(pb.open::float, 0) * 10000`. Returns (0.0, 0) if no fills.
   - `query_kill_switch_events(engine, start, end) -> list[dict]` -- from cmc_risk_events WHERE event_type LIKE 'kill_switch%', excluding rows WHERE reason LIKE '%V1 EXERCISE%' for real event counting
   - `query_kill_switch_exercise_events(engine, start, end) -> list[dict]` -- from cmc_risk_events WHERE reason LIKE '%V1 EXERCISE%' (exercise-specific)
   - `run_full_audit(engine, start, end) -> AuditSummary` -- placeholder that returns a simple dataclass (will be implemented fully in Plan 02). For now: `AuditSummary(n_anomalies=0, n_signed_off=0, all_signed_off=True)`.

5. **`AuditSummary` dataclass** -- `n_anomalies: int`, `n_signed_off: int`, `all_signed_off: bool`

6. **`build_gate_scorecard(engine, start_date: date, end_date: date) -> list[GateResult]`**
   Returns list of 7 GateResult objects:
   - BT-01 (Backtest Sharpe >= 1.0) -- hardcoded PASS with evidence from reports/bakeoff/BAKEOFF_SCORECARD.md. measured_value = "ema_trend(17,77): 1.401 | ema_trend(21,50): 1.397"
   - BT-02 (Backtest MaxDD <= 15%) -- hardcoded CONDITIONAL with mitigation string: "Structural: long-only BTC strategies face 70-75% bear-market drawdowns. Mitigation: 10% position fraction + 15% portfolio circuit breaker."
   - VAL-01 through VAL-05 -- use query helpers and scoring logic per research Pattern 1

Use `from sqlalchemy import text` and `from sqlalchemy.pool import NullPool` patterns. Use `from ta_lab2.db.config import resolve_db_url` for engine creation. All queries via `text()` with named bind params (`:start_date`, `:end_date`).

Create the `src/ta_lab2/scripts/validation/` scripts package.

**`run_preflight_check.py`** -- CLI that verifies 12+ conditions before starting the 14-day validation clock.

Structure:
```python
_PROJECT_ROOT = Path(__file__).resolve().parents[4]
```

Uses `_get_engine()` pattern from generate_bakeoff_scorecard.py (NullPool).

Define a list of `PreflightCheck` namedtuples:
```python
PreflightCheck = namedtuple("PreflightCheck", ["name", "query", "validator"])
```

Where `validator` is a callable that takes the query result and returns (passed: bool, detail: str).

**Checks to implement (all SQL via text()):**
1. DB connectivity -- `SELECT 1`
2. dim_executor_config has active rows -- `SELECT COUNT(*) FROM dim_executor_config WHERE is_active=TRUE` (expect >= 1)
3. Both EMA configs active -- `SELECT COUNT(*) FROM dim_executor_config WHERE is_active=TRUE AND signal_type='ema_crossover'` (expect >= 2)
4. dim_risk_state row exists -- `SELECT COUNT(*) FROM dim_risk_state WHERE state_id=1` (expect = 1)
5. trading_state is active -- `SELECT trading_state FROM dim_risk_state WHERE state_id=1` (expect 'active')
6. drift_paused is false -- `SELECT drift_paused FROM dim_risk_state WHERE state_id=1` (expect False)
7. dim_risk_limits row exists -- `SELECT COUNT(*) FROM dim_risk_limits WHERE asset_id IS NULL` (expect >= 1)
8. daily_loss_pct_threshold set -- `SELECT daily_loss_pct_threshold FROM dim_risk_limits WHERE asset_id IS NULL` (expect NOT NULL and > 0)
9. BTC price bars current (< 30 hours) -- `SELECT MAX(ts) FROM cmc_price_bars_multi_tf WHERE id=1 AND tf='1D'` (compare with now - 30h)
10. cmc_features current (< 30 hours) -- `SELECT MAX(ts) FROM cmc_features WHERE id=1 AND tf='1D'`
11. EMA data current (< 30 hours) -- `SELECT MAX(ts) FROM cmc_ema_multi_tf_u WHERE id=1 AND tf='1D'`
12. No orphaned orders -- `SELECT COUNT(*) FROM cmc_orders WHERE status IN ('created','submitted')` (expect 0)
13. Executor run log accessible -- `SELECT 1 FROM cmc_executor_run_log LIMIT 1` (no error)
14. cmc_drift_metrics accessible -- `SELECT 1 FROM cmc_drift_metrics LIMIT 0` (no error)
15. Slippage mode check -- `SELECT config_id, slippage_mode FROM dim_executor_config WHERE is_active=TRUE` -- WARN (not FAIL) if any config has slippage_mode='zero'. Print warning: "VAL-03 slippage measurement requires non-zero slippage mode"

For staleness checks (9-11): parse the MAX(ts) result as pd.Timestamp with UTC, compare with `datetime.now(timezone.utc) - timedelta(hours=30)`.

Output format:
```
============================================================
V1 VALIDATION PRE-FLIGHT CHECKLIST
============================================================
  [PASS] DB connectivity
  [PASS] Both EMA configs active (2 found)
  ...
  [WARN] Slippage mode: config_id=1 uses 'zero' mode
============================================================
Result: N PASS, M WARN, K FAIL
============================================================
```

Exit code: 0 if no FAILs, 1 if any FAIL. WARNs are informational, not blocking.

CLI: `python -m ta_lab2.scripts.validation.run_preflight_check [--db-url URL]`

argparse with optional `--db-url`. Uses `resolve_db_url()` as default.

Each check wrapped in try/except -- if a check raises (table missing, connection error), that check = FAIL with error message.

ALL file writes use `encoding="utf-8"`.
