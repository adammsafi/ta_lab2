Create `run_kill_switch_exercise.py` -- an interactive protocol script implementing the 8-step kill switch exercise.

**Project root:** `_PROJECT_ROOT = Path(__file__).resolve().parents[4]`

**Engine:** `_get_engine()` with NullPool, from `ta_lab2.db.config.resolve_db_url`.

**Imports from existing code:**
```python
from ta_lab2.risk.kill_switch import (
    activate_kill_switch, re_enable_trading, get_kill_switch_status
)
```

**`ExerciseStep` dataclass:**
- `step_num: int`
- `name: str`
- `timestamp: str` (ISO format)
- `result: str` (description of what happened)
- `passed: bool`

**`KillSwitchExercise` class:**

Constructor: `__init__(self, engine: Engine, operator: str)`

Stores `self._steps: list[ExerciseStep]` for evidence collection.

**`run_exercise(self) -> str`** -- returns path to evidence document. Implements:

**STEP 1: Pre-exercise snapshot**
- Query dim_risk_state: capture trading_state, halted_at, drift_paused
- Query cmc_orders: count orders in 'created' or 'submitted' status
- Query cmc_risk_events: capture MAX(event_ts)
- Record T0 = datetime.now(timezone.utc)
- Log step: "Pre-state: trading_state={}, open_orders={}, last_event={}"
- Verify trading_state == 'active'. If not, print WARNING and ask operator to confirm continue.

**STEP 2: Manual kill switch activation**
- Call `activate_kill_switch(self._engine, reason="V1 EXERCISE: manual kill switch test", trigger_source="manual")`
- Record T1 = datetime.now(timezone.utc)
- Compute manual_latency = (T1 - T0).total_seconds()
- Log step: "Manual trigger fired, latency={manual_latency:.3f}s"

**STEP 3: Validate manual trigger effects**
- Query dim_risk_state: verify trading_state='halted', halted_reason contains 'V1 EXERCISE'
- Query cmc_orders: verify no orders in 'created'/'submitted' status
- Query cmc_risk_events: verify new row with event_type='kill_switch_activated' and reason containing 'V1 EXERCISE'
- Log each check as sub-step with PASS/FAIL

**STEP 4: Manual re-enable**
- Print to stdout: `\n>>> Kill switch activated. Verify effects above.`
- Print: `>>> Press ENTER to re-enable trading...`
- Call `input()` to wait for operator
- Call `re_enable_trading(self._engine, reason="V1 EXERCISE: manual test complete", operator=self._operator)`
- Verify trading_state returns to 'active'
- Log step

**STEP 5: Engineer automatic trigger**
- Query current daily_loss_pct_threshold from dim_risk_limits (WHERE asset_id IS NULL AND strategy_id IS NULL)
- Store original_threshold
- Update to 0.001 (0.1%) via `UPDATE dim_risk_limits SET daily_loss_pct_threshold = 0.001 WHERE asset_id IS NULL AND strategy_id IS NULL`
- Print: `>>> Threshold lowered to 0.1%.`
- Print: `>>> Polling dim_risk_state.trading_state every 5s for up to 5 minutes...`
- Print: `>>> (You may need to trigger an executor run in another terminal)`
- **Polling loop:** Poll `SELECT trading_state FROM dim_risk_state WHERE state_id = 1` every 5 seconds for up to 60 iterations (5 minutes total). On each poll, print a dot (`.`) to show liveness. Break as soon as `trading_state = 'halted'`.
- If polling loop finds `trading_state = 'halted'`: print `>>> Auto-trigger confirmed! trading_state='halted'`, record T5 = datetime.now(timezone.utc), log step as PASS.
- If polling loop times out after 5 minutes without halt: print `>>> TIMEOUT: trading_state did not change to 'halted' within 5 minutes.`, print `>>> This may mean the executor has not run, or the threshold was not low enough.`, log step as FAIL with result "Auto-trigger timeout: trading_state remained 'active' after 5 minutes".

**STEP 6: Validate automatic trigger effects**
- If Step 5 FAILED (timeout): skip validation, log step as FAIL with result "Skipped: auto-trigger did not fire".
- If Step 5 PASSED: Query dim_risk_state: verify trading_state='halted'. Query cmc_risk_events: check for event with trigger_source containing 'daily_loss_stop' (or event_type containing appropriate value). Log each check.

**STEP 7: Restore thresholds + re-enable**
- `UPDATE dim_risk_limits SET daily_loss_pct_threshold = :original WHERE asset_id IS NULL AND strategy_id IS NULL`
- Verify restored via SELECT
- Print: `>>> Thresholds restored. Press ENTER to re-enable trading...`
- Call `input()`
- Call `re_enable_trading(self._engine, reason="V1 EXERCISE: auto-trigger test complete", operator=self._operator)`
- Verify trading_state='active'
- Log step
- **IMPORTANT:** Threshold restoration MUST happen even if Step 5/6 failed. Wrap Steps 5-6 in try/finally to guarantee restoration.

**STEP 8: Produce evidence document**
- Call `_generate_evidence_report()` which writes Markdown to `reports/validation/kill_switch_exercise/ks_exercise_{date}.md`
- Return path

**`_generate_evidence_report(self) -> str`:**

Markdown structure:
```markdown
# Kill Switch Exercise: {date}

## Summary
- Exercise type: Manual + Automatic (daily loss stop)
- Date: {date}
- Operator: {operator}
- Outcome: PASS / FAIL

## Manual Trigger Evidence
| Step | Timestamp | Result |
...

**Manual trigger latency: {X}ms**
**Target: < 5 seconds | Actual: {X}ms | Status: PASS/FAIL**

## Automatic Trigger Evidence
| Step | Timestamp | Result |
...

## Threshold Restoration Evidence
- Original threshold: {X}%
- Test threshold: 0.1%
- Restored at: {ts}
- Confirmed via: SELECT

## VAL-04 Gate Assessment
- Manual trigger: PASS/FAIL
- Automatic trigger: PASS/FAIL
- Recovery: PASS/FAIL
- Overall VAL-04: PASS/FAIL
```

All file writes use `encoding="utf-8"`.

**CLI argparse:**
- `--operator` (required): operator name for re_enable_trading
- `--db-url` (optional)
- `--output-dir` (optional, default `reports/validation/kill_switch_exercise`)
- `--skip-auto` (optional flag): skip the automatic trigger test (Steps 5-6) if operator only wants manual test
- `--poll-interval` (optional, default 5): seconds between polls during auto-trigger wait
- `--poll-timeout` (optional, default 300): max seconds to wait for auto-trigger

Invocation: `python -m ta_lab2.scripts.validation.run_kill_switch_exercise --operator asafi`

**Key constraints:**
- Use EXISTING event types only (kill_switch_activated, kill_switch_disabled). Do NOT create new event types.
- All exercise-related events use "V1 EXERCISE:" prefix in reason string for filtering.
- Re-enable is ALWAYS manual (operator + reason). Never implement auto-resume for kill switch.
- The script is interactive and prints prompts to stdout. It is NOT designed for automated/background execution.
- Threshold restoration is guaranteed via try/finally even on failure or timeout.
