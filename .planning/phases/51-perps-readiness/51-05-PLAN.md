    **_check_margin_gate(self, asset_id: int, strategy_id: int, order_side: str, limits: RiskLimits) -> Optional[str]:**
    - If order_side == 'sell': return None immediately (sells/closes always allowed -- reducing exposure is safe)
    - Query cmc_perp_positions for the current position matching asset+strategy
    - If no perp position exists: return None (no margin to check; spot-only)
    - Load margin tiers from cmc_margin_config for the position's venue+symbol
    - Compute margin utilization via compute_margin_utilization()

    **CRITICAL: Check thresholds from most severe to least severe to avoid dead code:**

    1. **Critical check FIRST (<=1.1x):**
       If margin_utilization <= limits.liquidation_kill_threshold (default 1.1x):
       - Log event type='liquidation_critical', trigger_source='margin_monitor'
       - Return "critical" (blocks order)

    2. **Warning check SECOND (<=1.5x):**
       If margin_utilization <= limits.margin_alert_threshold (default 1.5x):
       - Log event type='liquidation_warning', trigger_source='margin_monitor'
       - Return "warning" (log only -- does NOT block the order; warning is informational)

    3. **Proactive buffer check THIRD (<=2.0x):**
       If margin_utilization <= 2.0 (i.e., allocated margin is less than 2x maintenance margin):
       - Log event type='margin_alert', trigger_source='margin_monitor' with reason "margin below 2x buffer"
       - Return "buffer" (blocks order -- must maintain >= 2x maintenance margin to open new positions)

    4. **Safe (>2.0x):** return None

    In check_order() method, add Gate 1.6 call after the last existing gate before circuit breaker:
    ```python
    # Gate 1.6: Margin/liquidation check (perps only)
    margin_result = self._check_margin_gate(
        asset_id=asset_id, strategy_id=strategy_id, order_side=order_side,
        limits=limits,
    )
    if margin_result in ("critical", "buffer"):
        reason = (
            f"Liquidation critical: margin utilization at or below "
            f"{limits.liquidation_kill_threshold}x maintenance margin"
            if margin_result == "critical"
            else "Margin buffer insufficient: must maintain >= 2x maintenance margin to open new positions"
        )
        return RiskCheckResult(
            allowed=False,
            blocked_reason=reason,
        )
    # Note: "warning" result is logged but does NOT block -- order proceeds
    ```
