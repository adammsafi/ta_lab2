    **funding_adjuster.py** -- post-simulation P&L adjustment for perp backtests. Do NOT modify vectorbt callbacks or vbt_runner.py.

    1. `FundingAdjustedResult` dataclass:
       - equity_adjusted: pd.Series (equity curve with funding applied)
       - total_funding_paid: float (cumulative; negative = received)
       - total_return_adjusted: float
       - sharpe_adjusted: float
       - funding_payments_series: pd.Series (per-bar funding cash flows)

    2. `compute_funding_payments(position_timeline: pd.Series, funding_rates: pd.Series, is_short: bool = False) -> pd.Series`:
       - position_timeline: abs notional value at each bar (index = DatetimeIndex)
       - funding_rates: funding rate at each settlement (index = DatetimeIndex)
       - Aligns funding rates to position timeline via reindex+ffill
       - Payment = position_value * funding_rate
       - Sign convention: positive rate -> longs PAY (negative cash flow for long), shorts RECEIVE (positive cash flow for short)
       - If is_short: flip sign (shorts receive positive rate, pay negative rate)
       - Returns pd.Series of per-bar funding payments (negative = outflow)

    3. `load_funding_rates_for_backtest(engine, venue: str, symbol: str, start: datetime, end: datetime, mode: str = 'daily') -> pd.Series`:
       - mode='daily': SELECT from cmc_funding_rates WHERE tf='1d' (pre-rolled up daily sum)
       - mode='per_settlement': SELECT from cmc_funding_rates WHERE tf IN ('1h','4h','8h') (native granularity)
       - Returns pd.Series with DatetimeIndex and funding_rate values
       - If no data: log WARNING and return empty Series (zero funding)

    4. `FundingAdjuster` class:
       - __init__(engine, venue, symbol, mode='daily')
       - adjust(pf: vbt.Portfolio, is_short: bool = False) -> FundingAdjustedResult
         - Extract equity from pf.value()
         - Extract position_timeline from pf.value() (approximation: use equity as position value)
         - Load funding rates from DB via load_funding_rates_for_backtest
         - Compute funding payments
         - Adjust equity: equity_adj = equity - cumulative_funding (longs subtract positive funding paid)
         - Compute adjusted returns, Sharpe (annualized sqrt(365))
         - Return FundingAdjustedResult
       - CRITICAL: Strip timezone from funding rate index before joining with vbt equity (vbt outputs tz-naive index per MEMORY.md pitfall)

    5. `get_funding_rate_with_fallback(engine, venue, symbol, ts, tf='8h') -> Optional[float]`:
       - Exact match first, then cross-venue average within +/- 30 min window
       - This is the public API for other modules needing a single funding rate lookup

    - Test sign convention: long position + positive rate = negative payment (outflow)
    - Test sign convention: short position + positive rate = positive payment (inflow)
    - Test alignment: funding at different frequency than position timeline
    - Test zero funding rate: payments should all be 0
    - Test FundingAdjustedResult fields computed correctly from synthetic equity + payments
    - Mock engine for load_funding_rates_for_backtest tests (or test the pure functions only)

    **margin_monitor.py** -- margin model for perp position tracking.

    1. `MarginTier` dataclass:
       - notional_floor: Decimal (min position notional for this tier)
       - notional_cap: Decimal (max; Decimal('inf') for last tier)
       - initial_margin_rate: Decimal (e.g., 0.008 = 0.8%)
       - maintenance_margin_rate: Decimal (e.g., 0.004 = 0.4%)
       - max_leverage: int (e.g., 125)

    2. `MarginState` dataclass:
       - venue: str
       - symbol: str
       - position_value: Decimal (mark_price * abs(quantity))
       - leverage: Decimal (1x to 10x for V1)
       - margin_mode: str ('isolated' or 'cross')
       - initial_margin: Decimal (computed)
       - maintenance_margin: Decimal (computed)
       - margin_utilization: Decimal (current_margin / maintenance_margin ratio)
       - allocated_margin: Decimal (collateral held)
       - liquidation_price: Optional[Decimal] (estimated)
       - is_liquidation_warning: bool (margin_utilization <= 1.5)
       - is_liquidation_critical: bool (margin_utilization <= 1.1)

    3. `compute_margin_utilization(position_value: Decimal, allocated_margin: Decimal, leverage: Decimal, tiers: List[MarginTier], margin_mode: str = 'isolated') -> MarginState`:
       - Find applicable tier based on position_value (highest tier where notional_floor <= position_value)
       - initial_margin = position_value * im_rate
       - maintenance_margin = position_value * mm_rate
       - margin_utilization = allocated_margin / maintenance_margin (higher = safer; <1 = liquidation)
       - is_liquidation_warning = margin_utilization <= Decimal("1.5")
       - is_liquidation_critical = margin_utilization <= Decimal("1.1")
       - Estimate liquidation_price for isolated mode: entry_price * (1 - 1/leverage + mm_rate) for longs, entry_price * (1 + 1/leverage - mm_rate) for shorts. Pass side as optional param.
       - Return MarginState with all fields populated
       - Fallback: if no tiers, use conservative defaults (IM=10%, MM=5%)

    4. `load_margin_tiers(engine, venue: str, symbol: str) -> List[MarginTier]`:
       - SELECT from cmc_margin_config WHERE venue=:venue AND symbol=:sym ORDER BY notional_floor ASC
       - Convert to List[MarginTier] using Decimal(str(val)) pattern
       - Return empty list if no rows (caller uses defaults)

    5. `compute_cross_margin_utilization(positions: List[MarginState], total_wallet_balance: Decimal) -> Decimal`:
       - Cross margin: sum of all maintenance margins vs total wallet balance
       - utilization = total_wallet_balance / sum(mm for each position)
       - Returns the utilization ratio

    **tests/test_margin_monitor.py:**

    - Test warning threshold: margin_utilization = 1.5 -> warning = True
    - Test critical threshold: margin_utilization = 1.1 -> critical = True
    - Test safe margin: margin_utilization = 3.0 -> warning = False, critical = False
    - Test cross margin utilization with multiple positions
    - Test no tiers: fallback to conservative defaults (IM 10%, MM 5%)
    - Test liquidation price estimation for long and short positions
    - All tests use Decimal for precision
