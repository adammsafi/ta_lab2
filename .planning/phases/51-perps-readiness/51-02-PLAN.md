    Create `src/ta_lab2/scripts/perps/` package.

    **funding_fetchers.py** -- contains:

    1. `FundingRateRow` dataclass: venue (str), symbol (str), ts (datetime), tf (str), funding_rate (float), mark_price (Optional[float]=None), raw_tf (str='')

    2. `fetch_binance_funding(symbol='BTCUSDT', start_ms=None, end_ms=None, limit=1000) -> List[FundingRateRow]`:
       - GET https://fapi.binance.com/fapi/v1/fundingRate
       - 8h settlement. No auth. Limit 1000.
       - Parse: fundingTime (ms), fundingRate (float), markPrice (optional float)
       - symbol normalization: strip 'USDT' suffix for storage

    3. `fetch_hyperliquid_funding(coin='BTC', start_ms=0, end_ms=None) -> List[FundingRateRow]`:
       - POST https://api.hyperliquid.xyz/info with {"type": "fundingHistory", "coin": coin, "startTime": start_ms}
       - 1h settlement. No auth.
       - Parse: time (ms), fundingRate (string->float)

    4. `fetch_bybit_funding(symbol='BTCUSDT', start_ms=None, end_ms=None) -> List[FundingRateRow]`:
       - GET https://api.bybit.com/v5/market/funding/history
       - CRITICAL: If start_ms provided, end_ms MUST also be provided (Bybit constraint).
       - 8h settlement for BTC/ETH. Limit 200.
       - Parse: fundingRateTimestamp (string->int ms), fundingRate (string->float)

    5. `fetch_dydx_funding(market='BTC-USD', before_or_at=None, limit=100) -> List[FundingRateRow]`:
       - GET https://indexer.dydx.trade/v4/historicalFunding/{market}
       - 1h settlement. Cursor: effectiveBeforeOrAt (ISO datetime string).
       - Parse: effectiveAt (ISO string), rate (string->float)
       - Note: v4 only (Oct 2023+). Do NOT use v3 endpoint.

    6. `fetch_aevo_funding(instrument='BTC-PERP', start_ns=0, end_ns=None, limit=50) -> List[FundingRateRow]`:
       - GET https://api.aevo.xyz/funding-history
       - CRITICAL: Timestamps are NANOSECONDS (divide by 1e9, NOT 1e3).
       - 1h settlement. Limit 50.
       - Parse: response array items [instrument, timestamp_ns, rate, mark_price]

    7. `fetch_aster_funding(symbol='BTCUSDT', start_ms=None, end_ms=None, limit=1000) -> List[FundingRateRow]`:
       - GET https://fapi.asterdex.com/fapi/v1/fundingRate (mirrors Binance API exactly)
       - 8h settlement. Parse identical to Binance.
       - venue='aster' in FundingRateRow

    8. `fetch_lighter_funding(...)`: Stub that logs WARNING "Lighter funding rate API not confirmed; requires lighter-python SDK" and returns empty list. Include TODO comment referencing the open question from research.

    All fetchers:
    - Use `requests.get/post` with timeout=30
    - Call `resp.raise_for_status()`
    - Return `List[FundingRateRow]`
    - Handle empty responses gracefully (return [])
    - Log errors at WARNING level, do not crash on individual venue failure

  <name>Task 2: Refresh script with watermark pagination, daily rollup, and CLI</name>
  <files>

  <action>
    Create the main refresh CLI script that orchestrates all venue fetchers.

    **refresh_funding_rates.py** -- contains:

    1. `upsert_funding_rates(engine, rows: List[FundingRateRow]) -> int`:
       - Convert rows to DataFrame
       - Use temp table + INSERT...SELECT...ON CONFLICT (venue, symbol, ts, tf) DO NOTHING
       - Follow sync_utils.py pattern: DataFrame to_sql into temp table, then raw SQL INSERT...SELECT
       - Return count of rows inserted

    2. `get_watermark(engine, venue: str, symbol: str, tf: str) -> Optional[datetime]`:
       - SELECT MAX(ts) FROM cmc_funding_rates WHERE venue=:venue AND symbol=:sym AND tf=:tf
       - Return None if no rows exist (triggers full history backfill)

    3. `ingest_venue_full(engine, venue: str, symbols: List[str], dry_run: bool) -> int`:
       - For each symbol, get watermark, then paginate through history
       - **--dry-run behavior**: When dry_run=True, skip the DB watermark query entirely (use epoch 0 / None as start, equivalent to full backfill start). Log what would be fetched (venue, symbol, estimated start time) but do NOT call any fetcher functions or write to DB. This ensures --dry-run works without a DB connection for the watermark query.
       - Binance/Aster: paginate forward from watermark using startTime/endTime in batch_size*INTERVAL_MS windows
       - Hyperliquid: paginate forward from watermark using startTime
       - Bybit: paginate backward from now using endTime, stop at watermark. ALWAYS provide both startTime and endTime.
       - dYdX: paginate backward using effectiveBeforeOrAt cursor, stop at watermark
       - Aevo: paginate using offset parameter (start_ns from watermark), limit 50
       - Lighter: call stub (returns []), log skip
       - Sleep 0.1s between API calls to respect rate limits
       - Call upsert_funding_rates per batch
       - Log progress: "venue={venue} symbol={sym} fetched={n} inserted={m}"

    4. `compute_daily_rollup(engine, venue: str, symbol: str) -> int`:
       - Read all sub-day rates (tf != '1d') for venue/symbol
       - Resample to daily sum (UTC day boundary)
       - Upsert as tf='1d', raw_tf='rollup'
       - Return count inserted

    5. `get_funding_rate_with_fallback(engine, venue, symbol, ts, tf='8h') -> Optional[float]`:
       - Try exact match first
       - Fall back to cross-venue average within +/- 30 minutes window
       - Return None if no data available

    6. CLI via argparse:
       - `python -m ta_lab2.scripts.perps.refresh_funding_rates --all` : all venues, BTC+ETH
       - `--venue binance` : single venue
       - `--symbol BTC` : single symbol
       - `--dry-run` : show what would be fetched, no DB writes, no DB watermark query (uses epoch 0 as start)
       - `--rollup` : compute daily rollup after ingest
       - `--no-rollup` : skip rollup (default is to compute rollup)

    Use NullPool for engine (project pattern for scripts). Use logging module consistently. Do NOT wire into run_daily_refresh.py (standalone only per research decision).
  </action>
