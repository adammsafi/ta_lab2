  First, create the reports/data-economics/ directory if it does not exist.

  Run the following SQL queries against the local PostgreSQL database to gather real measurements. Use the project's DB connection (TARGET_DB_URL from db_config.env or environment). Execute via Python one-liner or a temporary script:

  1. Total database size:
     SELECT pg_size_pretty(pg_database_size(current_database()));

  2. Per-table breakdown (top 30 tables by size):
     SELECT relname AS table_name,
            pg_size_pretty(pg_total_relation_size(relid)) AS total_size,
            pg_size_pretty(pg_relation_size(relid)) AS table_size,
            pg_size_pretty(pg_indexes_size(relid)) AS index_size,
            n_live_tup AS live_rows
     FROM pg_stat_user_tables
     ORDER BY pg_total_relation_size(relid) DESC
     LIMIT 30;

  3. Table family aggregation -- group tables by prefix pattern to get per-family totals:
     SELECT
       CASE
         WHEN relname LIKE 'cmc_ema%' THEN 'EMA family'
         WHEN relname LIKE 'cmc_price_bars%' THEN 'Price Bars family'
         WHEN relname LIKE 'cmc_returns%' THEN 'Returns family'
         WHEN relname LIKE 'cmc_ama%' THEN 'AMA family'
         WHEN relname LIKE 'cmc_features%' THEN 'Features'
         WHEN relname LIKE 'cmc_regime%' THEN 'Regime family'
         WHEN relname LIKE 'cmc_signal%' OR relname LIKE 'cmc_backtest%' THEN 'Signal/Backtest family'
         WHEN relname LIKE 'dim_%' THEN 'Dimension tables'
         ELSE 'Other'
       END AS family,
       COUNT(*) AS table_count,
       pg_size_pretty(SUM(pg_total_relation_size(relid))) AS total_size,
       SUM(n_live_tup) AS total_rows
     FROM pg_stat_user_tables
     GROUP BY 1
     ORDER BY SUM(pg_total_relation_size(relid)) DESC;

  4. Row count for asset_data_coverage or dim_assets to confirm asset count:
     SELECT id, symbol FROM dim_assets ORDER BY id;
