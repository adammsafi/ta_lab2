Append two new query functions to the existing `queries/research.py`. Do NOT modify existing functions.

**1. load_ohlcv_features(_engine, asset_id: int, tf: str) -> pd.DataFrame**
- @st.cache_data(ttl=300)
- SELECT ts, open, high, low, close, volume, rsi_14 FROM public.features WHERE id=:id AND tf=:tf ORDER BY ts
- Convert ts to UTC via pd.to_datetime(utc=True)
- Return empty DataFrame if no rows

**2. load_ema_overlays(_engine, asset_id: int, tf: str, periods: list[int] | None = None) -> pd.DataFrame**
- @st.cache_data(ttl=300)
- SELECT ts, period, ema FROM public.ema_multi_tf_u WHERE id=:id AND tf=:tf AND alignment_source='multi_tf'
- If periods provided, add AND period = ANY(:periods) using text() with array binding
- If periods is None, load all available periods
- Convert ts to UTC
- ORDER BY period, ts
- Return empty DataFrame if no rows

CRITICAL: ema_multi_tf_u uses columns d1/d2/d1_roll/d2_roll (NOT ema_d1). The column we want for overlay is the actual EMA value. Per RESEARCH.md, the column is just `ema`. Use that.

For the periods parameter binding with SQLAlchemy text(), use:
```python
if periods:
    sql = text("... AND period = ANY(:periods)")
    params["periods"] = periods
```
Note: psycopg2 handles Python list -> PostgreSQL array for ANY().
