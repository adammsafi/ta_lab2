  <action>
Create `pages/11_backtest_results.py` following the established page pattern (no st.set_page_config, module-level docstring, try/except engine, sidebar controls outside fragment).

**Page structure:**

1. **Header**: st.header("Backtest Results"), st.caption with context about 109 assets / 13 strategies / 16 cost scenarios

2. **URL state (module level, outside fragment)**:
   - Read st.query_params for: view (leaderboard/strategy/asset), strategy, asset, tf, cv_method, cost_scenario
   - Set defaults: view="leaderboard", tf="1D", cv_method="purged_kfold", cost_scenario="spot_fee16_slip10"

3. **Sidebar controls (outside fragment)**:
   - View toggle: st.radio("View", ["Leaderboard", "Strategy-First", "Asset-First"], horizontal=True)
   - Timeframe: st.selectbox (only "1D" in current data, but future-proof)
   - CV Method: st.selectbox(["purged_kfold", "cpcv"])
   - Cost Scenario: st.selectbox with all 16 scenarios
   - Strategy filter: st.multiselect from load_bakeoff_strategies()
   - Asset filter: st.multiselect from load_bakeoff_assets() (show symbol, use asset_id)
   - Update st.query_params on widget changes

4. **Auto-refresh fragment** (@st.fragment, run_every=3600 since bakeoff data is static):

   **Leaderboard View:**
   - Load via load_bakeoff_leaderboard() with selected filters
   - Display as st.dataframe with columns: strategy_name, symbol (join from load_bakeoff_assets), sharpe_mean, sharpe_std, psr, dsr, max_drawdown_worst, turnover, trade_count_total, pbo_prob, experiment_name
   - Color-code PSR/DSR using st.column_config.NumberColumn with format + help text
   - PSR > 0.95 = strong, PSR > 0.50 = moderate (use column_config formatting)
   - Sort by sharpe_mean DESC by default
   - Apply strategy/asset multiselect filters client-side on the loaded DataFrame

   **Strategy-First View:**
   - Group by strategy_name
   - For each strategy: show expander with st.expander(strategy_name)
   - Inside expander: filtered DataFrame of all assets for that strategy, sorted by sharpe_mean DESC
   - Equity sparkline for top 3 assets per strategy (load fold_metrics, build sparkline)

   **Asset-First View:**
   - Group by asset (symbol)
   - For each asset: show expander with st.expander(symbol)
   - Inside expander: all strategies for that asset, sorted by sharpe_mean DESC

5. **Cost Scenario Comparison (below main view)**:
   - st.divider() + st.subheader("Cost Scenario Comparison")
   - Two selectboxes: pick strategy_name and asset (symbol)
   - Load via load_bakeoff_cost_matrix()
   - Display as pivoted table: rows = metrics (sharpe_mean, psr, dsr, max_drawdown_worst, turnover), columns = cost scenarios
   - st.dataframe with use_container_width=True

6. **Monte Carlo Summary Card**:
   - st.divider() + st.subheader("Monte Carlo Sharpe CI")
   - For the currently selected strategy+asset+cost_scenario, load fold_metrics via load_bakeoff_fold_metrics()
   - Extract per-fold Sharpe values from fold_metrics_json
   - Compute bootstrap CI: resample fold Sharpe values 1000 times, compute mean of each sample, report 5th and 95th percentile as CI
   - Display as st.metric cards: "Sharpe Mean", "95% CI Lower", "95% CI Upper"
   - Build and display equity sparkline from fold_metrics
   - chart_download_button for the sparkline

7. **CSV Download**:
   - st.download_button for the currently displayed leaderboard DataFrame

**Anti-patterns to avoid:**
- No st.set_page_config()
- No sidebar widgets inside @st.fragment
- No full-table load of strategy_bakeoff_results (always filter server-side)
- Use unique key= on every st.plotly_chart, st.dataframe, st.selectbox call
- Use .tolist() on any tz-aware datetime columns before Plotly
