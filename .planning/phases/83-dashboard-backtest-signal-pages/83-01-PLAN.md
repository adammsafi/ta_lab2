Create `queries/backtest.py` following the exact pattern from `queries/research.py` and `queries/experiments.py`:

**backtest.py functions:**

1. `load_bakeoff_leaderboard(_engine, tf="1D", cv_method="purged_kfold", cost_scenario="spot_fee16_slip10") -> pd.DataFrame`
   - SELECT strategy_name, asset_id, tf, params_json, cost_scenario, cv_method, sharpe_mean, sharpe_std, max_drawdown_worst, psr, dsr, turnover, trade_count_total, pbo_prob, experiment_name, computed_at FROM public.strategy_bakeoff_results WHERE tf=:tf AND cv_method=:cv_method AND cost_scenario=:cost_scenario ORDER BY sharpe_mean DESC
   - Convert computed_at to UTC datetime

2. `load_bakeoff_for_asset(_engine, asset_id: int, tf="1D") -> pd.DataFrame`
   - Same columns, filtered by asset_id and tf (all cost scenarios and cv_methods for that asset)
   - ORDER BY cost_scenario, cv_method, sharpe_mean DESC

3. `load_bakeoff_cost_matrix(_engine, strategy_name: str, asset_id: int, tf="1D", cv_method="purged_kfold") -> pd.DataFrame`
   - SELECT cost_scenario, sharpe_mean, sharpe_std, max_drawdown_worst, psr, dsr, turnover, trade_count_total
   - WHERE strategy_name=:strategy_name AND asset_id=:asset_id AND tf=:tf AND cv_method=:cv_method
   - ORDER BY cost_scenario

4. `load_bakeoff_strategies(_engine) -> list[str]`
   - SELECT DISTINCT strategy_name ORDER BY strategy_name

5. `load_bakeoff_assets(_engine) -> pd.DataFrame`
   - SELECT DISTINCT sbr.asset_id, da.symbol FROM strategy_bakeoff_results sbr JOIN dim_assets da ON da.id = sbr.asset_id ORDER BY da.symbol

6. `load_bakeoff_fold_metrics(_engine, strategy_name: str, asset_id: int, tf="1D", cost_scenario="spot_fee16_slip10", cv_method="purged_kfold") -> list[dict]`
   - SELECT fold_metrics_json FROM strategy_bakeoff_results with all filters
   - Return the parsed list of dicts (JSONB auto-deserialized by psycopg2, do NOT json.loads())
   - Return empty list if no rows

**signals.py functions:**

1. `load_active_signals(_engine, strategy_type: str | None = None) -> pd.DataFrame`
   - UNION ALL across three signal tables (signals_ema_crossover, signals_rsi_mean_revert, signals_atr_breakout)
   - Add a literal 'ema_crossover'/'rsi_mean_revert'/'atr_breakout' AS strategy_type column to each SELECT
   - WHERE position_state = 'open'
   - Optional strategy_type filter applied after UNION (or pre-filter individual SELECTs)
   - JOIN dim_assets da ON da.id = combined.id for symbol column
   - Columns: id, venue_id, symbol, ts, signal_id, direction, entry_price, entry_ts, pnl_pct, feature_snapshot, regime_key, strategy_type
   - Convert ts, entry_ts to UTC
   - ttl=300

2. `load_signal_history(_engine, asset_id: int | None = None, strategy_type: str | None = None, days: int = 90) -> pd.DataFrame`
   - Same UNION ALL pattern but no position_state filter, add WHERE ts >= NOW() - :days interval
   - Include exit_price, exit_ts, position_state columns
   - Optional asset_id and strategy_type filters
   - Convert all timestamp columns to UTC
   - ttl=300

3. `load_signal_strategies(_engine) -> list[str]`
   - Return hardcoded ["ema_crossover", "rsi_mean_revert", "atr_breakout"] (no table to query)

4. `load_dim_signals(_engine) -> pd.DataFrame`
   - SELECT signal_id, signal_name, signal_type, params FROM public.dim_signals ORDER BY signal_id
   - ttl=3600

CRITICAL: For the UNION ALL queries in signals.py, each sub-SELECT must have identical column lists. The three signal tables share the same 17-column schema (verified in RESEARCH.md). Use `NOW() - make_interval(days => :days)` for PostgreSQL interval arithmetic with parameterized days.

**2. build_equity_sparkline(fold_metrics: list[dict], height=150) -> go.Figure**

As specified in RESEARCH.md Pattern 6:
- fold_metrics: list of dicts from fold_metrics_json, each with keys: fold_idx, total_return, test_start, test_end
- Compute cumulative equity: start at 1.0, multiply by (1 + total_return) per fold
- x_vals = [first fold test_start] + [each fold test_end]
- y_vals = [1.0] + [cumulative values]
- go.Scatter mode="lines", line color green(0,200,100)
- template="plotly_dark", height=height, showlegend=False, margin=dict(l=0, r=0, t=0, b=0)
- Handle empty fold_metrics (return empty figure with annotation)

- Actually, use go.Scatter with mode="lines" for each signal period: draw a thick line from entry_ts to exit_ts at y=strategy_type
