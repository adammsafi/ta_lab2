**Page structure:**

1. **Header**: st.header("Signal Browser"), st.caption about live signal monitoring across 3 generators

2. **URL state (module level)**:
   - Read st.query_params for: view (cards/table/heatmap), strategy_type, asset, direction
   - Defaults: view="table", strategy_type=None, direction=None

3. **Sidebar controls (outside fragment)**:
   - View toggle: st.radio("Active Signal View", ["Dashboard Cards", "Live Table", "Heatmap Grid"], horizontal=True)
   - Strategy filter: st.multiselect("Strategy", ["ema_crossover", "rsi_mean_revert", "atr_breakout"])
   - Direction filter: st.selectbox("Direction", ["All", "long", "short"])
   - Asset search: st.text_input("Search asset")
   - History days: st.select_slider("History (days)", [7, 14, 30, 60, 90], value=30)
   - Update st.query_params on change

4. **Signal strength computation** (helper function at module level):
   - Define `compute_signal_strength(feature_snapshot: dict | None) -> int`
   - If feature_snapshot is None, return 50 (neutral)
   - CRITICAL: Use `.get()` with defaults for ALL dictionary key accesses to prevent KeyError. Not all signal types have the same feature_snapshot keys.
   - Extract available features with defensive access:
     - close = feature_snapshot.get("close", None)
     - fast_ema = feature_snapshot.get("fast_ema", None)
     - slow_ema = feature_snapshot.get("slow_ema", None)
     - rsi_14 = feature_snapshot.get("rsi_14", None)
     - atr_14 = feature_snapshot.get("atr_14", None)
   - Score components (only contribute if the required keys are present):
     - EMA separation (only if close, fast_ema, slow_ema are all not None): abs(fast_ema - slow_ema) / close * 100, clamp to 0-30 points
     - RSI extremity (only if rsi_14 is not None): abs(rsi_14 - 50) / 50 * 30, clamp to 0-30 points (more extreme = higher)
     - ATR magnitude (only if close and atr_14 are both not None): atr_14 / close * 100 * 10, clamp to 0-20 points
     - Base: 20 points
   - Return sum clamped to 0-100
   - This is Claude's discretion per CONTEXT.md -- the exact formula can be adjusted

5. **Auto-refresh fragment** (@st.fragment(run_every=300)):

   **Summary metrics row (always visible above views):**
   - Load active signals via load_active_signals()
   - st.columns(4): Total Active, Long, Short, By Strategy (comma-separated counts)
   - Apply sidebar filters (strategy, direction, asset search) to the active DataFrame

   **Dashboard Cards View:**
   - For each active signal, render a card using st.container with border=True:
     - Top: Symbol (bold) + Strategy type badge + Direction (green=long, red=short)
     - Middle: Entry price, Entry time (relative: "2d ago"), Current PnL % (if available)
     - Bottom: Signal strength bar (st.progress), Regime key
   - Layout: st.columns(3) to show 3 cards per row
   - Limit to first 30 signals to avoid rendering overload; show "Showing 30 of N" caption

   **Live Table View:**
   - st.dataframe with columns: symbol, strategy_type, direction, entry_price, entry_ts, signal_strength, regime_key, signal_id
   - Add signal_strength column computed from feature_snapshot
   - st.column_config for direction (color-coded), signal_strength (ProgressColumn 0-100)
   - Sortable, use_container_width=True

   **Heatmap Grid View:**
   - Build a pivot: rows = unique symbols (sorted), columns = strategy types
   - Cell value: direction ("long"/"short") or empty
   - Color: green for long, red for short, gray for no signal
   - Render using go.Heatmap with text annotations showing direction
   - chart_download_button for the heatmap

   - CSV download button
