**B. dim_data_sources table:**

Use `op.execute()` with raw SQL (matching the dim_venues migration pattern). Schema:

```sql
CREATE TABLE public.dim_data_sources (
    source_key        TEXT PRIMARY KEY,
    source_name       TEXT NOT NULL UNIQUE,
    source_table      TEXT NOT NULL,
    venue_id          SMALLINT NOT NULL REFERENCES public.dim_venues(venue_id),
    default_venue     TEXT NOT NULL,
    ohlc_repair       BOOLEAN NOT NULL DEFAULT FALSE,
    has_market_cap    BOOLEAN NOT NULL DEFAULT FALSE,
    has_timehigh      BOOLEAN NOT NULL DEFAULT FALSE,
    id_loader_sql     TEXT,
    src_cte_template  TEXT NOT NULL,
    join_clause       TEXT,
    id_filter_sql     TEXT NOT NULL,
    ts_column         TEXT NOT NULL DEFAULT 'timestamp',
    conflict_columns  TEXT NOT NULL,
    src_name_label    TEXT NOT NULL,
    description       TEXT
);
```

Key column notes:
- `source_key`: Short identifier ('cmc', 'tvc', 'hl')
- `conflict_columns`: Comma-separated PK column list for ON CONFLICT clause. **IMPORTANT:** Use the CURRENT PK columns (post venue_id migration), NOT the pre-migration ON CONFLICT clauses from the builder files:

  - HL: `id,venue_id,tf,bar_seq,timestamp` (same PK)
- `src_cte_template`: The full CTE SQL template with `{placeholder}` style substitution points
- `id_filter_sql`: WHERE clause fragment template for filtering by IDs (e.g., `s.id = ANY(%(ids)s)`)
- `ts_column`: Source timestamp column name ('timestamp' for CMC/TVC, 'ts' for HL)
- `venue_id`: FK to dim_venues. CMC=1, TVC=11, HL=2

Seed with 3 rows (CMC, TVC, HL) using the SQL templates extracted from reading the builder files. For the SQL template TEXT values, use dollar-quoting (`$$...$$`) in the INSERT to avoid escaping issues with single quotes inside SQL.
