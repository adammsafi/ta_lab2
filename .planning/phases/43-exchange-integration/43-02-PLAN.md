Rewrite `src/ta_lab2/connectivity/coinbase.py` completely. The new implementation must:

**Class structure:**
```python
class CoinbaseExchange(ExchangeInterface):
    BASE_HOST = "api.coinbase.com"
    SANDBOX_HOST = "api-sandbox.coinbase.com"
```

**Constructor:**
- Accept `api_key: str = None, api_secret: str = None, config: ExchangeConfig = None, **kwargs`
- If `config` is provided, use config.api_key, config.api_secret, config.environment
- If config is None, default environment to "production"
- Store `self.environment` and select base host accordingly
- Build `self.base_url = f"https://{host}"`
- Call `super().__init__(api_key, api_secret, **kwargs)`

**JWT signing method `_build_jwt(self, method: str, path: str) -> str`:**
- Import `jwt` (PyJWT), `time`, `secrets`
- Import `load_pem_private_key` from `cryptography.hazmat.primitives.serialization`
- Load the EC private key from `self.api_secret.encode("utf-8")` with `password=None`
- Build URI: `f"{method} {self.BASE_HOST}{path}"` (use BASE_HOST or SANDBOX_HOST based on environment -- always the HOST not the full URL)
- JWT payload: `{"sub": self.api_key, "iss": "cdp", "nbf": int(time.time()), "exp": int(time.time()) + 120, "uri": uri}`
- JWT headers: `{"kid": self.api_key, "nonce": secrets.token_hex(16)}`
- Return `jwt.encode(payload, private_key, algorithm="ES256", headers=headers)`

**Helper `_authenticated_request(self, method: str, path: str, data: dict = None) -> dict`:**
- Build JWT token via `_build_jwt(method.upper(), path)`
- Set Authorization header: `Bearer {token}`
- Full URL: `f"{self.base_url}{path}"`
- For GET: `self.session.get(url, headers=headers)`
- For POST: `self.session.post(url, headers=headers, json=data)`
- Call `response.raise_for_status()`
- Return `response.json()`

**Helper `_requires_auth(self)` -- raises AuthenticationError if api_key or api_secret is empty/None.**

**Public API methods (NO auth required, use existing @handle_api_errors decorator):**

`_normalize_pair(self, pair: str) -> str`:
- Keep existing: `pair.replace("/", "-").upper()`

`get_ticker(self, pair: str) -> Dict[str, float]`:
- Use the public Advanced Trade endpoint: GET `/api/v3/brokerage/products/{product_id}/ticker`
- This endpoint does NOT require JWT auth (truly public, unlike /best_bid_ask which requires auth)
- Parse response to extract bid, ask, and compute last_price as midpoint: `mid = (bid + ask) / 2`
- Return `{"last_price": mid, "bid": bid, "ask": ask}`

`get_order_book(self, pair: str, depth: int) -> Dict[str, List[List[float]]]`:
- Use Advanced Trade endpoint: GET `/api/v3/brokerage/product_book?product_id={product_id}&limit={depth}`
- Parse bids and asks arrays
- Return `{"bids": [[price, qty], ...], "asks": [[price, qty], ...]}`

`get_historical_klines(self, pair: str, interval: str, start_time: int, end_time: int) -> List[list]`:
- Use Advanced Trade endpoint: GET `/api/v3/brokerage/products/{product_id}/candles`
- Map interval to granularity: {"1m": "ONE_MINUTE", "5m": "FIVE_MINUTE", "15m": "FIFTEEN_MINUTE", "30m": "THIRTY_MINUTE", "1h": "ONE_HOUR", "6h": "SIX_HOUR", "1d": "ONE_DAY"}
- Pass params: start (ISO 8601 string), end (ISO 8601 string), granularity
- Parse candles array: each candle has {start, low, high, open, close, volume}
- Return list of [timestamp, open, high, low, close, volume] sorted ascending

**Authenticated methods (require auth, use @handle_api_errors):**

`get_account_balances(self) -> Dict[str, float]`:
- Call `_requires_auth()`
- GET `/api/v3/brokerage/accounts` with pagination (cursor-based)
- Return `{currency: float(available_balance)}` for all accounts with balance > 0
- Handle pagination: if `has_next` is True, follow `cursor` parameter

`get_open_orders(self, pair: str = None) -> List[dict]`:
- Call `_requires_auth()`
- GET `/api/v3/brokerage/orders/historical/batch` with `order_status=OPEN`
- If pair provided, add `product_id` filter
- Return list of order dicts with: order_id, product_id, side, type, size, price, status

`place_market_order(self, pair: str, side: str, amount: float) -> Dict[str, str]`:
- Call `_requires_auth()`
- POST `/api/v3/brokerage/orders` with body:
  ```json
  {
    "client_order_id": str(uuid.uuid4()),
    "product_id": self._normalize_pair(pair),
    "side": side.upper(),
    "order_configuration": {
      "market_market_ioc": {"quote_size": str(amount)}
    }
  }
  ```
- For SELL side, use `"base_size"` instead of `"quote_size"`
- Return `{"order_id": response["success_response"]["order_id"], "status": "submitted"}`

`place_limit_order(self, pair: str, side: str, price: float, amount: float) -> Dict[str, str]`:
- Call `_requires_auth()`
- POST `/api/v3/brokerage/orders` with body:
  ```json
  {
    "client_order_id": str(uuid.uuid4()),
    "product_id": self._normalize_pair(pair),
    "side": side.upper(),
    "order_configuration": {
      "limit_limit_gtc": {
        "base_size": str(amount),
        "limit_price": str(price)
      }
    }
  }
  ```
- Return `{"order_id": response["success_response"]["order_id"], "status": "submitted"}`

`cancel_order(self, order_id: str) -> Dict[str, str]`:
- Call `_requires_auth()`
- POST `/api/v3/brokerage/orders/batch_cancel` with body: `{"order_ids": [order_id]}`
- Return `{"order_id": order_id, "status": "cancelled"}`

**Error handling:**
- Wrap authenticated methods with `@handle_api_errors` decorator
- In `_authenticated_request`, log the method+path at DEBUG level before calling
- On HTTP error, the decorator will catch and re-raise as appropriate exception type

CRITICAL: The existing coinbase.py public methods (get_ticker, get_order_book, get_historical_klines) use the old Pro API URLs. ALL paths must be updated to Advanced Trade API paths. Do not preserve any old Pro API URLs.
