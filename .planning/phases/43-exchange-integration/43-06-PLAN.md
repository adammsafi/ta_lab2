**tests/test_coinbase_auth.py:**

Test Coinbase JWT auth using mocked crypto. These tests verify the JWT structure without needing a real EC private key.

1. `test_build_jwt_structure` -- Mock `load_pem_private_key` and `jwt.encode` to verify:
   - payload has sub, iss="cdp", nbf, exp, uri keys
   - exp is ~120s after nbf
   - uri format is "METHOD host/path"
   - headers have kid and nonce
   - algorithm is "ES256"
   Use `unittest.mock.patch` on the jwt.encode call to capture arguments.

2. `test_build_jwt_sandbox_host` -- With environment="sandbox", URI uses sandbox host

3. `test_authenticated_request_adds_bearer` -- Mock session.get/post to verify Authorization header starts with "Bearer "

4. `test_requires_auth_no_key` -- CoinbaseExchange(api_key=None).get_account_balances() raises AuthenticationError

5. `test_requires_auth_no_secret` -- CoinbaseExchange(api_key="key", api_secret=None).get_account_balances() raises AuthenticationError

6. `test_ticker_uses_advanced_trade_url` -- Mock session.get, verify URL contains "/api/v3/brokerage/" not "/products/"

7. `test_config_integration` -- CoinbaseExchange(config=ExchangeConfig(venue="coinbase", api_key="k", api_secret="s", environment="production")) uses production host

**tests/test_kraken_auth.py:**

Test Kraken HMAC-SHA512 signing with known test vectors.

1. `test_sign_known_vector` -- Use a known api_secret (base64-encoded), urlpath, nonce, and data to verify the HMAC-SHA512 output matches expected value. Compute expected value independently:
   ```python
   import urllib.parse, hashlib, hmac, base64
   api_secret_b64 = "dGVzdF9zZWNyZXRfa2V5X2Zvcl9rcmFrZW4="  # base64 of "test_secret_key_for_kraken"
   nonce = "1234567890000"
   data = {"nonce": nonce, "pair": "XBTUSD"}
   urlpath = "/0/private/Balance"
   encoded = (nonce + urllib.parse.urlencode(data)).encode()
   message = urlpath.encode() + hashlib.sha256(encoded).digest()
   expected = base64.b64encode(hmac.new(base64.b64decode(api_secret_b64), message, hashlib.sha512).digest()).decode()
   ```
   Create a KrakenExchange with that api_secret and verify _sign returns expected.

2. `test_private_post_sends_headers` -- Mock session.post to verify API-Key and API-Sign headers are present

3. `test_private_post_checks_error` -- Mock response with `{"error": ["EGeneral:Some error"], "result": {}}` and verify InvalidRequestError is raised

4. `test_nonce_is_milliseconds` -- Mock time.time, verify nonce in data matches ms-resolution timestamp

5. `test_requires_auth_no_key` -- KrakenExchange().get_account_balances() raises AuthenticationError

6. `test_config_integration` -- KrakenExchange(config=ExchangeConfig(venue="kraken", api_key="k", api_secret="s")) has correct credentials
