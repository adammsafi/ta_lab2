  <name>Task 1: Create ExchangeConfig dataclass</name>
  <files>src/ta_lab2/connectivity/exchange_config.py</files>
  <action>
Create `src/ta_lab2/connectivity/exchange_config.py` with an `ExchangeConfig` frozen dataclass:

```python
from dataclasses import dataclass, field
from typing import Literal, Optional
```

Fields:
- `venue: str` -- exchange name (e.g. "coinbase", "kraken")
- `environment: Literal["sandbox", "production"]` -- defaults to "sandbox"
- `api_key: str` -- defaults to ""
- `api_secret: str` -- defaults to ""
- `passphrase: Optional[str]` -- defaults to None (not used by Coinbase Advanced Trade or Kraken, kept for backward compat)
- `env_file: Optional[str]` -- defaults to None

Methods:

`validate(self)` -- raises ValueError if api_key is empty or api_secret is empty. Message should include the venue name for clarity.

`@classmethod from_env_file(cls, venue: str, env_file: str, environment: str = "sandbox") -> "ExchangeConfig"`:
- Parse the env_file using a simple key=value line reader (do NOT add python-dotenv dependency -- project does not use it).
- Strip surrounding quotes from values (single or double).
- Skip blank lines and lines starting with #.
- Look for keys: `API_KEY`, `API_SECRET`, `PASSPHRASE` (case-insensitive after stripping).
- Return an ExchangeConfig instance with parsed values.
- Raise FileNotFoundError if env_file does not exist.

`@property def is_sandbox(self) -> bool` -- returns True if environment == "sandbox".

The dataclass should NOT be frozen (needs to allow mutation for testing and runtime overrides).

Why NOT frozen: ExchangeConfig instances may need api_key/secret set after construction (e.g., from environment variables at runtime).

<task type="auto">
  <name>Task 2: Create Alembic migration + reference DDL for exchange tables</name>
  <files>alembic/versions/XXXX_exchange_tables.py, sql/exchange/080_exchange_price_feed.sql, sql/exchange/081_paper_orders.sql</files>
  <action>
First create the `sql/exchange/` directory if it does not exist.

Generate a new Alembic revision: `alembic revision -m "exchange_tables"` from the project root. The filename will contain an auto-generated revision hash (XXXX is a placeholder).

Set `down_revision = '8d5bc7ee1732'` (Phase 41 head, asset_stats_and_correlation_tables).

In upgrade(), create TWO tables:

**Table 1: exchange_price_feed** in public schema:
- `feed_id` UUID PK with `server_default=sa.text('gen_random_uuid()')`, nullable=False
- `fetched_at` TIMESTAMPTZ NOT NULL, `server_default=sa.text('now()')`
- `exchange` TEXT NOT NULL
- `pair` TEXT NOT NULL
- `environment` TEXT NOT NULL, `server_default='production'`
- `bid` NUMERIC nullable
- `ask` NUMERIC nullable
- `mid` NUMERIC nullable
- `last_price` NUMERIC NOT NULL
- `bar_close` NUMERIC nullable
- `bar_ts` TIMESTAMPTZ nullable
- `discrepancy_pct` NUMERIC nullable
- `threshold_pct` NUMERIC nullable
- `exceeds_threshold` BOOLEAN nullable

CHECK constraint named `chk_exchange`: `exchange IN ('coinbase','kraken','binance','bitfinex','bitstamp')`

Indexes:
- `idx_exchange_price_feed_pair_ts` on (exchange, pair, fetched_at DESC)
- `idx_exchange_price_feed_exceeds` on (exceeds_threshold, fetched_at DESC) WHERE exceeds_threshold = TRUE (partial index)

**Table 2: paper_orders** in public schema:
- `order_uuid` UUID PK with `server_default=sa.text('gen_random_uuid()')`, nullable=False
- `created_at` TIMESTAMPTZ NOT NULL, `server_default=sa.text('now()')`
- `signal_id` INTEGER nullable
- `asset_id` INTEGER nullable
- `exchange` TEXT NOT NULL
- `pair` TEXT NOT NULL
- `side` TEXT NOT NULL
- `order_type` TEXT NOT NULL
- `quantity` NUMERIC NOT NULL
- `limit_price` NUMERIC nullable
- `stop_price` NUMERIC nullable
- `exchange_payload` sa.Text nullable (store JSON as text -- avoids JSONB import complexity; parse with json.loads when needed)
- `status` TEXT NOT NULL, `server_default='paper'`
- `environment` TEXT NOT NULL, `server_default='sandbox'`
- `client_order_id` TEXT nullable

CHECK constraints:
- `chk_side`: `side IN ('buy', 'sell')`
- `chk_order_type`: `order_type IN ('market', 'limit', 'stop')`
- `chk_status`: `status IN ('paper', 'submitted', 'cancelled')`

In downgrade(): Drop indexes first, then drop both tables (paper_orders first, then exchange_price_feed).

CRITICAL: Do NOT use any UTF-8 box-drawing characters in comments (Windows encoding pitfall per MEMORY.md). Use plain ASCII dashes or equals signs only.

Also create reference DDL files:
- `sql/exchange/080_exchange_price_feed.sql` -- CREATE TABLE + COMMENT ON TABLE
- `sql/exchange/081_paper_orders.sql` -- CREATE TABLE + COMMENT ON TABLE

These are documentation-only files matching project convention. They are NOT executed by Alembic.
  </action>
