  <action>
Create `src/ta_lab2/scripts/features/refresh_ctf.py` -- the standalone CLI for CTF feature refresh.

**Module docstring:** Include usage examples matching the roadmap success criteria commands.

**Data structures:**

1. `CTFWorkerTask` dataclass (frozen):
   - `asset_id: int`
   - `db_url: str`
   - `venue_id: int`
   - `alignment_source: str`
   - `yaml_path: str | None`
   - `base_tf_filter: list[str] | None` (None = all from YAML)
   - `ref_tf_filter: list[str] | None` (None = all from YAML)
   - `indicator_filter: list[str] | None` (None = all active)
   - `dry_run: bool`

2. `CTFRefreshResult` dataclass:
   - `asset_id: int`
   - `rows_written: int`
   - `duration_seconds: float`
   - `error: str | None`

**Module-level worker function** `_ctf_worker(task: CTFWorkerTask) -> dict`:
- MUST be module-level (not a method) for pickling compatibility
- Creates its own NullPool engine inside the function
- Instantiates CTFFeature with CTFConfig
- Applies filtering:
  - If `task.base_tf_filter` or `task.ref_tf_filter` is set: load the YAML config, filter `timeframe_pairs` in-memory to only matching entries, write a temp YAML file, pass its path via `CTFConfig(yaml_path=temp_path)`. Clean up temp file in finally block.
  - If `task.indicator_filter` is set: after loading dim_ctf_indicators, pre-filter the list. To do this cleanly, override CTFFeature._dim_indicators cache directly after load: `feature._dim_indicators = [ind for ind in feature._load_dim_ctf_indicators() if ind['indicator_name'] in task.indicator_filter]`.
- If `task.dry_run`: skip compute_for_ids, instead just log what WOULD be computed (how many TF pairs, how many indicators, for which asset_id). Return rows_written=0.
- Calls `feature.compute_for_ids(ids=[task.asset_id])` (one asset per worker)
- Returns dict: `{"asset_id": ..., "rows": ..., "duration": ..., "error": ...}`
- Always disposes engine in finally block

**State management functions:**

`_should_skip_asset(engine, asset_id, venue_id, base_tf, ref_tf, indicator_id, alignment_source) -> bool`:
- Query ctf_state for this exact scope
- Query source table(s) for MAX(ingested_at) or MAX(computed_at) from public.ctf WHERE same scope
- If ctf_state.updated_at >= source freshness, return True (skip)
- If no state row exists, return False (must compute)

`_update_ctf_state(engine, asset_id, venue_id, base_tf, ref_tf, indicator_id, alignment_source, last_ts, row_count)`:
- Upsert into ctf_state table
- Uses INSERT ... ON CONFLICT DO UPDATE pattern

`_reset_ctf_state(engine, ids, venue_id)`:
- DELETE FROM ctf_state WHERE id = ANY(ids) AND venue_id = venue_id
- Used by --full-refresh

`_delete_ctf_rows(engine, ids, venue_id)`:
- DELETE FROM public.ctf WHERE id = ANY(ids) AND venue_id = venue_id
- Used by --full-refresh before recomputing

NOTE: The incremental skip logic is complex at the per-indicator/per-TF-pair level. For Plan 02, implement a SIMPLER approach: skip at the per-asset level. Check if ANY ctf_state row for this asset has updated_at older than the source table's max timestamp for this asset. If all scopes are fresh, skip the entire asset. If any are stale, recompute all scopes for this asset. This avoids the complexity of per-scope skipping while still providing meaningful incremental behavior. The worker function should check this BEFORE calling compute_for_ids.

**CLI argument parsing** (parse_args function):

```
--ids         Comma-separated asset IDs (e.g., '1,52,1027'). Mutually exclusive with --all.
--all         Process all assets from price_bars_multi_tf_u. Mutually exclusive with --ids.
--base-tf     Space-separated base TFs to process (e.g., '1D 7D'). Default: all from YAML.
--ref-tfs     Space-separated ref TFs to filter (e.g., '7D 30D'). Default: all from YAML.
--indicators  Space-separated indicator names (e.g., 'rsi_14 macd_12_26'). Default: all active.
--full-refresh  Recompute everything. Deletes existing ctf rows and resets state.
--workers     Number of parallel workers. Default: min(6, cpu_count()).
--dry-run     Report what would be computed without writing to DB.
--venue-id    Venue ID filter. Default: 1 (CMC_AGG).
--log-level   DEBUG|INFO|WARNING|ERROR. Default: INFO.
```

Use `add_mutually_exclusive_group(required=True)` for --ids/--all (matching run_all_feature_refreshes.py pattern).

`--base-tf` uses `nargs='+'` so user can pass `--base-tf 1D 7D`.
`--ref-tfs` uses `nargs='+'`.
`--indicators` uses `nargs='+'`.

**load_ids function:** Same pattern as run_all_feature_refreshes.py. If --all, query `SELECT DISTINCT id FROM public.price_bars_multi_tf_u WHERE venue_id = :venue_id AND alignment_source = 'multi_tf' ORDER BY id`. If --ids, parse comma-separated.

**main function:**

1. Parse args, setup logging (basicConfig with format matching other scripts)
2. Validate TARGET_DB_URL
3. Create engine (standard, not NullPool -- NullPool is for workers only)
4. Load IDs via load_ids
5. If --full-refresh: call _delete_ctf_rows and _reset_ctf_state for all IDs
6. Build CTFWorkerTask list (one per asset_id)
7. Determine effective workers: `args.workers or min(6, cpu_count())`
8. If workers == 1 or len(ids) == 1: run sequentially (call _ctf_worker directly, with tqdm wrapping the loop)
9. Else: use `multiprocessing.Pool(processes=effective_workers, maxtasksperchild=1)` with `pool.imap_unordered(_ctf_worker, tasks)`
10. Wrap the result iterator with `tqdm(total=len(tasks), desc="CTF refresh", unit="asset")` for progress bar
11. Accumulate results: total_rows, n_done, n_errors, n_skipped, per-asset timing
12. After all workers complete: print summary table (asset count, rows written, errors, duration, skipped)
13. If any errors: log warnings with asset_id and error message
14. Return 0 if no errors, 1 if any errors

**Also expose a callable function for Plan 03 pipeline integration:**

```python
def refresh_ctf_step(
    engine,
    ids: list[int],
    tf: str = "1D",
    venue_id: int = 1,
    workers: int | None = None,
) -> RefreshResult:
    """Run CTF refresh as a pipeline step. Returns RefreshResult for pipeline summary."""
```

This function:
- Imports RefreshResult from run_all_feature_refreshes (or redefines a compatible one)
- Calls the same worker logic as main() but without CLI parsing
- Takes engine and ids directly
- Uses `tf` as the base_tf filter (so pipeline can run CTF for the current pipeline TF)
- Returns a RefreshResult with table="ctf", rows_inserted, duration_seconds, success, error

**Logging:** Use standard Python logging. Per-indicator detail is already handled by CTFFeature internals (it logs at DEBUG level). The CLI should log at INFO level: per-asset completion, progress summary every 10 assets.

**tqdm usage:** `from tqdm import tqdm`. Wrap the result iterator: `for result in tqdm(results_iter, total=len(tasks), desc="CTF refresh", unit="asset"):`. For sequential mode, wrap the task list: `for task in tqdm(tasks, desc="CTF refresh", unit="asset"):`.

**Error handling:** Log-and-continue per asset. If one asset fails, others proceed. Failures are collected and reported in the summary. The overall script returns exit code 1 if ANY asset failed.

**Important implementation notes:**
- Use `from __future__ import annotations` at top
- Import `multiprocessing.Pool` and `cpu_count` (not ProcessPoolExecutor)
- `maxtasksperchild=1` is REQUIRED (Windows project convention)
- Worker creates engine with `NullPool` inside function body
- Pass `str(TARGET_DB_URL)` to worker tasks (not the engine object)
- The temp YAML file approach for base_tf/ref_tf filtering: use `tempfile.NamedTemporaryFile(mode='w', suffix='.yaml', delete=False)`, write filtered config, pass path to CTFConfig, delete in finally block
