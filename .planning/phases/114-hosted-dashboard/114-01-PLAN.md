Implementation details:
- _ssh_cmd() and _vm_psql() helpers identical to sync_cmc_from_vm.py
- _sync_full_replace(table_name, columns): local COPY TO STDOUT CSV -> SSH -> VM: TRUNCATE + COPY FROM STDIN CSV
- _sync_incremental(table_name, columns, watermark_col, conflict_cols, update_cols): local COPY TO STDOUT with WHERE watermark_col > last_sync -> SSH -> VM staging table -> INSERT ON CONFLICT DO UPDATE
- _get_vm_watermark(table_name, watermark_col): SSH psql to get MAX(watermark_col) from VM table
- For `features` table: batch by id (SELECT DISTINCT id) to avoid huge single transactions
- main() iterates all tables, prints per-table row counts and timing
- CLI args: --dry-run (report only), --full (ignore watermarks, re-sync everything), --table TABLE (sync single table)
- Sync log: INSERT into hyperliquid.sync_log on VM after each table sync (reuse existing sync_log table)
- Print summary at end: tables synced, total rows, total time

Use ta_lab2.io.get_engine for local DB connection.
Do NOT hardcode column lists for every table -- use information_schema or SQLAlchemy inspection to get column names dynamically. Only hardcode watermark_col, conflict_cols (PK), and update strategy per table.
