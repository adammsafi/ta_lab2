<task type="auto">
  <name>Task 1: Create nginx config template</name>
  <files>deploy/vm/nginx_streamlit.conf</files>
  <action>
Create nginx server block configuration for Streamlit behind SSL with basic auth.

The config must include:

**HTTP server block (port 80):**
- Redirect all traffic to HTTPS (301)
- Exception: /.well-known/acme-challenge/ for certbot HTTP-01 challenge

**HTTPS server block (port 443):**
- ssl_certificate and ssl_certificate_key paths (Let's Encrypt default: /etc/letsencrypt/live/DOMAIN/)
- Use placeholder `DOMAIN_PLACEHOLDER` that setup script will sed-replace

- auth_basic "ta_lab2 Dashboard" at SERVER level (NOT location level -- research finding: location-level auth breaks WebSocket connections)
- auth_basic_user_file /etc/nginx/.htpasswd

- Root location / block with ALL of:
  - proxy_pass http://127.0.0.1:8501
  - proxy_http_version 1.1
  - proxy_set_header Upgrade $http_upgrade (WebSocket)
  - proxy_set_header Connection "upgrade" (WebSocket)
  - proxy_set_header Host $host
  - proxy_set_header X-Real-IP $remote_addr
  - proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for
  - proxy_set_header X-Forwarded-Proto $scheme
  - proxy_buffering off (REQUIRED for Streamlit streaming)
  - proxy_read_timeout 86400 (long-lived WebSocket)
  - proxy_send_timeout 86400

CRITICAL anti-patterns to avoid (from research):
- Do NOT put WebSocket headers in a separate location block -- Streamlit uses /_stcore/stream and this path can change
- Do NOT omit proxy_buffering off -- causes infinite "Please wait..." spinner
- Do NOT put auth_basic in location block -- breaks WebSocket auth handshake
  </action>

<task type="auto">
  <name>Task 2: Create nginx + SSL setup script</name>
  <files>deploy/vm/setup_nginx_ssl.sh</files>
  <action>
Create setup_nginx_ssl.sh to run on the VM. Must accept domain name as argument.

```bash
#!/usr/bin/env bash
set -euo pipefail
DOMAIN="${1:?Usage: $0 <domain>}"
```

Steps:
1. Install nginx and certbot: `apt-get install -y nginx certbot python3-certbot-nginx`
2. Create initial htpasswd user:
   - Install apache2-utils (for htpasswd command)
   - Prompt for username and password (read -p / read -sp)
   - `htpasswd -cb /etc/nginx/.htpasswd "$USERNAME" "$PASSWORD"`
   - Print instructions for adding more users: `htpasswd -b /etc/nginx/.htpasswd <user> <pass>`
3. Copy nginx_streamlit.conf to /etc/nginx/sites-available/streamlit
   - sed replace DOMAIN_PLACEHOLDER with $DOMAIN
4. Create symlink in sites-enabled, remove default site
5. Test nginx config: `nginx -t`
6. Configure firewall (CRITICAL -- dual-layer for Oracle Cloud):
   - iptables: Insert rules BEFORE the REJECT rule (position matters!):
     ```bash
     # Find the line number of the REJECT rule in INPUT chain
     REJECT_LINE=$(iptables -L INPUT --line-numbers -n | grep -i reject | head -1 | awk '{print $1}')
     iptables -I INPUT $REJECT_LINE -p tcp --dport 80 -j ACCEPT
     iptables -I INPUT $REJECT_LINE -p tcp --dport 443 -j ACCEPT
     ```
   - Save with netfilter-persistent: `apt-get install -y iptables-persistent && netfilter-persistent save`
   - Print WARNING that OCI Security List must also allow 80/443 (script cannot do this -- requires OCI console)
7. Obtain SSL certificate: `certbot --nginx -d $DOMAIN --non-interactive --agree-tos --email admin@$DOMAIN`
   - If certbot fails (DNS not pointed yet), print instructions for manual retry
8. Verify certbot auto-renewal timer: `systemctl list-timers | grep certbot`
9. Reload nginx: `systemctl reload nginx`
10. Print summary: domain, https URL, how to add users, certbot renewal status

Make idempotent: check if nginx already installed, check if cert already exists, etc.
  </action>
