  <action>
Create deploy_dashboard.sh that orchestrates the full deployment from the local PC.

```bash
#!/usr/bin/env bash
set -euo pipefail
DOMAIN="${1:?Usage: $0 <domain>}"
```

VM connection constants (same as sync scripts):
- VM_HOST=161.118.209.59
- VM_USER=ubuntu
- SSH_KEY=~/Downloads/oracle_sg_keys/ssh-key-2026-03-10.key
- REMOTE_DIR=/opt/ta_lab2

Steps:
1. **Rsync codebase to VM:**
   ```bash
   rsync -avz --delete \
     -e "ssh -i $SSH_KEY -o StrictHostKeyChecking=accept-new" \
     --exclude '.git' --exclude '__pycache__' --exclude '.planning' \
     --exclude 'node_modules' --exclude '.mypy_cache' --exclude '.ruff_cache' \
     ./ ${VM_USER}@${VM_HOST}:${REMOTE_DIR}/src/
   ```

2. **Rsync deploy scripts:**
   ```bash
   rsync -avz -e "ssh -i $SSH_KEY" \
     deploy/vm/ ${VM_USER}@${VM_HOST}:${REMOTE_DIR}/deploy/
   ```

3. **Run setup_dashboard_env.sh on VM:**
   ```bash
   ssh -i $SSH_KEY ${VM_USER}@${VM_HOST} "sudo bash ${REMOTE_DIR}/deploy/setup_dashboard_env.sh"
   ```

4. **Run setup_nginx_ssl.sh on VM:**
   ```bash
   ssh -i $SSH_KEY ${VM_USER}@${VM_HOST} "sudo bash ${REMOTE_DIR}/deploy/setup_nginx_ssl.sh $DOMAIN"
   ```

5. **Push initial data from local:**
   ```bash
   python -m ta_lab2.scripts.etl.sync_dashboard_to_vm --full
   ```

6. **Verify Streamlit is running:**
   ```bash
   ssh -i $SSH_KEY ${VM_USER}@${VM_HOST} "systemctl is-active streamlit"
   ```

7. **Verify nginx is running:**
   ```bash
   ssh -i $SSH_KEY ${VM_USER}@${VM_HOST} "systemctl is-active nginx"
   ```

8. **Print deployment summary:**
   - URL: https://$DOMAIN
   - Status of streamlit and nginx services
   - Reminder: Ensure OCI Security List allows TCP 80 and 443
   - Reminder: Ensure DNS A record points $DOMAIN to $VM_HOST
   - How to add users: ssh to VM, htpasswd -b /etc/nginx/.htpasswd <user> <pass>
   - How to sync data: python -m ta_lab2.scripts.etl.sync_dashboard_to_vm
   - How to update code: re-run this script (rsync handles incremental)

Include a --code-only flag that skips steps 3-4 (just rsync + data sync, for code updates after initial deploy).
Include a --data-only flag that skips steps 1-4 (just run sync_dashboard_to_vm).
