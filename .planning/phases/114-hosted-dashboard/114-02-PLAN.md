Create `deploy/vm/` directory with three files:

**1. setup_dashboard_env.sh** -- Idempotent bootstrap script to run on the VM (via SSH or manually).

```bash
#!/usr/bin/env bash
set -euo pipefail
```

Steps the script must perform:
- Install system packages: python3.11-venv, python3-pip (if not present)
- Create /opt/ta_lab2/ directory (owned by ubuntu user)
- Create Python venv at /opt/ta_lab2/venv
- Clone or update ta_lab2 repo from git (or rsync from local -- use rsync approach since repo may be private). For now, assume code is rsynced to /opt/ta_lab2/src/ separately. Script installs from that path.
- pip install -e /opt/ta_lab2/src[dashboard] (or whatever the install group is -- check setup.cfg/pyproject.toml for dashboard extras; if none, just pip install -e .)
- Create /opt/ta_lab2/db_config.env with DB_HOST=127.0.0.1, DB_PORT=5432, DB_NAME=hyperliquid, DB_USER=hluser, DB_PASS=hlpass (same DB as HL collector). Script should NOT overwrite if file exists (guard with -f check).
- Create public schema tables on VM DB: run `alembic upgrade head` from the ta_lab2 directory (uses db_config.env). If alembic fails (missing tables that only exist via manual migration), fall back to pg_dump --schema-only from local and pipe to VM. Document both approaches in comments.
- Copy streamlit_config.toml to /opt/ta_lab2/.streamlit/config.toml
- Install and enable streamlit.service systemd unit
- Print status summary

Make the script idempotent: re-running should not break anything (use mkdir -p, pip install is idempotent, CREATE TABLE IF NOT EXISTS, etc.).

**2. streamlit.service** -- Systemd unit file:

```ini
[Unit]
Description=ta_lab2 Streamlit Dashboard
After=postgresql.service network.target
Wants=postgresql.service

[Service]
Type=simple
User=ubuntu
Group=ubuntu
WorkingDirectory=/opt/ta_lab2/src/src/ta_lab2/dashboard
EnvironmentFile=/opt/ta_lab2/db_config.env
ExecStart=/opt/ta_lab2/venv/bin/streamlit run app.py \
    --server.address 127.0.0.1 \
    --server.port 8501 \
    --server.headless true
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

CRITICAL: --server.address 127.0.0.1 (NEVER 0.0.0.0). nginx handles external access.

**3. streamlit_config.toml** -- Streamlit config for behind-proxy deployment:

```toml
[server]
enableCORS = false
enableXsrfProtection = false
headless = true
address = "127.0.0.1"
port = 8501

[browser]
gatherUsageStats = false
```

enableCORS=false and enableXsrfProtection=false are REQUIRED when behind nginx reverse proxy (research finding).
