  <action>
Create `src/ta_lab2/scripts/analysis/run_phase103_ic.py` with:

**Purpose:** End-to-end script that:
1. Refreshes extended TA features (writes new indicator values to the features/ta table)
2. Runs IC sweep for the ~35 new feature columns
3. Applies FDR control at 5% across all new indicator IC results
4. Promotes FDR passers to dim_feature_registry (lifecycle='promoted')
5. Logs FDR rejects to dim_feature_registry (lifecycle='deprecated') with IC and p-values preserved

**Step 1: Refresh features.**
- Use TAFeature to compute all active indicators (including the 20 new ones) for all assets
- This writes to the features/ta output table via the existing BaseFeature.write_features() path
- The migration from 103-02 already seeded dim_indicators and added the columns

Alternatively, if the TAFeature pipeline is slow or complex to invoke from a script, call the existing refresh script:
```python
# Option A: Direct invocation
from ta_lab2.scripts.features.ta_feature import TAFeature, TAConfig
config = TAConfig(tf="1d", alignment_source="multi_tf")
feature = TAFeature(engine, config)
feature.compute_for_ids(ids=active_ids)
```
```python
# Option B: Shell out to existing refresh
import subprocess
subprocess.run(["python", "-m", "ta_lab2.scripts.features.run_all_feature_refreshes", "--ta", "--all-tfs"], check=True)
```

Choose whichever is consistent with how the project normally refreshes features. Check existing scripts for the pattern.

**Step 2: Run IC sweep.**
- Invoke the existing IC sweep machinery for the ~35 new feature column names
- The sweep reads from the features table and computes Spearman IC vs forward returns
- Phase 102 wiring auto-logs results to trial_registry (no extra code needed)
- If run_ic_sweep.py supports `--features` argument, use it: `--features willr_14,cci_20,...`
- Otherwise, import the IC computation function directly and call it

**Step 3: Apply FDR.**
- After IC sweep populates trial_registry, query trial_registry for all Phase 103 indicator results
- Import `fdr_control` from `ta_lab2.analysis.multiple_testing`
- Apply FDR at alpha=0.05 across all ~35 feature columns' IC p-values
- This returns a mask of which features pass

**Step 4: Promote survivors.**
- For each FDR passer: upsert to dim_feature_registry with lifecycle='promoted'
  ```sql
  INSERT INTO public.dim_feature_registry
      (feature_name, lifecycle, promoted_at, promotion_alpha, best_ic, best_horizon, updated_at)
  VALUES (:feature_name, 'promoted', now(), 0.05, :best_ic, :best_horizon, now())
  ON CONFLICT (feature_name) DO UPDATE SET
      lifecycle = 'promoted', promoted_at = EXCLUDED.promoted_at,
      promotion_alpha = EXCLUDED.promotion_alpha,
      best_ic = EXCLUDED.best_ic, best_horizon = EXCLUDED.best_horizon,
      updated_at = EXCLUDED.updated_at
  ```

**Step 5: Log rejects.**
- For each FDR reject: upsert to dim_feature_registry with lifecycle='deprecated'
  ```sql
  INSERT INTO public.dim_feature_registry
      (feature_name, lifecycle, updated_at)
  VALUES (:feature_name, 'deprecated', now())
  ON CONFLICT (feature_name) DO UPDATE SET
      lifecycle = 'deprecated', updated_at = EXCLUDED.updated_at
  ```

**Step 6: Print summary.**
- Print count of passers and rejects
- Print the top 5 IC values among passers
- Print total distinct indicator_name count from trial_registry for Phase 103

**CLI:** Use argparse with:
- `--skip-refresh`: Skip Step 1 (if features already computed)
- `--fdr-alpha`: FDR threshold (default 0.05)
- `--dry-run`: Run sweep and FDR but don't write to dim_feature_registry

**CRITICAL checks before writing:**
1. Verify Phase 102 is complete: `SELECT COUNT(*) FROM trial_registry` must not error
2. Verify `from ta_lab2.analysis.multiple_testing import fdr_control` imports successfully
3. Verify dim_feature_registry schema: check if column is `lifecycle` or `is_active` (research says lifecycle)

Use `db_config.env` or `TARGET_DB_URL` for the engine, following the project's standard pattern.
