  3. **Add trading_state + daily loss check in _run_strategy** (at the very start of the method body, before the signal_table lookup on line ~247):
     ```python
     # Risk gate: check if trading is halted
     if self.risk_engine._is_halted():
         self.logger.warning(
             "PaperExecutor: trading halted for config=%s -- skipping",
             config.config_name,
         )
         self._write_run_log(config, status="halted")
         return {
             "signals_read": 0,
             "orders_generated": 0,
             "fills_processed": 0,
             "skipped_no_delta": 0,
             "skipped_rejected": 0,
         }

     # Risk gate: check daily loss and trigger kill switch if exceeded
     if self.risk_engine.check_daily_loss():
         self.logger.warning(
             "PaperExecutor: daily loss limit triggered for config=%s -- halting",
             config.config_name,
         )
         self._try_telegram_alert(
             f"DAILY LOSS LIMIT triggered for {config.config_name}"
         )
         self._write_run_log(config, status="halted", error="daily_loss_limit")
         return {
             "signals_read": 0,
             "orders_generated": 0,
             "fills_processed": 0,
             "skipped_no_delta": 0,
             "skipped_rejected": 0,
         }
     ```

     NOTE: Using `_is_halted()` directly is acceptable here -- it's a simple DB read and the alternative (duplicating the query) is worse. The research confirmed this approach.

  4. **Add check_order gate in _process_asset_signal** (after the delta threshold check on line ~431, before the dry_run check on line ~433, insert):
     ```python
     # Risk gate: check order through RiskEngine
     side_for_risk: str = "buy" if delta > 0 else "sell"
     current_position_value = current_qty * current_price
     risk_result = self.risk_engine.check_order(
         order_qty=abs(delta),
         order_side=side_for_risk,
         fill_price=current_price,
         asset_id=asset_id,
         strategy_id=config.config_id,
         current_position_value=current_position_value,
         portfolio_value=portfolio_value,
     )
     if not risk_result.allowed:
         self.logger.info(
             "_process_asset_signal: risk gate blocked order for asset_id=%d: %s",
             asset_id,
             risk_result.blocked_reason,
         )
         return {"skipped_no_delta": True}
     # Use adjusted quantity from risk engine (may be scaled down)
     delta = risk_result.adjusted_quantity if delta > 0 else -risk_result.adjusted_quantity
     ```

     IMPORTANT: After this block, the existing code at the dry_run check (line ~433) and the CanonicalOrder creation (line ~444-445) must use the potentially-adjusted `delta`. The existing code already derives `side` and `order_qty` from `delta`:
     ```python

     order_qty = abs(delta)
     ```
     So no further changes are needed there -- the adjusted delta flows through naturally.
