  **1. Create `_load_signal_scores(engine, asset_ids, active_features, tf, venue_id=1)` helper:**

  Parameters:
  - `engine`: SQLAlchemy engine
  - `asset_ids`: list[int] -- the asset IDs from prices.columns (these are already mapped to int IDs in the existing code)
  - `active_features`: list[dict] -- output of parse_active_features() (has name/indicator/params_hash/source keys)
  - `tf`: str -- timeframe (e.g. "1D")
  - `venue_id`: int -- default 1

  Returns: `pd.DataFrame` with index=asset_ids, columns=feature_names, values=latest feature values.

  Logic:
  - Initialize result DataFrame with NaN (index=asset_ids, columns=[f["name"] for f in active_features])
  - For each feature in active_features:
    - If `feature["source"] == "features"` (bar-level like ret_is_outlier):
      ```sql
      SELECT f.id AS asset_id, f.{feature_col} AS val
      FROM public.features f
      WHERE f.id = ANY(:ids)
        AND f.tf = :tf
        AND f.venue_id = :venue_id
      ORDER BY f.id, f.ts DESC
      ```
      Then take the LAST row per asset_id (latest ts). Use DISTINCT ON for efficiency:
      ```sql
      SELECT DISTINCT ON (f.id) f.id AS asset_id, f.{feature_col} AS val
      FROM public.features f
      WHERE f.id = ANY(:ids) AND f.tf = :tf AND f.venue_id = :venue_id
      ORDER BY f.id, f.ts DESC
      ```
      Validate the column name exists first (same information_schema check as staleness monitor).

    - If `feature["source"] == "ama_multi_tf_u"` (AMA feature):
      Use `d1` column (first derivative -- stationary, momentum-oriented, recommended by research):
      ```sql
      SELECT DISTINCT ON (a.id) a.id AS asset_id, a.d1 AS val
      FROM public.ama_multi_tf_u a
      WHERE a.id = ANY(:ids)
        AND a.tf = :tf
        AND a.venue_id = :venue_id
        AND a.indicator = :indicator
        AND LEFT(a.params_hash, 8) = :params_hash
        AND a.alignment_source = 'multi_tf'
        AND a.roll = FALSE
      ORDER BY a.id, a.ts DESC
      ```

  - For each query result, fill the corresponding column in the result DataFrame.
  - Replace remaining NaN with 0.0 (neutral signal -- research recommendation).
  - Wrap each feature query in try/except so one failure doesn't break all features.
  - Log how many features loaded successfully vs failed.
