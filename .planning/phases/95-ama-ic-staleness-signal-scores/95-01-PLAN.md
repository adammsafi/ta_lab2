  Three changes to this file:

  1. **Bump MAX_ACTIVE_FEATURES from 10 to 20.** There are 20 active features in feature_selection.yaml (18 AMA + 2 bar-level).

  2. **Replace `_load_active_features()` with `parse_active_features()` from bakeoff_orchestrator.** The existing `_load_active_features()` returns a flat list of name strings. Replace it with an import of `parse_active_features` from `ta_lab2.backtests.bakeoff_orchestrator`, which returns dicts with keys: `name`, `indicator`, `params_hash`, `source` (either `"ama_multi_tf_u"` or `"features"`). Update `ICStalenessMonitor.run()` to call `parse_active_features()` and pass the parsed feature dicts (not just name strings) to `_check_one`. Remove the old `_load_active_features()` function.

  3. **Rewrite `_load_close_and_feature()` to branch on feature source.** Accept a parsed feature dict instead of a feature_name string. Two branches:

     **Branch A -- bar-level features (source="features"):** Keep the existing logic: check `information_schema.columns` for the feature column in `features` table, then `SELECT ts, {column}, close FROM features WHERE id=:id AND tf=:tf AND venue_id=:venue_id ORDER BY ts`.

     **Branch B -- AMA features (source="ama_multi_tf_u"):** No column check needed (the column is always `ama`). Use this query pattern (from run_ic_sweep.py):
     ```sql
     SELECT a.ts, a.ama AS feature_value, f.close
     FROM public.ama_multi_tf_u a
     INNER JOIN public.features f
         ON f.id = a.id AND f.ts = a.ts AND f.tf = a.tf AND f.venue_id = a.venue_id
     WHERE a.id = :asset_id
       AND a.venue_id = :venue_id
       AND a.tf = :tf
       AND a.indicator = :indicator
       AND LEFT(a.params_hash, 8) = :params_hash
       AND a.alignment_source = 'multi_tf'
       AND a.roll = FALSE
     ORDER BY a.ts
     ```
     The `indicator` and `params_hash` values come from the parsed feature dict.

     Return type stays the same: `tuple[pd.Series, pd.Series] | None` where first is feature values, second is close prices, both indexed by UTC-aware ts.

  **CRITICAL filters for ama_multi_tf_u:** Always include `alignment_source = 'multi_tf'` AND `roll = FALSE` AND `venue_id = :venue_id`. Missing any causes duplicate rows that break rolling IC computation.

  **Update `_check_one()`:** Change its signature from `(self, conn, feature_name: str, asset_id: int)` to `(self, conn, feature_info: dict, asset_id: int)` where feature_info has keys name/indicator/params_hash/source. Extract `feature_name = feature_info["name"]` for logging. Pass the full dict to `_load_close_and_feature()`.

  **Update `ICStalenessMonitor.run()`:** Replace `features = _load_active_features(...)` with:

  ```python
      active_features = parse_active_features()
  except Exception as exc:

      return 1
  ```
  Limit to `active_features[:MAX_ACTIVE_FEATURES]`. Log count. Iterate `active_features` passing each dict to `_check_one()`.
