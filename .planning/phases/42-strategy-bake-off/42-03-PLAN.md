1. **Constants**:
```python
WEIGHT_SCHEMES = {
    "balanced":      {"sharpe": 0.35, "max_dd_abs": 0.30, "psr": 0.25, "turnover_inv": 0.10},
    "risk_focus":    {"sharpe": 0.25, "max_dd_abs": 0.45, "psr": 0.20, "turnover_inv": 0.10},
    "quality_focus": {"sharpe": 0.30, "max_dd_abs": 0.25, "psr": 0.35, "turnover_inv": 0.10},
    "low_cost":      {"sharpe": 0.30, "max_dd_abs": 0.25, "psr": 0.25, "turnover_inv": 0.20},
}

V1_GATES = {
    "min_sharpe": 1.0,
    "max_drawdown_pct": 15.0,  # max DD as positive percentage (|DD| <= 15%)
}
```

2. **compute_composite_score(metrics_df: pd.DataFrame, weights: dict) -> pd.DataFrame**:
   - Input: DataFrame with columns: strategy_name, sharpe_mean, max_drawdown_mean, psr, turnover. Each row is one strategy (aggregated across folds, for a specific cost_scenario).
   - Normalize each metric to [0, 1] using min-max normalization:
     - `sharpe_norm`: higher is better. `(val - min) / (max - min + 1e-10)`
     - `max_dd_abs_norm`: less negative is better. `(|max| - |val|) / (|max| - |min| + 1e-10)` where max/min refer to the absolute values
     - `psr_norm`: already in [0, 1], use directly. Clip to [0, 1] for safety.
     - `turnover_inv_norm`: lower turnover is better. `(max - val) / (max - min + 1e-10)`
   - Handle edge case: if only 1 strategy, all normalized scores = 0.5 (no relative ranking possible)
   - Handle NaN: if psr is NaN, treat as 0.0 for scoring purposes (worst case)
   - Compute weighted sum: `composite = sum(weight_k * norm_k for k in weights)`
   - Return input DataFrame with additional columns: sharpe_norm, max_dd_abs_norm, psr_norm, turnover_inv_norm, composite_score

3. **apply_v1_gates(metrics_df: pd.DataFrame) -> pd.DataFrame**:
   - Add column `passes_v1_gates`: True if `sharpe_mean >= V1_GATES["min_sharpe"]` AND `abs(max_drawdown_mean) <= V1_GATES["max_drawdown_pct"] / 100`
   - Add column `gate_failures`: list of which gates failed (e.g. ["sharpe < 1.0"] or ["max_dd > 15%"])
   - Return augmented DataFrame

4. **rank_strategies(metrics_df: pd.DataFrame, weights: dict) -> pd.DataFrame**:
   - Call compute_composite_score, then sort by composite_score descending
   - Add `rank` column (1-based)
   - Return sorted DataFrame

5. **sensitivity_analysis(metrics_df: pd.DataFrame) -> pd.DataFrame**:
   - For each weight scheme in WEIGHT_SCHEMES:
     - Rank strategies using rank_strategies()
     - Record top-2 strategy names for this scheme
   - Return a summary DataFrame with columns: strategy_name, rank_balanced, rank_risk_focus, rank_quality_focus, rank_low_cost, n_times_top2, robust (True if n_times_top2 >= 3)
   - Log which strategies are robustly top-2

6. **blend_signals(position_series_list: list[pd.Series], weights: list[float] | None = None) -> pd.Series**:
   - Ensemble/blending contingency function (for use if no single strategy passes V1 gates)
   - Takes multiple position series (values in {-1, 0, 1}) and blends them
   - If weights is None, use equal weights
   - Return blended position: `np.sign(sum(w_i * pos_i))` -- majority vote
   - This is the simple path; IC-weighted blending would require more infrastructure

7. **load_bakeoff_metrics(engine, asset_id: int, tf: str, cv_method: str = "purged_kfold", cost_scenario: str | None = None) -> pd.DataFrame**:
   - Query strategy_bakeoff_results for the given asset/tf/cv_method
   - If cost_scenario is None, use the "baseline" scenario: spot_maker_10bps (middle ground)
   - Return DataFrame ready for scoring

All functions should work on DataFrames, not directly on the DB. The load function provides the DB-to-DataFrame bridge.
