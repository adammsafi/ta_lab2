<task type="auto">
  <name>Task 1: Build run_ic_sweep.py batch IC evaluation script</name>
  <files>src/ta_lab2/scripts/analysis/run_ic_sweep.py</files>
  <action>
Create `src/ta_lab2/scripts/analysis/run_ic_sweep.py` that orchestrates a broad IC sweep:

1. **Asset-TF discovery**: Query `asset_data_coverage` table to find all (asset_id, tf) pairs. Filter by minimum row count (default 500 bars, configurable via `--min-bars`). Log skipped sparse combinations.

2. **Feature column discovery**: Use `get_columns(engine, "public.cmc_features")` to discover all columns. Exclude non-feature columns using the existing `_NON_FEATURE_COLS` frozenset from `ic.py` (id, ts, tf, close, open, high, low, volume, ingested_at).

3. **Train window**: Use the full available date range per asset-TF as the train window (query min(ts), max(ts) from cmc_features for each asset-TF pair). This is the discovery phase -- we want max signal detection, not OOS validation (that's Plan 02's job).

4. **IC computation loop**: For each qualifying (asset_id, tf) pair:
   - Load features + close from cmc_features via `load_feature_series` pattern (direct SQL with `pd.read_sql`)
   - Call `batch_compute_ic(features_df, close, train_start, train_end, horizons=[1, 2, 3, 5, 10, 20, 60], return_types=["arith", "log"], tf_days_nominal=...)`
   - For BTC (id=1) and ETH (id=1027) on 1D: ALSO call `compute_ic_by_regime()` using `load_regimes_for_asset()` for trend_state and vol_state regime columns
   - Persist results via `save_ic_results(conn, rows, overwrite=True)` -- overwrite=True since we want fresh results

5. **Result summary**: After all computations, query cmc_ic_results to produce a feature ranking table:
   - Group by feature, compute mean |IC| and mean IC-IR across all asset-TF pairs at horizon=1, return_type='arith'
   - Sort by mean |IC-IR| descending
   - Print top-20 features to console
   - Save ranking CSV to `reports/bakeoff/feature_ic_ranking.csv`

6. **CLI interface**:
   - `python -m ta_lab2.scripts.analysis.run_ic_sweep --all` (full sweep)
   - `python -m ta_lab2.scripts.analysis.run_ic_sweep --assets 1 1027 --tf 1D` (targeted)
   - `--min-bars 500` (default; minimum bars for IC analysis)
   - `--regime` flag to include regime-conditional IC (default: only for BTC/ETH 1D)
   - `--dry-run` to list qualifying asset-TF pairs without computing
   - `--overwrite` flag (default True for bake-off; controls save_ic_results overwrite parameter)

7. **Connection management**: Use NullPool engine (project pattern for scripts). Single connection for all operations.

8. **DimTimeframe lookup**: Use `DimTimeframe` from `ta_lab2.time.dim_timeframe` to get `tf_days_nominal` for each TF. Fall back to 1 if lookup fails.

9. **Progress logging**: Log each (asset_id, tf) pair as it starts and completes with timing. Report total IC calculations completed at end.

CRITICAL PITFALLS TO AVOID:
- Use `pd.to_datetime(df["ts"], utc=True)` when loading data -- Windows mixed-tz issue
- Use `encoding='utf-8'` if reading any SQL files
- The `batch_compute_ic` function auto-excludes 'close' from feature_cols, so explicitly pass the discovered feature_cols list
- `save_ic_results` requires `horizon_days` key in each row dict; compute as `horizon * tf_days_nominal`
