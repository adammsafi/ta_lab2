  <name>Task 1: Build run_ic_sweep.py batch IC evaluation script (cmc_features + AMA tables)</name>
  <files>src/ta_lab2/scripts/analysis/run_ic_sweep.py</files>
  <action>
Create `src/ta_lab2/scripts/analysis/run_ic_sweep.py` that orchestrates a broad IC sweep across TWO data sources:

**Source A: cmc_features (bar-level features)**

1. **Asset-TF discovery**: Query `asset_data_coverage` table to find all (asset_id, tf) pairs. Filter by minimum row count (default 500 bars, configurable via `--min-bars`). Log skipped sparse combinations.

2. **Feature column discovery**: Use `get_columns(engine, "public.cmc_features")` to discover all columns. Exclude non-feature columns using the existing `_NON_FEATURE_COLS` frozenset from `ic.py` (id, ts, tf, close, open, high, low, volume, ingested_at). Also exclude metadata columns: alignment_source, tf_days, asset_class, updated_at, has_price_gap, has_outlier.

3. **Train window**: Use the full available date range per asset-TF as the train window (query min(ts), max(ts) from cmc_features for each asset-TF pair). This is the discovery phase -- we want max signal detection, not OOS validation (that's Plan 02's job).

4. **IC computation loop**: For each qualifying (asset_id, tf) pair:
   - Load features + close from cmc_features via `load_feature_series` pattern (direct SQL with `pd.read_sql`)
   - Call `batch_compute_ic(features_df, close, train_start, train_end, horizons=[1, 2, 3, 5, 10, 20, 60], return_types=["arith", "log"], tf_days_nominal=...)`
   - For BTC (id=1) and ETH (id=1027) on 1D: ALSO call `compute_ic_by_regime()` using `load_regimes_for_asset()` for trend_state and vol_state regime columns
   - Persist results via `save_ic_results(conn, rows, overwrite=True)` -- overwrite=True since we want fresh results

**Source B: AMA indicators (from cmc_ama_multi_tf_u)**

5. **AMA column discovery**: AMA data lives in `cmc_ama_multi_tf_u` with PK (id, ts, tf, indicator, params_hash, alignment_source). The evaluatable numeric columns are: `ama`, `d1`, `d2`, `d1_roll`, `d2_roll`, `er` (er is KAMA-only, NULL for DEMA/TEMA/HMA).

6. **AMA asset-TF-indicator combos**: Query `SELECT DISTINCT id, tf, indicator, params_hash FROM cmc_ama_multi_tf_u` to discover all available (asset_id, tf, indicator, params_hash) combinations. For each combo, count rows; skip if < `--min-bars`.

7. **AMA IC computation**: For each qualifying (asset_id, tf, indicator, params_hash) combo:
   - Load AMA columns + close price (close from cmc_features or cmc_price_bars_multi_tf for the same asset/tf/ts)
   - Build a features DataFrame with columns named as `{indicator}_{params_hash}_{col}` (e.g., `KAMA_abc123_ama`, `KAMA_abc123_d1`, `KAMA_abc123_er`) to disambiguate across indicator/params combos
   - Call `batch_compute_ic()` with these columns against close
   - Persist results. Use the disambiguated column name as the `feature` field in cmc_ic_results so AMA features can be ranked alongside cmc_features columns.

8. **AMA returns as features (optional, if time allows)**: The `cmc_returns_ama_multi_tf` table has pre-computed return columns (ret_arith_ama, ret_log_ama, z-scores). These can also be evaluated as features. If the sweep for Sources A+B completes within time budget, extend to include AMA return columns following the same pattern as step 7.

**Common infrastructure:**

9. **CLI interface**:
   - `python -m ta_lab2.scripts.analysis.run_ic_sweep --all` (full sweep: cmc_features + AMA)
   - `python -m ta_lab2.scripts.analysis.run_ic_sweep --assets 1 1027 --tf 1D` (targeted)
   - `--min-bars 500` (default; minimum bars for IC analysis)
   - `--regime` flag to include regime-conditional IC (default: only for BTC/ETH 1D)
   - `--dry-run` to list qualifying asset-TF pairs without computing
   - `--overwrite` flag (default True for bake-off; controls save_ic_results overwrite parameter)
   - `--skip-ama` to skip AMA table sweep (useful for faster iteration on cmc_features only)

10. **Result summary**: After all computations, query cmc_ic_results to produce a feature ranking table:
   - Group by feature, compute mean |IC| and mean IC-IR across all asset-TF pairs at horizon=1, return_type='arith'
   - Sort by mean |IC-IR| descending
   - Print top-20 features to console (including any AMA-derived features that rank)
   - Save ranking CSV to `reports/bakeoff/feature_ic_ranking.csv`

11. **Connection management**: Use NullPool engine (project pattern for scripts). Single connection for all operations.

12. **DimTimeframe lookup**: Use `DimTimeframe` from `ta_lab2.time.dim_timeframe` to get `tf_days_nominal` for each TF. Fall back to 1 if lookup fails.

13. **Progress logging**: Log each (asset_id, tf) pair and each AMA (asset_id, tf, indicator, params_hash) combo as it starts and completes with timing. Report total IC calculations completed at end.

CRITICAL PITFALLS TO AVOID:
- Use `pd.to_datetime(df["ts"], utc=True)` when loading data -- Windows mixed-tz issue
- Use `encoding='utf-8'` if reading any SQL files
- The `batch_compute_ic` function auto-excludes 'close' from feature_cols, so explicitly pass the discovered feature_cols list
- `save_ic_results` requires `horizon_days` key in each row dict; compute as `horizon * tf_days_nominal`
- AMA tables have `indicator` and `params_hash` columns in PK -- must handle the extra dimensionality when loading data
- `er` column is NULL for non-KAMA indicators -- filter out all-NULL columns before IC computation
- When joining AMA data with close prices, match on (id, ts, tf) and pick alignment_source='multi_tf' for consistency
