  <files>src/ta_lab2/scripts/analysis/select_strategies.py, reports/bakeoff/STRATEGY_SELECTION.md</files>
  <action>
Create `src/ta_lab2/scripts/analysis/select_strategies.py` that:

1. **Load all bakeoff results** from strategy_bakeoff_results table and the scoring CSVs from reports/bakeoff/.

2. **Apply selection rules** from CONTEXT.md:
   - Top-2 by composite score under the "balanced" weighting scheme
   - Must be robust: rank in top-2 under >= 3 of 4 weighting schemes (from sensitivity analysis)
   - If tied, prefer the strategy with higher PSR (statistical significance)
   - If no strategy passes V1 gates (Sharpe >= 1.0, Max DD <= 15%):
     a. Try blending top-2 signals using `blend_signals()` from composite_scorer.py
     b. Re-evaluate blended strategy metrics
     c. If blend still fails, select top-2 anyway and document that V1 gate was not met with recommendation to adjust

3. **Extract final parameters** for each selected strategy:
   - Use the parameter set that was used in the bake-off evaluation (from params_json in strategy_bakeoff_results)
   - These are FIXED parameters evaluated via walk-forward -- NOT re-optimized
   - Document that parameter selection methodology is walk-forward validation, not in-sample optimization

4. **Compute expected performance range** per strategy:
   - Mean +/- std of Sharpe across folds (from fold_metrics_json)
   - Mean +/- std of Max DD across folds
   - Best-case and worst-case fold performance
   - PSR confidence level

5. **Cost sensitivity analysis** per strategy:
   - How does Sharpe degrade from 5bps to 10bps to 20bps slippage?
   - Spot vs perps: how much does funding cost reduce Sharpe?
   - What is the break-even slippage level where Sharpe drops below 1.0?
