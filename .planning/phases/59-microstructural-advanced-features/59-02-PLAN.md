  <action>
Create `src/ta_lab2/features/microstructure.py` with all algorithm implementations from the research document. Organize into 5 sections with clear docstrings:

**Section 1: Fractional Differentiation (MICRO-01)**
- `ffd_weights(d: float, size: int = 1000, threshold: float = 1e-2) -> np.ndarray` -- compute FFD weights; threshold=1e-2 for practical ~12-weight window at d=0.4
- `frac_diff_ffd(series: np.ndarray, d: float = 0.4, threshold: float = 1e-2) -> np.ndarray` -- apply FFD to price series; returns NaN for first (width-1) bars
- `find_min_d(close: np.ndarray, min_d: float = 0.1, max_d: float = 1.0, n_steps: int = 20, adf_threshold: float = -2.9) -> float` -- find minimum d achieving stationarity via ADF test on log(close)

**Section 2: Liquidity Impact Measures (MICRO-02)**
- `amihud_lambda(close: np.ndarray, volume: np.ndarray, window: int = 20) -> np.ndarray` -- Amihud (2002) illiquidity: |return|/dollar_volume rolling mean
- `kyle_lambda(close: np.ndarray, volume: np.ndarray, window: int = 20) -> np.ndarray` -- Kyle (1985): OLS(delta_price ~ signed_volume) rolling
- `hasbrouck_lambda(close: np.ndarray, volume: np.ndarray, window: int = 20) -> np.ndarray` -- Hasbrouck (2009): OLS(delta_price ~ signed_sqrt(dollar_volume)) rolling

All three return np.ndarray of same length as close, with NaN for the first `window` bars. Use `scipy.stats.linregress` for OLS in Kyle and Hasbrouck (per research: numerically stable).

**Section 3: Rolling ADF / SADF Proxy (MICRO-03)**
- `_adf_tstat(log_prices: np.ndarray, lags: int = 1) -> float` -- ADF t-statistic via `scipy.linalg.lstsq`; return NaN if insufficient data (< lags+5 points)
- `rolling_adf(log_prices: np.ndarray, window: int = 63, lags: int = 1) -> np.ndarray` -- rolling window ADF t-stat; NaN for first `window` bars

Use `scipy.linalg.lstsq(X, y, check_finite=False)` for numerical stability. Use `np.linalg.pinv(X.T @ X)` for covariance matrix. Guard against near-zero variance with `max(cov[1,1], 1e-20)`.

**Section 4: Entropy Features (MICRO-04)**
- `quantile_encode(arr: np.ndarray, n_bins: int = 10) -> np.ndarray` -- map continuous values to discrete symbols via quantile binning; handle duplicates with np.unique on quantile edges
- `shannon_entropy(encoded: np.ndarray) -> float` -- Shannon entropy in nats: -sum(p * log(p + 1e-12))
- `lempel_ziv_complexity(s: list) -> int` -- count distinct sub-phrases (Lempel-Ziv 76 algorithm)
- `rolling_entropy(returns: np.ndarray, window: int = 50, n_bins: int = 10) -> tuple[np.ndarray, np.ndarray]` -- returns (shannon_vals, lz_vals) both shape (len(returns),); NaN for first `window` bars. LZ normalized by log2(window).

CRITICAL: Entropy must be computed on RETURN series, not price levels (per research pitfall #3).

**Section 5: Non-Linear Codependence (MICRO-05)**
- `distance_correlation(x: np.ndarray, y: np.ndarray) -> float` -- Szekely (2007) formula; builds n x n distance matrices; returns 0.0 if denominator is zero
- `pairwise_mi(x: np.ndarray, y: np.ndarray, n_neighbors: int = 3) -> float` -- mutual information via `sklearn.feature_selection.mutual_info_regression` with random_state=42
- `variation_of_information(x_encoded: np.ndarray, y_encoded: np.ndarray) -> float` -- H(X) + H(Y) - 2*I(X;Y) using `sklearn.metrics.mutual_info_score` for joint MI

Each function should have a clear docstring with: purpose, parameter descriptions, return type, and the paper reference (e.g., "Amihud (2002)").

Module-level docstring should explain: "Core mathematical implementations for microstructural features. All functions are pure (no DB/IO). Used by MicrostructureFeature and CodependenceFeature scripts."

Imports: numpy, scipy.linalg, scipy.stats (linregress), sklearn.feature_selection (mutual_info_regression), sklearn.metrics (mutual_info_score). All are already installed per research verification.

  <action>
Create `tests/unit/features/test_microstructure.py` with pytest tests covering all 5 feature classes. Use synthetic data (no DB required).

**FFD tests:**
- `test_ffd_weights_count`: ffd_weights(d=0.4, threshold=1e-2) returns ~12 weights (between 8 and 20)
- `test_ffd_weights_sum`: first weight is 1.0, subsequent weights are negative and decreasing in absolute value
- `test_frac_diff_ffd_output_shape`: output length equals input length; first (width-1) values are NaN
- `test_frac_diff_ffd_stationarity`: for a random walk input, FFD at d=0.4 produces lower autocorrelation than d=0 (raw prices)
- `test_find_min_d_returns_value`: for a random walk, returns d in (0.1, 1.0]; for stationary series (np.random.randn), returns d <= 0.2
- `test_ffd_threshold_pitfall`: threshold=1e-5 yields significantly more weights than threshold=1e-2 (verifying the research pitfall)

**Liquidity tests:**
- `test_amihud_lambda_shape`: output same length as close; first `window` values are NaN
- `test_amihud_lambda_positive`: for positive prices and volumes, non-NaN values are non-negative
- `test_kyle_lambda_shape`: output same length as close
- `test_hasbrouck_lambda_shape`: output same length as close
- `test_liquidity_handles_zero_volume`: no crash when volume contains zeros (guarded by +1 or similar)

**Rolling ADF tests:**
- `test_adf_tstat_stationary`: for np.random.randn(500), t-stat should be < -2.9 (reject unit root)
- `test_adf_tstat_random_walk`: for np.cumsum(np.random.randn(500)), t-stat should be > -2.9 (fail to reject)
- `test_rolling_adf_shape`: output same length as input; first `window` values NaN
- `test_rolling_adf_explosive`: for an exponentially growing series (np.exp(0.01 * np.arange(200))), rolling ADF should produce some values > 1.5

**Entropy tests:**
- `test_quantile_encode_range`: encoded values in [0, n_bins-1]
- `test_shannon_entropy_uniform`: uniform distribution has higher entropy than peaked distribution
- `test_shannon_entropy_degenerate`: single-value array has entropy near 0
- `test_lempel_ziv_complexity_repetitive`: [0,0,0,0,...] has lower complexity than random sequence
- `test_rolling_entropy_shape`: both outputs same length as input; first `window` values NaN
- `test_entropy_on_returns_vs_prices`: entropy on returns produces meaningful differentiation (per research pitfall)

**Codependence tests:**
- `test_distance_correlation_independent`: for truly independent x, y (np.random.randn with different seeds), dcor should be < 0.2
- `test_distance_correlation_nonlinear`: for y = x^2 + noise, dcor should be > 0.3 (Pearson would be near 0)
- `test_pairwise_mi_dependent`: for correlated series, MI > 0
- `test_pairwise_mi_independent`: for independent series, MI should be near 0 (< 0.1)
- `test_variation_of_information_self`: VI(x, x) should be near 0
- `test_variation_of_information_independent`: VI for independent encoded series should be > 0

Use `np.random.seed(42)` or `np.random.default_rng(42)` for reproducibility. Each test function should have a brief docstring.

Check if `tests/unit/features/` directory exists; create `__init__.py` if needed.
