Create `src/ta_lab2/scripts/features/microstructure_feature.py` following the VolatilityFeature pattern exactly.

**MicrostructureConfig (frozen dataclass extending FeatureConfig):**
- `feature_type: str = "microstructure"`
- `output_table: str = "cmc_features"` -- writes directly to cmc_features (like daily_features_view does), NOT to a separate table
- `null_strategy: str = "forward_fill"`
- `add_zscore: bool = False` -- microstructure features don't need z-scores in the base computation
- `ffd_threshold: float = 1e-2` -- FFD weight cutoff
- `liquidity_window: int = 20` -- window for Kyle/Amihud/Hasbrouck
- `adf_window: int = 63` -- rolling ADF window
- `entropy_window: int = 50` -- rolling entropy window
- `entropy_bins: int = 10` -- quantile bins for entropy encoding

**MicrostructureFeature(BaseFeature):**

`SOURCE_TABLE = "public.cmc_price_bars_multi_tf_u"`
`TS_COLUMN = "time_close"`

`load_source_data(ids, start, end)`:
- Query `cmc_price_bars_multi_tf_u` for (id, time_close AS ts, open, high, low, close, volume, tf, alignment_source, venue, venue_rank) WHERE id = ANY(:ids) AND tf = :tf AND alignment_source = :as
- Order by id, ts
- Return DataFrame

`compute_features(df_source)`:
- Process each (id, venue) group separately (same pattern as vol_feature.py)
- For each group, extract close and volume as numpy arrays
- Compute all features:
  1. **FFD (MICRO-01):** `find_min_d(close)` to get optimal d, then `frac_diff_ffd(np.log(close + 1e-10), d=d_opt)` for close_fracdiff. Store d_opt as close_fracdiff_d (constant for all bars in this refresh).
  2. **Liquidity (MICRO-02):** `kyle_lambda(close, volume, window)`, `amihud_lambda(close, volume, window)`, `hasbrouck_lambda(close, volume, window)`
  3. **Rolling ADF (MICRO-03):** `rolling_adf(np.log(close + 1e-10), window=63)` for sadf_stat. Derive `sadf_is_explosive = sadf_stat > 1.5`
  4. **Entropy (MICRO-04):** Compute log returns `np.diff(np.log(close + 1e-10))`, prepend NaN for alignment. Then `rolling_entropy(returns_valid, window, n_bins)` for entropy_shannon, entropy_lz.
- Concatenate results per group into output DataFrame with columns: id, ts, tf, alignment_source, venue, venue_rank, close_fracdiff, close_fracdiff_d, kyle_lambda, amihud_lambda, hasbrouck_lambda, sadf_stat, sadf_is_explosive, entropy_shannon, entropy_lz

`get_feature_columns()`:
- Return: `["close_fracdiff", "close_fracdiff_d", "kyle_lambda", "amihud_lambda", "hasbrouck_lambda", "sadf_stat", "sadf_is_explosive", "entropy_shannon", "entropy_lz"]`

`get_output_schema()`:
- Return schema matching the 9 columns + PK columns (id, ts, tf, alignment_source, venue, venue_rank)
- All DOUBLE PRECISION except sadf_is_explosive (BOOLEAN)

**Override `add_normalizations` and `add_outlier_flags`:**
- Override both to be no-ops (return df unchanged) -- these microstructure features are already informative without z-scores and don't need per-column outlier flags. This matches the TAFeature pattern which also overrides these methods.

**Override `write_to_db`:**
- Instead of the base class DELETE+INSERT for the full table, this feature writes to specific columns of cmc_features that is already populated by daily_features_view.
- Use UPDATE approach: For each row, UPDATE cmc_features SET close_fracdiff=:v1, close_fracdiff_d=:v2, ... WHERE id=:id AND ts=:ts AND tf=:tf AND alignment_source=:as.
- If UPDATE affected 0 rows (the bar doesn't exist in cmc_features yet), skip it -- microstructure features are supplemental to the main cmc_features rows.
- Use executemany with batches of 5000 for performance.
- Return count of rows updated.
