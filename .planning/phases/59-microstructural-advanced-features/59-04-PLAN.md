Create `src/ta_lab2/scripts/features/codependence_feature.py` as a standalone script (NOT a BaseFeature subclass, because the output is pairwise not per-bar).

**Data loading function:**
`load_return_series(engine, ids: list[int], tf: str, window_bars: int) -> dict[int, np.ndarray]`:
- Query `cmc_returns_bars_multi_tf_u` for each id: `SELECT id, "timestamp" AS ts, ret_log FROM cmc_returns_bars_multi_tf_u WHERE id = ANY(:ids) AND tf = :tf AND roll = FALSE ORDER BY id, ts DESC LIMIT :window_bars`
- Return dict mapping id -> np.ndarray of ret_log values (most recent `window_bars` bars, chronological order)
- Skip assets with fewer than `window_bars * 0.5` non-NULL return bars (insufficient data)

**Pair generation:**
`generate_pairs(ids: list[int]) -> list[tuple[int, int]]`:
- Generate all unique pairs (id_a, id_b) where id_a < id_b (avoid duplicates and self-pairs)
- For 50 assets, this is 1225 pairs

**Computation function:**
`compute_codependence(x: np.ndarray, y: np.ndarray) -> dict`:
- Align x and y to common non-NaN indices
- If fewer than 30 overlapping observations, return dict with all NaN
- Compute:
  1. `pearson_corr`: np.corrcoef(x, y)[0, 1]
  2. `distance_corr`: from microstructure.distance_correlation
  3. `mutual_info`: from microstructure.pairwise_mi
  4. `variation_of_info`: encode both with quantile_encode(10 bins), then microstructure.variation_of_information
- Return dict: {pearson_corr, distance_corr, mutual_info, variation_of_info, n_obs}

**Main computation loop:**
`refresh_codependence(engine, ids: list[int], tf: str = "1D", window_bars: int = 252) -> int`:
- Load return series for all ids
- Generate pairs
- Process pairs SEQUENTIALLY (per research: 34MB per pair for distance corr, avoid OOM)
- Log progress every 100 pairs: "{done}/{total} pairs computed"
- Collect results as list of dicts with: id_a, id_b, tf, window_bars, computed_at (use single datetime for all rows in this batch), pearson_corr, distance_corr, mutual_info, variation_of_info, n_obs
- Write to DB: INSERT all rows at once using pandas to_sql (append mode). DO NOT delete prior rows -- the computed_at in PK preserves historical snapshots.
- Return row count

**Write to DB:**
- Build DataFrame from results list
- `df.to_sql("cmc_codependence", engine, schema="public", if_exists="append", index=False, method="multi", chunksize=5000)`
- This is append-only; each refresh creates a new snapshot (identified by computed_at)

**CLI:**
- argparse with: --ids (comma-separated) or --all, --tf (default 1D), --window (default 252 bars), --log-level, --dry-run
- --dry-run: load data and print pair count, but don't compute or write
- Load engine from TARGET_DB_URL
- Load IDs (query `SELECT DISTINCT id FROM cmc_returns_bars_multi_tf_u WHERE tf = :tf`)
- Call refresh_codependence
- Print summary: "Computed {N} pairs for {M} assets, {R} rows written to cmc_codependence"
- Print timing: "Total: {T:.1f}s ({T/N:.3f}s per pair)"
