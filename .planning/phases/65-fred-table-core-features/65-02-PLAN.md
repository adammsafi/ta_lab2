**refresh_macro_features.py:**
CLI script following the same patterns as other refreshers in the project (run_all_bar_builders.py, etc.).

Key components:

1. **Watermark query:**
   ```python
   WARMUP_DAYS = 60  # covers 45d monthly ffill + 20d rolling window
   def get_compute_window(engine) -> tuple[str, str]:
       """Return (start_date, end_date) for the next compute run."""
       with engine.connect() as conn:
           max_date = conn.execute(text("SELECT MAX(date) FROM fred.fred_macro_features")).scalar()
       if max_date is None:
           start = "2000-01-01"  # full history on first run
       else:
           start = (pd.Timestamp(max_date) - pd.Timedelta(days=WARMUP_DAYS)).strftime("%Y-%m-%d")
       end = pd.Timestamp.now("UTC").strftime("%Y-%m-%d")
       return start, end
   ```

2. **FRED staleness check:**
   ```python
   def check_fred_staleness(engine, warn_hours: float = 48.0) -> tuple[bool, str]:
       """Check if fred.series_values has recent data."""
   ```
   Query `SELECT MAX(date) FROM fred.series_values WHERE series_id = 'DFF'`. Compare to now(). Return (is_fresh, message). Log WARNING if stale, but do NOT raise. Warn-and-continue per project convention.

3. **Upsert function:**
   ```python
   def upsert_macro_features(engine, df: pd.DataFrame) -> int:
   ```
   - Reset index so 'date' becomes a column
   - Convert date column to `datetime.date` objects (not pd.Timestamp -- psycopg2 binding)
   - Handle numpy scalar conversion with `_to_python()` helper per project gotcha (hasattr(v, 'item'))
   - Use temp table + INSERT...ON CONFLICT pattern:
     - `CREATE TEMP TABLE _macro_staging (LIKE fred.fred_macro_features INCLUDING DEFAULTS) ON COMMIT DROP`
     - Write DataFrame to _macro_staging via `to_sql(method="multi")`
     - `INSERT INTO fred.fred_macro_features SELECT * FROM _macro_staging ON CONFLICT (date) DO UPDATE SET` all columns except 'date', plus `ingested_at = now()`
   - Return row count

4. **Main function with argparse:**
   - `--dry-run`: Compute features but do not write to DB. Print row count and sample.
   - `--verbose`: Enable DEBUG logging.
   - `--full`: Ignore watermark, recompute full history from 2000-01-01.
   - `--start-date` / `--end-date`: Override compute window (useful for backfill).
   - Uses `from ta_lab2.io import get_engine` for DB connection.
   - Uses `from ta_lab2.macro.feature_computer import compute_macro_features` for computation.
   - Pipeline: check_fred_staleness -> get_compute_window -> compute_macro_features -> upsert_macro_features
   - Log: rows computed, rows upserted, date range, elapsed time.
   - Exit code 0 on success, 1 on failure.

5. **Numpy/pandas type safety:**
   Before writing to DB, convert DataFrame values to native Python types. NaN -> None for nullable columns. This is CRITICAL per the project gotcha about numpy scalars and psycopg2.
   ```python
   df = df.where(df.notna(), None)  # NaN -> None for SQL
   ```

Entry point: `if __name__ == "__main__": sys.exit(main())`
