End-to-end execution and verification. No new files created -- this task runs existing code and validates output.

1. **Run Alembic migration** (if not already applied):
   ```bash
   alembic upgrade head
   ```
   Verify table exists:
   ```sql
   SELECT column_name, data_type FROM information_schema.columns
   WHERE table_schema = 'fred' AND table_name = 'fred_macro_features'
   ORDER BY ordinal_position;
   ```
   Expect 27 columns.

2. **Run full macro feature computation:**
   ```bash
   python -m ta_lab2.scripts.macro.refresh_macro_features --full --verbose
   ```
   This should compute from 2000-01-01 to today, loading all 11 series from fred.series_values, forward-filling, computing derived features, and upserting.

3. **Verify FRED-01** (table exists with PK (date)):
   ```sql
   SELECT COUNT(*) as total_rows, MIN(date) as first_date, MAX(date) as last_date
   FROM fred.fred_macro_features;
   ```
   Expect thousands of rows spanning from earliest FRED data to today.

4. **Verify FRED-02** (forward-fill with source_freq provenance):
   ```sql
   -- Check that weekends have values (forward-filled, not gaps)
   SELECT date, walcl, source_freq_walcl, days_since_walcl
   FROM fred.fred_macro_features
   WHERE EXTRACT(DOW FROM date) IN (0, 6)  -- Sunday=0, Saturday=6
   ORDER BY date DESC
   LIMIT 5;
   ```
   Expect: non-NULL walcl on weekends, source_freq_walcl = 'weekly', days_since_walcl > 0.

5. **Verify FRED-03** (net liquidity):
   ```sql
   SELECT date, walcl, wtregen, rrpontsyd, net_liquidity,
          walcl - wtregen - rrpontsyd as expected_nl
   FROM fred.fred_macro_features
   WHERE walcl IS NOT NULL AND wtregen IS NOT NULL AND rrpontsyd IS NOT NULL
   ORDER BY date DESC
   LIMIT 5;
   ```
   Expect: net_liquidity = walcl - wtregen - rrpontsyd (exact match).

6. **Verify FRED-04** (rate spreads):
   ```sql
   SELECT date, dff, irstci01jpm156n, us_jp_rate_spread,
          dff - irstci01jpm156n as expected_spread
   FROM fred.fred_macro_features
   WHERE dff IS NOT NULL AND irstci01jpm156n IS NOT NULL
   ORDER BY date DESC
   LIMIT 5;
   ```
   Expect: us_jp_rate_spread matches DFF - IRSTCI01JPM156N.

7. **Verify FRED-05** (yield curve):
   ```sql
   SELECT date, t10y2y, yc_slope_change_5d
   FROM fred.fred_macro_features
   WHERE t10y2y IS NOT NULL AND yc_slope_change_5d IS NOT NULL
   ORDER BY date DESC
   LIMIT 5;
   ```
   Expect: yc_slope_change_5d is non-NULL for dates after the first 5 days of T10Y2Y data.

8. **Verify FRED-06** (VIX regime):
   ```sql
   SELECT vix_regime, COUNT(*) as days
   FROM fred.fred_macro_features
   WHERE vix_regime IS NOT NULL
   GROUP BY vix_regime
   ORDER BY days DESC;
   ```
   Expect: Three categories (calm, elevated, crisis) with calm having the most days.
   ```sql
   -- Spot-check thresholds
   SELECT date, vixcls, vix_regime
   FROM fred.fred_macro_features
   WHERE vixcls BETWEEN 14 AND 16
   ORDER BY date DESC
   LIMIT 5;
   ```
   Expect: vixcls < 15 -> calm, vixcls >= 15 -> elevated.

9. **Verify FRED-07** (dollar strength):
   ```sql
   SELECT date, dtwexbgs, dtwexbgs_5d_change, dtwexbgs_20d_change
   FROM fred.fred_macro_features
   WHERE dtwexbgs IS NOT NULL AND dtwexbgs_5d_change IS NOT NULL
   ORDER BY date DESC
   LIMIT 5;
   ```
   Expect: non-NULL values for all three columns on recent dates.

10. **Verify incremental mode works:**
    ```bash
    python -m ta_lab2.scripts.macro.refresh_macro_features --verbose
    ```
    Should run in seconds (only computing from watermark + warmup), not re-processing entire history. Check logs for "Computed N rows" where N is small (< 100 for recent dates).

11. **Verify idempotency:**
    Run the same command again. Row count in fred.fred_macro_features should not change. Upsert overwrites existing rows without duplicating.

If WTREGEN was skipped (user typed "skip wtregen"), verify that net_liquidity IS NULL but all other features are populated correctly. Log a note that WTREGEN can be added later and the pipeline re-run with --full.
