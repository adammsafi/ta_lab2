Create `src/ta_lab2/macro/` package with four files:

**fred_reader.py:**
- `SERIES_TO_LOAD`: list of 11 series IDs needed for FRED-03 through FRED-07: WALCL, WTREGEN, RRPONTSYD, DFF, IRSTCI01JPM156N, ECBDFR, DGS10, IRLTLT01JPM156N, T10Y2Y, VIXCLS, DTWEXBGS
- `load_series_wide(engine, series_ids, start_date, end_date) -> pd.DataFrame`: Query fred.series_values with `WHERE series_id = ANY(:ids) AND date >= :start AND date <= :end`, pivot to wide (index=date, columns=series_id, values=value), convert index to DatetimeIndex via `pd.to_datetime`, reindex to full calendar range with `pd.date_range(min, max, freq='D')`. Return wide DataFrame with uppercase column names matching FRED series IDs.
- Use `from ta_lab2.io import get_engine` for DB access pattern reference, but the function accepts an engine parameter (caller provides).
- Use `sqlalchemy.text()` for parameterized queries. Use `pd.read_sql()` for reading.
- CRITICAL: Use `.tolist()` or explicit UTC handling per project gotcha about pandas tz-aware series.

**forward_fill.py:**
- `FFILL_LIMITS` dict: weekly series (WALCL=10, WTREGEN=10), monthly series (IRSTCI01JPM156N=45, IRLTLT01JPM156N=45), daily series (DFF=5, DGS10=5, T10Y2Y=5, VIXCLS=5, DTWEXBGS=5, ECBDFR=5, RRPONTSYD=5)
- `SOURCE_FREQ` dict: maps series to "weekly"/"monthly"/"daily" strings
- `ffill_with_source_date(series, limit) -> tuple[pd.Series, pd.Series]`: Forward-fill a series and return (filled_values, source_observation_dates). Track which index date provided the original observation, then ffill the source_date series with the same limit. This enables days_since computation: `(row_date - source_date).days`.
- `forward_fill_with_limits(df_wide, tracked_series=None) -> tuple[pd.DataFrame, dict[str, pd.Series]]`: Apply per-column ffill with FFILL_LIMITS. For columns in `tracked_series` (default: ["WALCL", "WTREGEN"]), use ffill_with_source_date and return the source_date series in a dict. For all other columns, use plain `df[col].ffill(limit=N)`.

**feature_computer.py:**
- `compute_derived_features(df) -> pd.DataFrame`: Takes a forward-filled wide DataFrame (uppercase FRED column names) and adds derived columns:
  - FRED-03: `net_liquidity = WALCL - WTREGEN - RRPONTSYD`. If any input is NaN after ffill (component exhausted its fill limit), net_liquidity stays NaN for that row.
  - FRED-04: `us_jp_rate_spread = DFF - IRSTCI01JPM156N`, `us_ecb_rate_spread = DFF - ECBDFR`, `us_jp_10y_spread = DGS10 - IRLTLT01JPM156N`
  - FRED-05: `yc_slope_change_5d = T10Y2Y.diff(5)` (5-day delta of yield curve spread)
  - FRED-06: `vix_regime = pd.cut(VIXCLS, bins=[0, 15, 25, float('inf')], labels=['calm', 'elevated', 'crisis'])`. Cast to str. Set to None where VIXCLS is NaN.
  - FRED-07: `dtwexbgs_5d_change = DTWEXBGS.diff(5)`, `dtwexbgs_20d_change = DTWEXBGS.diff(20)`
  Returns the DataFrame with new columns added.

- `compute_macro_features(engine, start_date, end_date) -> pd.DataFrame`: Orchestrates the full pipeline:
  1. Call `load_series_wide(engine, SERIES_TO_LOAD, start_date, end_date)`
  2. Call `forward_fill_with_limits(df_wide, tracked_series=["WALCL", "WTREGEN"])`
  3. Call `compute_derived_features(df_filled)`
  4. Rename uppercase columns to lowercase (matching DB column names)
  5. Add source_freq provenance columns: source_freq_walcl="weekly", source_freq_wtregen="weekly", source_freq_irstci01jpm156n="monthly", source_freq_irltlt01jpm156n="monthly"
  6. Compute days_since_walcl and days_since_wtregen from source_date Series: `(df.index - source_dates).dt.days`
  7. Return DataFrame with index=date, columns matching fred.fred_macro_features schema

**__init__.py:**
- Import and re-export public API: `load_series_wide` from fred_reader, `forward_fill_with_limits` from forward_fill, `compute_macro_features` from feature_computer.

IMPORTANT: Handle missing WTREGEN gracefully. If WTREGEN has no rows in fred.series_values, the pivot will not produce a WTREGEN column. In compute_derived_features, if 'WTREGEN' not in df.columns, set net_liquidity to NaN and log a WARNING. Do NOT crash the entire pipeline.
