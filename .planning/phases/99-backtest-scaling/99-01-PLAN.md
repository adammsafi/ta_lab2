**Part A: mass_backtest_state table (BT-01)**
```sql
CREATE TABLE IF NOT EXISTS public.mass_backtest_state (
    id              SERIAL PRIMARY KEY,
    strategy_name   TEXT NOT NULL,
    asset_id        INTEGER NOT NULL,
    params_hash     TEXT NOT NULL,
    tf              TEXT NOT NULL,
    cost_bps        NUMERIC NOT NULL,
    status          TEXT NOT NULL DEFAULT 'pending',
    started_at      TIMESTAMPTZ,
    completed_at    TIMESTAMPTZ,
    error_msg       TEXT,
    created_at      TIMESTAMPTZ DEFAULT now(),
    UNIQUE(strategy_name, asset_id, params_hash, tf, cost_bps)
);
```
Add CHECK constraint: `status IN ('pending', 'running', 'done', 'error')`.
Add indexes: `idx_mass_backtest_state_status` on `(status)` and `idx_mass_backtest_state_strategy` on `(strategy_name, asset_id)`.
Add COMMENT on table explaining its purpose.

**Part B: backtest_trades partitioning (BT-02)**
This is a multi-step DDL change because backtest_trades has a UUID PK and FK to backtest_runs, and PostgreSQL requires the partition key be in the PK.

Steps in upgrade():
1. Check if backtest_trades has any rows. Store count for logging.
2. Add `strategy_name TEXT` column to backtest_trades (allow NULL initially).
3. Populate `strategy_name` from `backtest_runs.signal_type` via UPDATE...FROM JOIN:
   ```sql
   UPDATE public.backtest_trades t
   SET strategy_name = r.signal_type
   FROM public.backtest_runs r
   WHERE t.run_id = r.run_id AND t.strategy_name IS NULL
   ```
4. For any remaining NULL strategy_name rows (orphans), set to 'unknown'.
5. ALTER COLUMN strategy_name SET NOT NULL.
6. Rename existing table: `ALTER TABLE public.backtest_trades RENAME TO backtest_trades_old`.
7. Create new partitioned table with SAME columns plus strategy_name:
   ```sql
   CREATE TABLE public.backtest_trades (
       trade_id        UUID NOT NULL DEFAULT gen_random_uuid(),
       run_id          UUID NOT NULL,
       strategy_name   TEXT NOT NULL,
       entry_ts        TIMESTAMPTZ NOT NULL,
       entry_price     NUMERIC NOT NULL,
       exit_ts         TIMESTAMPTZ,
       exit_price      NUMERIC,
       direction       TEXT NOT NULL,
       size            NUMERIC,
       pnl_pct         NUMERIC,
       pnl_dollars     NUMERIC,
       fees_paid       NUMERIC,
       slippage_cost   NUMERIC,
       created_at      TIMESTAMPTZ DEFAULT now(),
       PRIMARY KEY (trade_id, strategy_name),
       CHECK (direction IN ('long', 'short'))
   ) PARTITION BY LIST (strategy_name);
   ```
   NOTE: The FK to backtest_runs is deliberately dropped from the partitioned table. PostgreSQL cannot enforce cross-partition FK checks with the old PK structure. The run_id column is kept for application-level linking.
8. Create default partition: `CREATE TABLE public.backtest_trades_default PARTITION OF public.backtest_trades DEFAULT`.
9. Create named partitions for known strategies:
   ```sql
   CREATE TABLE public.backtest_trades_ema_trend PARTITION OF public.backtest_trades FOR VALUES IN ('ema_trend');
   CREATE TABLE public.backtest_trades_rsi_mean_revert PARTITION OF public.backtest_trades FOR VALUES IN ('rsi_mean_revert');
   CREATE TABLE public.backtest_trades_breakout_atr PARTITION OF public.backtest_trades FOR VALUES IN ('breakout_atr');
   CREATE TABLE public.backtest_trades_macd_crossover PARTITION OF public.backtest_trades FOR VALUES IN ('macd_crossover');
   CREATE TABLE public.backtest_trades_ama_momentum PARTITION OF public.backtest_trades FOR VALUES IN ('ama_momentum');
   CREATE TABLE public.backtest_trades_ama_mean_reversion PARTITION OF public.backtest_trades FOR VALUES IN ('ama_mean_reversion');
   CREATE TABLE public.backtest_trades_ama_regime_conditional PARTITION OF public.backtest_trades FOR VALUES IN ('ama_regime_conditional');
   CREATE TABLE public.backtest_trades_ctf_threshold PARTITION OF public.backtest_trades FOR VALUES IN ('ctf_threshold');
   ```
10. Re-insert data from old table: `INSERT INTO public.backtest_trades SELECT * FROM public.backtest_trades_old`.
11. Recreate indexes on the partitioned table:
    ```sql
    CREATE INDEX IF NOT EXISTS idx_backtest_trades_run ON public.backtest_trades(run_id);
    CREATE INDEX IF NOT EXISTS idx_backtest_trades_entry_ts ON public.backtest_trades(entry_ts);
    CREATE INDEX IF NOT EXISTS idx_backtest_trades_strategy ON public.backtest_trades(strategy_name);
    ```
12. Add COMMENT on partitioned table.

For downgrade():
1. Drop mass_backtest_state table.
2. Drop new partitioned backtest_trades.
3. Rename backtest_trades_old back to backtest_trades.
4. Drop strategy_name column from backtest_trades.

Use `op.execute()` for raw SQL (not op.create_table) because Alembic's ORM doesn't natively support PARTITION BY LIST. Wrap entire upgrade in a transaction. Use `encoding='utf-8'` convention for any string operations.

IMPORTANT: Do NOT add mc_sharpe_lo/hi/median columns to backtest_metrics -- they already exist from migration c3d4e5f6a1b2. Research confirms this (Pitfall 4).
