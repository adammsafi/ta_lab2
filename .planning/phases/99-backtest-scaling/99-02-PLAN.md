**A. Create `src/ta_lab2/signals/ctf_threshold.py`:**

A new signal adapter following the exact same pattern as `ema_trend.py`, `rsi_mean_revert.py`, etc.

The `make_signals` function signature MUST be:
```python
def make_signals(
    df: pd.DataFrame,
    **params
) -> tuple[pd.Series, pd.Series, Optional[pd.Series]]:
```

Parameters (extracted from `**params`):
- `feature_col: str` -- name of the CTF feature column in df (e.g. "ret_arith_365d_divergence")
- `entry_threshold: float` -- threshold above which to enter (for long), default 0.0
- `exit_threshold: float` -- threshold below which to exit (for long), default 0.0
- `direction: str` -- "long" or "short", default "long"
- `holding_bars: int` -- optional max holding period, default 0 (no limit). When > 0, force exit after holding_bars bars.

Logic:
- For "long": entry when `feature > entry_threshold`, exit when `feature < exit_threshold`
- For "short": entry when `feature < entry_threshold`, exit when `feature > exit_threshold`
- If holding_bars > 0: overlay a time-based exit (set exits=True at holding_bars after each entry)
- Handle missing feature_col gracefully: raise KeyError with clear message listing available CTF columns
- fillna(0.0) on the feature column before comparison to avoid NaN propagation

Include module docstring explaining this is a Phase 99 CTF threshold crossover signal adapter.
Include `__all__ = ["make_signals"]`.
