Create `src/ta_lab2/dashboard/queries/asset_stats.py` following the exact patterns from queries/pipeline.py and queries/research.py.

**Functions (all with @st.cache_data(ttl=300) decorator):**

1. `load_asset_stats_latest(tf: str = "1D") -> pd.DataFrame`:
   - Query cmc_asset_stats for the latest ts per (id, tf)
   - Use: `SELECT DISTINCT ON (id) * FROM cmc_asset_stats WHERE tf = :tf ORDER BY id, ts DESC`
   - Join with dim_assets for symbol names
   - Return DataFrame with symbol as index, all stat columns

2. `load_corr_latest(tf: str = "1D", window: int = 90) -> pd.DataFrame`:
   - Query cmc_corr_latest materialized view: `SELECT * FROM cmc_corr_latest WHERE tf = :tf AND window = :window`
   - Join with dim_assets (twice, for id_a and id_b) to get symbol names
   - Return DataFrame with columns: symbol_a, symbol_b, pearson_r, spearman_r, pearson_p, spearman_p, n_obs

3. `load_asset_stats_timeseries(asset_id: int, tf: str = "1D") -> pd.DataFrame`:
   - Query full time series: `SELECT * FROM cmc_asset_stats WHERE id = :id AND tf = :tf ORDER BY ts`
   - Return DataFrame indexed by ts

4. `load_asset_symbols() -> dict[int, str]`:
   - Query dim_assets for id -> symbol mapping
   - Cached separately (long TTL)

All functions use `from ta_lab2.dashboard.db import get_engine` for NullPool engine.
Use `import streamlit as st` lazily inside function body if needed (consistent with charts.py lazy import pattern for non-Streamlit contexts). Actually for query modules, direct import at top is fine since they are only used in dashboard context.
