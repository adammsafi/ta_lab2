  <action>
Run `alembic revision -m "asset_stats_and_correlation_tables"` from the project root to generate the migration file. Then implement upgrade() and downgrade() functions.

The migration creates 5 objects in upgrade():

1. **cmc_asset_stats** table (wide format):
   - PK: (id INTEGER, ts TIMESTAMPTZ, tf TEXT)
   - Per-window columns for windows 30, 60, 90, 252 (all NUMERIC, nullable):
     - mean_ret_{W}, std_ret_{W}, sharpe_raw_{W}, sharpe_ann_{W}
     - skew_{W}, kurt_fisher_{W}, kurt_pearson_{W}, max_dd_window_{W}
   - That is 8 columns x 4 windows = 32 stat columns
   - Additional columns: max_dd_from_ath (NUMERIC, nullable), rf_rate (NUMERIC, nullable)
   - ingested_at (TIMESTAMPTZ, server_default=now(), NOT NULL)
   - Index: idx_asset_stats_id_tf_ts on (id, tf, ts) for time-series queries

2. **cmc_cross_asset_corr** table (long format):
   - PK: (id_a INTEGER, id_b INTEGER, ts TIMESTAMPTZ, tf TEXT, window INTEGER)
   - Columns: pearson_r, pearson_p, spearman_r, spearman_p (all NUMERIC nullable), n_obs (INTEGER nullable)
   - ingested_at (TIMESTAMPTZ, server_default=now(), NOT NULL)
   - CHECK constraint: chk_corr_pair_order CHECK (id_a < id_b)
   - Index: idx_corr_pair_tf_window_ts on (id_a, id_b, tf, window, ts)

3. **cmc_asset_stats_state** table:
   - PK: (id INTEGER, tf TEXT)
   - last_timestamp TIMESTAMPTZ nullable
   - updated_at TIMESTAMPTZ server_default=now() NOT NULL

4. **cmc_cross_asset_corr_state** table:
   - PK: (id_a INTEGER, id_b INTEGER, tf TEXT)
   - last_timestamp TIMESTAMPTZ nullable
   - updated_at TIMESTAMPTZ server_default=now() NOT NULL

5. **cmc_corr_latest** materialized view:
   ```sql
   CREATE MATERIALIZED VIEW public.cmc_corr_latest AS
   SELECT DISTINCT ON (id_a, id_b, tf, window)
       id_a, id_b, ts, tf, window,
       pearson_r, pearson_p, spearman_r, spearman_p, n_obs
   FROM public.cmc_cross_asset_corr
   ORDER BY id_a, id_b, tf, window, ts DESC
   ```
   - Unique index: idx_corr_latest_pk on (id_a, id_b, tf, window)
   - Use op.execute() for the CREATE MATERIALIZED VIEW since Alembic has no native op for it

downgrade() drops all 5 objects in reverse order (materialized view first, then tables).

CRITICAL: Set down_revision = "6f82e9117c58" (current Alembic head). Use schema="public" on all op.create_table calls. No UTF-8 box-drawing characters in comments (Windows cp1252 pitfall). Follow exact pattern from existing migrations (6f82e9117c58, c3b718c2d088).
