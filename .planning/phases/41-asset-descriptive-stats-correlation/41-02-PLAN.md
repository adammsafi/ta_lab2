**Constants:**
- WINDOWS = [30, 60, 90, 252]
- DEFAULT_RF = 0.0
- TABLE_NAME = "cmc_asset_stats"
- STATE_TABLE = "cmc_asset_stats_state"
- SOURCE_TABLE = "cmc_returns_bars_multi_tf"

**Core computation function: `compute_asset_stats(df, tf_days, windows, rf)`**
- Input: DataFrame with at least `ret_arith` column, sorted by ts ascending
- Uses pandas rolling with min_periods=window (NULL until full window, NO partial windows)
- For each window W:
  - mean_ret_{W} = rolling(W).mean()
  - std_ret_{W} = rolling(W).std(ddof=1)
  - sharpe_raw_{W} = (mean - rf) / std (use .replace(0, np.nan) on std to avoid div-by-zero)
  - sharpe_ann_{W} = sharpe_raw * sqrt(365.0 / tf_days) -- use tf_days_nominal from DimTimeframe
  - skew_{W} = rolling(W).skew() -- pandas native, bias-corrected Fisher skewness
  - kurt_fisher_{W} = rolling(W).kurt() -- CRITICAL: pandas .kurt() returns Fisher/excess (normal=0)
  - kurt_pearson_{W} = kurt_fisher_{W} + 3.0 -- Pearson (normal=3), consistent with psr.py
  - max_dd_window_{W} = rolling max drawdown within trailing W bars
- max_dd_from_ath = expanding drawdown from all-time-high (not windowed)
- rf_rate = rf constant for all rows

**Rolling max drawdown helper: `_rolling_max_drawdown(ret_series, window)`**
- Uses rolling(window, min_periods=window).apply(_mdd, raw=True)
- _mdd(arr): cumprod(1 + arr), maximum.accumulate for peak, eq/peak - 1.0, return min
- Returns negative fractions (consistent with project convention in performance.py)

**Current drawdown from ATH: `_current_drawdown_from_ath(ret_series)`**
- eq = (1 + ret_arith.fillna(0)).cumprod()
- ath = eq.cummax()
- return eq / ath - 1.0

**Watermark-based incremental refresh:**
- Read last_timestamp from cmc_asset_stats_state per (id, tf)
- Load ret_arith from cmc_returns_bars_multi_tf WHERE id=:id AND tf=:tf AND roll=FALSE AND ts > (last_ts - lookback)
  - lookback = max(WINDOWS) = 252 bars before last_ts (to recompute correctly)
  - If no state exists (first run), load entire history
- Compute stats on loaded data
- DELETE existing rows in cmc_asset_stats for (id, tf) WHERE ts >= first_new_ts
- INSERT computed rows
- UPDATE cmc_asset_stats_state with new last_timestamp

**Worker function for multiprocessing:**
- `_worker(task, db_url)` processes one (id, tf) pair
- Creates NullPool engine per worker (required for multiprocessing)
- Returns (id, tf, n_rows, success, error_msg) tuple

**CLI (argparse):**
- `--ids` (comma-separated or "all", default "all")
- `--tf` (single TF filter, optional -- runs all TFs if omitted)
- `--windows` (comma-separated window sizes, default "30,60,90,252")
- `--rf` (risk-free rate, default 0.0)
- `--full-rebuild` (ignore watermarks, recompute all)
- `--dry-run` (show what would run without executing)
- `--continue-on-error` (don't stop on individual (id,tf) failures)
- `--workers` (number of parallel workers, default 4)
- `--db-url` (override DB URL)

**Main flow:**
1. Parse args, resolve DB URL
2. Load asset IDs from dim_assets (or filter by --ids)
3. Load TF list from dim_timeframe (or filter by --tf)
4. Build task list: all (id, tf) combinations
5. Run tasks via multiprocessing.Pool with _worker
6. Print summary (total rows inserted, failures)

Use `from ta_lab2.scripts.refresh_utils import resolve_db_url, parse_ids` for DB URL and ID parsing.
