The existing run_all_stats_runners.py has two registration mechanisms:

1. **`STATS_TABLES` list** (line 51-58): Table names queried for aggregate PASS/WARN/FAIL status via `query_stats_status()`. These are stats OUTPUT tables (e.g., `price_bars_multi_tf_stats`, `cmc_features_stats`) -- they contain rows with a `status` column and `checked_at` timestamp written by individual stats runner scripts.

2. **`ALL_STATS_SCRIPTS` list** (line 83-120): `StatsScript` dataclass entries defining which runner scripts to invoke as subprocesses. Each entry has a `module` path to a Python module with a `main()` that runs quality checks and writes results to a stats output table.

The desc stats tables (`cmc_asset_stats`, `cmc_cross_asset_corr`) do NOT have existing stats runner scripts yet. To register them properly:

**Step A: Create a minimal desc stats quality runner.**

The existing stats runners follow a pattern: they query source data tables, compute quality metrics (row counts, null fractions, value bounds, freshness), and write PASS/WARN/FAIL rows to a `*_stats` output table. However, creating a full stats runner with its own output table is heavyweight for Phase 41.

Instead, add lightweight inline quality checks directly to run_all_stats_runners.py as a new function:

```python
def check_desc_stats_quality(engine) -> list[dict]:
    """
    Run lightweight quality checks on desc stats tables.
    Returns list of {table, check, status, detail} dicts.
    """
```

Implement these checks using direct SQL queries:
- `cmc_asset_stats`: row count > 0 (WARN if empty), NULL fraction for std_ret_90 < 50% (WARN if high), no mean_ret_252 values with |value| > 0.5 (WARN if implausible)
- `cmc_cross_asset_corr`: row count > 0 (WARN if empty), zero rows where id_a >= id_b (FAIL if any -- pair order violation), all pearson_r between -1 and 1 (FAIL if out of bounds), n_obs consistency (WARN if any NULL n_obs)

**Step B: Call the check function after all stats scripts complete.**

In `run_all_stats()`  (around line 376, after the "Query DB for aggregate PASS/WARN/FAIL status" section), add:

```python
# Run desc stats quality checks (lightweight, inline)
desc_stats_checks = []
if not dry_run:
    try:
        desc_stats_checks = check_desc_stats_quality(engine)
        # Print results
        for check in desc_stats_checks:
            status_icon = "[OK]" if check["status"] == "PASS" else f"[{check['status']}]"
            print(f"  {status_icon} {check['table']}.{check['check']}: {check['detail']}")
    except Exception as e:
        logger.warning(f"Desc stats quality checks failed: {e}")
```

Incorporate desc stats check results into the overall status determination: if any check has status "FAIL", set overall_status to "FAIL". If any check has "WARN", escalate to "WARN" if current status is "PASS".
