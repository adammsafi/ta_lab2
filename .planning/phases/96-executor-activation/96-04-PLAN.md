  <action>
    Create `src/ta_lab2/scripts/executor/run_parity_report.py` — a standalone CLI that computes and stores live-vs-backtest parity per strategy.

    **CLI interface:**
    ```
    python -m ta_lab2.scripts.executor.run_parity_report
    python -m ta_lab2.scripts.executor.run_parity_report --window 30
    python -m ta_lab2.scripts.executor.run_parity_report --strategy ema_trend_17_77_paper_v1
    python -m ta_lab2.scripts.executor.run_parity_report --dry-run
    ```
    Arguments:
    - `--window` (default: 30) — rolling window in days for Sharpe computation
    - `--strategy` (optional) — filter to a single strategy config_name
    - `--dry-run` — compute and print but do not write to DB
    - `--db-url` (optional) — override DB URL
    - `--verbose` — debug logging

    **Computation logic:**

    For each active strategy in `dim_executor_config`:

    1. **Fill-to-fill Sharpe** (execution quality):
       - Query round-trip returns from `fills` + `orders` for this strategy_id within the window:
         ```sql
         WITH closed_positions AS (
             SELECT o.asset_id, o.strategy_id,
                    f.fill_price, f.fill_qty, o.side, f.filled_at
             FROM fills f
             JOIN orders o ON f.order_id = o.order_id
             WHERE o.strategy_id = :strategy_id
               AND f.filled_at >= :window_start
         )
         ```
       - Group by asset_id to find entry/exit pairs (buy then sell for long, sell then buy for short)
       - Compute return per round-trip: `(exit_price - entry_price) / entry_price`
       - Annualized Sharpe: `mean(returns) / std(returns) * sqrt(252 / mean_hold_days)` (use numpy)
       - If fewer than 2 round-trips: set `live_sharpe_fill = NULL`

    2. **Mark-to-market daily Sharpe** (portfolio-level):
       - For each day in the window, compute portfolio daily PnL:
         - Look up open positions from `positions` table for this strategy_id
         - Get daily close from `price_bars_multi_tf_u` (1D, alignment_source='multi_tf')
         - daily_pnl_per_asset = (today_close - prev_close) / prev_close * position_value
         - Sum across assets for total daily portfolio return
       - Annualized Sharpe: `mean(daily_returns) / std(daily_returns) * sqrt(252)`
       - If fewer than 5 MTM days: set `live_sharpe_mtm = NULL`

    3. **Backtest Sharpe** reference (IMPORTANT: use cross-validated aggregate, NOT single-asset):
       - Query `strategy_bakeoff_results` for the cross-validated aggregate Sharpe matching this strategy's signal_type:
         ```sql
         SELECT sharpe_mean FROM strategy_bakeoff_results
         WHERE signal_type = :signal_type
         ORDER BY created_at DESC LIMIT 1
         ```
       - `strategy_bakeoff_results.sharpe_mean` is the cross-sectional mean Sharpe from the bake-off, which is the correct multi-asset reference for parity comparison.
       - Do NOT use `backtest_runs` here -- that table stores per-asset single-run rows which are meaningless noise for multi-asset strategies.
       - If no bakeoff result found: set `bt_sharpe = NULL`

    4. **Compute ratios:** `ratio_fill = live_sharpe_fill / bt_sharpe`, `ratio_mtm = live_sharpe_mtm / bt_sharpe` (NULL if either is NULL)

    5. **Persist:** INSERT into `strategy_parity` with all computed fields

    **Output format** (printed to console):
    ```
    ============================================================
    STRATEGY PARITY REPORT (30-day window)
    ============================================================
    Strategy: ema_trend_17_77_paper_v1
      Fill-to-fill Sharpe : 1.23 (BT: 1.71, ratio: 0.72)
      MTM daily Sharpe    : 0.98 (BT: 1.71, ratio: 0.57)
      Fills in window     : 14
      MTM days            : 28
    ---
    Strategy: rsi_mean_revert_14_paper_v1
      Fill-to-fill Sharpe : n/a (< 2 round-trips)
      ...
    ============================================================
    ```

    **Edge cases:**
    - No fills yet (fresh executor): print "No fills in window" and INSERT row with NULL Sharpe values
    - No backtest reference: print "No backtest reference" and set bt_sharpe=NULL
    - Division by zero (std=0): set Sharpe=NULL
    - Use `NullPool` for engine (project convention)
    - ASCII-only output (no Unicode box characters — Windows compatibility)

  <action>
    Create `src/ta_lab2/scripts/executor/run_pnl_attribution.py` — a standalone CLI that computes PnL attribution separating alpha from multi-asset-class beta.

    **CLI interface:**
    ```
    python -m ta_lab2.scripts.executor.run_pnl_attribution
    python -m ta_lab2.scripts.executor.run_pnl_attribution --period 30
    python -m ta_lab2.scripts.executor.run_pnl_attribution --start 2026-03-01 --end 2026-03-30
    python -m ta_lab2.scripts.executor.run_pnl_attribution --dry-run
    ```
    Arguments:
    - `--period` (default: 30) — lookback days from today
    - `--start` / `--end` (optional) — explicit date range (overrides --period)
    - `--dry-run` — compute and print but do not write to DB
    - `--db-url` (optional) — override DB URL
    - `--verbose` — debug logging

    **Computation logic:**

    1. **Classify positions by asset class:**
       - Query all fills in the period joined with asset metadata:
         ```sql
         SELECT DISTINCT o.asset_id, o.strategy_id
         FROM fills f
         JOIN orders o ON f.order_id = o.order_id
         WHERE f.filled_at BETWEEN :start AND :end
         ```
       - Asset class mapping (hardcoded for Phase 96, extensible later):
         - Check `hl_assets` table: if asset has `asset_type='perp'` -> class='perp'
         - Otherwise: class='crypto' (default for all CMC + HL spot)
         - 'equity' class reserved for Phase 97 (FRED macro indices)

    2. **For each asset class, compute beta vs benchmark:**

       Per-asset-class benchmarks (from CONTEXT.md):
       - 'crypto': benchmark = BTC (id=1 in `price_bars_multi_tf_u`)
       - 'perp': benchmark = underlying spot price (same id, but if unavailable use BTC)
       - 'all' (blended): weighted combination of per-class benchmarks

       Computation:
       - Load daily returns for all positions: daily_close from `price_bars_multi_tf_u` for each asset_id
       - Load benchmark daily returns: daily_close for benchmark asset
       - Position-weighted portfolio daily return for this asset class
       - Beta: `np.cov(portfolio_returns, benchmark_returns)[0,1] / np.var(benchmark_returns)` (standard OLS beta, 2 lines — do NOT hand-roll a regression)
       - `beta_pnl = beta * benchmark_total_return * portfolio_start_value`
       - `alpha_pnl = total_pnl - beta_pnl`
       - `sharpe_alpha = mean(alpha_daily_returns) / std(alpha_daily_returns) * sqrt(252)` where `alpha_daily_returns = portfolio_returns - beta * benchmark_returns`

    3. **Persist:** INSERT into `pnl_attribution` per asset class (crypto, perp, all):
       ```sql
       INSERT INTO pnl_attribution (period_start, period_end, asset_class, benchmark,
                                     total_pnl, beta_pnl, alpha_pnl, beta, sharpe_alpha, n_positions)
       VALUES (:start, :end, :asset_class, :benchmark, ...)
       ```

    **Output format** (printed to console):
    ```
    ============================================================
    PNL ATTRIBUTION REPORT (2026-03-01 to 2026-03-30)
    ============================================================
    Asset Class: crypto (benchmark: BTC)
      Positions    : 45
      Total PnL    : $1,234.56
      Beta         : 0.82
      Beta PnL     : $987.65
      Alpha PnL    : $246.91
      Alpha Sharpe : 0.45
    ---
    Asset Class: perp (benchmark: BTC)
      Positions    : 12
      Total PnL    : -$123.45
      Beta         : 1.15
      Beta PnL     : -$89.23
      Alpha PnL    : -$34.22
      Alpha Sharpe : -0.12
    ---
    BLENDED PORTFOLIO
      Total PnL    : $1,111.11
      Beta PnL     : $898.42
      Alpha PnL    : $212.69
      Alpha Sharpe : 0.38
    ============================================================
    ```

    **Edge cases:**
    - No fills in period: print "No fills in period" and skip (do not INSERT empty row)
    - Only one asset class has positions: compute for that class + 'all' = same values
    - Zero variance in benchmark: set beta=NULL, alpha_pnl=total_pnl
    - No benchmark price data: skip that asset class with warning
    - Use `NullPool` for engine (project convention)
    - ASCII-only output (no Unicode box characters)
    - Use `numpy` for beta calculation (already in project deps)
