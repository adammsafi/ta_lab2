  <name>Task 1: Add AMA tables to _ALLOWED_TABLES and extend _load_inputs with filters support</name>
  <files>src/ta_lab2/experiments/runner.py</files>
  <action>
1. Add 4 AMA tables to the `_ALLOWED_TABLES` frozenset (lines 42-57):
   - cmc_ama_multi_tf
   - cmc_ama_multi_tf_u
   - cmc_returns_ama_multi_tf
   - cmc_returns_ama_multi_tf_u

2. Do NOT add AMA tables to `_TABLES_WITHOUT_TF` -- they DO have a tf column.

3. Extend the `_load_inputs` method to support an optional `filters` dict on each
   input entry. The YAML spec allows:
   ```yaml
   inputs:
     - table: cmc_ama_multi_tf_u
       columns: [er]
       filters:
         indicator: kama
         params_hash: d47fe5cc
   ```

   In the `_load_inputs` method, after the existing table/columns validation loop,
   read `inp.get("filters", {})`. For each key-value pair in filters:
   - Validate the key is a safe identifier (reuse `_is_safe_identifier`).
   - Append ` AND {key} = :{key}` to the SQL string.
   - Add the value to the params dict.

   This keeps the existing behavior for all current features (no filters key = no
   change) while enabling AMA tables that require indicator and params_hash filtering.

   Implementation detail: Insert the filter handling AFTER the existing `if table in
   _TABLES_WITHOUT_TF:` block but BEFORE `pd.read_sql`. Build a list of extra
   WHERE clauses from filters, then append them to sql_str. Example:

   ```python
   # Handle optional filters (e.g., indicator, params_hash for AMA tables)
   filters = inp.get("filters", {})
   for fkey, fval in filters.items():
       if not _is_safe_identifier(fkey):
           raise ValueError(
               f"Filter key '{fkey}' contains unsafe characters. "
               "Only alphanumeric and underscore are allowed."
           )
       sql_str += f' AND "{fkey}" = :filter_{fkey}'
       params[f"filter_{fkey}"] = fval
   ```

   Note: Use `filter_` prefix on param names to avoid collisions with existing
   params (id, tf, start, end). Quote filter keys with double-quotes in SQL to
   handle any reserved words.
  </action>

  </verify>
