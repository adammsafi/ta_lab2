**Step A: Create `src/ta_lab2/dashboard/queries/experiments.py`**

Follow the exact pattern of queries/research.py. Use `@st.cache_data(ttl=300)` and
`_engine` parameter (underscore-prefixed so st.cache_data skips engine hashing).
NullPool engine provided by caller.

Functions to create:

1. `load_experiment_results(_engine, feature_name: str | None = None) -> pd.DataFrame`
   Query cmc_feature_experiments. If feature_name is None, load all rows. Otherwise
   filter by feature_name. Return columns: feature_name, asset_id, tf, horizon,
   return_type, ic, ic_p_value, ic_p_value_bh, ic_ir, n_obs, wall_clock_seconds,
   train_start, train_end, computed_at. Order by feature_name, asset_id, horizon.

2. `load_experiment_feature_names(_engine) -> list[str]`
   Return DISTINCT feature_name from cmc_feature_experiments ORDER BY feature_name.

3. `load_experiment_summary(_engine) -> pd.DataFrame`
   Aggregate query: for each feature_name, compute count of experiments, count of
   BH-significant results (ic_p_value_bh < 0.05), mean absolute IC, and latest
   computed_at. Return columns: feature_name, n_experiments, n_significant,
   mean_abs_ic, latest_run. Order by n_significant DESC, mean_abs_ic DESC.

**Step B: Create `src/ta_lab2/dashboard/pages/5_experiments.py`**

Follow the exact pattern of 3_research_explorer.py. Do NOT call
st.set_page_config() -- it is called in app.py.

Page structure:

1. Header: `st.header("Feature Experiments")`
   Caption: "Explore experimental feature IC results with BH significance"

2. Engine init (same try/except pattern as research explorer)

3. Summary table section:
   - `st.subheader("Experiment Summary")`
   - Call `load_experiment_summary(engine)`
   - If empty, show `st.info("No experiment results found. Run experiments first.")`
   - Otherwise display with `st.dataframe(summary_df, use_container_width=True)`

4. Feature detail section:
   - Load feature names via `load_experiment_feature_names(engine)`
   - `st.selectbox("Select feature", feature_names)`
   - Load results via `load_experiment_results(engine, selected_feature)`
   - Display results table with columns: asset_id, tf, horizon, return_type, ic,
     ic_p_value, ic_p_value_bh, ic_ir, n_obs
   - Highlight significant rows: add a `significant` boolean column where
     `ic_p_value_bh < 0.05`, then use `st.dataframe` with column_config to
     show a checkmark or color for the significant column.
   - Add CSV download button (same pattern as research explorer)

5. BH significance breakdown section:
   - `st.subheader("BH Significance by Horizon")`
   - If results exist, group by horizon, compute fraction of BH-significant
     rows per horizon, display as a simple bar chart via `st.bar_chart()`.
