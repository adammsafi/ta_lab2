    Step 6: Compute and save MAE/MFE
    CRITICAL FIX: Use `trade_id` PK for the UPDATE key, NOT `(run_id, entry_ts)`. The cmc_backtest_trades table has a `trade_id UUID PRIMARY KEY` column that is auto-generated by gen_random_uuid() on INSERT. Two trades can share the same entry_ts (e.g., multiple signals on same bar), so entry_ts is NOT unique within a run.

    Approach: After inserting trades in step 2 (via to_sql), query back the trade_ids for this run_id along with entry_ts to match them to the trades_df:
    ```python
    # 6. Compute MAE/MFE for closed trades
    try:
        from ta_lab2.analysis.mae_mfe import compute_mae_mfe, _load_close_prices
        if not result.trades_df.empty:
            close_prices = _load_close_prices(self.engine, result.asset_id, result.start_ts, result.end_ts)
            trades_with_excursion = compute_mae_mfe(result.trades_df, close_prices)

            # Fetch trade_ids inserted in step 2, ordered by entry_ts to match trades_df order
            trade_id_rows = conn.execute(text(
                "SELECT trade_id, entry_ts FROM public.cmc_backtest_trades "
                "WHERE run_id = :run_id ORDER BY entry_ts, created_at"
            ), {"run_id": result.run_id}).fetchall()

            # Match by positional index (both ordered by entry_ts)
            for i, (trade_id, _entry_ts) in enumerate(trade_id_rows):
                if i < len(trades_with_excursion):
                    row = trades_with_excursion.iloc[i]
                    mae_val = row.get('mae')
                    mfe_val = row.get('mfe')
                    if mae_val is not None and mfe_val is not None:
                        conn.execute(text(
                            "UPDATE public.cmc_backtest_trades SET mae = :mae, mfe = :mfe "
                            "WHERE trade_id = :trade_id"
                        ), {"mae": float(mae_val), "mfe": float(mfe_val), "trade_id": str(trade_id)})
            logger.info(f"MAE/MFE computed for {len(trades_with_excursion)} trades")
    except Exception as e:
        logger.warning(f"MAE/MFE computation failed (non-fatal): {e}")
    ```

    Step 7: Monte Carlo CI

    ```python
    # 7. Monte Carlo trade resampling

    except Exception as e:
        logger.warning(f"Monte Carlo computation failed (non-fatal): {e}")
    ```

  <action>
    Create 2 standalone CLI scripts for running analytics on existing backtest runs (retroactive analysis):

    1. `src/ta_lab2/scripts/analysis/run_quantstats_report.py`:
       - argparse: `--run-id` (required UUID), `--output` (optional, default reports/tearsheets/{run_id}.html)
       - APPROACH (FIXED -- no re-running backtest): Reconstruct daily portfolio returns from trade records and close prices. This avoids re-executing the backtest simulation (which risks duplicate DB records and is fragile).
       - Implementation:
         a. Load run metadata from cmc_backtest_runs (asset_id, start_ts, end_ts, signal_type)
         b. Load trades from cmc_backtest_trades for the run_id
         c. Load daily close prices from cmc_features for the asset/date range
         d. Reconstruct a daily equity curve:
            - Start with equity = 1.0
            - For each trade: mark entry/exit dates, compute daily returns within trade as close[t]/close[t-1] - 1
            - Outside trades: daily return = 0 (flat/cash)
            - Combine into a single daily returns Series
         e. Call generate_tear_sheet(returns, benchmark_returns, output_path, title)
         f. UPDATE cmc_backtest_runs SET tearsheet_path = output_path WHERE run_id = run_id (--write flag)
       - Print: output path, metrics count
       - Module invocation: `python -m ta_lab2.scripts.analysis.run_quantstats_report --run-id <UUID>`

    2. `src/ta_lab2/scripts/analysis/run_monte_carlo.py`:
       - argparse: `--run-id` (required UUID), `--n-samples` (default 1000), `--seed` (default 42)
       - Load trades from cmc_backtest_trades for that run_id
       - Call monte_carlo_trades(trades_df, n_samples, seed)
       - Print results: Sharpe CI, median, n_trades
       - Optionally UPDATE cmc_backtest_metrics with MC results (--write flag)
       - Module invocation: `python -m ta_lab2.scripts.analysis.run_monte_carlo --run-id <UUID>`

    Both scripts follow existing CLI patterns (NullPool engine, logging setup, argparse).

    CRITICAL for run_quantstats_report.py: Do NOT call BacktestEngine.run_backtest() or re-execute the simulation. This would insert duplicate trades/metrics into the database. Instead, reconstruct the equity curve from the already-persisted trade records. The reconstruction is straightforward: during a long trade window [entry_ts, exit_ts], daily return = close[t]/close[t-1] - 1. Outside any trade window, daily return = 0.
