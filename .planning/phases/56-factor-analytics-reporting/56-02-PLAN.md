<task type="auto">
  <name>Task 2: Create quantstats_reporter.py module</name>
  <files>src/ta_lab2/analysis/quantstats_reporter.py</files>
  <action>
    Create `src/ta_lab2/analysis/quantstats_reporter.py` with:

    1. `generate_tear_sheet(portfolio_returns, benchmark_returns, output_path, title)` function:
       - Takes `pd.Series` portfolio daily returns (from `pf.returns()`)
       - Takes `pd.Series` benchmark daily returns (BTC pct_change), OR `None` for no benchmark
       - Strips tz from both DatetimeIndex if present (QuantStats requires tz-naive). Use conditional check: only strip if `.index.tz is not None` to avoid double-strip TypeError.
       - CRITICAL: Handle benchmark=None gracefully. If benchmark_returns is None, call `qs.reports.html(ret, output=output_path, title=title, periods_per_year=365, compounded=True)` WITHOUT the benchmark parameter. QuantStats will produce a valid tear sheet -- it simply omits benchmark-relative metrics (alpha, beta, correlation, information ratio). Do NOT pass an empty Series as benchmark -- this causes QuantStats internal errors.
       - If benchmark_returns is a non-None, non-empty Series: call `qs.reports.html(ret, benchmark=bench, output=output_path, title=title, periods_per_year=365, compounded=True)`
       - Creates output directory with `os.makedirs(os.path.dirname(output_path), exist_ok=True)`
       - Returns the output_path string on success
       - Wraps in try/except ImportError to gracefully handle missing quantstats (log warning, return None)

    2. `_load_btc_benchmark_returns(engine, start_ts, end_ts)` helper:
       - BTC_ID = 1 (CoinMarketCap ID for BTC)
       - Query: `SELECT ts, close FROM public.cmc_features WHERE id = :btc_id AND tf = '1D' AND ts >= :start AND ts <= :end ORDER BY ts`
       - Fix tz: `pd.to_datetime(df["ts"], utc=True)` then `.dt.tz_localize(None)` for QuantStats compat
       - Convert to returns: `close_series.pct_change().dropna()`
       - CRITICAL: If the query returns empty results (no BTC data for the date range), return `None` (NOT an empty Series). This signals to callers that no benchmark is available, and they should pass benchmark=None to generate_tear_sheet(). Log a warning: "No BTC benchmark data found for date range {start_ts} to {end_ts}"
       - Return `pd.Series` with tz-naive DatetimeIndex, or `None` if no data

    3. `_strip_tz(series)` private helper:
       - If `series.index.tz is not None`: return series with `series.index = series.index.tz_localize(None)`
       - Else: return series unchanged
       - Handles the common tz-stripping pattern to avoid repetition

    Module-level: `import logging`, `logger = logging.getLogger(__name__)`. Use lazy import for quantstats inside `generate_tear_sheet` (so module imports don't fail when qs not installed).

    CRITICAL: benchmark parameter to `qs.reports.html` must be a returns Series (pct_change), NOT a price Series. The `_load_btc_benchmark_returns` function handles this conversion.

    CRITICAL: Empty/missing BTC data must NOT cause errors. The chain is: _load_btc_benchmark_returns returns None -> caller passes benchmark=None to generate_tear_sheet -> QuantStats generates benchmark-free tear sheet. An empty pd.Series passed as benchmark to QuantStats causes internal division-by-zero or index alignment errors.
  </action>
