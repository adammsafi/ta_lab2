       - Input: `trades_df` DataFrame with columns: entry_ts, exit_ts, entry_price, direction (from cmc_backtest_trades)
       - Input: `close_series` pd.Series of close prices indexed by timestamp (tz-naive, covers full backtest window)
       - Filter: Skip trades where `exit_ts` is NULL/NaT (open positions). Set mae=None, mfe=None for these.
       - For each closed trade:
         a. Convert entry_ts and exit_ts to tz-naive Timestamps (strip tz if present)
         b. Slice close_series: `close_series.loc[entry_ts:exit_ts]`

         d. For LONG direction:
            - MAE = float(window.min() / entry_price - 1.0)  (most negative = worst drawdown)
            - MFE = float(window.max() / entry_price - 1.0)  (most positive = best unrealized gain)
         e. For SHORT direction:
            - MAE = float(entry_price / window.max() - 1.0)  (negative when price goes against short)
            - MFE = float(entry_price / window.min() - 1.0)  (positive when price drops)

       - Use `direction.lower()` for comparison (vbt uses mixed case)

    Create `src/ta_lab2/analysis/monte_carlo.py` with:

    1. `monte_carlo_trades(trades_df, n_samples=1000, seed=42)` function:
       - Input: `trades_df` DataFrame with `pnl_pct` column (from cmc_backtest_trades for one run_id)
       - Guard: If fewer than 10 trades, return dict with all None CI values + n_trades + n_samples. Log warning.
       - Extract `pnl_pct` values, drop NaN, convert from percentage to decimal (divide by 100)
       - Use `numpy.random.default_rng(seed)` for reproducibility
       - For each of N=1000 samples:
         a. Resample n_trades PnL values WITH replacement: `rng.choice(pnl_pct, size=n_trades, replace=True)`
         b. Compute sample Sharpe: `mean / std(ddof=1) * sqrt(365)` (crypto annualization)
         c. Skip sample if std == 0 (constant returns)
       - Compute 95% CI: `np.percentile(sharpe_arr, [2.5, 97.5])`
       - Return dict:
         ```python
         {
             'mc_sharpe_lo': float(ci_lo),
             'mc_sharpe_hi': float(ci_hi),
             'mc_sharpe_median': float(np.median(sharpe_arr)),
             'mc_n_samples': n_samples,
             'n_trades': n_trades,
         }
         ```

    2. `monte_carlo_returns(returns_series, n_samples=1000, seed=42)` function:
       - Alternative entry point for returns-based MC (e.g., from pf.returns())
       - Same resampling logic but operates on daily return Series instead of trade PnL
       - Returns same dict structure
       - Guard: minimum 30 returns observations

    Module docstring: "Monte Carlo trade-level resampling for Sharpe ratio confidence intervals. Anti-overfitting tool."
    Import: numpy, logging. Fixed seed ensures reproducibility.

    CRITICAL from RESEARCH.md: Minimum 10 trades for trade-level MC. Return None for CI when below threshold.
  </action>
