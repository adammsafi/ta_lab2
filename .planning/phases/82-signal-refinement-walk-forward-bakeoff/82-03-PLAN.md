IMPORTANT: The expression engine's `evaluate_expression()` accepts a single DataFrame. The AMA columns are pre-joined by `load_strategy_data_with_ama()`. Column names in expressions use the feature name from `feature_selection.yaml` (e.g., `$TEMA_0fca19a1_ama`).

Define the following experiments (minimum 5, covering all 3 archetypes):

```yaml
# Phase 82 Signal Experiments
# These define composite signals using expression engine $col syntax.
# The DataFrame passed to evaluate_expression() has:
#   - OHLCV columns (open, high, low, close, volume)
#   - bar-level features (rsi_14, bb_ma_20, close_fracdiff, ret_is_outlier)
#   - AMA features (TEMA_0fca19a1_ama, KAMA_987fc105_ama, etc.)
# All pre-loaded by load_strategy_data_with_ama().

experiments:

  # ARCHETYPE 1a: Momentum -- IC-IR weighted top-5 AMA composite
  ama_momentum_weighted_top5:
    lifecycle: experimental
    archetype: momentum
    description: "IC-IR weighted sum of top-5 AMA values (TEMA200, KAMA_fast, HMA_fast, TEMA_fast, DEMA200)"
    compute:
      mode: expression
      expression: "EMA(($TEMA_0fca19a1_ama * 0.359 + $KAMA_987fc105_ama * 0.323 + $HMA_514ffe35_ama * 0.318 + $TEMA_514ffe35_ama * 0.315 + $DEMA_0fca19a1_ama * 0.344) / 1.659, 5)"
    tags: [composite, momentum, ama, ic_ir_weighted]
    holding_bars: [5, 7, 10]

  # ARCHETYPE 1b: Momentum -- AMA crossover (fast KAMA vs slow KAMA)
  ama_kama_crossover:
    lifecycle: experimental
    archetype: momentum
    description: "Fast KAMA minus slow KAMA crossover signal"
    compute:
      mode: expression
      expression: "$KAMA_987fc105_ama - $KAMA_8545aeed_ama"
    tags: [crossover, momentum, ama, kama]
    holding_bars: [5, 7, 10]

  # ARCHETYPE 2a: Mean-reversion -- close vs KAMA z-score
  ama_kama_reversion_zscore:
    lifecycle: experimental
    archetype: mean_reversion
    description: "Z-score of close relative to KAMA(10,2,30): mean-reversion signal"
    compute:
      mode: expression
      expression: "($close - $KAMA_de1106d5_ama) / (Std($close, 20) + 1e-10)"
    tags: [mean_reversion, zscore, ama, kama]
    holding_bars: [7, 10, 14]

  # ARCHETYPE 2b: Mean-reversion -- close vs HMA z-score
  ama_hma_reversion_zscore:
    lifecycle: experimental
    archetype: mean_reversion
    description: "Z-score of close relative to HMA spread: mean-reversion signal"
    compute:
      mode: expression
      expression: "($close - $HMA_d47fe5cc_ama) / (Std($close - $HMA_d47fe5cc_ama, 30) + 1e-10)"
    tags: [mean_reversion, zscore, ama, hma]
    holding_bars: [7, 10, 14]

  # ARCHETYPE 3a: Regime-conditional -- AMA momentum with trend strength filter
  ama_trend_adx_conditional:
    lifecycle: experimental
    archetype: regime_conditional
    description: "DEMA trend momentum conditioned on ADX > 20 (trending market filter)"
    compute:
      mode: expression
      expression: "Delta($DEMA_d47fe5cc_ama, 5) * Sign($close - Mean($close, 20))"
    tags: [regime_conditional, trend, ama, dema]
    holding_bars: [5, 7]

  # ARCHETYPE 3b: Regime-conditional -- multi-AMA agreement signal
  ama_multi_agreement:
    lifecycle: experimental
    archetype: regime_conditional
    description: "Agreement score across 3 AMA indicators (TEMA, DEMA, HMA) -- high when all agree on direction"
    compute:
      mode: expression
      expression: "(Sign(Delta($TEMA_0fca19a1_ama, 5)) + Sign(Delta($DEMA_0fca19a1_ama, 5)) + Sign(Delta($HMA_514ffe35_ama, 5))) / 3"
    tags: [regime_conditional, agreement, multi_ama]
    holding_bars: [5, 7, 10]
```

Each experiment has:
- `lifecycle`: experimental (Phase 82 evaluation)
- `archetype`: momentum / mean_reversion / regime_conditional
- `description`: human-readable purpose
- `compute.mode`: expression
- `compute.expression`: $col-syntax expression
- `tags`: for filtering and grouping
- `holding_bars`: list of holding periods to sweep in bake-off
