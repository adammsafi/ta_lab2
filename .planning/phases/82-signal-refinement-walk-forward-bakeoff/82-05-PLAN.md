**In run_bakeoff.py:**

1. Add `--per-asset-weights` flag:
   ```python
   parser.add_argument(
       "--per-asset-weights",
       action="store_true",
       help="Run AMA momentum with per-asset IC-IR weights alongside universal weights",
   )
   ```

2. When `--per-asset-weights` is True, for each asset, load that asset's per-asset IC-IR weights from `load_per_asset_ic_weights()` and create a modified param set for `ama_momentum` that uses asset-specific weights. Register this as a separate strategy name like `ama_momentum_perasset` so it appears as a distinct strategy in results.

3. The implementation approach:
   - Load per-asset IC weight matrix once (before the asset loop)
   - For each asset, if the asset has per-asset weights, create a `partial(ama_momentum_signal, weights=asset_weights)` and register as strategy `ama_momentum_perasset`
   - This runs alongside the universal-weight `ama_momentum` strategy so the comparison is direct

**In bakeoff_orchestrator.py:**

4. Update `BakeoffOrchestrator.run()` to accept optional `per_asset_weight_fn` callback that modifies strategy params per asset. OR simpler: accept a `per_asset_strategies` dict alongside the standard `strategies` dict that maps `(strategy_name, asset_id)` to modified param grids.

Actually, simplest approach: in run_bakeoff.py, build the strategies dict per-asset when `--per-asset-weights` is active, rather than modifying the orchestrator. Create a wrapper function:

```python
def _make_weighted_ama_momentum(weights: list[float]):
    """Create ama_momentum_signal with custom IC-IR weights baked in."""
    from ta_lab2.signals.ama_composite import ama_momentum_signal
    from functools import partial
    return partial(ama_momentum_signal, weights=weights)
```

Then in the asset loop, if per-asset weights are available for this asset, add `ama_momentum_perasset` to the strategies dict with the asset-specific weights.

**IMPORTANT per-asset IC weight overfitting guard (RESEARCH.md Pitfall 4):**
- Approach (a): Compute per-asset IC-IR weights on a held-out pre-bakeoff period (first year of data). Apply FIXED weights during bake-off.
- Implementation: When loading per-asset weights, pass `end_date = data_start + 365 days` to filter ic_results to only the first year. This ensures weights are out-of-sample relative to the bake-off period.
- Add a `--ic-weight-cutoff` flag defaulting to "2024-01-01" that limits the IC data used for weight computation.

5. Add `_run_phase82_bakeoff()` helper function that encapsulates the full Phase 82 execution sequence:
