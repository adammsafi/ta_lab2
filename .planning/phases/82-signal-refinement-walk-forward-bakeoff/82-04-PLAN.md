```python
def load_per_asset_ic_weights(
    engine: Engine,
    features: list[str],
    tf: str = "1D",
    horizon: int = 1,
    return_type: str = "arith",
) -> pd.DataFrame:
    """Load per-asset IC-IR weights from ic_results.

    Returns DataFrame with index=asset_id, columns=feature_names, values=ic_ir.
    Normalizes weights per asset so they sum to 1.0.
    Falls back to universal weights (from feature_selection.yaml) where
    per-asset data is missing.

    Parameters
    ----------
    engine : Engine
        SQLAlchemy engine.
    features : list[str]
        Feature names to load IC-IR for.
    tf : str
        Timeframe (default "1D").
    horizon : int
        IC horizon (default 1, i.e., 1-bar forward return).
    return_type : str
        Return type used in IC calculation (default "arith").

    Returns
    -------
    pd.DataFrame
        Rows = asset_id, columns = feature names, values = normalized IC-IR weights.
    """
```

Implementation:
1. Query `ic_results` for the given features, tf, horizon, return_type, regime_col='all', regime_label='all'
2. Pivot to wide format: asset_id x feature_name
3. Fill missing IC-IR values with the universal mean IC-IR from feature_selection.yaml
4. Clip negative IC-IR to 0 (negative IC means inverse signal -- handle separately)
5. Normalize per row so weights sum to 1.0
6. Return DataFrame

Also add a helper `load_universal_ic_weights()` that reads feature_selection.yaml and returns a dict of feature_name -> normalized IC-IR weight (for use as fallback and baseline comparison).
