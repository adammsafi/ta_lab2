Create `src/ta_lab2/scripts/backtests/select_strategies.py` -- a CLI script that loads Phase 82 bake-off results, applies statistical gates, ranks survivors, and generates a selection report.

**CLI interface:**
```python
parser = argparse.ArgumentParser(description="Apply statistical gates and select strategies from bake-off results.")
parser.add_argument("--experiment-prefix", default="phase82", help="Filter by experiment_name prefix")
parser.add_argument("--dsr-gate", type=float, default=None, help="DSR threshold (auto-calibrated if not set)")
parser.add_argument("--pbo-gate", type=float, default=0.50, help="PBO threshold (default: 0.50)")
parser.add_argument("--max-drawdown", type=float, default=0.15, help="Max drawdown cap (default: 15%%)")
parser.add_argument("--min-trades", type=int, default=50, help="Minimum trade count (default: 50)")
parser.add_argument("--output", default="reports/bakeoff/phase82_results.md", help="Report output path")
parser.add_argument("--cv-method", default="purged_kfold", help="CV method to use for gates (default: purged_kfold)")
```

**Gate application logic:**

1. Load all Phase 82 results from strategy_bakeoff_results:
   ```sql
   SELECT * FROM strategy_bakeoff_results
   WHERE experiment_name LIKE :prefix || '%'
     AND cv_method = :cv_method
   ORDER BY strategy_name, asset_id, cost_scenario
   ```

2. DSR gate calibration (if --dsr-gate not set):
   - Compute the DSR distribution across all results
   - Set threshold at the 75th percentile of DSR values (adaptive, not hardcoded 0.95)
   - Print: "DSR distribution: mean={}, median={}, p75={}, p90={}. Using p75={} as gate."
   - If p75 > 0.95, use 0.95. If p75 < 0.50, warn that signal quality is low.

3. Apply gates sequentially:
   ```python
   def apply_gates(df, dsr_gate, pbo_gate, max_dd, min_trades):
       initial = len(df)
       df = df[df['trade_count_total'] >= min_trades]
       after_trades = len(df)
       df = df[df['max_drawdown_worst'].abs() <= max_dd]
       after_dd = len(df)
       df = df[df['dsr'] > dsr_gate]  # NaN DSR fails gate
       after_dsr = len(df)
       df = df[(df['pbo_prob'] < pbo_gate) | (df['pbo_prob'].isna())]  # NaN PBO passes
       after_pbo = len(df)
       return df, {
           "initial": initial, "after_trades": after_trades,
           "after_dd": after_dd, "after_dsr": after_dsr, "after_pbo": after_pbo,
       }
   ```

4. Rank survivors using composite_scorer:
   - Import `rank_strategies` and `sensitivity_analysis` from `composite_scorer`
   - Score survivors under all 4 weight schemes (balanced, risk_focus, quality_focus, low_cost)
   - Identify "robust" strategies: those in top-5 under at least 3/4 schemes

5. Per-asset IC weight comparison:
   - Group results by strategy_name ('ama_momentum' vs 'ama_momentum_perasset')
   - Compare mean Sharpe, mean PSR, mean DSR across the same asset set
   - t-test or Wilcoxon test on paired Sharpe values to assess significance
   - Report: "Per-asset IC weights: {improvement/no improvement} (mean Sharpe delta = {}, p-value = {})"

6. Print summary table to console:
   ```
   === PHASE 82 STRATEGY SELECTION ===

   Gates applied:
     - Min trades: 50 (removed N)
     - Max drawdown: 15% (removed N)
     - DSR > 0.XX (removed N, calibrated from p75)
     - PBO < 0.50 (removed N)
   Survivors: N out of M initial

   Top strategies (balanced scoring):
   Rank  Strategy                 Asset  Sharpe  MaxDD   PSR    DSR    PBO
   1     ama_momentum              ALL    X.XXX  -X.XX  X.XXX  X.XXX  X.XXX
   ...
   ```

**CONTEXT.md survivor policy:** "ALL strategies that pass the gates go to paper trading (not capped at top N)." Honor this -- do not cap survivors. But rank them for documentation.
