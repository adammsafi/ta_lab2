  <name>Task 2: Add CI docs job and extend version-check to include mkdocs.yml</name>
  <files>

  <action>
    **1. Add a `docs` job** to `.github/workflows/ci.yml` after the existing `version-check` job:

    ```yaml
      docs:
        runs-on: ubuntu-latest
        steps:
          - name: Checkout
            uses: actions/checkout@v4
          - name: Set up Python
            uses: actions/setup-python@v5
            with:
              python-version: "3.12"
          - name: Install docs deps
            run: pip install mkdocs "mkdocs-material>=9.0,<9.7" "mkdocstrings[python]>=0.24"
          - name: Build docs (strict)
            run: mkdocs build --strict
    ```

    This job is independent of all other jobs (test, lint, format, mypy, version-check) and can run in parallel.

    **2. Extend the version-check job** to also verify mkdocs.yml version consistency:

    Update the `Check version consistency` step to also extract and compare the mkdocs.yml version:

    ```yaml
      - name: Check version consistency
        run: |
          PYPROJECT_VER=$(grep -oP '^version = "\K[^"]+' pyproject.toml)
          README_VER=$(head -1 README.md | grep -oP 'v\K[0-9]+\.[0-9]+\.[0-9]+')
          MKDOCS_VER=$(grep -oP 'site_name:.*v\K[0-9]+\.[0-9]+\.[0-9]+' mkdocs.yml)
          if [ "$PYPROJECT_VER" != "$README_VER" ] || [ "$PYPROJECT_VER" != "$MKDOCS_VER" ]; then
            echo "::error::Version mismatch: pyproject.toml=$PYPROJECT_VER, README=$README_VER, mkdocs=$MKDOCS_VER"
            exit 1
          fi
          echo "Version consistent: $PYPROJECT_VER"
    ```

    The only change from the existing version-check is adding the `MKDOCS_VER` extraction line and including it in the comparison.
  </action>
