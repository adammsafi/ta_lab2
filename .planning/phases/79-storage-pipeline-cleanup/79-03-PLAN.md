<objective>
Remove dead REST API routes and stale ChromaDB client from MCP memory server.

Purpose: The MCP memory server has two API surfaces: (1) MCP tools via `mcp_server.py` (functional, uses Mem0/Qdrant), and (2) REST routes `/api/v1/memory/*` via `api.py` (dead, still call ChromaDB-based modules, return HTTP 500). Nothing consumes the REST routes. The `client.py` file contains a ChromaDB PersistentClient singleton that is only referenced by `api.py` routes and a few other legacy modules. This plan removes the dead REST routes and deletes `client.py`.

Output: Clean server with MCP-only surface, no dead ChromaDB code.
</objective>

  **Step 1: Gut api.py -- remove all /api/v1/memory/* route handlers.**

  The `create_memory_api()` function in `api.py` defines 10 REST endpoints that all import from `client.py` (ChromaDB) and fail with HTTP 500. Remove ALL the route handler functions inside `create_memory_api()`:
  - GET `/api/v1/memory/stats`
  - POST `/api/v1/memory/search`
  - POST `/api/v1/memory/context`
  - GET `/api/v1/memory/types`
  - GET `/api/v1/memory/health`
  - GET `/api/v1/memory/health/stale`
  - POST `/api/v1/memory/health/refresh`
  - POST `/api/v1/memory/conflict/check`
  - POST `/api/v1/memory/conflict/add`

  Keep the `create_memory_api()` function itself (it's called by `server.py`), but make it create a bare FastAPI app with ONLY the `/health` endpoint. The health endpoint should use Qdrant (via `mem0_client`), not ChromaDB.

  Also remove the Pydantic request/response model classes that were only used by the dead routes (MemorySearchRequest, MemorySearchResponse, MemoryStatsResponse, ContextInjectionRequest, ContextInjectionResponse, HealthReportResponse, StaleMemoryResponse, RefreshRequest, ConflictCheckRequest, PotentialConflict, ConflictCheckResponse, AddWithConflictRequest, ConflictResolutionResponse). BUT keep MemoryResult if it's used elsewhere (check imports first).

  Keep the `app = create_memory_api()` line at the bottom (uvicorn entry point).

  **Step 2: Simplify server.py.**

  `server.py` currently:
  1. Imports `create_memory_api` from api.py
  2. Imports `mcp` from mcp_server.py
  3. Creates FastAPI app via `create_memory_api(lifespan=mcp_app.lifespan)`
  4. Mounts MCP at `/mcp/`
  5. Replaces the health endpoint with a Qdrant-aware one

  After gutting api.py, server.py should:
  1. Create a minimal FastAPI app directly (no need to import from api.py anymore)
  2. Mount MCP at `/mcp/`
  3. Add Qdrant health check at `/health`

  This means server.py no longer needs to import from api.py. If api.py is only consumed by server.py, we could delete api.py entirely. However, the `app = create_memory_api()` line in api.py serves as an alternate uvicorn entry point (`uvicorn ...memory.api:app`). Keep api.py as a minimal file with just the bare FastAPI app + health check for backward compat.

  **Step 3: Remove `from .client import ...` from `api.py`.**

  After removing the dead routes, api.py should have NO imports from `.client`.
