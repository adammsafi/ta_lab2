<objective>
Prune NULL first-observation rows from AMA returns and prevent new ones.

Purpose: AMA returns tables contain ~4% dead-weight rows where all return columns are NULL (first observation per series where LAG() has no prior value). Bar returns and EMA returns already filter these out (`WHERE prev_close_u IS NOT NULL` / `WHERE prev_ts_u IS NOT NULL`), but AMA returns (`refresh_returns_ama.py`) inserts every row from pass2 without filtering. This plan adds the same filter to AMA returns and runs a one-time cleanup DELETE.

Output: Updated `refresh_returns_ama.py` with first-observation filter, NULL rows purged from `returns_ama_multi_tf_u`, row counts logged.
</objective>

  **Step 2: Fix the SQL template in refresh_returns_ama.py.**

  The `_INSERT_SQL` template (around line 106-151) currently ends with:
  ```sql
  SELECT *, '{alignment_source}'::text AS alignment_source FROM pass2
  ON CONFLICT ...
  ```

  Change this to filter out first-observation rows. In pass1, `delta1` is computed as `ama - LAG(ama, 1) OVER w`. On the first observation per series, `LAG()` returns NULL, so `delta1` is NULL. The correct filter is:

  ```sql
  SELECT *, '{alignment_source}'::text AS alignment_source FROM pass2
  WHERE delta1 IS NOT NULL
  ON CONFLICT ...
  ```

  This mirrors the pattern in bar returns (`WHERE prev_close_u IS NOT NULL`) and EMA returns (`WHERE prev_ts_u IS NOT NULL`). We filter on `delta1` from pass1 because it's the simplest indicator of "has a prior observation" -- if `LAG(ama, 1)` returned NULL, then `delta1 = ama - NULL = NULL`.

  IMPORTANT: `delta1` is a column from pass1 that propagates through pass2 as `delta1_ama_roll` (since pass2 renames it). However, looking at the SQL more carefully:

  In pass2, the columns are renamed: `delta1 AS delta1_ama_roll`. So the column available in the final SELECT from pass2 is `delta1_ama_roll`, not `delta1`.

  The correct filter should be:
  ```sql
  SELECT *, '{alignment_source}'::text AS alignment_source FROM pass2
  WHERE delta1_ama_roll IS NOT NULL
  ON CONFLICT ...
  ```

  This ensures we skip the first observation per (id, venue_id, tf, indicator, params_hash) series where there's no prior AMA value.

  **Step 3: DELETE existing NULL rows from returns_ama_multi_tf_u.**

  Run a batched DELETE to remove existing first-observation NULL rows. Batch by alignment_source to keep transactions manageable:

  DELETE FROM returns_ama_multi_tf_u

    AND ret_arith_ama_roll IS NULL
    AND ret_log_ama_roll IS NULL
    AND delta1_ama IS NULL
