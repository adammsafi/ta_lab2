<objective>
Migrate the vol sub-phase to polars.

Purpose: Vol sub-phase is medium risk -- rolling std/var accumulates float precision differences, and ATR uses EWM which has a critical divergence between pandas and polars (pandas ewm skips NaN, polars does not). Research identified the fix: chain .fill_nan(None) after computing TR before ewm_mean. This plan implements polars-native vol functions and wires them into VolatilityFeature.

Output:
- Polars-native volatility functions in vol.py
- VolatilityFeature polars compute path
- Regression verification: all vol columns match within 1e-10
</objective>

  Add polars-native volatility functions to vol.py. Keep all existing pandas functions UNCHANGED. Add new functions at the bottom of the file in a clearly marked section.

  Import polars conditionally at the top of vol.py:
  ```python
  try:
      import polars as pl
      HAVE_POLARS = True
  except ImportError:
      pl = None
      HAVE_POLARS = False
  ```

  Implement these polars-native functions (all operate on a single-group polars DataFrame, already sorted by ts):

  1. `add_parkinson_vol_polars(lf: pl.DataFrame, high_col, low_col, windows, annualize, periods_per_year) -> pl.DataFrame`:
     - Compute `hl_sq = (ln(high/low))^2` using pl.col expressions
     - For each window: `rolling_mean(window, min_samples=window)` then sqrt * annualization
     - CRITICAL: use `min_samples=` NOT `min_periods=` (renamed in polars 1.21+)
     - Output column names match pandas: `vol_parkinson_{w}`

  2. `add_garman_klass_vol_polars(...)` -- same pattern with GK formula:
     `rs = 0.5 * ln(h/lo)^2 - (2*ln(2)-1) * ln(c/o)^2`

  3. `add_rogers_satchell_vol_polars(...)` -- RS formula:
     `rs = ln(h/c)*ln(h/o) + ln(lo/c)*ln(lo/o)`

  4. `add_atr_polars(lf: pl.DataFrame, period, open_col, high_col, low_col, close_col) -> pl.DataFrame`:
     - Compute TR: max(h-lo, abs(h-prev_close), abs(lo-prev_close))
     - CRITICAL ATR DIVERGENCE FIX: After computing TR column, chain `.fill_nan(None)` before ewm_mean.
       This converts NaN (from shift producing NaN on first row) to polars null, which ewm_mean skips (matching pandas behavior).
     - `ewm_mean(alpha=1/period, adjust=False, min_samples=1)`
     - Output: `atr_{period}`

  5. `add_rolling_vol_from_returns_polars(lf: pl.DataFrame, close_col, windows, annualize, periods_per_year, ddof, prefix, returns_col=None) -> pl.DataFrame`:
     - If returns_col exists in DataFrame, use it; otherwise compute log returns from close
     - For each window: `rolling_std(window, min_samples=window, ddof=ddof)` then annualize
     - Output: `{prefix}_log_roll_{w}`

  Each function takes a pl.DataFrame for a single (id, venue_id) group (pre-sorted by ts) and returns a pl.DataFrame with new columns appended. Use pl.col expressions and with_columns chaining.

  In polars_feature_ops.py, add a convenience function:
  `pandas_to_polars_df(df: pd.DataFrame, ts_col: str = "ts") -> pl.DataFrame`:
    Strips tz, converts to polars. Used before calling polars-native functions.
  `polars_to_pandas_df(pl_df: pl.DataFrame, ts_col: str = "ts") -> pd.DataFrame`:
    Converts back to pandas, restores UTC on ts_col.
