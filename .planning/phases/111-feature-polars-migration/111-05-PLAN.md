  CTF migration is the most complex because it involves merge_asof for cross-timeframe joins. The key challenge is timezone handling at the polars boundary.

  1. Read src/ta_lab2/features/cross_timeframe.py to understand the CTFFeature class and its merge_asof usage. Find where pd.merge_asof is called.

  2. In cross_timeframe.py, add a polars path for the merge_asof operation:
     - When use_polars is True (passed via CTFConfig or a parameter):
       a. Strip UTC from both DataFrames' timestamp columns using normalize_timestamps_for_polars
       b. Convert both to polars DataFrames
       c. Use pl.DataFrame.join_asof() with:
          - `on=` the timestamp column (must be sorted)
          - `by=` the group columns (id, venue_id)
          - `strategy="backward"` (matches pandas merge_asof default)
       d. Convert result back to pandas
       e. Restore UTC on timestamp columns
     - The rest of the CTF computation (slope, divergence, agreement, crossover features) may use simple arithmetic on the joined result. If they use pandas rolling/groupby, apply the same polars_sorted_groupby pattern. If they are simple column arithmetic, leave as pandas (overhead of conversion not worth it for element-wise ops).

  3. In refresh_ctf.py:
     - Add --use-polars flag to CLI argparse
     - Pass use_polars to CTFWorkerTask and through to CTFFeature/CTFConfig
     - In _ctf_worker, pass use_polars when creating CTFConfig

  4. CRITICAL TIMEZONE FIX for join_asof:
     - polars join_asof requires both DataFrames sorted by the `on` column
     - polars Datetime('us', 'UTC') and Datetime('us') are different types -- join will fail if mixed
     - Solution: normalize both to tz-naive before join, restore after
     - Use the normalize_timestamps_for_polars / restore_timestamps_from_polars helpers from polars_feature_ops.py

  5. If the CTF computation within cross_timeframe.py has additional pandas operations (indicator loop from YAML, computing slope/divergence per indicator pair), assess each:
     - Simple arithmetic (slope = (base - ref) / ref): leave as pandas, no benefit from polars
     - GroupBy + rolling: use polars_sorted_groupby
     - The YAML indicator loop that iterates over indicator pairs: keep the loop structure, just optimize the merge_asof inside

  6. Also add --use-polars propagation in run_all_feature_refreshes.py for the CTF step (deferred from Plan 04).
