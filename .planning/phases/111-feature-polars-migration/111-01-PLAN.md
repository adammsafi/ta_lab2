  2. Create src/ta_lab2/features/polars_feature_ops.py with these utilities:
     - `HAVE_POLARS` constant (try import polars, fallback False)
     - `normalize_timestamps_for_polars(df: pd.DataFrame, ts_col: str = "ts") -> pd.DataFrame`:
       Strips UTC timezone from timestamp column before polars conversion (polars Datetime('us','UTC') vs pandas datetime64[ns, UTC] causes issues). Returns modified copy.
     - `restore_timestamps_from_polars(df: pd.DataFrame, ts_col: str = "ts") -> pd.DataFrame`:
       Restores UTC on the timestamp column after converting back from polars. Uses pd.to_datetime(utc=True).
     - `polars_groupby_apply(df: pd.DataFrame, group_cols: list[str], sort_col: str, apply_fn, ts_col: str = "ts") -> pd.DataFrame`:
       The core pattern: takes a pandas DataFrame, groups by group_cols, sorts each group by sort_col, applies apply_fn (which receives and returns a pandas DataFrame for a single group), and concatenates results. Uses polars over() for the sort (pl.col(sort_col).sort_by(sort_col).over(group_cols)) to guarantee ordering, then falls back to pandas groupby for the actual apply_fn call (since the kernels are numba/numpy). This is the "Pattern 2" from research: polars replaces the outer groupby loop sorting, but the inner computation stays in numpy/numba.

     Actually, for cycle_stats and rolling_extremes, the numba kernels operate on numpy arrays, so polars value is in faster sort + partition, not in the kernel itself. The correct pattern is:
     - Convert pandas -> polars
     - Sort by [group_cols, sort_col] (single vectorized sort)
     - Convert back to pandas (now guaranteed sorted)
     - Do pandas groupby with the apply_fn (which calls numba kernels)
     This avoids the per-group .sort_values() call that pandas does inside each groupby iteration.

     Implement `polars_sorted_groupby(df: pd.DataFrame, group_cols: list[str], sort_col: str, apply_fn, ts_col: str = "ts") -> pd.DataFrame`:
     ```python
     def polars_sorted_groupby(df, group_cols, sort_col, apply_fn, ts_col="ts"):
         """Sort with polars, then apply per-group function."""
         if not HAVE_POLARS:
             # Fallback: pure pandas
             results = []
             for keys, grp in df.groupby(group_cols):
                 grp = grp.sort_values(sort_col).reset_index(drop=True)
                 results.append(apply_fn(grp))
             return pd.concat(results, ignore_index=True)

         import polars as pl
         df_work = normalize_timestamps_for_polars(df, ts_col)
         pl_df = pl.from_pandas(df_work)
         # Single vectorized sort across all groups
         pl_df = pl_df.sort(group_cols + [sort_col])
         df_sorted = pl_df.to_pandas()
         df_sorted = restore_timestamps_from_polars(df_sorted, ts_col)

         # Now groupby on pre-sorted data (no per-group sort needed)
         results = []
         for keys, grp in df_sorted.groupby(group_cols, sort=False):
             grp = grp.reset_index(drop=True)
             results.append(apply_fn(grp))
         return pd.concat(results, ignore_index=True)
     ```
