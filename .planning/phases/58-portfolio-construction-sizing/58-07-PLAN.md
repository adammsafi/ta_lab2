     - If stop_ladder is not None and stop_ladder.enabled:
       - Call: `triggers = stop_ladder.check_triggers(current_price=float(row["close"]), entry_price=entry_price, side=1, asset_id=int(id_), strategy="atr_breakout", already_triggered=already_triggered_set)`

       - `side=1` because ATR breakout is long-only (see breakout_atr.py: "Short side omitted by default")

         - Add the tier key (e.g., "sl_1", "tp_2") to already_triggered_set
         - Create an exit record with:
           - `direction = "close"` (partial or full depending on cumulative size_frac)
           - `exit_type = f"stop_ladder_{trigger['type']}_{trigger['tier']}"` (e.g., "stop_ladder_sl_1", "stop_ladder_tp_2")
           - `size_frac = trigger["size_frac"]` -- store in the record for downstream position sizing
         - Log: `logger.debug("Stop ladder %s tier %d triggered for id=%d at price=%.2f", trigger["type"], trigger["tier"], id_, current_price)`

   IMPORTANT: The existing _transform_signals_to_records already has a per-asset position state machine tracking open/closed positions. The stop ladder check should integrate INTO this existing state machine, not replace it. It adds an additional exit condition: when the position is open and no channel/ATR exit fired, check stop ladder tiers.

   If the existing code does not track entry_price per position, you may need to store it when a new position opens (entry_signal is True). Look at the existing state tracking logic carefully and integrate minimally.

   If integrating into the existing state machine is complex, a simpler approach is acceptable: After _transform_signals_to_records produces its records DataFrame, do a SECOND pass over the price data for assets with open positions, calling check_triggers for each bar, and append any triggered exit records to the DataFrame. This avoids modifying the existing state machine.
