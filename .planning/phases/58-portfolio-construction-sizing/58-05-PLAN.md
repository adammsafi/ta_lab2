2. Create `src/ta_lab2/scripts/portfolio/refresh_portfolio_allocations.py`.

**CLI pattern:** argparse with:
- `--ids` (asset IDs or 'all', default 'all')
- `--tf` (timeframe, default '1D')
- `--regime` (force regime label, optional)
- `--lookback` (override lookback_days, optional)
- `--dry-run` (compute but don't write to DB)
- `--config` (path to portfolio.yaml, default 'configs/portfolio.yaml')
- `--no-bl` (skip Black-Litterman, just run MV/CVaR/HRP)
- `--no-sizing` (skip bet sizing, write raw optimizer weights as final_weight)

**main() function flow:**
1. Parse args, load config via `load_portfolio_config(args.config)`.
2. Resolve DB URL via `resolve_db_url()` (from `ta_lab2.scripts.refresh_utils`).
3. Create engine with NullPool (pattern from existing scripts).
4. Load asset IDs: if 'all', query `SELECT DISTINCT id FROM cmc_price_bars_multi_tf WHERE tf = :tf`.
5. Load price matrix: query `cmc_price_bars_multi_tf` for (ids, tf), pivot to wide format
   (DatetimeIndex x asset_id columns, values = close price). Use lookback_days window.
   Filter assets with < lookback_days * 0.5 data points (insufficient history).
6. Load market caps: query latest market_cap from `cmc_price_bars_multi_tf` at max(ts) for
   each asset WHERE tf = '1D'. (BL Pitfall 5: same timestamp for all assets.)
7. Load regime: if --regime provided, use that. Otherwise query latest regime from
   `cmc_regimes` for the universe. Use modal regime label (most common across assets).
8. Run `PortfolioOptimizer().run_all(prices, regime_label)`.
9. If not --no-bl: Load signal scores from `cmc_signals_daily` (latest per asset). Load IC-IR
   from `cmc_ic_results` (latest per signal_type). Compute base_vol from prices. Run
   `BLAllocationBuilder().run(prices, mcaps, signal_scores, ic_ir, base_vol, S=result['S'])`.
   Add 'bl' weights to result.
10. If not --no-sizing: Load signal probabilities (signal_score as proxy for prob, normalized).
    Run `BetSizer().scale_weights(active_weights, probs, sides)`.
    Set final_weight = sized weights. Otherwise final_weight = raw weights.
11. Persist to `cmc_portfolio_allocations`: For each optimizer ('mv', 'cvar', 'hrp', 'bl'),
    INSERT rows with ON CONFLICT (ts, optimizer, asset_id) DO UPDATE.
    Set is_active=True for the active optimizer, False for others.
    config_snapshot = JSON of the config used.
12. Log summary: N assets, active optimizer, condition number, total weight sum, turnover.

**DB write pattern:** Use the project's standard upsert (temp table + ON CONFLICT DO UPDATE):
```python
with engine.begin() as conn:
    for optimizer_name, weights in all_weights.items():
        if weights is None:
            continue
        rows = [
            {"ts": rebalance_ts, "optimizer": optimizer_name,
             "is_active": optimizer_name == active,
             "regime_label": regime_label, "asset_id": aid,
             "weight": w, "final_weight": final_weights.get(aid, w),
             "signal_score": signal_scores_dict.get(aid),
             "condition_number": result["condition_number"],
             "config_snapshot": json.dumps(config)}
            for aid, w in weights.items() if w > 1e-6
        ]
        # INSERT ... ON CONFLICT DO UPDATE
```

Add `if __name__ == "__main__": main()` guard.

1. Create `src/ta_lab2/scripts/portfolio/run_portfolio_backtest.py`.

**CLI pattern:** argparse with:
- `--start` (backtest start date, required)
- `--end` (backtest end date, default today)
- `--tf` (timeframe, default '1D')
- `--strategy` (choices: 'topk_dropout', 'equal_weight', 'per_asset', 'all'; default 'all')
- `--config` (path to portfolio.yaml)
- `--output` (directory for results, default 'artifacts/portfolio_backtest')

**main() function flow:**
1. Load config, DB engine (NullPool), prices, signals, IC-IR.
2. Generate date range from start to end at frequency.
3. For each rebalance date:
   a. Slice price history up to that date (lookback_days window).
   b. Compute signal scores as of that date.
   c. Run optimizer and determine active weights.
   d. Apply TopkDropout selection if strategy = 'topk_dropout'.
   e. Compute turnover cost via TurnoverTracker.
   f. Track portfolio value: apply returns from rebalance date to next rebalance date.
4. **Strategies to compare:**
   - `topk_dropout`: PortfolioOptimizer + TopkDropoutSelector + BetSizer
   - `equal_weight`: 1/N allocation across all assets in universe
   - `per_asset`: Best per-asset strategy from Phase 42 bake-off (load from cmc_backtest_metrics)
5. Compute metrics per strategy: Sharpe, max drawdown, total return, annualized return,
   cumulative turnover cost. Use `ta_lab2.analysis.performance` functions (sharpe, max_drawdown, etc.).
6. Print comparison table. Optionally save to CSV in output directory.
7. Log: "TopkDropout Sharpe: X, EW Sharpe: Y, improvement: Z%"
