  <action>
Create `src/ta_lab2/portfolio/topk_selector.py` with `TopkDropoutSelector` class.

**Constructor:**
- Accepts optional config dict. Extracts topk_selection section: topk (10), n_drop (2).

**Method: `select(scores, current_holdings) -> tuple[set, set]`**
- `scores`: pd.Series indexed by asset_id, values are signal scores (higher = more bullish).
- `current_holdings`: set of currently held asset IDs.
- Implementation from 58-RESEARCH.md Pattern 6:
  1. Rank assets by score descending.
  2. Identify top_assets = set of top-K by score.
  3. Find held_below_threshold = held assets NOT in top_assets.
  4. Sort held_below_threshold by score ascending (worst first).
  5. to_sell = worst n_drop from held_below_threshold (or fewer if less available).
  6. candidates_to_buy = assets in top_assets not currently held, ordered by score desc.
  7. to_buy = first len(to_sell) from candidates_to_buy (buy same count as sell to maintain K).
- Return (to_buy: set, to_sell: set).

**Method: `get_target_universe(scores) -> set`**
- Returns the top-K asset IDs by score (ignoring current holdings). Useful for initial
  portfolio construction.

**Edge cases:**
- If current_holdings is empty (initial portfolio): to_sell is empty, to_buy = top-K assets.
- If fewer than topk assets have scores: use all available.
- Log the selection: "TopK: {topk}, dropping {n_sell}, buying {n_buy}".
  </action>

1. Create `src/ta_lab2/portfolio/cost_tracker.py` with `TurnoverTracker` class.

**Constructor:**
- Accepts optional config dict. Extracts fee_bps from rebalancing section (default 10.0).

**Method: `compute(old_weights, new_weights, portfolio_value) -> dict`**
- `old_weights`: dict[asset_id, float] of previous weights.
- `new_weights`: dict[asset_id, float] of target weights.
- `portfolio_value`: float total portfolio notional.
- Compute turnover_pct = sum of |new - old| across all assets (union of keys).
- Compute cost_pct = turnover_pct * fee_bps / 10000.
- Compute notional_cost = portfolio_value * cost_pct.
- Return dict:
  ```python
  {"turnover_pct": turnover_pct, "cost_pct": cost_pct, "notional_cost": notional_cost,
   "n_buys": count_new_or_increased, "n_sells": count_removed_or_decreased}
  ```

**Method: `track(ts, old_weights, new_weights, gross_return, portfolio_value) -> dict`**
- Calls compute(), then adds decomposed returns:
  ```python
  {"ts": ts, "gross_return": gross_return, "cost_pct": cost_pct,
   "net_return": gross_return - cost_pct, "turnover_pct": turnover_pct,
   "notional_cost": notional_cost}
  ```
- Appends to internal `self.history` list for cumulative tracking.

**Property: `cumulative_costs() -> float`**
- Sum of all notional_cost in history.

2. Create `src/ta_lab2/portfolio/rebalancer.py` with `RebalanceScheduler` class.

**Constructor:**
- Accepts optional config dict. Extracts rebalancing section: mode, frequency, drift_threshold, turnover_penalty.

**Method: `should_rebalance(current_ts, last_rebalance_ts, current_weights, target_weights, new_signal_arrived=False) -> bool`**
- Three modes:
  - **time_based:** Check if `current_ts - last_rebalance_ts >= frequency`. Parse frequency
    string ('1D', '7D', '30D') to timedelta.
  - **signal_driven:** Return True if `new_signal_arrived` is True.
  - **threshold_based:** Compute max absolute drift = max(|current - target|) across all
    assets. Return True if max_drift > drift_threshold.
- If mode is time_based but drift_threshold > 0, also check threshold overlay (rebalance
  if EITHER time OR drift triggers).

**Method: `parse_frequency(freq_str) -> timedelta`**
- Parse '1D' -> timedelta(days=1), '7D' -> timedelta(days=7), '1W' -> timedelta(weeks=1),
  '30D' -> timedelta(days=30), '1M' -> timedelta(days=30).

Do NOT update __init__.py here -- plan 58-05 consolidates all exports.

Create `src/ta_lab2/portfolio/stop_ladder.py` with a `StopLadder` class that implements
multi-tier stop-loss and take-profit exit scaling, configurable at per-asset x per-strategy
granularity. This is the PORT-05 requirement from CONTEXT.md: "Stop laddering: Configurable
at per-asset x per-strategy granularity."

Reference: VectorBT PRO stop laddering concept (58-RESEARCH.md Section 3.6):
```
sl_stop = [0.02, 0.03, 0.05]  # Scale out at 2%, 3%, 5% adverse move
tp_stop = [0.03, 0.05, 0.10]  # Scale out at 3%, 5%, 10% favorable move
```

**Constructor:**
- Accepts optional `config: dict = None`. If None, calls `load_portfolio_config()`.
- Extracts `stop_laddering` section from config:
  - `enabled`: bool (default True)
  - `defaults.sl_stops`: list[float] -- adverse move thresholds (e.g., [0.02, 0.03, 0.05])
  - `defaults.tp_stops`: list[float] -- favorable move thresholds (e.g., [0.03, 0.05, 0.10])
  - `defaults.sl_sizes`: list[float] -- fraction of position to exit at each SL tier (must sum to 1.0)
  - `defaults.tp_sizes`: list[float] -- fraction of position to exit at each TP tier (must sum to 1.0)
  - `per_strategy_overrides`: dict[str, dict] -- strategy_name -> partial override of sl_stops/tp_stops/etc.
  - `per_asset_overrides`: dict[str, dict] -- asset_id (as string) -> partial override
- Validates: len(sl_stops) == len(sl_sizes) and len(tp_stops) == len(tp_sizes) for each tier set.
- Validates: sl_sizes sum to ~1.0 and tp_sizes sum to ~1.0 (tolerance 0.01).

**Method: `get_tiers(asset_id: int, strategy: str | None = None) -> dict`**
- Resolves the effective stop ladder config for a specific asset + strategy combination.
- Resolution order (later overrides earlier):
  1. Start with global `defaults`.
  2. Apply `per_strategy_overrides[strategy]` if strategy provided and key exists.
  3. Apply `per_asset_overrides[str(asset_id)]` if key exists.
  4. Apply combined key `per_asset_overrides[f"{asset_id}:{strategy}"]` if exists (most specific).
- Return dict:
  ```python
  {
      "sl_stops": [0.02, 0.03, 0.05],
      "tp_stops": [0.03, 0.05, 0.10],
      "sl_sizes": [0.33, 0.33, 0.34],
      "tp_sizes": [0.33, 0.33, 0.34],
  }
  ```

**Method: `compute_exit_schedule(entry_price: float, side: int, asset_id: int, strategy: str | None = None) -> dict`**
- `entry_price`: float, the position entry price.
- `side`: +1 for long, -1 for short.
- Calls `get_tiers(asset_id, strategy)` to resolve config.
- Computes concrete exit price levels and sizes:
  - For long positions (side=+1):
    - SL levels: `entry_price * (1 - sl_stop)` for each sl_stop
    - TP levels: `entry_price * (1 + tp_stop)` for each tp_stop
  - For short positions (side=-1):
    - SL levels: `entry_price * (1 + sl_stop)` for each sl_stop
    - TP levels: `entry_price * (1 - tp_stop)` for each tp_stop
- Return dict:
  ```python
  {
      "sl_levels": [
          {"price": 49000.0, "size_frac": 0.33, "tier": 1},
          {"price": 48500.0, "size_frac": 0.33, "tier": 2},
          {"price": 47500.0, "size_frac": 0.34, "tier": 3},
      ],
      "tp_levels": [
          {"price": 51500.0, "size_frac": 0.33, "tier": 1},
          {"price": 52500.0, "size_frac": 0.33, "tier": 2},
          {"price": 55000.0, "size_frac": 0.34, "tier": 3},
      ],
      "entry_price": 50000.0,
      "side": 1,
  }
  ```

**Method: `check_triggers(current_price: float, entry_price: float, side: int, asset_id: int, strategy: str | None = None, already_triggered: set | None = None) -> list[dict]`**
- Given the current price, check which stop ladder tiers have been triggered.
- `already_triggered`: set of tier indices already executed (to avoid re-triggering).
- Returns list of newly triggered tiers:
  ```python
  [{"type": "sl", "tier": 1, "price": 49000.0, "size_frac": 0.33},
   {"type": "tp", "tier": 2, "price": 52500.0, "size_frac": 0.33}]
  ```
- For long side=+1: SL triggers when current_price <= sl_level, TP triggers when current_price >= tp_level.
- For short side=-1: SL triggers when current_price >= sl_level, TP triggers when current_price <= tp_level.

All logging via `logging.getLogger(__name__)`. Log tier resolution when overrides are applied.
  </action>
