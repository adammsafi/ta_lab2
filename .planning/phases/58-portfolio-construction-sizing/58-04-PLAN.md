Create `src/ta_lab2/portfolio/topk_selector.py` with `TopkDropoutSelector` class.

**Constructor:**
- Accepts optional config dict. Extracts topk_selection section: topk (10), n_drop (2).

**Method: `select(scores, current_holdings) -> tuple[set, set]`**
- `scores`: pd.Series indexed by asset_id, values are signal scores (higher = more bullish).
- `current_holdings`: set of currently held asset IDs.
- Implementation from 58-RESEARCH.md Pattern 6:
  1. Rank assets by score descending.
  2. Identify top_assets = set of top-K by score.
  3. Find held_below_threshold = held assets NOT in top_assets.
  4. Sort held_below_threshold by score ascending (worst first).
  5. to_sell = worst n_drop from held_below_threshold (or fewer if less available).
  6. candidates_to_buy = assets in top_assets not currently held, ordered by score desc.
  7. to_buy = first len(to_sell) from candidates_to_buy (buy same count as sell to maintain K).
- Return (to_buy: set, to_sell: set).

**Method: `get_target_universe(scores) -> set`**
- Returns the top-K asset IDs by score (ignoring current holdings). Useful for initial
  portfolio construction.

**Edge cases:**
- If current_holdings is empty (initial portfolio): to_sell is empty, to_buy = top-K assets.
- If fewer than topk assets have scores: use all available.
- Log the selection: "TopK: {topk}, dropping {n_sell}, buying {n_buy}".

1. Create `src/ta_lab2/portfolio/cost_tracker.py` with `TurnoverTracker` class.

**Constructor:**
- Accepts optional config dict. Extracts fee_bps from rebalancing section (default 10.0).

**Method: `compute(old_weights, new_weights, portfolio_value) -> dict`**
- `old_weights`: dict[asset_id, float] of previous weights.
- `new_weights`: dict[asset_id, float] of target weights.
- `portfolio_value`: float total portfolio notional.
- Compute turnover_pct = sum of |new - old| across all assets (union of keys).
- Compute cost_pct = turnover_pct * fee_bps / 10000.
- Compute notional_cost = portfolio_value * cost_pct.
- Return dict:
  ```python
  {"turnover_pct": turnover_pct, "cost_pct": cost_pct, "notional_cost": notional_cost,
   "n_buys": count_new_or_increased, "n_sells": count_removed_or_decreased}
  ```

**Method: `track(ts, old_weights, new_weights, gross_return, portfolio_value) -> dict`**
- Calls compute(), then adds decomposed returns:
  ```python
  {"ts": ts, "gross_return": gross_return, "cost_pct": cost_pct,
   "net_return": gross_return - cost_pct, "turnover_pct": turnover_pct,
   "notional_cost": notional_cost}
  ```
- Appends to internal `self.history` list for cumulative tracking.

**Property: `cumulative_costs() -> float`**
- Sum of all notional_cost in history.

2. Create `src/ta_lab2/portfolio/rebalancer.py` with `RebalanceScheduler` class.

**Constructor:**
- Accepts optional config dict. Extracts rebalancing section: mode, frequency, drift_threshold, turnover_penalty.

**Method: `should_rebalance(current_ts, last_rebalance_ts, current_weights, target_weights, new_signal_arrived=False) -> bool`**
- Three modes:
  - **time_based:** Check if `current_ts - last_rebalance_ts >= frequency`. Parse frequency
    string ('1D', '7D', '30D') to timedelta.
  - **signal_driven:** Return True if `new_signal_arrived` is True.
  - **threshold_based:** Compute max absolute drift = max(|current - target|) across all
    assets. Return True if max_drift > drift_threshold.
- If mode is time_based but drift_threshold > 0, also check threshold overlay (rebalance
  if EITHER time OR drift triggers).

**Method: `parse_frequency(freq_str) -> timedelta`**
- Parse '1D' -> timedelta(days=1), '7D' -> timedelta(days=7), '1W' -> timedelta(weeks=1),
  '30D' -> timedelta(days=30), '1M' -> timedelta(days=30).
