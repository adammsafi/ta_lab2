  <name>Task 1: Portfolio optimizer covariance override for high-correlation regime</name>
  <files>src/ta_lab2/portfolio/optimizer.py, configs/portfolio.yaml</files>
  <action>
**Modify `src/ta_lab2/portfolio/optimizer.py`:**

Add a method `_apply_high_corr_override(self, S: pd.DataFrame) -> pd.DataFrame` to the PortfolioOptimizer class that:

1. Reads the latest `high_corr_flag` from `cmc_cross_asset_agg`: `SELECT high_corr_flag, avg_pairwise_corr_30d FROM cmc_cross_asset_agg ORDER BY date DESC LIMIT 1`.
2. If `high_corr_flag` is False or NULL, or if the config switch `high_corr_override.enabled` is False, return S unchanged.
3. If `high_corr_flag` is True: inflate off-diagonal covariance entries by blending toward a fully-correlated matrix. The blend formula: `S_adjusted = (1 - blend_factor) * S + blend_factor * S_full_corr`, where `S_full_corr` has the same diagonal (variances) as S but off-diagonal entries set to `sqrt(var_i * var_j)` (i.e., correlation = 1.0 for all pairs). `blend_factor` comes from `configs/cross_asset_config.yaml -> portfolio_override.high_corr_blend_factor` (default 0.3).
4. Log a warning when the override is applied: "High-correlation regime detected (avg_pairwise_corr=%.3f); inflating off-diagonal covariance by blend_factor=%.2f".
5. Return `S_adjusted`.

**Call site:** In `run_all()`, after computing `S` via `risk_models.CovarianceShrinkage(prices_window).ledoit_wolf()` (around line 201) and BEFORE the condition number check, call `S = self._apply_high_corr_override(S)`. This way the inflated covariance is what gets checked for ill-conditioning and passed to all 3 optimizers.

**Config loading:** The PortfolioOptimizer `__init__` already loads `portfolio.yaml`. Add loading of `cross_asset_config.yaml` for the blend factor. Use a try/except so the optimizer still works without the cross-asset config (graceful degradation -- log info and skip override).

**DB connection for flag lookup:** Use the same engine pattern as `_resolve_tf_days()` -- try `os.environ.get("TARGET_DB_URL")` and if not available, skip the override with a debug log. The flag lookup should be lightweight (single row, indexed column).

**Modify `configs/portfolio.yaml`:**

Add a new section at the bottom:

```yaml
# --- High-correlation regime covariance override ---
# When cross-asset high_corr_flag is True, inflate off-diagonal covariance
# to reduce illusory diversification benefit.
high_corr_override:
  enabled: true                  # Master switch
  blend_factor: 0.3              # Blend toward full-correlation matrix (0=no change, 1=full corr)
```

NOTE: The blend_factor is duplicated between portfolio.yaml and cross_asset_config.yaml. portfolio.yaml is the authoritative source for the optimizer; cross_asset_config.yaml has the same default for documentation. The optimizer reads from portfolio.yaml.

**Important:**
- Do NOT break existing optimizer behavior. If the cmc_cross_asset_agg table doesn't exist or has no data, the override silently does nothing.
- Do NOT add hard dependencies on cross_asset_config.yaml from the optimizer -- it reads its own config from portfolio.yaml.
- Wrap the entire override in try/except so DB connection issues never break optimization.

2. Filter `sign_flip_df` to rows where `sign_flip_flag == True`.
3. If no sign flips, return 0.

   Asset: {asset_name} (id={asset_id})
   Macro Var: {macro_var}
   Correlation: {prev_corr_60d:.3f} -> {corr_60d:.3f}
   Date: {date}
