Follow the same structure as `regime_classifier.py`: module docstring, imports, constants, YAML config loader, `_to_python()` helper, computation functions, DB upsert methods.

**Config loading:**
- Load from `configs/cross_asset_config.yaml` using the same `project_root()` pattern as regime_classifier.py.
- Function: `load_cross_asset_config(yaml_path=None) -> dict`

**XAGG-01 + XAGG-02: `compute_cross_asset_corr(engine, config, start_date=None, end_date=None) -> pd.DataFrame`**
- Load daily close-to-close returns from `cmc_returns_bars_multi_tf` (tf='1D', roll=FALSE, ret_arith column) for ALL assets.
- ALSO load daily prices from `tvc_price_histories` and compute simple close-to-close returns for TVC assets. Merge both return sets into a single DataFrame with columns = asset identifiers.
- BTC/ETH 30d correlation: `df[['BTC','ETH']].rolling(window).corr()` using pandas rolling correlation. Use id=1 for BTC, id=52 for ETH (standard in cmc_price_histories7). Look these up from the DB if possible; hardcode as fallback.
- Average pairwise correlation: For each date, compute the mean of all pairwise 30d rolling correlations. Use `df.rolling(window).corr()` to get the full correlation matrix, then extract the upper triangle mean (excluding diagonal).
- `high_corr_flag`: True when `avg_pairwise_corr_30d > config['cross_asset']['high_corr_threshold']`.
- `n_assets`: Count of assets with non-null returns on that date.
- Return DataFrame with columns: date, btc_eth_corr_30d, avg_pairwise_corr_30d, high_corr_flag, n_assets.

**DB upsert function:**
- `upsert_cross_asset_agg(engine, df) -> int` -- temp table + INSERT...ON CONFLICT(date) DO UPDATE SET all columns.

**Watermark support:**
- `get_watermark(engine, table_name) -> Optional[date]` -- SELECT MAX(date) FROM {table_name}.
- When `start_date` is None, use watermark minus warmup (max rolling window from config, e.g., 90 days) for lookback.

**Also include stubs** for the XAGG-03 and XAGG-04 functions (raise NotImplementedError with docstring) so imports work. These will be filled in by the next task.

**Important anti-patterns to avoid:**
- Do NOT forward-fill crypto returns to fill FRED gaps (RESEARCH.md pitfall).
- Do NOT use `cmc_cross_asset_corr` table directly (different granularity -- per-pair per-TF vs aggregate per-date).
- Do NOT compute on sub-daily data (daily alignment only).
- Use `_to_python()` helper for ALL float values before DB binding (numpy scalar safety).
- Use `pd.to_datetime(utc=True)` for timestamp parsing, NOT `parse_dates` parameter.

Fill in the XAGG-03 and XAGG-04 stubs created in Task 1 and add their upsert functions.

**XAGG-03: `compute_funding_rate_agg(engine, config, start_date=None, end_date=None) -> pd.DataFrame`**
- Load daily rollup funding rates from `cmc_funding_rates` WHERE `tf = '1d'` (the daily rollup rows). Group by (date, symbol).
- For each (date, symbol): compute simple average of `funding_rate` across venues. `vwap_funding_rate` = NULL for now (start with simple avg only per RESEARCH.md). `n_venues` = count of non-null venues. `venues_included` = comma-separated list of venue names.
- Z-scores: For each symbol, compute 30d and 90d rolling z-scores of `avg_funding_rate` using `rolling(window).mean()` and `rolling(window).std()` (ddof=1).
- Extract date from ts using `ts::date` in SQL or `pd.Timestamp.normalize()` in Python.
- Return DataFrame with columns: date, symbol, avg_funding_rate, vwap_funding_rate, n_venues, zscore_30d, zscore_90d, venues_included.

**XAGG-04: `compute_crypto_macro_corr(engine, config, start_date=None, end_date=None) -> tuple[pd.DataFrame, pd.DataFrame]`**
- Returns a tuple: (crypto_macro_corr_df, macro_regime_update_df).
- Load daily returns for ALL tradeable assets from `cmc_returns_bars_multi_tf` (tf='1D', roll=FALSE, ret_arith).
- Load macro variable time series from `fred.fred_macro_features`: columns defined in `config['crypto_macro']['macro_var_columns']` mapping (vix->vixcls, dxy->dtwexbgs, hy_oas->bamlh0a0hym2, net_liquidity->net_liquidity).
- For each (asset, macro_var): compute 60d rolling Pearson correlation between asset daily returns and macro variable daily changes (use `.diff()` for VIX/DXY/HY_OAS; use level for net_liquidity since it already represents a flow).
- Sign-flip detection: For each (asset, macro_var), compare current `corr_60d` and `prev_corr_60d` (the prior day's value). `sign_flip_flag = True` when `prev_corr_60d > sign_flip_threshold AND corr_60d < -sign_flip_threshold` OR `prev_corr_60d < -sign_flip_threshold AND corr_60d > sign_flip_threshold`.
- `corr_regime` label: 'correlated' if `abs(corr_60d) > sign_flip_threshold`, 'decorrelated' if `abs(corr_60d) <= sign_flip_threshold`, 'flipping' if `sign_flip_flag = True`.
- `crypto_macro_corr_df`: DataFrame with columns: date, asset_id, macro_var, corr_60d, prev_corr_60d, sign_flip_flag, corr_regime.
- `macro_regime_update_df`: For each date, determine the most adverse crypto-macro state across all assets and macro vars. If ANY sign_flip_flag is True for that date -> 'flipping'. Else if majority are 'correlated' -> 'correlated'. Else -> 'decorrelated'. DataFrame with columns: date, crypto_macro_corr (TEXT label).

**DB upsert functions:**
- `upsert_funding_rate_agg(engine, df) -> int` -- temp table + INSERT...ON CONFLICT(date, symbol) DO UPDATE SET all columns.
- `upsert_crypto_macro_corr(engine, df) -> int` -- temp table + INSERT...ON CONFLICT(date, asset_id, macro_var) DO UPDATE SET all columns.
- `update_macro_regime_corr(engine, df) -> int` -- UPDATE cmc_macro_regimes SET crypto_macro_corr = :label WHERE date = :date for each row. Use batch parameterized UPDATE.

All upsert functions follow the temp table pattern from `refresh_cmc_cross_asset_corr.py`: create temp table, COPY/INSERT data, then INSERT...SELECT...ON CONFLICT. Use `_to_python()` on all float values before binding. Use `_sanitize_dataframe()` from regime_classifier.py pattern.

**Part A: Create CLI script `src/ta_lab2/scripts/macro/refresh_cross_asset_agg.py`**

Follow the exact CLI pattern of `refresh_macro_regimes.py`:

**argparse flags:**
- `--dry-run`: Compute all 4 XAGG outputs but do NOT write to DB. Print summary of what would be written.
- `--full`: Recompute from earliest available data (ignore watermarks).
- `--verbose`: Set logging to DEBUG.
- `--start-date` / `--end-date`: Custom date range.
- `--config`: Override YAML config path.
- `--skip-corr`: Skip XAGG-01/02 (cross-asset correlation).
- `--skip-funding`: Skip XAGG-03 (funding rate aggregation).
- `--skip-macro-corr`: Skip XAGG-04 (crypto-macro correlation).

**main() flow:**
1. Parse args, configure logging (same pattern as refresh_macro_regimes.py).
2. Load config via `load_cross_asset_config()`.
3. Create engine via `from ta_lab2.io import get_engine; engine = get_engine()`.
4. Determine date range: if `--full`, start from 2020-01-01. Else use watermarks.
5. Run each XAGG computation in sequence (unless skipped):
   a. XAGG-01/02: `compute_cross_asset_corr()` -> `upsert_cross_asset_agg()` (unless dry-run)
   b. XAGG-03: `compute_funding_rate_agg()` -> `upsert_funding_rate_agg()` (unless dry-run)
   c. XAGG-04: `compute_crypto_macro_corr()` -> `upsert_crypto_macro_corr()` + `update_macro_regime_corr()` (unless dry-run)
6. For each step, log: rows computed, rows upserted, elapsed time.
7. On dry-run: print summary DataFrame head(10) and shape for each output.
8. Return 0 on success, 1 on failure.

**Usage docstring** (in module docstring):
```

python -m ta_lab2.scripts.macro.refresh_cross_asset_agg --dry-run    # compute only
python -m ta_lab2.scripts.macro.refresh_cross_asset_agg --full       # full recompute
```
