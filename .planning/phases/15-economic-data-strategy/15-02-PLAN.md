```markdown
# Economic Data Packages Archive

**Archived:** 2026-02-03
**Phase:** 15-economic-data-strategy
**Decision:** Archive (not integrate)

## Contents

This archive contains two custom economic data packages that were evaluated for integration into ta_lab2:

### fredtools2 (167 lines)
PostgreSQL-backed FRED data ingestion with CLI.

**Purpose:** Fetch FRED series and release metadata, store in PostgreSQL.
**Entry point:** `fred [init|releases|series]`
**Dependencies:** requests, psycopg2-binary, python-dotenv

### fedtools2 (659 lines)
ETL consolidation of Federal Reserve policy target datasets.

**Purpose:** Merge FEDFUNDS, DFEDTAR, DFEDTARL, DFEDTARU into unified daily dataset.
**Entry point:** `fedtools2 [--config] [--plot]`
**Dependencies:** pandas, numpy, pyyaml, matplotlib, sqlalchemy, python-dotenv

## Archive Rationale

1. **Zero usage in ta_lab2:** No imports from either package found in codebase
2. **Ecosystem alternatives superior:** fredapi and fedfred provide better functionality with active maintenance
3. **Maintenance burden:** Custom wrappers add technical debt without corresponding value
4. **Preservation without cost:** Archive preserves code for reference without ongoing maintenance

## Restoration

To restore these packages for use:

1. Copy package directory to project root or lib/
2. Install in development mode: `pip install -e ./fredtools2`
3. Set environment variables (see INTEGRATION_GUIDE.md)
4. Run entry point: `fred init` or `fedtools2`

## Ecosystem Alternatives

For new economic data integration, use established packages instead:

- **fredapi** (https://pypi.org/project/fredapi/) - Most established FRED client
- **fedfred** (https://pypi.org/project/fedfred/) - Modern alternative with async/caching

See INTEGRATION_GUIDE.md for detailed comparison and usage examples.

See manifest.json for complete file listing with SHA256 checksums.
```

# Economic Data Integration Guide

This guide documents ecosystem alternatives and restoration procedures for archived economic data packages.

## Recommended Approach: Use Ecosystem Standards

If economic data integration becomes necessary for ta_lab2, use established packages instead of restoring archived custom wrappers.

### fredapi (Recommended)

**Installation:**
```bash
pip install fredapi
```

**Usage:**
```python
from fredapi import Fred
import os

fred = Fred(api_key=os.getenv('FRED_API_KEY'))

# Fetch series
unemployment = fred.get_series('UNRATE')
fed_funds = fred.get_series('FEDFUNDS')

# Search series
results = fred.search('unemployment rate')

# Get metadata
info = fred.get_series_info('UNRATE')

# Historical data revisions (ALFRED)
vintage_data = fred.get_series_all_releases('UNRATE')
```

**Advantages over fredtools2:**
- Handles data revisions (ALFRED support)
- Full FRED API coverage (series, releases, categories, tags, sources)
- Active maintenance
- Built-in search capabilities

### fedfred (Modern Alternative)

**Installation:**
```bash
pip install fedfred
```

**Usage:**
```python
from fedfred import Fred

fred = Fred(api_key=os.getenv('FRED_API_KEY'))

# Async support
import asyncio
data = asyncio.run(fred.get_series_async('UNRATE'))

# Built-in caching
data = fred.get_series('UNRATE', use_cache=True)
```

**Advantages:**
- Async support for high-volume workflows
- Built-in rate limiting (120 calls/min)
- Automatic caching with TTL
- Pandas/Polars/Dask support

## Optional Dependency Pattern

If adding economic data to ta_lab2, use optional dependency pattern:

**pyproject.toml:**
