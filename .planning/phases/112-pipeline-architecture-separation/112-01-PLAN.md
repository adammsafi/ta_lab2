The migration adds `pipeline_name` to both `pipeline_run_log` and `pipeline_stage_log`:

```sql
-- pipeline_run_log: add pipeline_name with DEFAULT for backward compat
ALTER TABLE public.pipeline_run_log
    ADD COLUMN IF NOT EXISTS pipeline_name VARCHAR(30) NOT NULL DEFAULT 'daily';

-- pipeline_stage_log: add pipeline_name (nullable, informational)
ALTER TABLE public.pipeline_stage_log
    ADD COLUMN IF NOT EXISTS pipeline_name VARCHAR(30);

-- Index for per-pipeline dead-man queries and dashboard lookups
CREATE INDEX IF NOT EXISTS ix_pipeline_run_log_name_ts
    ON public.pipeline_run_log (pipeline_name, completed_at);
```

Downgrade:
```sql
DROP INDEX IF EXISTS ix_pipeline_run_log_name_ts;
ALTER TABLE public.pipeline_stage_log DROP COLUMN IF EXISTS pipeline_name;
ALTER TABLE public.pipeline_run_log DROP COLUMN IF EXISTS pipeline_name;
```

IMPORTANT: ASCII-only in migration file (Windows cp1252 compatibility). No box-drawing characters or Unicode.

Use `op.execute()` with raw SQL wrapped in `sa.text()` for the IF NOT EXISTS pattern (Alembic's `op.add_column` doesn't support IF NOT EXISTS). Or use a standard approach:

```python
def upgrade() -> None:

        sa.Column('pipeline_name', sa.VARCHAR(30), nullable=False, server_default='daily'))
    op.add_column('pipeline_stage_log',
        sa.Column('pipeline_name', sa.VARCHAR(30), nullable=True))
    op.create_index('ix_pipeline_run_log_name_ts', 'pipeline_run_log',
```
