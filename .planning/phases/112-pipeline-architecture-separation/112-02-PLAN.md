
**Argparse args:**
- `--ids` (required): Asset IDs or 'all'
- `--db-url`: Database URL (default: from resolve_db_url)
- `--dry-run`: Print commands without executing
- `--verbose`: Stream subprocess output
- `--continue-on-error`: Don't stop on stage failure
- `--no-sync-vms`: Skip VM sync stages
- `--source`: One of {cmc,tvc,hl,all} (default: all) -- passed to bar builders
- `-n/--num-processes`: Parallel processes for bar builders
- `--chain`: After completion, launch Features pipeline via subprocess

**main() structure:**
1. Parse args, resolve db_url, parse_ids
2. `pipeline_run_id = _start_pipeline_run(db_url, pipeline_name="data")`
3. Run each stage in order with _log_stage_start / _log_stage_complete bracketing
4. After each stage: check `not result.success and not args.continue_on_error` -> return 1
5. After each stage: check `_maybe_kill(...)` -> return 2
6. VM sync stages are skipped if `--no-sync-vms` (VM sync failures are warnings, not stops -- matching existing behavior)
7. `_complete_pipeline_run(...)` with final status
8. `print_combined_summary(results)`
9. If `args.chain` and all_success: launch Features pipeline via `subprocess.run([sys.executable, "-m", "ta_lab2.scripts.pipelines.run_features_pipeline", "--chain", "--ids", args.ids, "--db-url", db_url])`
10. Return 0 if all_success else 1

**if __name__ == "__main__": sys.exit(main())**

The script must work as `python -m ta_lab2.scripts.pipelines.run_data_pipeline --ids 1 --dry-run`.
