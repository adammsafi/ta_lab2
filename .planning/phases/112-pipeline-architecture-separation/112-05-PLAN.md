  <action>
Create `docs/PIPELINE_CONTRACTS.md` documenting the pipeline architecture (satisfies PIPE-04).

**Structure:**

1. **Overview** -- 5 pipelines, deployment topology (local vs VM), chain mechanism

2. **Pipeline Definitions** -- For each pipeline:
   - Entry point (python -m command)
   - Stages (ordered list)
   - Key CLI args
   - Reads (DB tables consumed as input)
   - Writes (DB tables produced as output)
   - Trigger (manual, chain, polling loop, timer)
   - Where it runs (local PC vs Oracle VM)

3. **Handoff Contracts:**
   - Data -> Features: `pipeline_run_log.pipeline_name='data' AND status='complete'` + price_bars_multi_tf rows exist for requested IDs
   - Features -> Signals: `pipeline_run_log.pipeline_name='features' AND status='complete'` + features table updated, regimes table updated
   - Signals -> sync_signals_to_vm: signals_* tables have fresh rows (gate not blocked)
   - sync_signals_to_vm -> Execution (VM): signals_* rows on VM DB, MAX(ts) > last executor run
   - Monitoring: Independent, reads pipeline_run_log + drift_metrics + stats tables

4. **Chain Mechanism:**
   - `run_full_chain.py` calls Data -> Features -> Signals -> sync_signals_to_vm via subprocess
   - Each pipeline also supports --chain flag for pairwise chaining
   - Chain halts on any pipeline failure + Telegram alert
   - Signal gate blocks sync_signals_to_vm (VM executor sees no new signals, does nothing)

5. **Failure Modes:**
   - Pipeline failure: chain halts, Telegram alert, stale signals remain valid for executor
   - VM unreachable: sync_signals_to_vm fails, executor runs on last-known signals
   - Kill switch: .pipeline_kill file stops current pipeline, logged as 'killed' in pipeline_run_log

6. **Quick Reference:**
   ```
   # Full chain (replaces run_daily_refresh.py --all)
   python -m ta_lab2.scripts.pipelines.run_full_chain --ids all

   # Individual pipelines
   python -m ta_lab2.scripts.pipelines.run_data_pipeline --ids all
   python -m ta_lab2.scripts.pipelines.run_features_pipeline --ids all
   python -m ta_lab2.scripts.pipelines.run_signals_pipeline
   python -m ta_lab2.scripts.pipelines.run_execution_pipeline
   python -m ta_lab2.scripts.pipelines.run_monitoring_pipeline

   # VM services (Phase 113 deploys these)
   python -m ta_lab2.scripts.pipelines.run_execution_pipeline --loop --poll-interval 300
   python -m ta_lab2.scripts.pipelines.run_monitoring_pipeline --paper-start 2026-01-01

   # Sync signals after ad-hoc Signals run
   python -m ta_lab2.scripts.etl.sync_signals_to_vm

   # Legacy (deprecated)
   python -m ta_lab2.scripts.run_daily_refresh --all --ids all
   ```

Keep the document concise -- reference code for details, don't duplicate implementation.
