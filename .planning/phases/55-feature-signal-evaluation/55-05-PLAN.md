1. **Build promotion decision table.** Query both cmc_ic_results and cmc_feature_experiments to build a unified view of each feature's evaluation status:

   For each feature:
   - mean_abs_ic_ir (from cmc_ic_results, aggregated across all assets and TFs at horizon=1)
   - n_significant_bh (from cmc_feature_experiments, count of horizons passing BH at alpha=0.05)
   - bh_passes (boolean: any horizon passes BH)
   - ic_rank (rank by mean_abs_ic_ir descending)
   - recommendation: "promote" if bh_passes AND mean_abs_ic_ir > 0.02 (feature has real predictive signal); "keep_experimental" if borderline; "deprecate_candidate" if mean_abs_ic_ir < 0.005 (no signal)

   Write to reports/evaluation/promotion_decisions.csv.

2. **Execute promotions for clearly passing features.** Use FeaturePromoter for features that unambiguously pass the BH gate and have meaningful IC:

   ```python
   from ta_lab2.experiments.promoter import FeaturePromoter
   # Promote top features that pass BH gate
   promoter = FeaturePromoter(engine)
   for feature_name in features_to_promote:
       try:
           promoter.promote_feature(feature_name, alpha=0.05)
           print(f"PROMOTED: {feature_name}")
       except PromotionRejectedError as e:
           print(f"REJECTED: {feature_name} - {e.reason}")
   ```

   IMPORTANT: Per CONTEXT.md, Claude's discretion on lifecycle policy. Recommended approach:
   - Auto-promote: Features with BH pass + IC-IR > 0.03 (strong signal)
   - Manual review: Features with BH pass + IC-IR 0.01-0.03 (weak signal, may be noise)
   - Deprecation candidate: Features with BH fail OR IC-IR < 0.005 (no signal)
   - Do NOT auto-deprecate: deprecation is non-destructive but should be reviewed

   Thresholds above are starting points; adjust based on the actual IC distribution observed in the data.

3. **Log all decisions.** For each feature, record in promotion_decisions.csv:
