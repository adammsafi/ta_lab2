1. **Inspect cmc_features columns.** Query the DB or read the DDL to get all ~112 column names from cmc_features. The remaining columns are the canonical feature set.

2. **Confirm EMA column name.** The DDL for cmc_ema_multi_tf_u shows the EMA value column is `ema` (NOT `value` or `ema_value`). Use `ema` in all EMA crossover YAML entries. Other columns: `ema_bar`, `d1`, `d2`, `d1_roll`, `d2_roll`.

3. **Derive AMA params_hash values.** Import from ama_params.py to get correct params_hash for each AMA variant:
   - KAMA: 3 variants (KAMA_CANONICAL/KAMA_FAST/KAMA_SLOW) with their params_hash
   - DEMA: 5 variants (periods 9, 10, 21, 50, 200) with their params_hash
   - TEMA: 5 variants (periods 9, 10, 21, 50, 200) with their params_hash
   - HMA: 5 variants (periods 9, 10, 21, 50, 200) with their params_hash
   Run a small Python script to print all params_hash values: `python -c "from ta_lab2.features.ama.ama_params import ALL_AMA_PARAMS; [print(p.indicator, p.label, p.params_hash[:8]) for p in ALL_AMA_PARAMS]"`

4. **Resolve EMA crossover column collision.** ExperimentRunner._load_inputs merges tables on ts using join. If two inputs from cmc_ema_multi_tf_u (period=9 and period=21) are loaded, both have column 'ema' -- causing a name collision. This MUST be resolved before writing the YAML.

   **Resolution approach: Use dotpath mode for EMA crossover features.** Create a small Python function (e.g., `ta_lab2.experiments.ema_crossover:compute_ema_spread`) that:
   - Accepts a DataFrame with `close` column (from cmc_price_bars_multi_tf_u)
   - Accepts `fast_period` and `slow_period` keyword arguments
   - Loads cmc_ema_multi_tf_u rows for the given periods in two separate queries
   - Computes the spread: `(ema_fast - ema_slow) / (close + 1e-10)`
   - Returns the spread as a pandas Series

   Define the most commonly used crossover pairs:
   - ema_cross_9_21: fast=9, slow=21 (short-term momentum)
   - ema_cross_10_50: fast=10, slow=50 (medium-term trend)
   - ema_cross_21_50: fast=21, slow=50 (intermediate trend)
   - ema_cross_50_200: fast=50, slow=200 (golden/death cross)

   Each uses dotpath compute mode:
   ```yaml
   ema_cross_9_21:
     lifecycle: experimental
     description: "EMA(9)/EMA(21) normalized spread"
     compute:
       mode: dotpath
       function: "ta_lab2.experiments.ema_crossover:compute_ema_spread"
       kwargs:
         fast_period: 9
         slow_period: 21
     inputs:
       - table: cmc_price_bars_multi_tf_u
         columns: [close]
     tags: [ema, crossover, trend]
   ```

   IMPORTANT: If ExperimentRunner's dotpath mode does not support kwargs or if the runner does not pass connection info needed for the second query, fall back to skipping EMA crossover features entirely. Document the limitation in the YAML as comments and update must_haves truth accordingly.

5. **Define adaptive RSI feature.** Use inline expression on cmc_ta_daily:
   ```yaml
   adaptive_rsi_normalized:
     lifecycle: experimental
     description: "RSI(14) normalized by rolling 20th/80th percentile window"
     compute:
       mode: inline
       expression: "(rsi_14 - rsi_14.rolling(100).quantile(0.20)) / (rsi_14.rolling(100).quantile(0.80) - rsi_14.rolling(100).quantile(0.20) + 1e-10)"
     inputs:
       - table: cmc_ta_daily
         columns: [rsi_14]
     tags: [rsi, adaptive, momentum]
   ```
