**PromotionRejectedError** -- Custom exception class inheriting from Exception. Include a `reason` attribute and the BH results DataFrame.

**FeaturePromoter class:**

Constructor: `__init__(self, engine, registry: FeatureRegistry | None = None)` -- stores engine and optional registry.

**check_bh_gate() method:**
```python
def check_bh_gate(
    self,
    ic_results_df: pd.DataFrame,
    alpha: float = 0.05,
    min_pass_rate: float = 0.0,
) -> tuple[bool, pd.DataFrame, str]:
```
- Filter NaN p-values from ic_p_value column (CRITICAL: false_discovery_control raises ValueError on NaN).
- If zero valid p-values, return (False, df, "No valid p-values").
- Apply `scipy.stats.false_discovery_control(valid_pvals, method='bh')`.
- Write adjusted values to `ic_p_value_bh` column.
- Count passing combos (ic_p_value_bh < alpha).
- Compute pass_rate = n_pass / n_valid.
- If min_pass_rate == 0.0, passes = n_pass > 0.
- Else passes = pass_rate >= min_pass_rate.
- Return (passes, df_with_bh, reason_string).

**promote_feature() method:**
```python
def promote_feature(
    self,
    feature_name: str,
    *,
    alpha: float = 0.05,
    min_pass_rate: float = 0.0,
    confirm: bool = True,
) -> str:  # returns migration stub path
```
1. Load experiment results from `cmc_feature_experiments WHERE feature_name = :name`.
2. If no results, raise ValueError("No experiment results found").
3. Call check_bh_gate(). If fails, raise PromotionRejectedError with reason.
4. Find best IC row (highest absolute IC among BH-significant rows).
5. If confirm, print promotion summary and prompt "Promote {feature_name}? [y/N]: ". If declined, return early.
6. Write to dim_feature_registry:
   - INSERT feature_name, lifecycle='promoted', promoted_at=now(), promotion_alpha, promotion_min_pass_rate, best_ic, best_horizon.
   - If registry available, also write description, compute_mode, compute_spec, input_tables, input_columns, tags, yaml_digest from spec.
   - ON CONFLICT (feature_name) DO UPDATE SET lifecycle='promoted', promoted_at, etc.
7. Call _generate_migration_stub() to create Alembic migration file.
8. Update dim_feature_registry with migration_stub_path.
9. Print instructions: "Migration stub created at: {path}. Run: alembic upgrade head. Then add computation to src/ta_lab2/features/promoted_features.py".
10. Return migration stub path.

**_generate_migration_stub() method:**
- Query live Alembic head: `SELECT version_num FROM alembic_version` (CRITICAL: do NOT hardcode revision -- Pitfall 4 from research).
- Generate rev_id via `uuid.uuid4().hex[:12]`.
- Write migration file to `alembic/versions/{rev_id}_promoted_{slug}.py`.
- The migration adds a NUMERIC nullable column to cmc_features.
- downgrade drops the column.
- Include a comment block: "MANUAL STEP: After running alembic upgrade head, wire this feature's computation into src/ta_lab2/features/ by creating a compute function and registering it."
- Use `encoding='utf-8'` when writing (Windows pitfall).
- Return filepath.

**deprecate_feature() method:**
- UPDATE dim_feature_registry SET lifecycle='deprecated', updated_at=now() WHERE feature_name = :name.
- Do NOT remove column from cmc_features (irreversible without downtime).
- Do NOT remove rows from cmc_feature_experiments (audit trail).
- Print: "Feature {name} deprecated. Column remains in cmc_features but will no longer be computed."

Update `__init__.py` to export FeaturePromoter and PromotionRejectedError.

IMPORTANT:
- Use _to_python() pattern for numpy scalar normalization when inserting into DB.
- Use NullPool engine (caller creates).
- The migration stub MUST query live alembic_version for down_revision (not hardcode).
- No UTF-8 box-drawing chars in generated migration stubs (Windows).

**promote_feature.py:**
Arguments:
- `--feature` (required): Feature name to promote.
- `--alpha`: BH significance threshold (default: 0.05).
- `--min-pass-rate`: Minimum fraction of combos that must pass BH (default: 0.0 = any).
- `--yes` / `-y`: Skip confirmation prompt.
- `--dry-run`: Show BH gate results without promoting.
- `--registry`: Path to features.yaml (default: configs/experiments/features.yaml) for description metadata.

Main flow:
1. Parse args. Create engine with NullPool + resolve_db_url().
2. Optionally load FeatureRegistry for metadata (try/except if YAML path doesn't exist -- promotion works without it).
3. Create FeaturePromoter(engine, registry).
4. If --dry-run, load results and call check_bh_gate(), print results, exit.
5. Call promoter.promote_feature(feature_name, alpha=alpha, min_pass_rate=min_pass_rate, confirm=not args.yes).
6. Catch PromotionRejectedError and print reason with non-zero exit code.
7. On success, print migration stub path and next steps.

Invocation: `python -m ta_lab2.scripts.experiments.promote_feature --feature vol_ratio_30_7 --yes`

**purge_experiment.py:**
Arguments:
- `--feature` (required): Feature name to purge.
- `--force`: Also delete from dim_feature_registry (default: only deprecate).
- `--yes` / `-y`: Skip confirmation prompt.
- `--dry-run`: Show what would be deleted without actually deleting.

Main flow:
1. Parse args. Create engine with NullPool + resolve_db_url().
2. Count rows in cmc_feature_experiments WHERE feature_name = :name.
3. Print: "Will delete {N} experiment result rows for {feature_name}."
4. If dim_feature_registry has an entry, print its lifecycle status.
5. If --dry-run, exit 0.
6. If not --yes, prompt "Purge experiment results for {feature_name}? [y/N]: ".
7. DELETE FROM cmc_feature_experiments WHERE feature_name = :name.
8. If --force, DELETE FROM dim_feature_registry WHERE feature_name = :name. Else UPDATE lifecycle='deprecated', updated_at=now().
9. Print summary: "{N} rows deleted from cmc_feature_experiments. Registry entry {deleted/deprecated}."

Invocation: `python -m ta_lab2.scripts.experiments.purge_experiment --feature vol_ratio_30_7 --yes`
