Create `runner.py` with the `ExperimentRunner` class:

**Constructor:** `__init__(self, registry: FeatureRegistry, engine)` -- stores registry and SQLAlchemy engine. Registry must already be loaded (assert `registry._features` is not empty).

**run() method signature:**
```python
def run(
    self,
    feature_name: str,
    asset_ids: list[int],
    tf: str,
    train_start: pd.Timestamp,
    train_end: pd.Timestamp,
    *,
    horizons: list[int] | None = None,
    return_types: list[str] | None = None,
    dry_run: bool = False,
) -> pd.DataFrame:
```

**run() implementation flow:**
1. Get spec from registry via `self.registry.get_feature(feature_name)`.
2. Get tf_days_nominal via `DimTimeframe` (from `ta_lab2.time.dim_timeframe`).
3. Start cost tracking: `tracemalloc.start()`, `t0 = time.perf_counter()`.
4. Open a SINGLE `with self.engine.connect() as conn:` block for all operations (NullPool temp table lifetime pitfall).
5. Inside the connection block:
   a. Create temp scratch table: `CREATE TEMP TABLE IF NOT EXISTS {scratch_name} (id INTEGER NOT NULL, ts TIMESTAMPTZ NOT NULL, tf TEXT NOT NULL, feature_name TEXT NOT NULL, feature_val DOUBLE PRECISION)`. Use `_exp_{safe_name}_{ts_suffix}` naming convention (truncate feature name to 20 chars, use last 8 digits of epoch).
   b. For each asset_id:
      - Call `_load_inputs(conn, spec, asset_id, tf, train_start, train_end)` to load input data.
      - If input_df is empty, skip.
      - Call `_compute_feature(spec, input_df)` to get feature_series.
      - Extract close series from input_df (needed for IC). If close not in inputs, load it separately from cmc_price_bars_multi_tf_u.
      - Write feature values to scratch table via batch INSERT.
      - Call `compute_ic(feature_series, close_series, train_start, train_end, horizons=horizons, return_types=return_types, tf_days_nominal=tf_days_nominal)`.
      - Add asset_id and tf columns to IC result.
      - Append to all_ic_rows list.
   c. `conn.commit()` for scratch table visibility during debug.
6. Stop cost tracking: wall_clock, peak_memory_mb.
7. If no results, return empty DataFrame.
8. Concatenate all IC rows. Apply BH correction across ALL rows via `_apply_bh_correction()`.
9. Add cost columns and metadata (feature_name, yaml_digest, train_start, train_end).
10. Return result DataFrame.

**_load_inputs() method:**
Load data from declared source tables using SQL. For each entry in `spec["inputs"]`, query `SELECT {columns} FROM {table} WHERE id = :id AND tf = :tf AND ts BETWEEN :start AND :end ORDER BY ts`. Handle tables without `tf` column (like cmc_vol, cmc_ta_daily) by omitting the tf filter. Always load `close` from `cmc_price_bars_multi_tf_u` if not already in inputs (needed for IC forward returns). Return a single DataFrame indexed by ts (join inputs by ts). Use `pd.to_datetime(utc=True)` for ts column (MEMORY.md pitfall).

CRITICAL: Validate table names against an allowlist of known tables to prevent SQL injection. Use parameterized queries for all values (id, tf, start, end). Only the table name and column names are injected via f-string (validated at registry load time).

**_compute_feature() method:**
Dispatch based on compute mode:
- `inline`: Inject DataFrame columns + resolved_params as local variables. Use `eval(expression, {"np": numpy, "pd": pandas, "__builtins__": {}}, local_vars)`. The result should be a pandas Series.
- `dotpath`: Split on ':', importlib.import_module + getattr, call fn(input_df). Return Series.

**_apply_bh_correction() method:**
Filter NaN p-values FIRST (false_discovery_control raises ValueError on NaN). Apply `scipy.stats.false_discovery_control(valid_pvals, method='bh')`. Write adjusted values back. This is applied ONCE across all rows, NOT per-asset.

**_write_to_scratch() method:**
Batch INSERT feature values into scratch table. Use `conn.execute(text(...), params_list)` with executemany pattern. Include id, ts, tf, feature_name, feature_val columns.

Update `src/ta_lab2/experiments/__init__.py` to export ExperimentRunner.

IMPORTANT:
- Use NullPool engine (caller creates it, runner just uses it).
- Import `compute_ic` from `ta_lab2.analysis.ic` -- the Phase 37 scoring engine.
- Use `DimTimeframe` for tf_days_nominal lookup (column is `tf_days_nominal` NOT `tf_days` -- MEMORY.md pitfall).
- Tables without tf column: cmc_vol, cmc_ta_daily, cmc_features have tf column. Check the spec's input table and handle accordingly.
- Ensure ts column is tz-aware UTC throughout (MEMORY.md pandas tz pitfall).

Create `src/ta_lab2/scripts/experiments/` directory with `__init__.py` (empty) and `run_experiment.py`.

**run_experiment.py** follows the Phase 37 `run_ic_eval.py` CLI pattern:

**Arguments:**
- `--feature` (required): Feature name from YAML registry.
- `--ids`: Comma-separated asset IDs. Default: all assets from `dim_assets`.
- `--tf`: Timeframe string (default: "1D").
- `--train-start` (required): Train window start date (YYYY-MM-DD).
- `--train-end` (required): Train window end date (YYYY-MM-DD).
- `--horizons`: Space-separated horizon ints (default: Phase 37 defaults [1,2,3,5,10,20,60]).
- `--yes` / `-y`: Skip confirmation prompt.
- `--dry-run`: Print results but do not write to DB.
- `--compare`: Compare against prior runs in cmc_feature_experiments (show delta IC).
- `--registry`: Path to features.yaml (default: `configs/experiments/features.yaml`).
- `--all-experimental`: Run all features with lifecycle=experimental (mutually exclusive with --feature).

**Main flow:**
1. Parse args. Create engine with NullPool + resolve_db_url().
2. Load FeatureRegistry from --registry path.
3. Resolve feature names: if --all-experimental, use registry.list_experimental(). If --feature, use [args.feature].
4. Resolve asset_ids: if --ids, parse comma-separated. Else query `SELECT id FROM dim_assets ORDER BY id`.
5. For each feature_name:
   a. Call `ExperimentRunner(registry, engine).run(feature_name, asset_ids, tf, train_start, train_end, ...)`.
   b. Print summary table using `result_df[display_cols].to_string(index=False)`. Display cols: feature_name, asset_id, tf, horizon, return_type, ic, ic_p_value, ic_p_value_bh, n_obs.
   c. Print cost summary: wall_clock_seconds, peak_memory_mb, n_rows_computed.
   d. If --compare, load prior results from cmc_feature_experiments for same feature and show delta IC.
   e. If not --dry-run and not --yes, prompt `"Write to cmc_feature_experiments? [y/N]: "`. If confirmed (or --yes), write results.
   f. Write results to cmc_feature_experiments using save_experiment_results() helper (INSERT with ON CONFLICT uq_feature_experiments_key DO UPDATE for overwrite).
6. Print total summary.

**save_experiment_results() helper function** (in run_experiment.py or a separate db_helpers module):
- Takes conn and result_df.
- For each row in result_df, INSERT into cmc_feature_experiments with ON CONFLICT (uq_feature_experiments_key) DO UPDATE SET ic=EXCLUDED.ic, etc.
- Use `_to_python()` pattern from Phase 37 for numpy scalar normalization (hasattr(v, 'item') check).
- Generate experiment_id via gen_random_uuid() server-side (omit from INSERT).

**Invocation:** `python -m ta_lab2.scripts.experiments.run_experiment --feature vol_ratio_30_7 --train-start 2024-01-01 --train-end 2025-12-31 --tf 1D --yes`
