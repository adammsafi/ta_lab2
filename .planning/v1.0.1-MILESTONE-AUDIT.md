## Integration Check Results

**Report:** `.planning/v1.0.1-INTEGRATION-CHECK.md`

### Issue 1 — CRITICAL: MacroGateEvaluator Not Injected into PaperExecutor

Gate 1.7 (macro event/VIX/carry/credit gates) exists in RiskEngine but is permanently a no-op because `PaperExecutor.__init__` constructs `RiskEngine(engine)` without passing `macro_gate_evaluator`. The parameter defaults to `None`, making `_check_macro_gates()` return `('normal', 1.0)` unconditionally.

**Impact:** FOMC/CPI/NFP event gates, VIX spike gate, carry unwind gate, credit stress gate, and composite stress score NEVER reduce position sizing during live paper trading.

**Fix:** Inject `MacroGateEvaluator(engine)` when constructing RiskEngine in `paper_executor.py` and `run_paper_executor.py`.

### Issue 2 — MODERATE: evaluate_macro_gates Not in Daily Pipeline

The gate evaluation CLI exists but is not called from `run_daily_refresh.py`. The `dim_macro_gate_state` table is never refreshed automatically.

**Fix:** Add `evaluate_macro_gates` subprocess call to `run_daily_refresh.py` after macro regime refresh.

### Issue 3 — MODERATE: run_macro_alerts Not in Daily Pipeline

Telegram macro regime transition alerts exist but are only triggered manually.

**Fix:** Add `run_macro_alerts` subprocess call to `run_daily_refresh.py` after regime refresh.
