## Design Observations (Not Defects)

**1. FRED VM sync is a prerequisite step, not part of run_daily_refresh.py.**
sync_fred_from_vm.py must run before --macro to have fresh fred.series_values.
This is intentional per the VM strategy (.planning/VM-STRATEGY.md).
The freshness gate in MacroGateEvaluator._check_freshness_gate() degrades gracefully
when FRED data is stale rather than silently allowing full position sizing.

**2. macro_gates and macro_alerts run under run_macro_regimes_flag, not run_macro.**
If --macro is passed alone (without --macro-regimes), gate evaluation does not run.
This is correct design: gates read cmc_macro_regimes which is only fresh after
--macro-regimes runs. Running gates without fresh regime data would read stale state.

**3. cmc_regimes.l4_label is in the SQL schema (sql/regimes/080_cmc_regimes.sql line 18)
but not in any Alembic migration.**
This table predates the migration infrastructure.
The refresh script defensively uses _SCHEMA_COLS filtering at line 666 to handle column mismatches.
No action needed; this documents an existing pattern.
