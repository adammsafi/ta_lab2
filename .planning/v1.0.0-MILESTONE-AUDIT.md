## Cross-Phase Integration

### Chain 1: Signal → Order → Fill → Position — WIRED
Signal generators → SignalReader → PaperExecutor → CanonicalOrder → PaperOrderLogger → FillSimulator → OrderManager → cmc_positions. Full data flow verified.

**Gap:** RiskEngine.check_order() not called before order creation. Risk gates silently bypassed.

### Chain 2: Risk Controls → Kill Switch → Executor — PARTIALLY WIRED
RiskEngine.check_daily_loss() correctly calls activate_kill_switch(). Kill switch correctly sets dim_risk_state.trading_state='halted' and cancels pending orders.

**Break:** PaperExecutor never instantiates RiskEngine. No check_order(), check_daily_loss(), or kill-switch state check in the execution loop.

### Chain 3: Drift Monitor → Alert → Kill Switch — WIRED
DriftMonitor.run() → check_drift_threshold() → activate_drift_pause() → check_drift_escalation() → activate_kill_switch(). Full chain verified.

**Note:** Executor doesn't check drift_paused flag, but escalation to kill switch works.

### Chain 4: Feature Pipeline → ML Infrastructure — WIRED
cmc_features → run_feature_importance → compute_mda/sfi → ExperimentTracker.log_run() → cmc_ml_experiments. Full chain verified.

### Chain 5: Daily Refresh Orchestrator — WIRED BUT INCOMPLETE
Stages: bars → EMAs → AMAs → desc_stats → regimes → signals → portfolio → executor → drift → stats.

**Gap:** No cmc_features refresh stage. Signals depend on cmc_features but it's not refreshed by --all.

### Chain 6: Dashboard → DB Views — WIRED
Dashboard reads from dim_risk_state, cmc_risk_events, cmc_positions, cmc_drift_metrics. All data sources populated by their respective modules.

## E2E Flow Verification

### Flow 1: "Start paper trading from scratch" — PARTIAL
Steps 1-4 mechanically work but: (a) features must be refreshed separately, (b) risk controls not enforced during execution.

### Flow 2: "Risk event triggers safety" — BROKEN
RiskEngine.check_daily_loss() is never called by the executor. Kill switch can only be triggered manually or via drift escalation, not via daily loss monitoring.

### Flow 3: "Evaluate and improve features" — WORKING
IC sweep → feature importance → expression factors → Optuna → ExperimentTracker. Full ML experimentation loop wired.
