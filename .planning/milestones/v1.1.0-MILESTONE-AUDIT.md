### Dead Code Detail: _sync_1d_to_multi_tf

`refresh_price_bars_1d.py:576-612` contains `_sync_1d_to_multi_tf()` which INSERTs TVC/HL 1D bars into `public.price_bars_multi_tf` — a table dropped in Phase 78. Called at line 987 for non-CMC sources.

**Impact: None.** The function fails silently (`log.warning`, catches Exception). TVC/HL bars are not consumed via the multi-TF pipeline (multi-TF builders read from `cmc_price_histories7`, CMC-only). TVC/HL data is accessed via `price_bars_1d` directly.

**Recommendation:** Remove the function and its call site. Low priority — harmless dead code.

### Flow 2: New Data Source Onboarding
**Status: PARTIALLY MECHANICAL**

Adding a new source to `dim_data_sources` + running `refresh_price_bars_1d.py --source new` works for 1D bars. However, multi-TF rollup for non-CMC sources would require modifying the multi-TF builder to read from `price_bars_1d` instead of `cmc_price_histories7`. This is a known limitation — the multi-TF builder consolidation was explicitly out of scope for v1.1.0.

BAR-03 ("Adding a new data source requires only a new SourceSpec entry") is satisfied for 1D bar ingestion, which was the requirement scope.

### Flow 3: Incremental Refresh with Watermarks
**Status: WORKS**

All 6 table families maintain separate state tables with correct watermark tracking. All builders scope queries by `alignment_source` when reading from `_u` tables. State tables are preserved (not dropped with siloed data tables).
