### Phase 56: Factor Analytics & Reporting Upgrade
**Goal:** Upgrade strategy/feature evaluation with industry-standard analytics from QuantStats, Qlib, and VectorBT PRO — richer metrics, HTML tear sheets, cross-sectional normalization, and anti-overfitting tools.
**Depends on:** Phase 55 (needs IC infrastructure and backtest tables populated)
**Source:** Quant finance ecosystem review (`.planning/research/quant_finance_ecosystem_review.md`)
**Requirements:** ANALYTICS-01, ANALYTICS-02, ANALYTICS-03, ANALYTICS-04, ANALYTICS-05
**Scope:**
  - **QuantStats HTML tear sheets**: Integrate `qs.reports.html()` for every backtest run — 60+ metrics + 18 charts + benchmark comparison vs BTC
  - **IC decay analysis**: Extend IC computation to 2/5/10/20-bar forward return horizons; reveals predictive half-life per factor
  - **Rank IC & ICIR**: Add Spearman rank IC (outlier-robust) and IC Information Ratio (risk-adjusted IC) to `cmc_ic_results`
  - **Quintile group returns**: Rank all assets by factor score into 5 buckets, track cumulative return per bucket — gold standard monotonicity test
  - **Cross-sectional normalization**: CSZScoreNorm and CSRankNorm (normalize each asset vs all assets at same timestamp) — complementary to existing time-series z-scores
  - **MAE/MFE per trade**: Maximum Adverse/Favorable Excursion added to `cmc_backtest_trades` — reveals stop tightness and profit left on table
  - **Monte Carlo trade resampling**: Resample completed trades N=1000, compute Sharpe/CAGR confidence intervals — low-cost anti-overfitting check
  - **Additional metrics**: omega_ratio, smart_sharpe, probabilistic_sharpe, kelly_criterion, ulcer_index, skew, kurtosis, recovery_factor
**Success Criteria:**
  1. Every backtest run produces an HTML tear sheet with 60+ metrics and benchmark comparison
  2. IC results include Rank IC, ICIR, and IC decay at 5 horizons for all canonical features
  3. Cross-sectional z-scores and ranks computed and persisted alongside existing time-series z-scores
  4. MAE/MFE columns populated in `cmc_backtest_trades`; Monte Carlo CI reported per backtest run
  5. Quintile return charts available for any factor — monotonicity visually confirmed or rejected

### Phase 57: Advanced Labeling & Cross-Validation
**Goal:** Replace fixed-horizon return labels with adaptive triple barrier labeling, add meta-labeling for false positive reduction, and implement purged cross-validation to prevent data leakage in backtests.
**Depends on:** Phase 55 (needs feature/signal evaluation infrastructure), Phase 56 (benefits from richer analytics)
**Source:** MLFinLab (AFML), VectorBT PRO — `.planning/research/quant_finance_ecosystem_review.md`
**Requirements:** LABEL-01, LABEL-02, LABEL-03, LABEL-04
**Plans:** 6 plans
Plans:
- [ ] 57-01-PLAN.md — DDL + Alembic migration + triple barrier labeler library
- [ ] 57-02-PLAN.md — CUSUM event filter + trend scanning labels libraries
- [ ] 57-03-PLAN.md — Triple barrier refresh script (batch compute + DB persistence)
- [ ] 57-04-PLAN.md — CUSUM integration into all 3 signal generators
- [ ] 57-05-PLAN.md — Meta-labeling library + RF training/scoring pipeline
- [ ] 57-06-PLAN.md — CPCV Sharpe distribution runner
**Scope:**
  - **Triple barrier labeling**: Profit-taking barrier + stop-loss barrier + vertical (time) barrier — label = which barrier hit first {+1, -1, 0}. Replaces fixed-horizon returns for ML training
  - **Meta-labeling**: Primary model (EMA/RSI/breakout signals) picks direction; secondary model predicts whether to trade {0, 1}. Sized by confidence. Dramatically reduces false positives
  - **Purged K-Fold / CPCV**: `CombinatorialPurgedKFold` with purging + embargo — generates multiple OOS backtest paths for statistical testing of Sharpe. Prevents leakage from overlapping labels
  - **CUSUM event filter**: Event-driven sampling — only generate signals when cumulative deviation crosses threshold. Reduces noise trades
  - **Trend scanning labels**: OLS t-value based labeling as alternative to triple barrier for trend-following strategies. Raw t-value doubles as sample weight
**Success Criteria:**
  1. Triple barrier labeler produces {+1, -1, 0} labels for any (asset, tf) pair with configurable pt/sl multipliers and vertical barrier
  2. Meta-labeling pipeline: existing signals → direction, RF classifier → trade/no-trade with probability-based sizing
  3. CPCV produces distribution of OOS Sharpe ratios (not single point estimate) for each signal strategy
  4. CUSUM filter integrated as optional pre-filter for all 3 signal generators; reduces trade count by 20-40% while maintaining or improving Sharpe

### Phase 59: Microstructural & Advanced Features
**Goal:** Expand `cmc_features` with microstructural signals, stationarity-preserving transforms, bubble detection, and non-linear dependency measures — features unavailable in standard TA libraries.
**Depends on:** Phase 55 (needs feature evaluation to measure new features), Phase 56 (needs IC/quintile analysis to validate)
**Source:** MLFinLab — `.planning/research/quant_finance_ecosystem_review.md`
**Requirements:** MICRO-01, MICRO-02, MICRO-03, MICRO-04, MICRO-05
**Plans:** 5 plans
Plans:
- [ ] 59-01-PLAN.md — SQL migrations (9 microstructure columns on cmc_features + cmc_codependence table)
- [ ] 59-02-PLAN.md — Core math library (microstructure.py) + unit tests for all 5 feature classes
- [ ] 59-03-PLAN.md — MicrostructureFeature (BaseFeature subclass) for MICRO-01 through MICRO-04
- [ ] 59-04-PLAN.md — Codependence feature script (MICRO-05 pairwise computation + cmc_codependence)
- [ ] 59-05-PLAN.md — Orchestrator wiring + DDL update + full pipeline run for all assets
**Scope:**
  - **Fractional differentiation**: `frac_diff_ffd(close, d=0.4)` — makes prices stationary while preserving memory (d=0 raw, d=1 returns, d≈0.3-0.5 sweet spot). `plot_min_ffd()` finds optimal d per asset
  - **Kyle/Amihud lambda (bar-based)**: Price impact and illiquidity estimates from existing OHLCV + volume. `kyle_lambda`, `amihud_lambda`, `hasbrouck_lambda` columns added to `cmc_features`
  - **SADF bubble detection**: Supremum ADF test for explosive price behavior — feeds into `cmc_regimes` as additional regime signal. Detects crypto bubble formation
  - **Entropy features**: Shannon/Lempel-Ziv entropy on encoded price series — measures market predictability/randomness. Novel signal class
  - **Non-linear codependence**: Distance correlation, mutual information, variation of information — detects non-linear relationships between assets/TFs invisible to Pearson correlation. Extends `cmc_regime_comovement`
**Success Criteria:**
  1. Fractionally differentiated prices computed for all assets with auto-tuned d via ADF test; stored as feature columns
  2. Kyle/Amihud/Hasbrouck lambdas computed from OHLCV bars; added to `cmc_features` with IC scores showing predictive value
  3. SADF series computed for all assets; integrated into regime pipeline as bubble/explosive flag
  4. Entropy features (at least Shannon + Lempel-Ziv) computed and persisted; IC evaluated
  5. Distance correlation and mutual information matrices computed; compared to Pearson for regime comovement

### Phase 60: ML Infrastructure & Experimentation
**Goal:** Build the ML experimentation layer — config-driven factor definitions, feature importance, adaptive models that route by regime, concept drift handling, and experiment tracking.
**Depends on:** Phase 57 (needs purged CV), Phase 59 (needs expanded feature set), Phase 55 (needs feature experimentation framework)
**Source:** Qlib, MLFinLab, Jesse — `.planning/research/quant_finance_ecosystem_review.md`
**Requirements:** MLINFRA-01, MLINFRA-02, MLINFRA-03, MLINFRA-04, MLINFRA-05, MLINFRA-06
**Scope:**
  - **Expression engine for factors**: Config-driven factor definitions as strings (`"EMA($close, 12) / EMA($close, 26) - 1"`) parsed at runtime. YAML registry, no code changes per experiment. Extends feature experimentation framework
  - **MDA/SFI feature importance**: Permutation-based (MDA) and single-feature (SFI) importance with purged CV — identify which of 112+ `cmc_features` columns actually predict returns. Clustered FI for correlated features
  - **Regime-routed models (TRA pattern)**: `cmc_regimes` L0-L2 labels route to specialized sub-models per regime — mean-reversion in sideways, momentum in trending. Temporal Routing Adaptor architecture
  - **Concept drift models**: DoubleEnsemble (double ensemble across time) and ADARNN (adaptive RNN for non-stationary series) — models designed for crypto bull/bear regime shifts
  - **Experiment tracking**: Lightweight MLflow-equivalent against PostgreSQL — hyperparameter configs, metrics, artifacts linked per experiment run. Extends `cmc_backtest_runs` with full config tracking
  - **Optuna + Ray optimization**: Replace grid search with Tree-structured Parzen Estimator for intelligent hyperparameter search. Resumable, distributed, objective-selectable (Sharpe/Calmar/Sortino)
**Success Criteria:**
  1. Expression engine parses factor strings from YAML, evaluates against OHLCV data, and produces feature columns without Python code changes
  2. MDA feature importance report ranks all `cmc_features` columns by OOS predictive contribution; top/bottom features documented
  3. Regime-routed strategy backtested: per-regime sub-model vs single model; improvement in Sharpe or drawdown documented
  4. At least one concept drift model (DoubleEnsemble or ADARNN) trained and evaluated with purged CV; compared to static model baseline
  5. Experiment tracker persists full config + metrics for every run; queryable comparison dashboard shows parameter→performance mapping
  6. Optuna optimization produces better parameters than grid search on at least 1 strategy with documented efficiency gain
