# Milestone v1.0.1: Macro Regime Infrastructure

**Status:** SHIPPED 2026-03-03
**Phases:** 64-73
**Total Plans:** 29

## Overview

Wire FRED macro data into the regime/risk pipeline so trading decisions are conditioned on macroeconomic state, not just per-asset price action. 39 FRED series (208K rows) flow through macro feature computation, 4-dimensional regime classification (monetary policy, liquidity, risk appetite, carry), cross-asset aggregation, event risk gates, and observability dashboard.

## Milestone Summary

**Key Decisions:**
- Rule-based macro regime first, HMM as optional secondary (avoid model-on-model risk)
- Tighten-only semantics: macro layer can only reduce exposure, never increase
- Forward-fill with frequency-aware limits (45d monthly, 10d weekly) for mixed-frequency FRED data
- Separate macro tables (fred_macro_features, cmc_macro_regimes) rather than adding to cmc_features
- L4 slot in resolver chain for macro regime, preserving L0-L2 per-asset regime independence
- Event gates as size_mult reducers, not trade blockers (too aggressive to block entirely)

**Issues Resolved:**
- MacroGateEvaluator not injected into PaperExecutor (CRITICAL gap found by milestone audit, closed in Phase 73)
- Gate evaluation and macro alerts not wired into daily pipeline (closed in Phase 73)
- Requirements checkbox formatting inconsistency (cosmetic, closed in Phase 73)

**Issues Deferred:**
- Blended global liquidity proxy (Fed + ECB + BOJ) -- needs careful normalization
- Macro regime in backtest replay -- requires full historical FRED labels
- ETF flow proxy -- requires external data source
- On-chain metrics -- requires paid subscription

**Technical Debt Incurred:**
- None significant -- all integration gaps were closed before milestone completion
