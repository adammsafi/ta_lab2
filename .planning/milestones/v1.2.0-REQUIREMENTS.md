# Requirements: ta_lab2 v1.2.0 Analysis -> Live Signals

**Defined:** 2026-03-24
**Core Value:** Promote the best AMA-based strategies from research to paper trading with GARCH volatility forecasting, Black-Litterman portfolio allocation, full pipeline alert wiring, and cross-timeframe feature infrastructure.

## v1.2.0 Requirements

### Feature Selection & GARCH (Phases 80-81)

- [ ] **REQ-01**: `dim_feature_selection` has 15-25 active features with IC-IR >= 1.0 (AMA features expected to dominate)
  - **Verification**: `SELECT count(*) FROM dim_feature_selection WHERE tier = 'active'`
  - **Source**: Phase 80

- [ ] **REQ-02**: `garch_forecasts` table populated for all tracked assets (at least 1 forecast row per asset)
  - **Verification**: `SELECT count(DISTINCT id) FROM garch_forecasts` -- should match assets in price_bars_multi_tf
  - **Source**: Phase 81

- [ ] **REQ-03**: `garch_forecasts` latest materialized view or query returns non-stale data (within 48 hours)
  - **Verification**: `SELECT max(ts) FROM garch_forecasts` -- must be within 2 days of today
  - **Source**: Phase 81

### Strategy & Portfolio (Phases 82, 86)

- [ ] **REQ-04**: `strategy_bakeoff_results` populated with CPCV cross-validated results for at least 3 strategy configurations
  - **Verification**: `SELECT count(DISTINCT strategy_name) FROM strategy_bakeoff_results`
  - **Source**: Phase 82

- [ ] **REQ-05**: `stop_calibrations` table populated with at least 1 row from MAE/MFE analysis
  - **Verification**: `SELECT count(*) FROM stop_calibrations`
  - **Source**: Phase 86

- [ ] **REQ-06**: `portfolio_allocations` table populated with MV (Mean-Variance) weights that sum to approximately 1.0 per run
  - **Verification**: `SELECT method, sum(weight) as total FROM portfolio_allocations GROUP BY method, allocated_at HAVING sum(weight) BETWEEN 0.99 AND 1.01 LIMIT 5`
  - **Source**: Phase 86

### Dashboard (Phases 83-85)

- [ ] **REQ-07**: All existing dashboard pages load without error (landing, pipeline monitor, research explorer, asset stats, experiments, trading, risk, drift, executor, macro)
  - **Verification**: `streamlit run src/ta_lab2/dashboard/app.py` -- navigate to each page, no st.error() displayed
  - **Source**: Phase 83-85

- [ ] **REQ-08**: Strategy-first page, signal monitor page, and asset hub page (with candlestick charts) load and display data
  - **Verification**: Navigate to each page in Streamlit dashboard; charts render; no "No data" on all widgets
  - **Source**: Phase 83

- [ ] **REQ-09**: Hyperliquid perps page and AMA inspector page load and display data
  - **Verification**: Navigate to Hyperliquid perps page; perp list loads; AMA inspector shows curve for at least 1 asset
  - **Source**: Phase 84

### Pipeline Alerts (Phase 87)

- [ ] **REQ-10**: `pipeline_run_log` table exists and is populated after a full `run_daily_refresh --all` run
  - **Verification**: `SELECT run_id, started_at, completed_at, stages_json FROM pipeline_run_log ORDER BY started_at DESC LIMIT 3`
  - **Source**: Phase 87

- [ ] **REQ-11**: `signal_anomaly_log` table exists and is populated after a `run_signal_anomaly_gate` run (or `--all` pipeline)
  - **Verification**: `SELECT signal_table, status, z_score, checked_at FROM signal_anomaly_log ORDER BY checked_at DESC LIMIT 5`
  - **Source**: Phase 87

- [ ] **REQ-12**: IC staleness monitor runs without error and returns exit code 0 (no decay) or 1 (decay detected, non-fatal)
  - **Verification**: `python -m ta_lab2.scripts.analysis.run_ic_staleness_check --dry-run` -- exits 0 or 1, not 2 (error)
  - **Source**: Phase 87

### CTF Infrastructure (Phases 89-92)

- [ ] **REQ-13**: `ctf` table has computed cross-timeframe features for at least 2 assets x 1 base_tf (1D minimum)
  - **Verification**: `SELECT id, base_tf, count(*) as rows FROM ctf GROUP BY id, base_tf ORDER BY rows DESC LIMIT 5`
  - **Source**: Phases 89-91

- [ ] **REQ-14**: `dim_ctf_feature_selection` table has active and/or watch tier features after CTF IC sweep
  - **Verification**: `SELECT tier, count(*) FROM dim_ctf_feature_selection GROUP BY tier`
  - **Source**: Phase 92

### Integration & Go-Live (Phase 88)

- [ ] **REQ-15**: Smoke test passes (exit 0) on current data for default test assets (BTC, ETH, USDT, XRP)
  - **Verification**: `python -m ta_lab2.scripts.integration.smoke_test` -- exits 0 (all 26 checks pass)
  - **Source**: Phase 88

- [ ] **REQ-16**: Parity check with `--bakeoff-winners --pnl-correlation-threshold 0.90` runs without script error (exit 0 = pass, exit 1 = correlation below threshold but script ran correctly)
  - **Verification**: `python -m ta_lab2.scripts.executor.run_parity_check --bakeoff-winners --start 2025-01-01 --end 2025-12-31 --slippage-mode fixed --pnl-correlation-threshold 0.90`
  - **Source**: Phase 88

- [ ] **REQ-17**: Daily burn-in report generates and displays status for a given `--burn-in-start` date
  - **Verification**: `python -m ta_lab2.scripts.integration.daily_burn_in_report --burn-in-start 2026-03-24` -- prints report with ON TRACK / WARNING / STOP verdict
  - **Source**: Phase 88

- [ ] **REQ-18**: Operations manual Parts 2, 4, 7 updated for v1.2.0 pipeline (GARCH, stop calibration, portfolio, parity threshold, burn-in protocol, smoke test)
  - **Verification**: `grep -i "garch" docs/guides/operations/02_daily_pipeline.md` returns results; `grep -i "pnl-correlation-threshold" docs/guides/operations/04_paper_trading_and_risk.md` returns results; `grep -i "burn-in" docs/guides/operations/07_path_to_production.md` returns results
  - **Source**: Phase 88

- [ ] **REQ-19**: `docs/CHANGELOG.md` has a `[1.2.0]` section with Added/Changed entries covering Phases 80-92
  - **Verification**: `grep "\[1.2.0\]" docs/CHANGELOG.md` returns a result
  - **Source**: Phase 88
